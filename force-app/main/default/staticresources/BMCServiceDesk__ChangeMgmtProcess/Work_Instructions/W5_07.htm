Work Instructions for Change Coordinators

5.7.1

Specify in the Note field of the change that the new release did not pass customer testing, that additional work must be done to ensure that the release passes testing, and that the change must be approved again because the timing of the transfer to production had to be adjusted. Set the Scheduled Start date and Scheduled End date to the new implementation start and end date.

5.7.2

Set the Status field of the change back to Pending Approval. The change appears in the change manager's list of changes that are awaiting approval.
