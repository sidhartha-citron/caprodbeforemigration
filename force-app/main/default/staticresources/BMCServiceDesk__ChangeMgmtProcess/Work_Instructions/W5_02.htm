Work Instructions for Release Administrators

5.2.1

When the Status field of your task for the transfer of the new release changes from Registered to Assigned, open the task and read the instructions in the Description field.

5.2.2

If you are not ready to start working on the task, set the Status field of the task to Accepted.

Note:

If the task should be assigned to another release administrator, ask the change coordinator to change its assignment. To do so, set the Status field of the task to Rejected and add a Notes action saying why you rejected the task.

5.2.3

When you are ready to transfer the
new release to the test environment, set the Status field of the task to In Progress and perform the transfer.

Note:

If you cannot continue your work on the task (for example, because you have not been provided with the required access rights), set its Status field to Waiting for, and add a Notes action saying what you are waiting for and when you expect to complete the task.

5.2.4

When you complete the transfer of the new release to the test environment, set the Status field of the task to Completed.

If you could not complete the transfer successfully, set the Status field of the task to Failed and specify in the Result field why the transfer failed.

5.2.5

In the Duration (HH:MM) field, enter the time that you spent completing the task.
