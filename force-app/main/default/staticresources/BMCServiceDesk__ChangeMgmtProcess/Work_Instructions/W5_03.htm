Work Instructions for Customer Representatives

5.3.1

When the Status field of your task for testing the new release changes from Registered to Assigned, open the task and read the instructions in the Description field.

5.3.2

If you are not ready to start working on the task, set the Status field of the task to Accepted.

Note:

If the task should be assigned to another person, ask the change coordinator to change its assignment. To do so, set the Status field of the task to Rejected and add a Notes action saying why you rejected the task.

5.3.3

When you are ready to test the new release, set the Status field of the task to In Progress and perform the tests in the test environment.

5.3.4

Describe the tests and their results in a document and attach this document to the task.

Note:

If you cannot continue the work on the task (for example, it is temporarily impossible to access the test environment), set its Status field to Waiting for, and add a Notes action saying what you are waiting for and when you expect to complete the task.

5.3.5

When you finish testing the new release, if it passed all tests, set the Status field of the task to Completed.

If the new release did not pass all tests, set the Status field of the task to Failed and specify in the Resolution which tests caused the new release to fail.

5.3.6

In the Duration (HH:MM) field, enter the time that you spent completing the task.
