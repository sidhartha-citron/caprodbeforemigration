Work Instructions for Specialists

5.8.1

When the Status field of your task for completing a final preparation step changes from Registered to Assigned, open the task and read the instructions in the Description field.

Note:

The tasks for completing the final preparations include all steps required to ensure operational readiness. This can include tasks such as:

Creating a master copy of the new release so that the configuration administrator can place it in the DML

Adding storage capacity for the production and continuity environments

Delivering training to users

5.8.2

If you are not ready to start working on the task, set the Status field of the task to Accepted.

Note:

If the task should be assigned to another specialist, ask the change coordinator to change its assignment. To do so, set the Status field of the task to Rejected and add a Notes action saying why you rejected the task.

5.8.3

When you are ready to start the work, set the Status field of the task to In Progress and make the final preparations for transferring the new release into production.

Note:

If you cannot continue your work on the task (for example, because you are waiting for equipment to be delivered), set its Status field to Waiting for, and add a Notes action saying what you are waiting for and when you expect to complete the task.

5.8.4

As soon as you have completed the work, set the
Status field of the task to Completed.

If you cannot complete the task, set the Status field of the task to Failed and specify in the Result field why you could not complete the task successfully.

5.8.5

In the Duration (HH:MM) field, enter the time that you spent completing the task.
