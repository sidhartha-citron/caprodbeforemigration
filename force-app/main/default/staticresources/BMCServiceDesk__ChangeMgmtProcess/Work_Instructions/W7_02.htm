Work Instructions for Specialists

7.2.1

In the development environment, implement the change as discussed with the service owner (or on-duty manager if the service owner was not available).Test the change in a separate test environment if one is available. Otherwise, perform the tests in the development environment. If (automated) scripts for regression testing are available for the application, run them. Use the development environment to fix any errors that occurred during testing. Request help from other specialists as needed.

7.2.2

After you confirm that the correction works and no new issues have been introduced, contact the release administrator and request the transfer of the new release to the test environment.

7.2.3

Add a Notes action to the support request specifying what you changed in the application, which tests you performed, and who you asked to transfer the new release.

7.2.4

In the Duration (HH:MM) field of the Notes action record, enter the time  that you spent developing and testing the solution.
