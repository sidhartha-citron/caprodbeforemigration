Work Instructions for Change Coordinators

5.5.1

Review the test result documentation from the customer representatives who discovered that the new release is not functioning as required.

5.5.2

Insert the necessary additional implementation tasks for correcting the new release and for getting it tested again by the customer representatives. If the additional work will delay the transfer to production, also add a new approval task for each approver.

Note:

When multiple tasks must be assigned to the same person, if these tasks are to be completed in sequence without interruption, you can create a single task to cover all these tasks.

5.5.3

Link each new task to the CIs that will change when the task is executed. To do so, click Link CI on the Configuration Items tab under Supporting Information.

5.5.4

For each new implementation task, determine the most appropriate member of your group (in terms of skills, availability, and access rights) to execute it. Select this person by using the Assign To option of the toolbar of the task.

If an implementation task must be completed by another group, work with the coordinator of the group to determine the most appropriate member (in terms of skills, availability, and access rights) to do the work and agree on a realistic completion target. Select this person by using the Assign To option of the toolbar. However, if the group coordinator does not yet know who should complete the task, use the Assign To Queue option to select only the group's queue. In that case, the group coordinator can select the most appropriate person in the Staff field after the task's status has been set to Assigned.

If an implementation task must be completed by an external organization, work with your contact person at this organization to determine a realistic implementation target. Use the Assign To option to select yourself in the Staff field of the task and add a Notes action record about the external organization that will be asked to do the work. Later, when the work can start, inform the supplier and enter the reference number under which the supplier has registered this assignment in the same Notes action record.

5.5.5

Ensure that the date and time specified in the Due date field of the new tasks is achievable. If necessary, also update the Due date field of tasks that were planned
earlier and that need to be completed after the new tasks have been executed.

5.5.6

Ensure that the Status field of the new tasks is set to Registered.  If it was not necessary to adjust the due date of the task for the transfer to production,  set the Status field of the tasks with which the change implementation should now continue to Assigned.
