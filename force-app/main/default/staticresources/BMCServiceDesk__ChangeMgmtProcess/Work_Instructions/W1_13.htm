Work Instructions for Change Coordinators

1.10.1

Open the Change List tab by selecting Change Requests in the Change Management section of the Navigator Bar. Open a new change request form by clicking the New button in the toolbar.

1.10.2

Click the Template button and select the change template from the list of available templates for the implementation of the requested change. Using a template ensures that the fields of the change are populated with values from the selected change template, and that the necessary tasks are created and linked to the change when it is saved.

1.10.3

Ensure that you are selected in the Profile field of the change.

1.10.4

On the Configuration Items tab, ensure that the correct   services have been linked.

1.10.5

In the Description field, adjust the short description of the change as needed, and summarize the requirements by adding a Notes action to the change.

1.10.6

Save the change to generate its tasks and to allow it to be related to a support request or problem.

1.10.7

If the change request was received from Incident Management as a support request, create a change request and select its Incident tab. Click the Actions button, select Link Incident, and link the existing incident to the new change request. Add a Notes action to the support request saying that the "Responsibility for coordination has been accepted by Change Management." In the Duration (HH:MM) field, enter the time that you spent reviewing the support request.

1.10.8

If the change request was received from Problem Management as a problem, select its Changes tab. Click the Actions button, select Link Change Request, and link the new change to the problem. Add a Notes action to the problem saying that the "Responsibility for coordination has been accepted by Change Management." In the Duration (HH:MM) field, enter the time that you spent reviewing the problem.
