Work Instructions for Change Coordinators

1.9.1

Open a new change by selecting Change Requests in the Change Management section of the Navigator Bar.

1.9.2

In the toolbar, click the Template button and select the change template that will best facilitate the registration of the requested change. Using a template ensures that the fields of the change are populated with values from the selected change template, and that the necessary tasks are created and linked to the change when it is saved.

1.9.3

Ensure that you are selected in the Profile field of the change.

1.9.4

On the Configuration Items tab in Supporting Information, link the services that will be affected if the change is implemented. Select configuration items of class BMC_Service.

1.9.5

In the Description field, provide a short description of the change, and summarize the requirements by adding a Notes action to the change.

1.9.6

In the Reason for Change field, select the main reason why the change implementation was requested.

1.9.7

Save the change to generate its tasks and to allow it to be related to a support request, problem, or attachment.

1.9.8

If the change request was received from Incident Management as a support request, create a change request and select its Incident tab. Click the Actions button, select Link Incident, and link the existing incident to the new change request. Add Notes action to the support request saying that the "Responsibility for coordination has been accepted by Change Management." In the Duration (HH:MM) field, enter the time that you spent reviewing the support request.

1.9.9

If the change request was received from Problem Management as a problem, select its Changes tab. Click the Actions button, select Link Change Request, and link the new change to the problem. Add a Notes action to the problem saying that the "Responsibility for coordination has been accepted by Change Management." In the Duration (HH:MM) field, enter the time that you spent reviewing the problem.
