Work Instructions for Change Coordinators

2.5.1

Develop a change implementation plan by using the results of the Change Assessment analysis. The objective is to create a plan that minimizes both the risk of failure and the impact on the customers.

Do this by adding the necessary implementation tasks to the change.

2.5.2

For non-standard changes, ensure that the change plan includes an implementation task for the following actions:

Testing the change in a separate environment (which is also relevant for infrastructure changes, when practical)

Production testing, following the transfer to production

For the service level administrator, ensuring that the service level management information is updated if a service, service catalog item, or service level agreement is added or adjusted by the change

For the configuration administrator, ensuring that the
CMDB is updated if the change causes the
CMDB information to become out -of-date

For the capacity manager, ensuring that the capacity utilization threshold is updated if the change results in a new service infrastructure, or adjusting the available capacity levels of an existing service infrastructure

Informing the requesters and approvers of the change before the change is closed

Note:

If multiple tasks must be assigned to the same person and these tasks are to be completed in sequence without interruption, you can create a single task to cover all these tasks.
