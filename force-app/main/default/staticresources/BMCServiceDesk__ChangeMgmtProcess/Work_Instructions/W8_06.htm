Work Instructions for Change Coordinators

8.6.1

Review the details of the support requests for which the emergency change was implemented.

8.6.2

Open a new change by selecting Change Requests in the Change Management section of the Navigator Bar.

8.6.3

Select Emergency in the Change Type field of the change.

8.6.4

On the Configuration Item tab of the change, link a service and select a CI of class name BMC_BusinessService  that was affected by the incident for which the emergency change was implemented.

8.6.5

Enter a short description of the emergency change in the Description field, and summarize what has changed in the Information update field of the change.

8.6.6

Ensure that the Status field of the new change is set to In Progress, and save the change.

8.6.7

Select the Incidents tab of the change. Click on the Add button and link the incidents for which the emergency change was implemented to the change.

8.6.8

Click on the Tasks tab of the change. If the CMDB must be updated, assign the task for the update to the configuration administrator of your group and set the task’s Status field to Assigned. If the CMDB does not require an update, set the Status field of this task to Cancelled and specify in its Result field why the CMDB update is not required.
