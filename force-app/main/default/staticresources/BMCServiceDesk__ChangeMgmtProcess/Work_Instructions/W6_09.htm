Work Instructions for Change Coordinators

6.9.1

After the work on the change is completed, set  the Status field of any task that does not need to be completed (for example, because the implementation of the change was rejected by its approvers) to Cancelled. Specify in the Result field why the task was cancelled.

6.9.2

For all tasks that are linked to the change, verify that they have reached their final status (that is, Failed, Completed, or Cancelled). Then, select the appropriate completion code in the Completion code field of the change.

Note:

You can set the change to Completed only when all tasks related to the change have reached their final status.

6.9.3

Ensure that the Status field of the change is set to Completed.

6.9.4

Ensure that the actual end date of the implementation is registered in the Actual end date field.
