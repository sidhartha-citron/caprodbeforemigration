Work Instructions for Change Coordinators

6.7.1

For the production test task whose Status field was set to Failed by the specialist, review the information in the Resolution field.

6.7.2

Insert the necessary additional implementation tasks to ensure that all change requirements are met. Submit the change record for approval again.

Note:

When multiple tasks must be assigned to the same person, if these tasks are to be completed in sequence without interruption, you can create a single task to cover all these tasks.

6.7.3

If the additional work will cause one or more customer organizations to receive different functionality or to experience a period of service degradation or service outage (regardless of whether it occurs during service hours), add information about these organizations to the implementation tasks whose execution affects them. Do to so, add a Notes action to the tasks.

6.7.4

Link each implementation task to the CIs that will change when the task is executed. To do so, select Link CI  on the Configuration Items tab of the tasks.

6.7.5

In the Description field of each implementation task, specify the expected downtime duration. Normally the downtime should be set to None, except for tasks that cause the service to become unavailable when they are executed. Specify the downtime duration regardless of whether the downtime occurs during service hours.

6.7.6

For each implementation task, determine the most appropriate member of your group (in terms of skills, availability, and access rights) to execute it. Select this person by using the Assign To option of the toolbar.

If an implementation task must be completed by another group, work with the coordinator of the group to determine the most appropriate member (in terms of skills, availability, and access rights) to do the work and agree on a realistic completion target. Select this person by using the Assign To option of the toolbar. However, if the group coordinator does not yet know who should complete the task, use the Assign To Queue option to select only the group’s queue. In that case, the group coordinator can select the most appropriate person in the Staff field after the task's status has been set to Assigned.

If an implementation task must completed by an external organization, work with your contact person at this organization to determine a realistic implementation target. Use the Assign To option of the toolbar to assign the task to yourself and add a Notes action record about the external organization that will be asked to do the work. Later, when the work can start, inform the supplier and enter the reference number under which the supplier has registered this assignment in the same Notes action record.

6.7.7

Ensure that the date and time specified in the Due date field of the tasks is achievable. Also update the
Due date field of tasks that were planned earlier and must be completed after the new tasks have been executed, as needed.

6.7.8

Ensure that the Status field of the new tasks is set to Registered.

6.7.9

Add a Note action record indicating that the change needs to be approved again because additional work is required to ensure that all change requirements are met.

6.7.10

Set the Status field of the change back to To Be Approved. The change appears in the change manager's list of changes that are awaiting approval.
