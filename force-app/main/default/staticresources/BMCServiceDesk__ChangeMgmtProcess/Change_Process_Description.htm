Description


The Change Management process consists of eight procedures: six for implementing planned changes, and two for implementing emergency changes.

1. Change Registration is used by change coordinators when they are managing change requests.

2. Change Planning is used by change coordinators and specialists to prepare implementation plans for planned changes.

3. Change Approval is used by the change manager and change approvers (that is, customer representatives and service owners) to approve planned changes.

4. Infrastructure Change Implementation is used by specialists to implement infrastructure changes.

5. Application Change Implementation is used by specialists, release administrators, customer representatives, and change coordinators to implement application changes.

6. Planned Change Closure is used by specialists when they perform a final production test after a change has been implemented, and by change coordinators when they close a change request.

7. Emergency Change Implementation is used by specialists and release administrators to implement changes to resolve incidents.

8. Emergency Change Closure is used by specialists and change coordinators to complete and close emergency changes.

For details about these procedures, click on the boxes that represents the procedures. The graphical representation of a procedure appears with its description in the right-hand pane.
