Work Instructions for Release Coordinators

2.2.1

After the change request submission deadline for the next CAB meeting has passed, set the Status field of the task for preparing and facilitating the CAB meeting back to In Progress.

2.2.2

Prepare a list that summarizes the details of each change request that is to be reviewed by the CAB. Like the tasks, these requests are also linked to the change that acts as the placeholder for the service's change requests that are awaiting CAB review.

Note:

This list of change requests must provide the following columns of information:

  • Request or problem number

  • Name and organization of the requester

  • Description of the change request

  • Estimated implementation effort

  • Estimated implementation cost

2.2.3

Email the finalized list of change requests that must be reviewed during the upcoming CAB meeting to all CAB staff members. This allows them to discuss the change requests internally before the meeting.

2.2.4

In the task for preparing and facilitating the CAB meeting, add a Notes action saying that you have sent the finalized list of change requests to all CAB staff members and that the work will continue when the CAB meeting starts.

2.2.5

Set the Status field of the task for preparing and facilitating the CAB meeting to Waiting for.

2.2.6

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent finalizing the list of change requests.
