Work Instructions for Release Coordinators

4.1.1

Open the task to prepare the requirement documents for change coordinators. This task is generated by the change template when the change for the coordination of the release is registered.

4.1.2

Update the Due Date field of the task by specifying a date and time that can be achieved.

4.1.3

Ensure that you are selected in the Owner field of the task.

4.1.4

If you are not ready to prepare the requirement documents for the change coordinators, change the Status field of the task from Registered to Accepted. When you are ready to prepare the requirement documents, set the Status field of the task to In Progress.

4.1.5

Prepare a document for each change coordinator. In each document, detail the requirements that the change coordinator has agreed to fulfil.

4.1.6

Email each document to the change coordinator for whom it is prepared. In each email, ask the change coordinator to plan a change for the fulfillment of the requirements. Also, specify the date on which you expect the change to be implemented. Ensure that this date is achievable. Finally, ensure that you include the release number so that all change coordinators can use it to link their change to the release.

Note:

When you are asking for multiple changes to be implemented, ensure that the requested changes get implemented in the correct order and at the right moment (if this is important for the overall success of the release implementation).

4.1.7

Add a Notes action to list the names of the requirement documents. Also, indicate which document is sent to which change coordinator and specify that you are waiting for the changes to be registered and linked to the release.

4.1.8

Set the Status field of the task to Waiting for.

4.1.9

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent preparing the requirement documents.

4.1.10

Set the Status field of the release to In Progress.
