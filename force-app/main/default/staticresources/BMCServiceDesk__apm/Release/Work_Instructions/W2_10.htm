Work Instructions for Release Coordinators

2.10.1

In the Linked Tasks section of the change request to which the change requests for the release are linked, open the task for splitting the release into changes. This task was generated by the change template when the change was registered.

2.10.2

Update the Due Date field of the task by specifying a date and time that can be achieved.

2.10.3

Ensure that you are selected in the Owner field of the task.

2.10.4

If you are not ready to split the release into changes, change the Status field of the task from Registered to Accepted. When you are ready, set the Status field of the task to In Progress.

2.10.5

Review the requirements of the change requests that you have linked to the new release of the service to determine which of its supporting services must be changed.

2.10.6

Schedule a meeting with the change coordinator of the service for which the new release is to be implemented, and with the change coordinators of the supporting services that must also be changed.

2.10.7

Add a Notes action in the task for splitting the release into changes to specify the meeting date and the names of the participating change coordinators. Set the Status field of the task to Waiting for.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent scheduling the meeting.

2.10.8

When the meeting with the change coordinators starts, set the Status field to In Progress.

2.10.9

Review the change requests that have been linked to the new release with these change coordinators. Ensure that the estimated cost and effort for each change request are realistic.

2.10.10

Work with the change coordinators to divide the responsibility for fulfilling the different requirements amongst them. Divide the responsibilities in such a way that all change coordinators get the responsibility for requirements that their group can implement.

2.10.11

Draft a high-level implementation plan that indicates the duration of each change that will be coordinated by the change coordinators to fulfill their requirements, and the dependencies among these changes.

2.10.12

Attach the draft plan to the task for splitting the release into changes, and in the Resolution field, specify that you have attached the draft plan.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent preparing the draft plan.

Set the Status field of this task to Completed.
