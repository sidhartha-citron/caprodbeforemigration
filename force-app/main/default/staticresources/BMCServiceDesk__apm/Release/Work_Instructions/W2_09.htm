Work Instructions for Release Coordinators

2.9.1

From the Remedyforce Console tab, click View > Releases. To open a new release, click New.

2.9.2

Enter a short description of the release in the Release Title field and summarize the requirements of the release in the Description field.

2.9.3

Select the correct category in the Category field. Select the correct Release Type, Subtype, and Deployment Type.

2.9.4

Specify the impact and urgency.

2.9.5

Set the Status field of the release to Registered.

2.9.6

Specify achievable release dates in the Due Date, Start Implementation, End Implementation, Start Deployment, and End Deployment fields, and save the release.

2.9.7

Select the service for which the release is to be implemented in the Linked Services and Configuration Items section.

2.9.8

To open a new change request, click Select and Link Change Requests to Release in the Linked Change Requests section.

2.9.9

Select the release coordination template in the Template field and save the change. This action ensures that the necessary fields of the change are filled and that the following tasks are created and linked to the change when it is saved:

  • Splitting the release into changes

  • Business justification for the release

  • Preparation of requirement documents for the change coordinators

  • Update of the requesters and approvers of the release

  • Post-implementation meeting

2.9.10

Ensure that you are selected in the Initiator ID field and Owner of the change request.

2.9.11

Edit the short description of the change request in the Description field.

2.9.12

Set the Status field of the change request to In Progress and save the change.

2.9.13

Select the service for which the release is to be implemented in the Linked Services and Configuration Items section of the change request.

2.9.14

In the Linked Releases section, click Select and Link Releases to Change Request.

2.9.15

In the Linked Requests section of the change to link any request, whose requirements are to be fulfilled by the new release, click New Link. Similarly, click Select and Link Problems to Change Request in the Linked Problems section of the change to link any problem, which proposed structural solution to be implemented by the new release.

2.9.16

Open the change that acts as the placeholder for the service's change requests that are awaiting CAB review. Remove the link between this placeholder change request and each request and problem that you related to the change for the coordination of the new release. To remove a link with a support request, in the Linked Requests section of the placeholder change, and click Del in the Action column. To remove a link with a problem, in the Linked Problems section of the placeholder change, select Del in the Action column.
