Work Instructions for Release Coordinators

4.2.1

Follow up on the change requests that you sent to the change coordinators by ensuring that the requested changes are registered in the service management application and that the changes are linked to the release.

4.2.2

If a change is not yet registered, remind the responsible change coordinator to register the change.

4.2.3

If a change coordinator forgot to link the change to the release, open the change and link it to the release in the Linked Releases section.

4.2.4

In the Resolution field of the task for preparing the requirement documents, specify that the requested changes are registered in the service management application and that those changes are linked to the release. Set the Status of the task to Completed.

4.2.5

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent following up on the requirements documents.

Note:

When a change coordinator informs you that one of the related changes has been completed, continue with step 4.3.1.
