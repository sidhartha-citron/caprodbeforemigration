Work Instructions for Release Coordinators

1.2.1

If the change request does not require the involvement of Release Management, continue with 1.3.1. Otherwise, go to 1.4.1.

Note:

Change requests that require implementation of a non-standard change must be handled by Release Management. This excludes change requests that have been submitted for the prevention or fix of a problem, provided that their implementation:

  • Can be coordinated by a single change coordinator

 

  • Does not change the functionality of a service

 

  • Does not require any additional funding
