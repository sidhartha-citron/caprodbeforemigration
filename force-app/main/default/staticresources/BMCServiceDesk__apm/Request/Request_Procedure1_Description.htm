Procedure 1: Request Registration


When a user contacts the service desk, the service desk analyst links the user's record to a new request. Next, the service desk analyst determines the nature of the user's request by asking the user how the service desk analyst can help the user if the user contacted the service desk by phone or in person. If the user contacted the service desk in writing (for example, email, fax, paper form, and so on), the service desk analyst determines the nature of the request by reading through the information.

If the user contacted the service desk about an earlier submitted request, the service desk analyst searches for the request. If the request is already completed or set to ClosedMail, but the user is not satisfied with the solution provided, the service desk analyst sets the status to assigned. The request appears in the To Do list of the staff and group that provided the original solution. If the user wants an update on the request's status, the service desk analyst informs the user of the progress.

If the user contacted the service desk to submit a new request, the service desk analyst determines if a request template is available for the request.

If a request template is available, the service desk analyst uses the template to create the request. The service desk analyst reads the instructions from the template and follows them. The instructions specify the information to obtain from the user and how to proceed after receiving the information (for example, how to resolve the request, or assign to which group).

If a request template is not available for the request, the service desk analyst creates the request. The service desk analyst links the applicable service and configuration item to the request.

If the user reports an incident, the service desk analyst determines the impact and urgency level based on the following:

  • Information provided by the user

  • Information about the configuration of the service instance

  • Other incidents that are currently open for the impacted service

The impact and urgency level determine the priority level that is assigned to requests for incident resolution.

If the user reports that a critical or core service has failed for several users, the service desk analyst escalates the incident to the service owner. The service owner activates the Major Incident Team to resolve the Major Incident.
