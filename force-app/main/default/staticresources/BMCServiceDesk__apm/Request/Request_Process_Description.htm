Process description

The Request Management process consists of eight procedures for handling requests.

  • Request Registration is used by service desk analysts when they register requests from users.

  • Request Assignment is used by service desk analysts when they assign requests to groups and to the service desk manager in case of complaints and compliments. Service desk analysts also use this procedure when they handle requests that are rejected by group coordinators. Group coordinators use this procedure after requests are assigned to their groups. Group coordinators use this procedure to assign these requests to a specific change coordinator or specialist for resolution.

  • Compliment and Complaint Handling is used by the service desk manager when the service desk manager satisfactorily completes requests of the Complaint or Compliment categories.

  • Request Tracking is used by service desk analysts when they track the progress of requests.

  • Request Resolution by Specialist is used by specialists to complete assigned requests.

  • Request Fulfillment Handling is used by service desk analysts or specialists to complete requests that can be handled by using a predefined workflow.

  • Incident Escalation Handling is used by service owners when an incident is escalated. The service owner of the affected service uses this procedure to determine how the incident can be efficiently resolved.

  • Request Closure is used by service desk analysts when they resolve and close completed requests.
