Work Instructions for Service Owners

7.4.1

When it is clear that the resolution of the incident is going to take a long time and that the recovery of the affected service at its continuity site is the most efficient and reliable way to resolve the incident, continue with 7.5.1. If this is not the case, go to 7.6.1.

Note:

The recovery of the service at its continuity site should not be considered when it is possible to resolve the incident by:

  • Improving security measures (for example, updating virus definitions, applying a security patch, and so on..)

  • Restoring a backup or a previous software release at the production site (for example, in case of data corruption or an application change implementation that was unsuccessful)

  • Replacing hardware that has failed to continue service delivery from the production site
