Work Instructions for Service Desk Analysts

1.14.1

If the request type of the request is Incident, find the open requests for the same service and use the information from the user to determine the extent to which the service is impacted. Select the appropriate impact level in the Impact field. The following options are available:

  • Low

-

Service degraded for one user

  • Medium

-

Service down for one user or degraded for several users

  • High

-

Service down for several users

  • Major

-

Critical or core service down for several users. This impact level might not be available out-of-the-box.

Select the appropriate urgency level in the Urgency field. The following options are available:

  • Low

-

Responding using standard operating procedures and as time allows

  • Medium

-

Responding using standard procedures and operating within normal supervisory management structures

  • High

-

Immediate response required


Note:

If the request type is incident, do not select the None - No Degradation of Service impact level.

This impact level might not be available by default.

Note:

If the request type is not incident, set the impact level to None - No Degradation of Service.

This impact level might not be available by default.

Note:

Service management automatically determines the appropriate priority for requests by using the standard Priority Matrix or the customer's SLA. After determining the appropriate priority for the request, service management uses the priority to calculate the due date and time for the request.
