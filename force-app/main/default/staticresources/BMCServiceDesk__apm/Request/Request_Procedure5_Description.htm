Procedure 5: Request Resolution by Specialist


After a request is assigned to a specialist of a group, the specialist reviews the request information and determines how to resolve the request.

If request fulfillment can handle a request, the specialist coordinates implementing the request.

The specialist escalates the request to the service owner when the request concerns an incident that cannot be resolved without Change Management because the resolution causes the following:

  • Service to become unavailable or degraded during service hours
  • Functionality of a service to change
  • CMDB to require an update

A group coordinator cannot determine if Change Management is required to resolve incidents because a specialist must first investigate the incident.

If Change Management is not required and the request is an incident that occurred because a supporting service failed, the specialist registers a related request and waits for the request to complete before continuing to resolve the request.

If the specialist determines that another specialist must resolve the request, the specialist notifies the appropriate group coordinator. Otherwise, the specialist resolves the request and updates the request.
