Process description


The Change Management process consists of eight procedures; six for implementing planned changes, and two for implementing emergency changes.

  • Change Registration is used by change coordinators when they are managing change requests.

  • Change Planning is used by change coordinators and specialists to prepare implementation plans for planned changes.

  • Change Approval is used by the change manager and change approvers (customer representatives and service owners) to approve planned changes.

  • Infrastructure Change Implementation is used by specialists to implement infrastructure changes.

  • Application Change Implementation is used by specialists, release administrators, customer representatives and change coordinators to implement application changes.

  • Planned Change Closure is used by specialists when they perform a final production test after a change has been implemented and by change coordinators when they close out a change.

  • Emergency Change Implementation is used by specialists and release administrators to implement changes to resolve incidents.

  • Emergency Change Closure is used by specialists and change coordinators to complete and close emergency changes.
