Work Instructions for Change Coordinators

6.9.1

After work on the change is completed, set the Status field of any task that does not need to be completed (for example, the implementation of the change was rejected by its approvers) to Cancelled. In the Resolution field, specify why the task is cancelled.

6.9.2

For all tasks that are linked to the change request, verify that the tasks have reached their final status (Failed, Completed, or Cancelled). Then, in the Closure Category field of the change, select the appropriate closure category.

Note:

You can set the change request to be completed only when all tasks related to the change request have reached their final status (Failed, Cancelled, or Completed).

6.9.3

Set the Status field of the change request to Completed.

6.9.4

Enter the actual end date of the implementation In the Actual End Date field.
