Work Instructions for Change Coordinators

1.2.1

Find the open changes for the service that is to be changed.

1.2.2

Determine whether a change has already been registered for a change request that includes the same requirements. If such a change exists, continue with 1.3.1.

1.2.3

When managing a change request that asks for the prevention or fix of a problem, determine whether a change has already been registered to which you can add the change requirements of the problem. You can do this under the following conditions:

The functionality of the service does not become different when the change requirements of the problem are fulfilled.

No additional funding is required to fulfill the change requirements of the problem.

The Status field of the existing change has not reached the To Be Approved level.

If adding the problem to an existing change is possible and efficient, continue with 1.3.1.

Note:

Combining change requests from Problem Management can save time, for example, when multiple problems have been passed to Change Management for fixing issues in an application. Linking the change requests to a single change ensures that only one change must be approved and only one new release of the application is tested and transferred.

1.2.4

For any other kind of change request, go to 1.4.1.
