Work Instructions for Change Coordinators

1.6.1

Inform the requester that the change request conflicts with an internal standard or policy.

1.6.2

If the change request was received from Request Management as a request, in the Resolution field, specify which internal standard or policy the support request conflicts with. Set the Closure Category field of the request to Unable to solve or in conflict with standards or policy, and set the Status field to Completed. Setting the Status field to Completed ensures that the service desk informs the person whose record is linked to the request in the Client ID field.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent updating the request.

1.6.3

If the change request was received from Problem Management as a problem, add a Notes action saying which internal standard or policy conflicts with the proposed structural solution. Set the Status field of the problem to Rejected so that the problem is returned to its problem coordinator.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent updating the problem.
