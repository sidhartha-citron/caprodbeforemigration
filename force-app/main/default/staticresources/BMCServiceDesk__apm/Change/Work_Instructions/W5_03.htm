Work Instructions for Customer Representatives

5.3.1

When the Status field of your task for testing the new release changes to Assigned, open the task and read the instructions in the Description field and Notes actions.

5.3.2

If you cannot start working on the task, set the Status field of the task to Accepted.

Note:

If the task must be assigned to another person, ask the change coordinator to change its assignment. To do so, set the Status field of the task to Rejected, and add a Notes action saying why it has been rejected.

5.3.3

When you are ready to test the new release, set the Status field of the task to In Progress, and perform the tests in the test environment.

5.3.4

Describe tests and their results in a document and attach this document to the task.

Note:

If you cannot continue working on the task (for example, it is temporarily impossible to access the test environment), set its Status field to Waiting for…, and add a Notes action to specify what you are waiting for and when you expect to complete the tests.

5.3.5

When you finish testing of the new release and if the release has passed all tests, set the Status field of the task to Completed.

If the new release did not pass all tests, set the Status field of the task to Failed, and in the Resolution field specify which tests failed.

5.3.6

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent completing the task.
