Work Instructions for Change Coordinators

1.12.1

From the Remedyforce Console tab, click View > Change Requests. To open a new change request, click New.

1.12.2

Ensure that you are selected in the Initiator ID field of the change request.

1.12.3

From the Template field, select the change template that helps you register the requested change. Using a template ensures that the fields of the change request are populated with values from the selected change template and that the necessary tasks are created and linked to the change when it is saved as specified in the template.

1.12.4

In the Category field, select Non-Standard - Approved Change Template not Available option. Also enter the main reason of implementing the change in the Reasons field of the change.

1.12.5

In the Description field, enter a short description of the change.

1.12.6

In the Change Type field, select the appropriate type.

1.12.7

In the Change Category field, select the appropriate category.

1.12.8

Save the change request to generate its tasks and relate it to a request, problem, release, or attachment.

1.12.9

Add a Notes action to summarize the requirements of the change.

1.12.10

In the Linked Services and CIs section, select the services that are affected if the change is implemented.

1.12.11

If the change request was received from Request Management as a request, open the request and link the change request in the Linked Change Requests section of the request. Add the following note in the Notes action in the request: Responsibility for coordination has been accepted by Change Management.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the request.

Set the Status field of the request to Change Pending.

1.12.12

If the change request was received from Problem Management as a problem, open the problem and link the change request in the Linked Change Requests section of the problem. Add the following note in the Notes action in the problem: Responsibility for coordination has been accepted by Change Management.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing of the problem.

Set the Status field of the problem to Change Pending.

1.12.13

If the change request originated from Release Management, link the release in the Linked Releases section of the change request. Attach the requirements documentation in the Notes & Attachments section of the change request.
