Work Instructions for Change Coordinators

1.13.1

From the Remedyforce Console tab, click View > Change Requests. To open a new change request, click New.

1.13.2

Ensure that you are selected in the Initiator ID field of the change request.

1.13.3

In the Template field, select the change template for the implementation of the requested change. Using a template ensures that the fields of the change are populated with values from the selected change template. It also ensures that the necessary tasks are created and linked to the change request when it is saved.

1.13.4

In the Description field, provide a short description of the change.

1.13.5

In the Change Type field, select the appropriate type.

1.13.6

In the Change Category field, select the appropriate category.

1.13.7

Save the change request to generate its tasks and to allow it to be related to a request, problem, or attachment.

1.13.8

Add a Notes action to summarize the requirements of the change.

1.13.9

In the Linked Services and CIs section, select the service that is affected if the change is implemented.

1.13.10

If the change request was received from Request Management as a request, open the request and link the change in the Linked Change Requests section of the request. Add the following note in the Notes action in the request: Responsibility for coordination has been accepted by Change Management.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the request.

Set the Status field of the request to Change Pending.

1.13.11

If the change request was received from Problem Management as a problem, open the problem and link the change in the Linked Change Requests section of the problem. Add the following note in the Notes action in the problem: Responsibility for coordination has been accepted by Change Management.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the problem.

Set the Status field of the problem to Change Pending.
