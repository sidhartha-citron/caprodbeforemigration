Work Instructions for Specialists

8.4.1

In the Configuration Item field, select the CI that caused the incident.

8.4.2

In the Resolution field, describe how the emergency change was implemented to resolve the support request.

8.4.3

If the root cause of the incident has been removed by the implementation of the emergency change, select Resolved in the Closure Category field. However, if the change implementation has not removed the root cause (for example, the incident has been resolved by rebooting a server), select Workaround in the Closure Category field.

8.4.4

If you think that similar incidents are likely to occur if the root cause is not removed, verify whether the problem that caused the incident has already been registered. If the problem has already been registered, link the request that you just resolved to this problem by selecting the existing problem in the Linked Problems section of the request. If the problem that caused the incident has not been registered, notify the problem coordinator (by email message or telephone) of the service that is affected. Provide the problem coordinator with the support request number under which the incident is registered and explain why you think that the incident can reoccur.

8.4.5

Set the Status field of the request to Completed, and perform a final review to ensure that the request is filled correctly.

8.4.6

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent resolving the support request, since the last time you entered a duration value.
