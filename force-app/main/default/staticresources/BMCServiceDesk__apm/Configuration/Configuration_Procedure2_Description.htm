Procedure 2, CI Receipt


If the task from Change Management or Request Management does not add new CIs to the CMDB, the configuration administrator goes directly to Procedure 4, CI Update.

On the other hand, if the task requests the registration of one or more CIs for which a formal purchase requisition was not submitted (for example, in the case of internally developed software), or if one or more new CIs have been ordered, the configuration administrator first checks the CIs after they have been delivered. The configuration administrator does this to ensure that the correct hardware, software, and software license certificates have been received, that no items are missing, and that the CIs are not damaged. If the delivery is not whole, the configuration administrator informs the supplier and subsequently informs the change coordinator of the delay by updating the task.

Conversely, if the CIs have been received in good condition, the configuration administrator checks the service management application to see if the contact details of the supplier are up to date. If the supplier of the CIs has not yet been registered, or if its contact details are no longer up to date, the configuration administrator ensures that the supplier information is registered or updated in Procedure 3, Supplier Information Maintenance.

After updating or registering the supplier information, the configuration administrator updates the status of the new CIs and adds any missing information (for example, the serial number) if the CIs are already registered in Procedure 1, CI Requisition. If a formal purchase requisition is not submitted for the new CIs; however, the configuration administrator registers the new CIs in the CMDB. The configuration administrator ensures that all attributes of the new CIs and the links between the new CIs and other CIs, services and users are correctly registered.

Having ensured that the new CIs are properly registered in the CMDB, the configuration administrator attaches a label with the CI code to each new CI when applicable.
