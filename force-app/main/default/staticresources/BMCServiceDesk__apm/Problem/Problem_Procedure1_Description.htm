Procedure 1, Request Review


The problem coordinator uses request information to identify problems in the assigned services.The problem coordinator can identify problems by using the search and reporting functionality of the service management application. A specialist can inform the problem coordinator about certain requests that, in the opinion of the specialist, represent a problem.

After the identification of a problem, the problem coordinator registers the problem in the service management application. The problem coordinator subsequently selects the appropriate impact and urgency level and links the requests that were caused by the problem.

The problem coordinator assigns the problem to the most appropriate specialist (in terms of skills, availability and access rights) for analysis.

The service desk can be automatically informed through email that a new problem has been identified after the problem has been saved.

When the problem coordinator finds requests that have not yet been linked to a problem, but which were caused by a previously identified problem, the problem coordinator links these requests to the problem.
