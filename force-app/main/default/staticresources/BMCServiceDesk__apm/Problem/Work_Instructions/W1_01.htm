Work Instructions for Problem Coordinators

1.1.1

At the beginning of the periodic request review, generate an overview of request information to identify the problems.

Create the Requests for Problem Management Review QuickView. In the Service field, select the service for which you are the problem coordinator. You get a list of requests that are linked to this service and meet the following conditions:

 

  • The requests are not related to a problem.

  • The Impact field of these requests is set to High, Low, or Medium.

  • The Closure Category field is set to Resolved, Unable to Reproduce, or Workaround.

1.1.2

If a specialist informs you about a new, potential problem, create an overview that includes the support requests that the specialist referred or any similar requests.
