Work Instructions for Problem Coordinators

3.4.1

Add a Notes action to explain why the analysis is not acceptable.

3.4.2

Determine which specialist is best suited (in terms of skills, availability, and access rights) to perform a better analysis. This could be the same specialist who performed the previous analysis of the problem. Use the Change option beside the Owner field to select specialist.

Note:

If the problem is linked to a CI that is part of a service instance that supports the service for which you act as the problem coordinator, contact the problem coordinator of the supporting service. Discuss the problem and if the problem coordinator agrees with your reasoning, assign the problem to the other problem coordinator by changing the name in the Owner field. In the Information field, summarize why you passed the responsibility of the problem to the other problem coordinator.

3.4.3

In the Notes action, Duration (HH:MM) field, enter the time that you spent on reviewing and assigning the problem.

3.4.4

Set the Status field to Assigned.
