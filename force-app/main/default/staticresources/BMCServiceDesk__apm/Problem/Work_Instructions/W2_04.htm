Work Instructions for Specialists

2.4.1

To find the root cause of the problem, review log files or search the internet for information.

2.4.2

When the root cause has been found, add this information in a Notes action and set the Status field to Known Error. Alternatively, add a Note action if the root cause is not found.

Note:

If for any reason you are not able to progress with the analysis (for example, because you are waiting for information from a supplier), add a Notes action to specify what you are waiting for and set the Status field to Waiting for.

2.4.3

Link the CI that is the root cause in the Linked Services and Configuration Items section. If the root cause was not found and it is not known which CI has caused the problem, select the special CI with the code N/A.

2.4.4

If the root cause has been found, review the knowledge article referenced in the Workaround field. If it is possible to propose a better workaround, add the information in the knowledge article.

2.4.5

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent on the problem since you last updated it.
