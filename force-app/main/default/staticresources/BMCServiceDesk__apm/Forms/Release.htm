Release


The following table lists the fields of the Release form and provides utilization guidelines for each field.

Field

Utilization

Release #

This field contains a unique release number. This number is generated by the application.

Release Details

Template

Use this field to select the template that is used for the release.

Release Title

Use this field to enter the release title.

Description

Use this field to enter a short description that summarizes the main reason for implementing the release. You can enter the reason for creating the release, such as Problem Prevention or Fix, New Business Requirement, or New Legal Requirement.

Category

Use this field to select the release category.

Release Type

Use this field to select the release type from the following options:

  • Major

  • Minor

  • Emergency

Subtype

Use this field to select the subtype from the following options:

  • Backlog

  • Delta

  • Full

  • Package

Deployment Type

Use this field to select the deployment type from the following options:

  • Phased

  • Non-phased

Testing completed

Use this field to set the date and time at which the testing was completed.

Status and Priority

Impact

Use this field to select the impact from the following options:

  • Low - Service Degraded for 1 User

  • Medium - Service Down for 1 User or Degraded for Several

  • High - Service Down for Several Users

Urgency

Use this field to select the urgency from the following options:

  • Low - Responding using standard operating procedures and as time allows

  • Medium - Responding using standard procedures and operating within normal supervisory management structures

  • High - Immediate response required

Priority

The application selects the priority after the impact and urgency levels have been selected.

Status

Use this field to select the appropriate status for the release from the following options:

Opened

The initial status of the release.

Registered

The release has been registered by its coordinator.

To Be Approved

A business case is being prepared or approved for the release.

Approved

The business case of the release has been approved.

In Progress

The requirements for the implementation of the release have been sent to the change coordinators.

Plan

The release is in the planning phase.

Deploy

The release is in the deployment phase.

Completed

The work on implementing the release has come to an end because of the reason specified in the Closure Category field.

Closed

The release implementation has been reviewed during its post-implementation meeting.

Closure Category

Use this field to select the appropriate closure category for the release after the release has been completed. The following options are available:

  • Business Case Rejected

  • Rejected

  • Rolled Back

  • Partially Implemented

  • Implemented

Inactive

Select this field if the release record is no longer valid but the record has to be retained for compliance with the records retention policy. For example, if a client has left the account, all the client's release records would be invalid.

Release Date and Time Details

Due Date

Use this field to select the date and time at which the changes that are linked with the release, except the change to which the change requests for the release are linked, must be completed.

Start Implementation

Use this field to select the date and time at which the implementation of the release must start.

End Implementation

Use this field to select the date and time at which the implementation of the release must be completed.

Start Deployment

Use this field to select the date and time at which the deployment of the release must start.

End Deployment

Use this field to select the date and time at which the deployment of the release must be completed.

Completion Date

Use this field to set the date and time at which the release must be completed.

Actual Implementation Start

Use this field to select the actual date and time at which the implementation of the release started.

Actual Implementation End

Use this field to select the actual date and time at which the implementation of the release was completed.

Actual Deployment Start

Use this field to select the actual date and time at which the deployment of the release started.

Actual Deployment End

Use this field to select the actual date and time at which the deployment of the release was completed.

Business and Financial Details

Budget Available

Use this field to specify whether budget for this release is available.

Business Justification

Use this field to select the appropriate business justification for the release. The following options are available:

  • Corporate Strategy

  • Business Unit Strategy

  • Maintenance

  • Defect

  • Upgrade

  • Enhancement

  • Customer Commitment

  • Sarbanes-Oxley

  • Other

Comments

Use this field to enter the business and financial comments.

Risk Rating

Use this field to select the appropriate risk rating for the release from the following options:

  • Low

  • Medium

  • High

Required Contracts Verified

Use this field to indicate if the required contracts are verified.

Required Licenses Verified

Use this field to specify if the required licenses are verified.

Required CIs Available

Use this field to specify if the required configuration items (CIs) are available.

Staff Assignment Details

Owner

This field displays the queue or staff to whom the release is assigned.

Release Coordinator

Use this field to select yourself as the release coordinator responsible for the release.

Related Lists

Action History

This section lists the history actions that have been registered for the release.

Linked Tasks

This section lists the tasks that are related to the release. Use this section to link tasks to the release.

Linked Problems

This section lists the problems that are related to the release. Use this section to link problems to the release.

Linked Incidents

This section lists the incidents that are related to the release. Use this section to link requests to the release.

Approval History

This section lists the approval records that are created for the release.

Linked Change Requests

This section lists the change requests that are related to the release. Use this section to link change requests to the release.

Linked Services and CIs

This section lists the services and CIs that are related to the release. Use this section to link the CIs and services that will be affected by the execution of the release.

Notes & Attachments

This section lists the documents that are related to the release. Use this section to attach files that provide additional information about the completion of the release.

Release History

This section lists the modifications that have been made to the release information.
