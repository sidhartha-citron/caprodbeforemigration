Request


The following table lists the fields of the Request form and provides utilization guidelines for each field.

Field

Utilization

Incident #

This field contains a unique request number. This number is generated by the application.

Client Details

Client ID

Use this field to select the client who submitted the request.

Client Type

Use this field to select the client type.

Preferred Contact Method

Use this field to select the method by which the customer prefers to be contacted. The following options are available:

  • Email

  • Mobile Phone

  • Work Phone

Account

Use this field to select the account of the client.

Call #

Use this field to enter the number of times the client of the request has called the service desk.

Incident Details

Incident Type

Use this field to select the request type from the following options:

  • Request for Change

  • Incident

  • Service Request

  • Complaint

  • Compliment

  • Information Request

  • Out of Scope

Incident Source

Use this field to record how the support request is submitted. The following options are available:

  • Email message

  • Email other

  • Event

  • Fax, Letter, Paper Form, or Memo

  • Mail Listen

  • Own Observation

  • Personal visit

  • Phone

  • Self Service

  • Walk up

  • Web Request Form

Template

Use this field to select a template that contains the information that helps you to register the request.

Category

Use this field to select the request category.

Description

Use this field to enter a short description of the request.

Examples for incidents:

  • Slow response time on <Service>

  • Error message using <Service>

  • Cannot log onto (or access) <Server or Service>

  • Job <Job name> failed with <Abend code>

  • <Customer code> router in <City> down

Example for requests:

How to <Requested information>

Examples for change requests:

  • Install <Software & Version> on <Workstation or Server Code>

  • Apply upgrade from <Current Software & Version> to <Requested Software & Version>

Examples for complaints and compliments:

  • Request <Request Number> not resolved in time

  • Change <Change Number> was implemented without approval

Request Definition

Use this field to select the request definition that you want to use to create a request.

Broadcast

Use this field to select an existing broadcast that is already registered and concerns the same outage. The current request inherits all the relevant information from the broadcast.

Resolution

Use this field to specify what has been done to complete the request. Also, specify detailed solution instructions (for example, if the client needs to adjust a setting) to provide the answer of request fulfillment. If it is not possible to provide a solution or workaround, describe why the request cannot be resolved.

First call resolution

Use this field to record if the support request is resolved in the first call made to the service desk.

Inactive

Use this field if the request is no longer valid but the record has to be retained for compliance with the records retention policy.

Status and Priority Details

Impact

Use this field to select the extent to which the service is impacted. The following options are available:

  • None - No Degradation of Service

  • Low - Service Degraded for 1 User

  • Medium - Service Down for 1 User or Degraded for Several

  • High - Service Down for Several Users

  • Major Incident

Note that a service is degraded when some of its functionalities do not function, or when the response time of the service is slow. A service is down when none of its functionality can be accessed.

Urgency

Use this field to select the urgency from the following options:

  • Low - Respond using standard operating procedures and as time allows

  • Medium - Respond using standard procedures and operating within normal supervisory management structures

  • High - Immediate response required

Priority

The application selects the priority after the impact and urgency levels have been selected.

Status

Use this field to select the status for the request from the following options:

Opened

The initial status of the request.

Rejected

The request must be assigned to another queue or staff.

Assigned

The responsibility for the resolution of the request has been assigned to a specific queue or staff.

Accepted

The request will be resolved when the assigned staff is ready to start working on it.

In Progress

The request is currently being resolved.

Waiting for…

The request is waiting for a task to be completed or approved, or for more information.

Waiting for customer

It is temporarily not possible to make further progress in the resolution of the request.

Pending

The request is pending.

Change Pending

A change request has been registered for implementing the requested change.

Completed

The work on the resolution of the request has ended because of the reason specified in the Closure Category field.

ClosedMail

The client has been informed through an email about how the request has been completed. The email mentions that the solution of the request must be verified.

Closed/No contact

The client has not responded and the request is closed.

Closed

The client has not responded to the "ClosedMail" email notification within 28 days. This value cannot be selected manually but is automatically selected by the service management application

Closure Category

Use this field to select the appropriate closure category for the request after the request has been completed. The following options are available:

  • Resolved

  • Unable to reproduce

  • Unable to Solve or in conflict with standards or policy

  • Withdrawn by requester

  • Workaround

Date and Time Details

Opened Date

This field is set to the date and time at which the request is created.

Responded Date

This field calculates the response date and time of the request as stated in the service level agreement (SLA).

Due Date

This field is set to the date and time at which the request must be completed, after the priority has been set. If an SLA is attached to the request, the due date is based upon the SLA.

Closed Date

This field is set to the date and time at which the support request status is set to Closed.

Service and CIs Details

Service

Use this field to select the service for which the client has submitted the request.
If the request does not concern a specific service or if it concerns a service that is not yet provided, select the special service N/A - Service does not (yet) exist or is out of scope.

Service Offering

Use this field to link a service offering of the associated service.

Configuration Item

Use this field to select the configuration item (CI) that is causing the service disruption (for a request for incident resolution), or that the client wants to know something about (in case of request fulfillment), or that the client wants to have changed (in case of a request for change).

If the CI has not yet been registered in the configuration management database (CMDB), select the special CI with the code NORECORD.

If the support request does not concern a specific CI (for example, in case of a complaint or compliment), select the special CI with the code N/A.

Outage Start

Use this field to enter the exact date and time at which the service became unavailable due to the incident described in this request.

Outage End

Use this field to enter the exact date and time at which the service became available again after the incident described in this request was resolved.

Assignment Details

Owner

This field displays the staff or queue to whom the request is assigned.

Related Lists

Action History

This section lists the history actions that have been registered for the request.

Linked Tasks

This section lists the tasks that have been registered for the request and is used for request fulfillment. Use this section to link tasks to the request.

Approval History

This section lists the approval history that has been registered for the request.

CIs

This section lists the CIs that are related to the request. Use this section to link CIs to the request.

Linked Change Requests

This section lists the change requests that are related to the request. Use this section to link change requests to the request.

Linked Problems

This section lists the problems that are related to the request. Use this section to link problems to the request.

Notes & Attachments

Use this section to attach files that provide additional information for the resolution of the request.

Incident History

This section lists the modifications that have been made to the request information.

Linked Releases

This section lists the releases that have been registered for the problem. Use this section to link releases to the request.
