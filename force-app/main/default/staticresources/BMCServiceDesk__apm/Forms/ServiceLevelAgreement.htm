Service Level Agreement


The following table lists the fields of the Service Level Agreement form and provides utilization guidelines for each field.

Field

Utilization

Agreement Title

Use this field to enter the name of the service level agreement (SLA). For example, Human Resource Management P21 (SAP) for UKPLANT.

Description

Use this field to enter the description of the SLA.

General Information

Agreement Type

Use this field to select the agreement type from the following options:

  • Service Level Agreement

  • Operational Level Agreement

  • Underpinning Contract

Service

Use this field to select the service for which you are creating the SLA.

Account

Use this field to select the organizations that are covered by the SLA. The following options are available:

  • Agreement applies to all accounts

  • Agreement applies to selected accounts

If you select Agreement applies to selected accounts, select the appropriate accounts.

Effective From

Use this field to specify the date on which the SLA becomes effective.

Effective Until

Use this field to specify the date before which the SLA must be renewed so that it does not expire.

Scheduled Review

Use this field to specify the date on which the SLA need to be reviewed.

Support Hours

Use this field to select the appropriate support hours for the SLA. These selected service hours are used for response time and resolution time calculations.

Agreement Owner

Use this field to select the user who acts as the service level manager.

Service Targets

Service Target Title

Use this field to enter service target title.

Description

Use this field to enter the description of the service target.

Service Target

Applies To

Use this field to select the module to which the target applies. The following options are available:

  • Incident

  • Change Request

Target Type

Use this field to select the target type from the following options:

  • Resolution Time

  • Response Time

Support Hours

Use this field to select the appropriate support hours for the service target. These selected service hours are used for the response time and resolution time calculations.

Target

Use this field to enter the target time of completing the service in days, hours, and minutes. Alternatively, use the Business Days field to set the target time of completing the service in business days.

Qualification

Conditions

Use this field to select the fields and values on which you want to set the criterion.

Measurement

Measurement

Use this field to select the status values on which the SLA clock must start, pause, and stop.

Milestones and Actions

Milestone

Use this field to select when an automated email notification must be sent. Select a milestone from the following options:

  • Percentage of Time Elapsed

  • Percentage of Time Remaining

  • Service Target is Canceled

  • Service Target is Met

  • Service Target is Missed

  • Service Target Warning Occurs

  • Time Elapsed

  • Time Remaining

Action

Use this field to select the email template that you want to send when a milestone is met. The following options are available:

  • Incident Percentage of Time Elapsed

  • Incident Percentage of Time Remaining

  • Incident Service Target is cancelled

  • Incident Service Target is met

  • Incident Service Target is missed

  • Incident Service Target warning

  • Incident time elapsed

  • Incident time remaining

  • Service Request Percentage of time remaining

  • Service Request Service Target is cancelled

  • Incident Service Target is cancelled

  • Service Request Service target is met

  • Service Request Service target is missed

  • Service Request Service target warning

  • Service Request time elapsed

  • Service Request time remaining

To

Use this field to specify which email addresses must receive the email message.

Cc

Use this field to specify which email addresses must be in the CC list of the email message.

Related Lists or Supporting Information

History

This tab lists the history actions that are registered for the agreement.

Incidents

This section lists the requests that are related to the SLA. Use this tab to link requests to the service.

Change Requests

This section lists the change requests that are related to the SLA. Use this tab to link change requests to the service.

Problems

This section lists the problems that are related to the SLA. Use this tab to attach problems to the SLA.

Documentation

This section lists the documents that are related to the SLA. Use this tab to attach any files that provide additional information about the completion of the SLA.

Auditing

This tab lists the modifications that have been made to the SLA information.
