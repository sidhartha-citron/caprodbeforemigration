Change Request


The following table lists the fields of the Change Request form and provides utilization guidelines for each field.

Field

Utilization

Change #

This field contains a unique change number.
This number is generated by the application after you save a change request.

Initiator Details

Initiator ID

Use this field to select yourself as the coordinator of the change.

Change Details

Template

Use this field to select a template that helps you to register the change request and its tasks.

Note that the tasks that are linked to the change request after the template has been applied are not shown until the change request has been saved.

Category

Use this field to select the correct category for the change request.

Description

Use this field to enter a short description about the change request.

Reasons

Use this field to enter the reason for the change.

Rollout Plan

Use this field to enter the procedures for implementing the change.

Back-out Plan

Use this field to enter the back-out plan for restoring the configuration item to its initial state before the change is implemented.

Approved

Use this field to specify if you want to create a pre-approved change request to ensure quick implementation.

Review Comments

Use this field to enter any comments that you want to track for the current change based on your review. These notes might include reasons why the change failed and how the failure might be avoided in future.

Change Type

Use this field to select the change type from the following options:

  • Emergency

  • Normal

  • Standard

Change Category

Use this field to assign the change category from the following options:

  • Major

  • Minor

  • Significant

Impact

Use this field to select the extent to which the service is impacted. The following options are available:

  • Low - Service Degraded for one User

  • Medium - Service Down for one User or Degraded for Several

  • High - Service Down for Several Users

Urgency

Use this field to select the urgency from the following options:

  • Low - Respond using standard operating procedures and as time allows

  • Medium - Respond using standard procedures and operating within normal supervisory management structures

  • High - Immediate response required

Priority

The application assigns priority after the impact and urgency levels have been selected.

Status

Use this field to select the appropriate status for the change request. The following options are available:

Opened

The initial status of the change request.

Registered

The change request is not yet ready for risk and impact analysis.

Risk & Impact

The risk and impact analysis is being conducted for the change request and the implementation plan is being prepared.

To Be Approved

The implementation plan has been prepared and needs to be reviewed by the change manager, after which approval needs to be requested.

Approval Requested

Approval has been requested for the change by the change manager.

Pending Approval

Approval is pending.

Approved

The change request has been approved.

In Progress

The change request is currently being implemented.

Rejected

The change request is rejected.

Completed

No further action can be taken for this change because it has been implemented, rolled back, or rejected by its approvers, as specified in the Closure Category field.

Closed

The change request is closed.

Closure Category

Use this field to select the appropriate closure category for the change request after the change request has been completed. The following options are available:

  • Withdrawn by Requester

  • Rejected

  • Rolled Back

  • Partially Implemented

  • Implemented

Inactive

Select this field if the change is no longer valid but the record has to be retained for compliance with the records retention policy.

Change Date and Time Details

Opened Date

This field is set to the date and time at which the change request is created.

Responded Date

This field is set to the date and time at which the response is received. This field records the date and time at which the staff communicated with the initiator about the change.

Review Date

Use this field to select the date on which the change request is scheduled to be reviewed, after the change request is closed.

Closed Date

Use this field to select the date on which the change request is closed.

Scheduled Start

Use this field to select the date on which the change request is scheduled to be implemented.

Scheduled End

Use this field to select the date on which the change request must be completed.

Actual Start Date

Use this field to select the date on which you start implementing the change.

Actual End Date

Use this field to select the actual end date of implementation.

Assignment Details

Owner

Use this field to assign change request to a queue or staff.

Related Lists

Action History

Use this section to add history actions that have been registered for the change request.

Linked Problems

Use this section to add and link related problems. The problems for which the change request has been prepared are listed in this section.

Linked Services and CIs

This section lists the configuration items (CIs) and services that are related to the change request. Use this section to link the services and CIs that are related to the change.

Change Assessments

This section lists the change assessments that are related to the change request. Use this section to create and link the change assessments that you want to register for the change.

Approval History

Use this list to approve a change request or to view the approval history that is registered for the change request.

Linked Tasks

This section lists the tasks that are related to the change request. Use this section to create and link tasks that have been registered for the change request.

Linked Incidents

This section lists the incidents that are related to the change request. Use this section to link incidents for which the change request has been prepared.

Notes & Attachments

This section lists the documents that are related to the change request. Use this section to attach any files that provide more information for the risk and impact analysis, approval, implementation, and rollback of the change request.

Change Request Audit

This section lists the modifications that have been made to the change information.

Linked Releases

This section lists the releases that are related to the change request. Use this section to link the releases that have been registered for the change request.
