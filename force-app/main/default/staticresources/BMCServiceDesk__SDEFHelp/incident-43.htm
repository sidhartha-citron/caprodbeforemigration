The steps for linking an object to another object are same on all the object forms that include incidents, tasks, problems, broadcasts, releases, and change requests.

This procedure uses the following example: Your Service Desk receives a phone call from a customer saying that the customer's laptop has crashed. An incident is logged for the customer. With investigation, the staff member determines that the RAM of the laptop has crashed. In this case, the RAM must be changed, so you must link a change request (to change the RAM) with the incident.

To link a record to the records of other objects

  1. Click the tab of the object to which you want to link another object.

    2. For the example, click the Remedyforce Console, Incident Console or Incidents tab.

  2. Click the record to which you want to link another object record.

    3. For the example, select the incident that was logged by the customer whose laptop crashed.

  3. If you are using the console layout UI, click Record Details.
  4. To link existing records to this record, in the Linked <object that you want to link> section, click Select and Link <Object 1> to <Object 2>.

    5. For the example, in the Linked Change Requests section, click Select and Link Change Request to Incident.

    Note: To create a new record and link it to the object to which you want to link another object (for the example, to create a change request and link it to the incident), click Create and Link <Object 1> to <Object 2>. In the button name, <Object> can be replaced with Broadcast, Change Request, Incident, Problem, and Release.

  5. From the lookup window that is displayed, select the check box for the required object record that you want to link.

    6. For the example, select the change request to change the RAM for the laptop.

    To select multiple records, select the check boxes for all the required records. Contact your system administrator to configure the required fields that you want to see in the lookup window.

    When you are linking multiple services or CIs to a record, note the following important points:

    • If you select the check box next to the Instance Name column, all configuration items are selected.
    • Clicking the Instance Name column header acts as a toggle to sort the column in ascending or descending order.
    • You can also search for a particular configuration item by entering the search criteria in the Search field and clicking Go!.
    • Clicking the Instance Name of a configuration item opens the configuration item in the Instance Editor form of the Remedyforce CMDB tab.
    • The lower section of the lookup window contains the First, Previous, Next, and Last buttons for navigating the pages. You can navigate through the pages and view all records. The current page number and the total number of pages are displayed in the lower right section of the lookup window. For example, if you are on the second page of a total of 10 pages, the following is displayed in the lower section:

      • Page 2 of 10

  6. (Optional) From the Records per page list in the lower-left section of the lookup window, select the number of records to be displayed per page.

    7. By default, 50 records are displayed per page.

  7. Click Link.

    8. Note: If you create a new incident from the Linked Incidents related list of an existing incident by using the New button, the new incident is not linked to the existing incident. Contact your system administrator to add the Incident field to the page layout (if you using the Incidents tab) or to a field set (if you are using the Remedyforce Console tab).
