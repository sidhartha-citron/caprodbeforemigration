The following table lists errors or issues that might occur when you import data from BMC Atrium CMDB and the solution that you must apply to resolve the error or issue:

Error or issue
Solution
The job does not appear in Atrium Integrator.
Ensure that you have saved your KTR and KJB files in a folder in Atrium Integrator.
Error setting value #2 [ARDelta_1 Integer] on prepared statement (Integer)
This error is generated if multiple records are created in the BMC Remedy AR System NGIE:Delta form for the running transformation. Delete the additional ARDelta entries that are created for the error transformation file by deleting the records.
To find the duplicate records for a transformation file, open the NGIE:Delta form in BMC Remedy AR System. In the TransName field, enter the transformation file name, and click Search.
Unable to query Salesforce
Check your Salesforce organization credentials in the Salesforce Upsert and Salesforce Input(CMDB_Classes) steps.
Did not find Remedy Application Service password for server <server name> in UDM:RAppPassword Form on server <server name>
Check your Atrium Server Connection credentials in the ARInput step.
Failed in writeToSalesForce: java.lang.IllegalArgumentException: The char '0x0' after 'Print' is not a valid XML character
Some unicode characters that cannot be parsed by XML parser are present in the any of the mapped BMC Atrium CMDB fields. Either delete these characters in data or delete the mapping of such BMC Atrium CMDB fields.
‘Oracle Database Server 10g Release 2’ is not valid for the type xsd: <data type>
Check the data type of the mapped fields in the Salesforce Upsert step.
