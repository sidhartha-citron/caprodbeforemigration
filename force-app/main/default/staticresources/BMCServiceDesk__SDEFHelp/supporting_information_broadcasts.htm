You can send messages to your entire account, selected profiles within the account, and to clients. You can send messages about work in progress, system status, planned work, and so on. You can also view messages that were broadcasted to you from other groups in your account.

To create a broadcast in the Visualforce page layout UI

  1. In the BMC Remedyforce home page, click Workspaces.
  2. From the list of Workspaces, expand Incident Management.
  3. In the Incident Management list, click Broadcasts.
  4. In the broadcasts list view, click New.
  5. Type a unique alphanumeric identifier that you want to assign to the current broadcast in the Broadcast ID field.
  6. In the Message field, type a short description of the outage or issue.

    7. The content of the Message field is broadcasted to all users in the banner of the browser.

  7. To display the content of the Message field in the banner of the browser, select the Scroll Message check box.

    8. The broadcast message can take up to five minutes to begin scrolling because the broadcast scrolling mechanism refreshes itself after every five minutes. If multiple users have logged on to BMC Remedyforce, each user can view the new broadcast message depending on when their broadcast scrolling mechanism had last refreshed. For example, if the broadcast scrolling mechanism had refreshed two minutes back for a staff member, this staff member can begin to view the broadcast message in the next three minutes.

    Note: The broadcast message appears in the banner of the browser only if the system administrator has enabled it.

  8. From the Expiration Date field, select the date when you want to stop displaying the broadcast message.
  9. In the Description field, type a detailed description of the outage or issue.

    10. This description is appended to the Description field of incidents associated to the current broadcast.

  10. In the Resolution field, type the steps that have been taken to resolve the outage or issue.

    11. This resolution is appended to the Resolution field of incidents that are associated to the broadcast.

  11. In the Details section, enter the following information:
    • Category—Select the category that you want to assign to the current broadcast.
    • Status—Select the status that you want to assign to the current broadcast.

      • You can track the progress of the broadcast from its stages of opening to closure by using this field.

      The Broadcast Progression Bar in the header of the Broadcast form is updated depending on the stage that is linked to the status you have selected. For example, if the Working status is linked to the In Process stage and you select Working as the status of the broadcast, then the In Process stage is selected in the Incident Progression Bar.

    • Impact—Select the impact that you want to assign to the current broadcast.

      • The level of impact and the urgency are used to calculate the priority of the broadcast.

    • Urgency—Select the urgency that you want to assign to the current broadcast.

      • The urgency and the level of impact are used to calculate the priority of the broadcast.

    • Priority—When you select urgency and impact values and save the record, the Priority field is filled in. The priority is calculated depending on the urgency and impact values that you have selected.
    • Closed Date—When you are closing a record, this field is filled with the date and time at which the broadcast is closed.
  12. In the Display Broadcast section, enter the following information:
    1. To Everyone—Select this check box to display the broadcast to all staff members in BMC Remedyforce.
    2. To Profile—Select the profile of the users to whom you want to display the broadcast message.

      3. Note: To enable this field, you must clear the To Everyone check box.

    3. Display in Self Service—Select this check box to display the broadcast to all the users of BMC Remedyforce when they log on to Self Service.
    4. If you select the Display in Self Service check box, select the appropriate option from the following:

      5. To users in all accounts: To display the broadcast to all the users in all the accounts.

      To users in these accounts only: To display the broadcast to the users of the selected accounts only. A list of the selected accounts is displayed below this option. Click the button next to these accounts to open the Select Accounts for Self Service Broadcast Display window. In this window, select the required accounts in the Available Accounts list, and click the right arrow to move the selected accounts from the Available Accounts list to the Selected Accounts list.

      You can make the broadcast available to a large number of accounts at one time. You can also search for a particular account by entering the search criteria in the Search field and clicking Search. To clear the search criteria in the Search field and refresh the list of accounts in the Available Accounts list, click Undo. You can use the * wildcard character to match one or more characters at the end of your search criteria, but not the beginning or the middle of your search criteria. You can navigate to the next or previous page of accounts by using Next Page and Previous Page next to the Undo. After you select the required accounts, click OK.

      Note: To display a broadcast to non-portal users, ensure that the Display in Self Service check box, the To users in all accounts option are selected or the accounts of the non-portal users are added to the To users in these accounts only field.
      If no accounts are displayed in the Available Accounts list, ensure that the Customer Portal feature is enabled in Salesforce. For more information about enabling Customer Portal, see https://na12.salesforce.com/help/doc/user_ed.jsp?loc=help&target=customer_portal_setting_up.htm&section=CustomerPortal.

  13. Click Save.

    14. The Post Date field is filled with the current date and time. This field records the date and time when the broadcast is created.

    Note: When you open a new form to create a record and you save the record, you cannot navigate to the next or previous record by using the Next Record and Previous Record buttons in the upper right section of the form. These buttons are disabled in a new form.

  14. After you have created the broadcast, you can add supporting information for the broadcast.

    15. For more information, see Supporting information for broadcasts.

  15. (Optional) To designate the current record as inactive, select the Inactive check box.

    16. You might need to make a record inactive because the record is no longer valid, but must be retained for compliance with the records retention policy. For example, the issue or outage might be related to a router that no longer exists or has been removed from the network.

    Note: Records marked as inactive are not included in any search performed on the database.

Supporting information for broadcasts

The Supporting Information tabs at the lower section of the broadcast form allow you to:

    • Track the history of the broadcast
    • Add new incidents
    • View audit information about the broadcast

The Broadcast form contains the following Supporting Information tabs:

    • History tab—Allows you to track the history of all actions that have been performed on the broadcast by viewing the existing user-defined and system-defined actions. You can also add new user-defined actions.
    • Incidents tab—Allows you to add new incidents that must be resolved to close the broadcast.
    • Auditing tab—Displays the history of changes made to fields in the form. The BMC Remedyforce system administrator enables field history tracking for required fields.

The Supporting Information section is not loaded in a new form. When you save the record, the Show Supporting Information link appears. Click this link to view the Supporting Information section. You can configure the Supporting Information section to always display by selecting the Always show Supporting Information for existing records check box. After displaying the Supporting Information section, click the Hide link to hide the Supporting Information section.
