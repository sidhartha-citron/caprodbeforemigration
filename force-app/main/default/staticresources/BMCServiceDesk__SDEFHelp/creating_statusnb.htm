The status of a record tracks its progress from the stages of opening to closure. BMC Remedyforce provides two default states: Open and Close. However, you can track more stages by creating status records and assigning descriptions that accurately describe each stage.

Although, you can create multiple statuses to assign to the Open and Close states, you can designate only one state as the default for any given status.

Note: You cannot edit or delete system status records.

To create a status record

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Status.
  4. In the Status List view, click New.
  5. In the Status field, type a unique identifier that you want to assign to the current status record.
  6. From the Stage list, select the stage that you want to map to the status.

    7. A stage is a visual indicator of the progress of a record from beginning to completion. For example, if you create a status of Assigned, you can select the Acknowledged stage for this status. When a user opens a record with the status of Assigned, the user can immediately see that the record is in the Acknowledged stage.

    The following stages are available:

    • Opened
    • Acknowledged
    • In Process
    • Closed
  7. In the State section, select one of the following options:
    • Select Open to assign Open as the state of this status record.
    • Select Close to assign Close as the state of this status record.
  8. (Optional) Select the Default Status check box to assign the current status as the default status when you create, open, or close applicable records.

    9. You can create multiple statuses with the Open and Close states, but you can designate only one status as the default status for the Open and Close states. When you select the Default Status check box for the current status record, the default status is removed from the previous default status record. When you create a new incident, task, broadcast, change request, or problem, their forms open with the Status field automatically filled with the status that is designated as the default Open status. Additionally, when you close an incident, task, broadcast, change request, or problem, the Close form appears with the Status field filled with the status that is designated as the default close status.

  9. To make this status available when you create records of the selected objects, in the Applies To section, select the required object check boxes.
  10. Click Save.

    11. Note: When you open a new form to create a record and you save the record, you cannot navigate to the next or previous record by using the Next Record and Previous Record buttons in the upper right section of the form. These buttons are disabled in a new form.

  11. (Optional) To designate the current record as inactive, select the Inactive check box.

    12. Note: Records marked as inactive are not included in any search performed on the database.
