Before importing data from BMC Client Management into BMC Remedyforce CMDB 2.0, you must meet the following requirements:

    • Install JRE. For more information, see Installing JRE.
    • Download the Pentaho Data Integration tool 5.0.1 or later. For more information, see Downloading the Pentaho Data Integration tool.
    • Register at the BMC Communities website. For more information, see Registering at the BMC Communities website.
    • Download the Pentaho package from the BMC Communities website. For more information, see Downloading the Pentaho package.
    • Perform the following tasks on the BMC Client Management server:
      • Configuring the BMC Remedyforce external integration on the BMC Client Management server
      • Enabling the REST web API on the BMC Client Management server
      • Installing an SSL certificate on the BMC Client Management server
    • Configure BMC Remedyforce to retain the original value in the Instance Name field of the imported CIs. For more information, see Configuring BMC Remedyforce to retain the original Instance Name value of CIs imported from an external source.

Downloading the Pentaho package

The Pentaho package for importing BMC Client Management data by using the REST web API is available on the BMC Communities website. To download the Pentaho package, click https://communities.bmc.com/docs/DOC-32929. This Pentaho package is available only for BMC Remedyforce CMDB 2.0.

Note: BMC recommends that you download the job files to the folder where you have unzipped the Pentaho Data Integration tool. Do not change the folder structure because doing so can interfere with the running of the batch files provided in the CombinedJobs folder.

Configuring the BMC Remedyforce external integration on the BMC Client Management server

To import data from BMC Client Management, you must configure the BMC Remedyforce external integration on the BMC Client Management server. In each Pentaho job file for importing data from BMC Client Management, you must provide the logon details of the external integration user that you configure by using the information in this section.

To configure the BMC Remedyforce external integration on the BMC Client Management server

  1. Log on as an administrator to the BMC Client Management server from which you want to import data to BMC Remedyforce.
  2. In the BMC Client Management server, navigate to Global Settings > External Integration.
  3. Click the Create BMC Remedyforce Integration toolbar icon.
  4. In the Properties dialog box, click Select an existing administrator.
  5. In the Assign an Administrator dialog box, select the user who is a BMC Client Management administrator.

    6. Note: Ensure that the user has the appropriate permissions to perform direct actions on all devices managed from BMC Remedyforce. Also, verify that the user password does not exceed 127 characters.

  6. To close the Assign an Administrator dialog box, click OK.
  7. To close the Properties dialog box, click OK.

Enabling the REST web API on the BMC Client Management server

  1. Ensure that you have configured the BMC Remedyforce external integration on the BMC Client Management server.
  2. Log on as an administrator to the BMC Client Management server from which you want to import data to BMC Remedyforce.
  3. In the BMC Client Management server, navigate to Global Settings > External Integration > Web Service Configuration.
  4. Click the Enable Web Service Module on Selected Device toolbar icon.
  5. In the Enable Web Service Module on Selected Device dialog box, select the BMC Client Management server with which you are configuring the integration.
  6. Click OK.

Installing an SSL certificate on the BMC Client Management server

To secure the transactions and encrypt the information passing between the BMC Client Management server and BMC Remedyforce, you must install a Secure Sockets Layer (SSL) certificate on the BMC Client Management server. For more information about SSL, see http://en.wikipedia.org/wiki/Secure_Sockets_Layer.

If your organization already has an SSL certificate, you can use it, or you can purchase an SSL certificate from an SSL vendor such as Go Daddy (http://www.godaddy.com/ssl-certificates.aspx) or Symantec (http://www.symantec.com/page.jsp?id=ssl-information-center). For more information, see http://wiki.developerforce.com/page/Outbound_Messaging_SSL_CA_Certificates.

After preparing the certificate request by creating a private key and Certificate Signing Request (CSR), you must purchase the SSL certificate. After you purchase the SSL certificate, you are ready to install it.

Important: For the integration to work, you must install an SSL certificate.

To install an SSL certificate on the BMC Client Management server

  1. Log on as an administrator to the BMC Client Management server from which you want to import data to BMC Remedyforce.
  2. Navigate to Global Settings > External Integration > Web Service Configuration.
  3. Click the Properties toolbar icon.
  4. Select the Enable Web Service check box.
  5. In the Web Service Port field, type the number of the port on which the web service should run.
  6. In the Web Service Thread Count field, type the number of threads that may be handled simultaneously.
  7. Click OK.
  8. To create a private key and CSR, perform the following actions:
    1. Click the Prepare Certificate Request toolbar icon.
    2. In the Prepare Certificate Request window, enter the appropriate details, and click Save Private Key.
    3. Browse to the location at which you want to save the private key.

      4. BMC recommends that you save the private key in the same folder in which you saved the SSL certificate.

    4. Click Save.
    5. In the Prepare Certificate Request window, click Save CSR.
    6. Browse to the location at which you want to save the CSR.

      7. BMC recommends that you save the CSR in the same folder in which you saved the SSL certificate and the private key.

    7. Click Save, and then click Close.
  9. Provide the private key and CSR to an SSL vendor and purchase the SSL certificate from the vendor.
  10. Click Install the certificate.
  11. In the Give a name to your certificate field, type a name for your certificate.
  12. In the Root Certificate section, click Browse to browse to the location at which the root certificate is located, and open the root certificate file.

    13. Some certificate providers might give you the root and intermediate certificates in one file. You can verify that you have added the correct certificate by clicking Details and viewing the content of the certificate. A root authority is self-signed; therefore, the Issuer and Subject fields must have the same value.

  13. If both certificates are present in one file, for the first certificate, delete the content from the -----BEGIN CERTIFICATE----- to the -----END CERTIFICATE----- markers. Click Details, and verify that the Issuer and Subject fields have the same value. If the values are different, delete the entire content and again open the root certificate file. Now, for the second certificate, delete the content from the -----BEGIN CERTIFICATE----- to the -----END CERTIFICATE----- markers. Click Details, and verify that the Issuer and Subject fields have the same value.
  14. Click Close.
  15. In the Intermediate Certificate section, click Browse to browse to the location at which the intermediate certificate is located, and open the intermediate certificate file.
  16. If both certificates are present in one file, for the first certificate, delete the content from the -----BEGIN CERTIFICATE----- to the -----END CERTIFICATE----- markers. Click Details, and verify that the Issuer and Subject fields have different values. If the values are the same, delete the entire content and again open the intermediate certificate file. Now, for the second certificate, delete the content from the -----BEGIN CERTIFICATE----- to the -----END CERTIFICATE----- markers. Click Details, and verify that the Issuer and Subject fields have different values.
  17. In the Final Certificate section, click Browse to browse to the location at which the final certificate is located, and open the final certificate file.
  18. To view the final certificate, click Details.
  19. Click Close.
  20. In the Final Certificate section, click Browse to browse to the location at which the private key is located, and open the private key file.

    21. You can verify that the private key has been added by checking for the -----BEGIN RSA PRIVATE KEY----- and -----END RSA PRIVATE KEY----- markers.

  21. In the Private Key Password field, type the private key password you entered while creating the private key.
  22. Click Install Certificates.
  23. In the confirmation dialog box, click Close.
  24. In the Install Certificate window, click Close.
  25. Click Refresh.

The SSL certificate is installed on the BMC Client Management server. The SSL certificate appears in the Web Service SSL Certificate File attribute.
