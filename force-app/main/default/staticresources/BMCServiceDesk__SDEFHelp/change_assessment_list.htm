Creating change assessments

No change to the IT infrastructure is without risk. Therefore, your change requests must minimize the severity of any impact and disruption. Change requests also need to be successful at the first attempt. The Change Manager and change assessor are responsible for assessing the technical, financial, and business risks and the benefit of the change request. Change approvers review the assessments made for the change request and use this information to approve or reject a change request.

To create a change assessment

  1. Navigate to the change request which you want to create a change assessment.

    2. For more information, see Navigating to a change request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, click Record Details and in the Change Assessment section, click New Change Assessment.
    • If you are using the Change Requests tab (Salesforce page layout), in the Change Assessment section, click New Change Assessment.
  3. From the Assessor ID field, select the assessor of the change request.
  4. Click Save.

    5. The change assessment appears in the My Change Assessment QuickView of the assessor.

Assessing change requests

If you are an assessor of a change assessment and a Change Manager has assigned a change assessment to you, it appears in My Change Assessments QuickView on the Remedyforce Dashboard tab. Open the required change assessment to assess it.

To assess a change request

  1. Click the Remedyforce Dashboard tab.
  2. From the list of All Available QuickViews, drag My change assessments in the Dashboard.
  3. From the list of change assessments, click the change assessment that you want to assess.
  4. On the Change Assessment page, in the Technical Risk field, select the technical risk of implementing the change request.
  5. In the Financial Risk field, select the financial risk of implementing the change request.
  6. In the Business Risk field, select the business risk of implementing the change request.
  7. In the Benefits of Change field, type the benefits of implementing the change request.
  8. In the Comments field, type your comments.

    9. The change approver can use these comments to decide about approving or rejecting the change request.

  9. Select the Assessment Completed check box when you have completed the change assessment.
  10. Click Save.

The date and time when you saved the change assessment is recorded in the Assessment Completed Date field.

Cloning change assessments

After you create a change assessment for a change request, you can clone the change assessment and assign them to different assessors.

The option to clone a change assessment is provided after creating a change assessment.

Deleting change assessments

If a change assessment is no longer applicable because you have received sufficient inputs from those most impacted by the change request and have decided to reject the change request, then you can delete the change assessment.

To delete a change assessment

  1. Navigate to the change request whose change assessment you want to delete.

    2. For more information, see Navigating to a change request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, click Record Details and in the Action column beside the change assessment that you want to delete, click Del.
    • If you are using the Change Requests tab (Salesforce page layout), in the Action column beside the change assessment that you want to delete, click Del.
  3. In the confirmation dialog box, click OK.
