After your master subscription application is validated and accepted, configure BMC Remedyforce and the BMC MyIT Administration Console (requests and calendars) to enable clients to use BMC MyIT.

Important: Before logging on to the BMC MyIT Administration Console, system administrators must enable the API for profiles. For more information, see the “Enable API for profiles” section in Table 22-2.

The following table provides an overview of the actions that you must perform to configure BMC Remedyforce and BMC MyIT. If you are upgrading to 20.14.02, you must also perform the configurations in Table 22-3.

Table 22-1:  Configuring BMC Remedyforce and BMC MyIT  
Step
Action

If you have not already added users in BMC Remedyforce, add or create them.

Note:  The users added in BMC Remedyforce can access BMC MyIT by using their client ID and password. To enable clients to log on to BMC MyIT, system administrators must initially log on to the BMC MyIT Administration Console.
  • Adding or editing users
  • Creating clients and staff members

If you have not already configured your BMC Remedyforce organization, configure the organization.

Note:  To configure your BMC Remedyforce organization for using with BMC MyIT, at a minimum you must configure roles, accounts, categories, status records, impacts, urgencies, priorities, templates, and workflows, and create configuration items.
  • Configuring BMC Remedyforce
  • Configuring workflows

Important: After configuring BMC Remedyforce to use BMC MyIT, do not modify the value of the Client ID field of a client on the Client form in BMC Remedyforce. The client whose record you modify is treated as a new client in BMC MyIT. The requests and appointments created by the client will be lost.
