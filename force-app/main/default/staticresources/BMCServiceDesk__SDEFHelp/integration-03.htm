When an event that needs attention of a staff is generated, an email must be sent to BMC Remedyforce to create an incident for the event. To send an email to BMC Remedyforce, create an action that sends an email when such an event is generated. To create the action, customize the event rule for the event that you want to track.

To customize event rules in BMC ProactiveNet Performance Management

  1. In the Operations Console, click Options.
  2. In the Options window, click Administration.
  3. Click Edit besides the required Event Rules entry.
  4. On the Edit Event Rule (Page 1/3) screen, click Next.
  5. On the Edit Event Rule (Page 2/3): Select Event/Monitor/Severity Information page, click Next.
  6. On the Edit Event Rule (Page 3/3): Action Information for Event Rule screen, in the From field, enter the email address from which an email is sent to BMC Remedyforce to create an incident.
  7. In the To field, enter the email address that you have configured in BMC Remedyforce.
  8. Click Add.
  9. Click Event Text Customization.
  10. In the Subject field, enter the subject of the email that is sent to BMC Remedyforce.
  11. In the Body field, enter the fields that you want to map to the incident that is created in BMC Remedyforce. Use the following format in the Body field:

    12. <keyword> <delimiter> <BMC ProactiveNet variable> <delimiter>

    <keyword> and <delimiter> are the keyword and delimiter that you select on the Remedyforce Administration > Configure Email > Incident - Email Settings page in BMC Remedyforce. If you are using the Visualforce page layout user interface (UI), the Incident - Email Settings tab is provided in Configuration > Application Settings. The value of <BMC ProactiveNet variable> is copied to the incident field that is mapped to the <keyword>. This mapping is specified on the Remedyforce Administration > Configure Email > Incident - Email Settings > Mapping Email Content to Incident Fields page in BMC Remedyforce.

    Examples of mapping are EVENTID##$EVENTID## and CATEGORY##General Information##, where EVENTID and CATEGORY are the keywords, $EVENTID is the BMC ProactiveNet variable, General Information is the category that you want to assign to the incident that you are creating, and ## is the delimiter.

    EVENTID and CATEGORY are mandatory fields. The EVENTID field is mapped to the BMCServiceDesk__Event_ID__c field in the Incident object in BMC Remedyforce, while the CATEGORY field is mapped to the Category field.

    The BMCServiceDesk__Event_ID__c field stores the unique event ID and is used as a reference from BMC Remedyforce to the event in BMC ProactiveNet Performance Management.

  12. Click Apply.
