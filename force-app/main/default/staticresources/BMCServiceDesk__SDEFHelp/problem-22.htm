You can order tasks linked to a problem by using the problem manifest. The manifest is available after you save a problem. The manifest provides a tabular view of all the tasks linked to a problem. You can set order numbers for linked tasks and also associate problem status with those tasks.

The following sections are provided:

    • Understanding the manifest
    • Ordering linked tasks and associating problem status

Understanding the manifest

You can use the manifest to establish a sequence for how the linked tasks must be carried out during the lifecycle of a problem.

You can also associate a problem status with the linked tasks to ensure that before a problem moves to another status, all the tasks with which the current status is associated are closed. For example, with tasks T1 and T2, Opened problem status is associated. In this case, you cannot change the status of the problem to In Progress or any other status until T1 and T2 are closed.

The following data about the tasks and problem are displayed in the problem manifest:

    • Problem Item: Problem ID
    • Item ID: Task ID
    • Description: Description of task
    • Status: Status of the task
    • Due Date: Due date of the task
    • Priority: Priority of the task
    • Assigned To: The user to whom the task is assigned
    • Problem Status: Status of the problem (only the status that belong to the Opened state)
    • Order: Sequence number of a task for a particular problem status

Ordering linked tasks and associating problem status

This section provides steps to order linked tasks and associate problem status with required tasks.

To order linked tasks and associate problem status

  1. Navigate to the required problem from which you want to set up order numbers for linked tasks and change requests.

    2. For more information, see Navigating to a problem record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Agent Tools menu, select Problem Manifest.
    • If you are using the Problems tab (Salesforce page layout), click Problem Manifest.

      • If the Problem Manifest button is not available, contact your system administrator to customize the layout of the problem form and add the button.

      All the linked tasks are displayed in a tabular format.

  3. To set the order number of a task, click the cell of the task in the Order Number column and select the order number.

    4. The selected order number defines the sequence number in which the task is performed.

  4. To associate a problem status with tasks, click the cell of the required task in the Problem Status column and select the required status.

    5. Remember the following points while setting up order number of a task and associating a status:

      • When you update the status of a problem, all the tasks from the current status must be completed. For example, if you change the status of a problem from Accepted to In Progress, then all the tasks of the Accepted status must be complete.
      • It is allowed that you order a task, but not select a problem status. However, if you select a problem status with a task, you must set order also.
      • A task is performed or implemented when the selected problem status is active.
  5. (Optional) To export the list of linked tasks to a CSV file format, click Export.
  6. (Optional) To print the list of linked tasks, click Print.
  7. Click Save.
