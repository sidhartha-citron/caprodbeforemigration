You can order tasks linked to a change request by using the change request manifest. The change request manifest is available after you save a change request. The manifest provides a tabular view of all the tasks linked to a change request. You can set order numbers for linked tasks and also associate change request status with those tasks.

The following sections are provided:

    • Understanding the manifest
    • Ordering linked tasks and associating change request status

Understanding the manifest

You can use the manifest to establish a sequence for how the linked tasks must be carried out during the lifecycle of a change request.

You can also associate a change request status with the linked tasks to ensure that before a change request moves to another status, all the tasks with which the current status is associated are closed. For example, with tasks T1 and T2, Opened change request status is associated. In this case, you cannot change the status of the change request to In Progress or any other status until T1 and T2 are closed.

The tasks that are defined in the template but are not yet created are displayed with Pending Creation status (referred to as future tasks). You can create such tasks if you create a change request by using a change request template. The order number of future tasks must be greater than the order number of the already-created tasks of the template.

If the linked tasks are ordered in the template that is used to create the change request and you set different order numbers for those tasks in the manifest, the ordering done in the manifest is followed.

The following data about the tasks and change requests are displayed in the change request manifest:

    • Change Item: Change request ID
    • Item ID: Task ID
    • Description: Description of the task
    • Status: Status of the task
    • Due Date: Due date of the task
    • Priority: Priority of the task
    • Assigned To: The user to whom the task is assigned
    • Change Status: Status of the change request
    • Order: Sequence number of a task for a particular change request status

Ordering linked tasks and associating change request status

This section provides steps to order linked tasks and associate change request status with required tasks.

To order linked tasks and associate change request status

  1. Navigate to the required record for which you want to set up order numbers for linked tasks and change requests.

    2. For more information, see Navigating to a change request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, click the Agent Tools menu, and select Change Manifest.

      • If the Change Manifest option is not available in the Agent Tools menu, contact your system administrator.

    • If you are using the Change Requests tab (Salesforce page layout), click Change Manifest.

      • If the Change Manifest button is not available, contact your system administrator to customize the layout of the incident form and add the button.

      All the linked tasks are displayed in a tabular format.

  3. To set the order number of a task, click the cell of the task in the Order Number column and select the order number.

    4. The selected order number defines the sequence number in which the task is performed.

  4. To associate a change request status with tasks, click the cell of the required task in the Change Status column and select the required status.

    5. Remember the following points while setting up order number of a task and associating a status:

      • When you change the status of a change request, all the tasks in the current status must be complete. For example, if you changed the status of a change request from Registered to Risk & Impact, then all the tasks and change requests from the Registered status must be complete.
      • The order number of future tasks must be greater than the order number of the already-created tasks.
      • It is allowed that you order a task, but not select a change request status. However, if you select a change request status with a task, you must set order also.
      • If you are ordering one future task, ensure that you order all the future tasks.
      • A task is performed or implemented when the selected change request status is active.
  5. (Optional) To export the list of linked tasks to a CSV file format, click Export.
  6. (Optional) To print the list of linked tasks, click Print.
  7. Click Save.
