This section describes how to link record of an object to a record of another object, for example, how to link a change request to an incident.

Assume that your Service Desk receives a phone call from a customer saying that the laptop has crashed. An incident is logged for the customer. With investigation, the staff member determines that the RAM of the laptop has crashed. In this case, you need to change the RAM, so you must link a change request with the incident to change the RAM.

For example, by clicking the Select and Link Tasks to <object> button that is available on the Linked Tasks section on the Remedyforce Console, Incident Console, Incidents, Change Requests, Problems, or Releases tabs, you can link the tasks that are not linked to any other record.

For change requests, incidents, problems, and releases, you can add a new task to a record (by clicking Create New Task) or create a new task and link it to a record (by clicking Create and Link Task to <Object>; Object can be replaced with Change Request, Incident, Problem, or Release).

For releases, to link the configuration items linked to the associated change requests, click Link Services & CIs from Linked Change Request.

Note: Services and configuration items (CIs) can also be linked to the records of these objects in the Linked Services and CIs section. To open a CI on the Remedyforce CMDB tab, click the CI in the Configuration Item~ column. Contact your system administrator if you do not see the Configuration Item~ column.

The Record Details that display change requests and problem linked to a knowledge article are not displayed on a knowledge article details page, by default. Contact your system administrator to display the required record details.

The following buttons are available for record details:

    • Select and Link <object 1> to <object 2>
    • Create and Link <object 1> to <object 2>
    • Create New <object>
