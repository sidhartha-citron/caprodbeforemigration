By default, the Category field is required and displayed on the Visualforce forms of the following objects:

    • Incident
    • Task
    • Broadcast
    • Change Request
    • Problem

The following two settings allow you to make the Category field optional or hide the field on the Visualforce forms:

    • Category Visible on Visualforce Forms: when you configure this setting to false, the Category field is not displayed on the Visualforce forms and the Close forms of the objects (both Visualforce and Salesforce page layout). By default, this setting is configured as true.
    • Category Optional on Visualforce Forms: when you configure this setting to true, the Category field is not required on the Visualforce forms. By default, this setting is configured as false.

To hide or make the Category field optional on the Visualforce forms

  1. Navigate to Setup > Develop > Custom Settings.
  2. In the Action column for Remedyforce Settings, click Manage.
  3. To make the Category field optional, in the Action column for Category Optional on Visualforce Forms, click Edit.
  4. In the value text box, enter TRUE and click Save.
  5. To hide the Category field on the Visualforce forms, in the Action column for Category Visible on Visualforce Forms, click Edit.
  6. In the value text box, enter FALSE and click Save.

    7. When you set the Category Visible on Visualforce Forms setting as false, the Category Optional on Visualforce Forms setting is considered as true in processing the Visualforce forms.

  7. If you have set the value field as TRUE for the Category Optional on Visualforce Forms setting, perform the following steps:
    1. Navigate to Setup > Create > Objects.
    2. Click Incidents.
    3. In the Validation Rules section, click Edit in the Action column for the Select Category or template or req def rule name.
    4. Clear the Active check box and click Save.
    5. Navigate to Setup > Create > Objects.
    6. Click Broadcasts.
    7. In the Validation Rules section, click Edit in the Action column for the Category Required rule name.
    8. Clear the Active check box and click Save.

      9. When you set the Category Visible on Visualforce Forms setting as false, the following behavior is observed:

    9. The Category field is invisible on the Visualforce forms of the existing records also.
    10. The Incident for Category option in the View menu of an incident record is enabled. For an existing record, when you click the Incident for Category option, a list of incidents where the category matches the category of the opened record is displayed.
    11. The Suggested Expert option is enabled in the Assign to menu. For an existing record, when you click the Suggested Expert option, a list of experts set for the category of the opened record is displayed.

      12. Note: After hiding the Category field from the Visualforce forms, if you want to display the field again, you must configure the value of the Category Visible on Visualforce Forms setting as TRUE and activate both the Category Required validation rule for the Incidents and Broadcasts objects.
