When staff members are linking an object to another object, such as linking change requests to an incident record, a pop-up window is displayed (for example, Link Change Requests to Incidents), in which the staff member can select multiple records (for example, multiple change requests to link to an incident). System administrators can configure the fields that are displayed in such pop-up windows.

A Column Headers field set is provided for each object (Incident, Problem, Change Request, Release, Task, Broadcast, and so on). You can configure fields in this field set for each object. When a staff member, change manager, or release coordinator is linking an object to another object, the values in the configured fields are displayed as columns in the pop-up windows.

The fields that you specify for the Incident object are also shown in the menu available on the Incidents Matching window. Staff members specify fields in this menu to find matching incident records.

To configure fields to be displayed when linking an object to another object

  1. Click the Remedyforce Administration tab.
  2. On the Home page, click the Manage Objects tile, and from the menu, select Create and Edit Objects.
  3. On the Create and Edit Objects page, from the Object list, select the required object, such as Incident, and then click Field Sets.
  4. In the Action column of the Field Sets section, click Edit for the Column Headers field set.
  5. Drag the required fields to the In the Field Set list.

    6. The fields configured for the Incident object are also shown in the Incidents Matching window when a staff member is finding matching records.

  6. Click Save.
