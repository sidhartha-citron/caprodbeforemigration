In the Defaults tab, you can define how a Self Service client works with incidents and service requests, select the Contact Us template, configure Internet and Intranet searching, create quick links, display the profile details of the logged on Self Service client, and enable Self Service access from mobile devices.

To define the default settings

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Self Service.
  3. In the Self Service list, click Self Service Settings.

    4. Note: In the BMC Remedyforce tabbed view, Self Service Settings is available in the Remedyforce Administration tab.

  4. In the Incidents section of the Defaults tab, perform any of the following actions:
    • To assign a default queue for all new incidents and service requests created in Self Service, perform the following actions:
    • Click the Incident Assignment to Queue list.
    • In the Select from Queue window, select the required queue.
    • To allow users to close incidents and service requests that they have created, select the Allow Users to Close Their Incidents check box and modify the Solution field in the service request, depending on the selection of one of the following options:
      • To allow users to close an incident or service request only after entering a solution in the Solution field, select Incident Resolution is Required.
      • To allow users to close an incident or service request without entering a solution in the Solution field, select Incident Resolution is Not Required.

        • While customizing the Self Service: Service Request form, if you remove the Resolution field from the In the Field Set list and select the Incident Resolution Required option, the Resolution field is not mandatory while closing the service request.

    • To allow users to reopen incidents and service requests that they have closed, select the Allow Users to Reopen Their Incidents check box.
    • To allow users to create a service request for an issue without searching for a solution, select the Display “Request Help” on the Home Page check box.
    • To limit the number of closed incidents and service requests that are displayed to users, select the Limit Closed Incidents (Recently Completed Requests) to check box.

      • If you clear this check box, users can view all closed incidents and service requests.

    • In the Days field, type the required number of days.

      • Self Service limits the display of closed incidents and service requests to only incidents and service requests that are closed in the entered number of days.

    • In the Site URL of Self Service field, enter the URL of the Self Service site.

      • This URL is used when you share a link to an incident with a client user in an email message. When the client user clicks this link, Self Service view of the incident is displayed.

      This URL is also used when you copy link to a knowledge article in the Resolution field of an incident. The client user can use this link to see the knowledge article in Self Service.

  5. In the Service Requests section of the Defaults tab, perform any of the following actions:
    • To configure the required view for the users, from the Default view field, select the required option. If you select Popular, then in Self Service, the Popular tab is active, by default. The Popular tab displays the Popular Service Requests and Popular Knowledge Articles panels. If you select Browse, then in Self Service, the Browse tab is active, by default. The Browse tab displays service requests or knowledge articles for the selected category in the category tree.
    • To display both Incident templates and service request definitions in the Popular Service Requests panel, select the Display incident templates and service request definitions together check box.

      • If you select this check box, the Report an Issue and Request a Service tabs do not appear in the Popular Service Requests panel. When you click Browse in the upper right section of the Self Service Home page, the Self Service client can browse through service requests only. Incident templates are not displayed.

      If you clear this check box, Incident templates are listed in the Report an Issue tab of the Popular Service Requests panel, and service request definitions are listed in the Request a Service tab of the Popular Service Requests panel. When you click Browse in the upper right section of the Self Service Home page, Incident templates are displayed in the Report an Issue tab and service requests are displayed in the Request a Service tab.

    • To view knowledge articles before Incident templates and service request definitions in Self Service, select the Knowledge articles have precedence over service requests in the layout check box.

      • If you select this check box, knowledge articles have precedence in the following sections:

      • The Popular Knowledge Articles panel appears on the left and the Popular Service Requests panel appears on the right in the Self Service home page.
      • When you click Browse in the upper right section of the Self Service Home page, the Knowledge Articles tab with knowledge articles in the Category tree is displayed. If you want to view service requests, click the Service Requests tab.
      • When you perform a search, the Knowledge Articles tab appears before the Report an Issue, Request a Service, and Web Search tabs in the search results.
    • To use the My Cart feature in Self Service, select the Enable My Cart check box.

      • If you select this check box, the My Cart feature becomes available in the following sections:

      • The Save To Cart button becomes available when you are submitting a service request.
      • The Add To Cart button becomes available when you are previewing a service request in the Service Request Preview window.
      • The My Cart portlet becomes available in Self Service.

        • Note: If Self Service clients have service requests in the My Cart portlet and you clear the Enable My Cart check box, the Self Service clients cannot check out these service requests because the My Cart portlet will not be available to them.

        If no active, online, available (the End Date has not occurred), and entitled service request definitions exist, the My Cart portlet does not appear in Self Service even if you select this check box.

    • To allow Self Service clients to preview the service request before creating the service request, select the Enable preview of service requests check box.

      • If you are installing BMC Remedyforce for the first time, this check box is cleared. You must select this check box if you want the Self Service clients to preview the service request. If you are upgrading from a previous version of BMC Remedyforce, this check box is selected and Self Service clients preview the service request before creating it.

  6. In the Contact Us section of the Defaults tab, perform any of the following actions:
    • To display the Contact Us link in Self Service, select Display Contact Us link.

      • If you clear the Display Contact Us link check box, the Message on the Header of the Contact Us Page and Incident Template for Contact Us Submissions are disabled.

    • In the Message on the Header of the Contact Us Page field, type the header message.

      • You can use the header message to provide instructions on how to use the Contact Us template.

    • To assign a default Contact Us template for all Self Service clients, perform the following actions:
      • Click the Incident Template for Contact Us Submissions list.
      • In the Select from Templates window, select the required Incident template.
  7. In the Search section of the Defaults tab, perform any of the following actions:
    • To allow users to search the intranet of their company for information, perform the following actions:
      • Select the Enable Intranet Searching check box.
      • In the Intranet URL field, type the Intranet URL.

        • This URL creates the search string that searches for matching values on the intranet of the company.

    • To allow users to search for matching values in the Internet by using the Google search engine, select the Enable Internet Searching check box.
    • To allow users to search for matching values in knowledge articles of BMC Remedyforce, select the Enable Remedyforce knowledge articles searching check box.

      • When you do not select this check box, the Popular Knowledge Articles panel is not displayed and the Self Service users do not get the link to knowledge articles if a Staff user wants to share it.

    • To allow users to search for matching values in knowledge articles of Salesforce, select the Enable Salesforce knowledge articles searching check box.

      • Select this check box if you have Salesforce knowledge license for the Salesforce organization where BMC Remedyforce is installed.

      For information about settings that you must configure to display Salesforce knowledge articles in Self Service, see Enabling Salesforce knowledge articles to be viewed in Self Service.

  8. To add quick links to the Corporate Links section of the Quick Links portlet, in the Quick Links section of the Defaults tab, perform the following actions:
    1. To display the Quick Links portlet in the Sidebar, select the Enable Quick Links Portlet check box.
    2. To add a quick link to the Quick Links portlet, click New.
    3. In the Name field, type the name of the quick link.

      4. The quick link appears with this name in the Quick Links portlet in the sidebar.

    4. In the URL field, type the URL of the quick link.

      5. When you click the quick link, Self Service opens this URL in a new window.

  9. In the Banner Configuration section of the Defaults tab, perform the following actions:
    1. To display the banner of the BMC Remedyforce application, select Display application banner.

      2. The application banner in Self Service consists of the banner that you select, the message that is being broadcasted for the client, and My Profile, Help, About, and Log Out links.

    2. To display the My Profile link in the banner, select the Display My Profile on application banner check box.

      3. The Display My Profile on application banner check box is enabled if the Display application banner check box is selected.

      The My Profile link allows the logged-on users to add or update their details.

    3. To enable Self Service users to control broadcast ticker speed, select the Allow user to control scrolling speed of broadcast ticker check box.

      4. If you select this check box, the Broadcast Ticker Speed field is displayed on the My Profile page in Self Service.

    4. Set the required speed of broadcast ticker speed in the BMC Remedyforce application in the Default ticker speed field.
  10. In the Mobile Configuration section of the Defaults tab, perform the following actions to enable Self Service users to access Self Service from their BlackBerry® and iPhone™ mobile devices:
    1. To allow Self Service users to access Self Service from mobile devices, select the Enable mobile access check box.

      2. Note: If Self Service is configured on a sandbox, you cannot connect to Self Service from BMC Remedyforce mobile app on your mobile device.

    2. In the Portal ID field, type the Portal ID of your Customer Portal.

      3. You can retrieve the Portal ID by navigating to Setup > Customize > Customer Portal > Settings, clicking Customer Portal in the Name column, and copying the Portal ID in the Portal Detail section.

      When a Self Service user access Self Service from the mobile device, the Organization ID and Portal ID are saved on the mobile device as part of the configuration data.

      You can download Self Service for mobile devices from the following locations:

      • BlackBerry: Download the Self Service application from ftp://ftp.bmc.com/pub/Remedyforce/BMCSDoF_SelfService_BlackBerry.zip and follow the instructions in the ReadMe.txt file to install the application.
      • iPhone: Download the Self Service app from http://itunes.apple.com/us/app/bmc-servicedesk-on-force-com/id370543399 and install it on your iPhone.

        • Tip: BMC recommends that you send an email message with the appropriate URL to your clients so that they can easily copy the URL from their email and paste the URL in the app.

        Staff members of BMC Remedyforce or other users in your company who have their own Salesforce license cannot access Self Service from an iPhone 3G, iPhone 4G, or BlackBerry Smartphone mobile device by using the BMC Remedyforce app. Only clients with the Customer Portal Manager Custom license can access Self Service from an iPhone 3G, iPhone 4G, or BlackBerry Smartphone mobile device that is running the BMC Remedyforce app.

        Additionally, if you have customized the Service Request form of Self Service, this custom form is not available when you access Self Service from an iPhone 3G, iPhone 4G, or BlackBerry Smartphone mobile device by using the BMC Remedyforce app.

        Note: You must send the Secure Web Address of the site you have configured for Self Service to your Self Service users. Self Service users can access Self Service from their BlackBerry and iPhone mobile devices by using the Secure Web Address only. They cannot access Self Service by using the Default Web Address of the site you have configured for Self Service. You can retrieve the Secure Web Address by navigating to Setup > Develop > Sites and copying the Secure Web Address in the Site Detail section.

  11. Click Save.

Enabling Salesforce knowledge articles to be viewed in Self Service

Only published Salesforce knowledge articles can be viewed in Self Service. Ensure that the following conditions are set for Salesforce knowledge articles that you want to display in Self Service:

    • The required Salesforce Article Type must be in Deployed state.
    • In the Article Type Permission section of the Article Type of the knowledge article, select the Read check box for the ServiceDesk Client profile.
    • Fields of the required Article Type must be set visible to the ServiceDesk Client profile.

      • For more information, see https://na12.salesforce.com/help/doc/user_ed.jsp?section=help&target=knowledge_article_types_create.htm&loc=help&hash=topic-title.

    • While creating a knowledge article, in the Channels area, select the Customer Portal check box.

      • For more information, see https://na12.salesforce.com/help/doc/user_ed.jsp?loc=help&target=knowledge_article_actions.htm&section=Salesforce_Knowledge.

    • In BMC Remedyforce, on the Configuration > Self Service Settings page, click the Search text box and ensure that the Enable Salesforce knowledge article searching check box is selected.
    • In BMC Remedyforce, on the Configuration > Global Search Settings page, ensure that the Search Salesforce knowledge articles check box is selected and click Refresh Article Types.
    • (Optional) To display article summary of Salesforce knowledge article, navigate to Setup > Customize > Knowledge > Settings and ensure that the Customer Portal check box is selected in the Article Summaries section.
    • (Optional) To search and view articles of more than one language, navigate to Setup > Customize > Knowledge > Settings and ensure that the required language is active in the Language Settings section.

Ability to add an attachment before creating a record in Self Service

You must enable Read permission for Temporary Attachment object for the following profiles:

    • ServiceDesk Change Manager
    • ServiceDesk Client
    • ServiceDesk Staff

For more information about how to enable permissions for a custom object for the required profiles, see Enabling permissions for a custom object.
