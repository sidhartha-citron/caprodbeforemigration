Using email conversations, you can create a single repository to store information about all the conversations that are part of a change request. You have access to all the information in one place and you can track all communications about any change request.

Note: If the size of the email conversation exceeds 32 KB, the first message in the email conversation is deleted and the latest messages are retained.

Sending an email message from a change request record

When an email message is sent by a user assigned to a change request, the message is stored in the Action History section as an Email Sent action record to allow audit trail tracking (in the Supporting Information section in the Visualforce page layout UI). When you open this email record, it opens the message in read-only mode in the Email Message window.

Each outbound email message has an identification token or reference number associated with it. By default, all outbound email messages contain two unique identifiers that the system uses to match the email message to a record in the system. One identifier is stored in the subject of the message, while the other identifier is stored in the body of the message.

Note: Do not delete the unique identifier that appears in your email message. Otherwise, the email conversation cannot be tracked.

When a user assigned to a change request receives reply on one of these email messages, BMC Remedyforce receives the message and scans the message body and subject for a unique identifier. If the identifier is found, BMC Remedyforce creates an Email Received action record in the Action History section to allow audit trail tracking (in the Supporting Information section in the Visualforce page layout UI). The initiator of the change request can reply to the message and the reply is added to the Action History record of the change request (or the History tab).

If you double-click an email record in the Action History section, the email record opens in the Email Message window.

To send an email message from a change request record

  1. Navigate to the change request from which you want to send an email message.

    2. For more information, see Navigating to a change request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Actions menu, select Email.
    • If you are using the Change Requests tab (Salesforce page layout), click Send Email.
    • If you are using the Visualforce page layout UI, click .

      • The To field is populated with the email address of the initiator of the change request. The Subject of the email message is populated with the change request number.

      The To field is enabled if the system administrator has configured it.

  3. To send an email message to users other than the initiator of the change request, perform the following actions:
    1. Click the lookup icon for the To field.
    2. In the Search Clients window, select the user to whom you want to send the email message.
    3. To add multiple users in the To fields, repeat step a and step b.

      4. Note: You can send an email message to a maximum of 100 users.

    4. Click the lookup icon for the Cc or Bcc field.

      5. The Bcc field is enabled if the system administrator has configured it.

    5. In the Search Clients window, select the user to whom you want to send a copy of the email message.
    6. To add multiple users in the Cc or Bcc field, repeat step d and step a.

      7. Note: You can also type the email address in the To, Cc, and Bcc fields. You must separate the email addresses with a semicolon (;). You can send a copy to a maximum of 25 users.

  4. (Optional) From the Email Template list, select the template for the email message.

    5. The Email Template field is enabled if the system administrator has configured it.

    If you select an email template, you cannot make changes to the Subject and Attachments of the email message. Contact your system administrator to create an email template of your requirements.

    In the out-of-the-box email templates, dates are displayed in the M/d/yyyy h:mm format. Contact your system administrator to customize the email template based on your needs.

    To preview the selected email template, click Preview. The fields in the email template are replaced with the actual content from the record. The Preview link is available if the system administrator has configured it.

  5. To copy the field values from the Change Request form, such as the value of the Description field in the email body, select Copy data from Change Request fields.

    6. When you select the Copy data from Change Requests fields check box and right-click in the email body, a list of all the fields available on the Change Request form that are accessible to the logged on user is displayed. Select a field to get the value of the selected field in the email body. If the field has a value, it is inserted in the email body.

    If the Copy data from Change Request fields check box is clear, you can paste data from clipboard in the email body.

  6. In the body of the email message, type the message that you want to send.

    7. You can add more information even after you selected a template. Contact your system administrator to enable you to add additional information in the email message even after selecting a template.

  7. (Optional) To add attachments to your email message, perform the following actions:
    1. Click Attach File.

      2. In the Add Attachments window, the files currently attached to the change request in the Notes & Attachments section are displayed. You can attach any of the files from the Notes & Attachments section, or browse to the required file and attach it.

      In the Visualforce page layout UI, the Files currently attached to the Change Request section displays the files that are added to the Documentation tab of the Supporting Information section. You can attach any of the files from the Documentation tab, or browse to the required file and attach it.

    2. In the Add Attachments window, select the appropriate option:
      • Select File, click Browse to browse to the location where the attachment you want to use is located, select the required attachment, and click Open.

        • Note: If the file name of an attachment exceeds 50 characters, you cannot attach the file. To attach such a file, rename the file with a file name that is less than 50 characters, and attach the file.

      • Select Files currently attached to the Change Request and from the Notes & Attachments section, select the required check boxes of the files.
      • In the Visualforce page layout UI, select Files currently attached to the Incident and from the Documentation tab, select the required check boxes of the files.

        • To select all check boxes, select the File check box.

    3. Click Add.

      4. The attached files appear in the Attachments section as hyperlinks. You can view any of the attachments by clicking the corresponding hyperlink to view the file in a new browser tab.

    4. (Optional) To remove an attachment, move the mouse over the attachment, click the Delete icon that is displayed.
    5. To complete adding attachments to your email message, repeat step b through step c.

      6. Note: The total size of all attachments cannot exceed 10 MB.

    6. (Optional) To save the attachments that are sent with the email message, select the Add the attachments to the Notes & Attachments section check box.

      7. To access these attachments, navigate to the Notes & Attachments section of the change request details page.

    7. (Optional) In the Visualforce page layout UI, to save the attachments that are sent with the email message in the change request, select the Add the attachments to the Documentation tab check box.

      8. You can access these attachments by navigating to the Documentation tab of the change request.

    8. Click OK.

      9. In the Email Message window, the attachments are listed in the Attachments field.

  8. Click Send.

    9. An Email Sent action record is created in the Action History section. The type of action record and the subject of the email message are added to the Description field of the action record.

    In the Visualforce page layout UI, an Email Sent action record is created in the History tab of the Supporting Information section. The type of action record and the subject of the email message are added to the Description field of the action record.

    The following information about the email message is recorded in the Notes field of the action record:

    • Organization-wide email address
    • Email addresses in the To field
    • Email addresses in the Cc field
    • Subject of the email message
    • Body of the email message
    • Email template name that was used

      • When you double-click the record in the Action History section, the email message is displayed in the Change Request History page. In the Visualforce page layout UI, when you double-click the action record, the email message opens as a read-only message in the Email Message window. The email addresses listed in the To field and the email addresses of the users who were sent a copy are listed in the Cc field. The attachments that are sent with this email message are listed in the body of the email message. A count of the attachments sent in the message and the file names of the attachments are listed in the Attachments sent with this email section of the email body.

      If the size of a single attachment or all attachments exceeds 3 MB, the attachments are converted to HTML files. When a user receives the email message, the email message contains HTML files that are linked to the attachments that are stored on the Salesforce server. When you open an HTML file from the email message, the HTML file opens in a new browser that directs you to the Salesforce server where the attachment is stored. In the browser, you are prompted to download the attachment from this server.

      Note: When you send the email message, a reference number is appended to the subject and the body of the email message. Do not change the subject while replying to the email message. If you change the reference number, the email message cannot be associated with the change request record.

Storage of email messages

When you are communicating with a user while working on a change request, all the email messages that you send or receive are stored in the Action History section of the change request records. In the Visualforce page layout UI, the email messages are saved in the History tab of the Supporting Information section. The email messages that you send are saved as Email Sent records; however, the messages that you receive are saved as Email Received records.

You can use both the Email Received and Email Sent records to send a reply to the client and other related users.

Replying to an email message

The reply to an email message contains the following:

    • email address of the client in the From field
    • the subject
    • the text of the reply from the client
    • the body of the original email message
    • a unique identifier
    • attachment, if any

For more information about attachments, see About attachments in email messages.

You can use both the Email Received and Email Sent records to send a reply to the client and other related users.

    2. For more information, see Navigating to a change request record.

    • If you are using the Change Requests tab (Salesforce page layout), in the Reply column of the Email Received (or Email Sent) action in the Action History section, click and in the Message tab, click .
    • If you are using the Visualforce page layout UI, in the History tab of the Supporting Information section, double-click the Email Sent (or Email Received) record and in the Message window, click .

      • The email message appears with the unique identifier included in the subject of the email message.

      The email address in the From field is copied to the To field of the email message that you are sending. If the email address of the initiator of the change request is not present in the To, the email address of the initiator is appended to the To field.

  3. (Optional) To reply to all the users that are part of the email conversation, click .

  5. If you need to add attachments, perform step a through step f to add attachments to the reply.

About attachments in email messages

If attachments are present in the reply, the attachments are stored in the Notes & Attachments section. You can view the attachments by double-clicking the required file. The body of the reply lists the count of the attachments received in the reply, and the file names of the attachments are listed in the Attachments received with this email section. For each email message sent by change manager, the count of the attachments sent in the email message and the file names of the attachments are listed in the Attachments sent with this email section.

In the Visualforce page layouts UI, the attachments are stored in the Documentation tab of the Supporting Information section. You can view the attachments by double-clicking the required file in the Documentation tab.

The attachments in the reply from the client are not stored in the Notes & Attachments section if the attachment meets one of the following conditions:

    • One of the attachments exceeds 25 MB.

      • The other attachments that are less than 25 MB are stored in the Notes & Attachments section (or the Documentation tab in Visualforce page layout UI).

    • The total size of all attachments exceeds the storage limit of your organization.

      • In the reply, the following message is displayed if the attachments exceed the storage limit: 

      You have exceeded the storage limit of your organization.
