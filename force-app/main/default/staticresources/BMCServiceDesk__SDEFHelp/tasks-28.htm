To close a task

  1. Navigate to the task that you want to close.

    2. For more information, see Navigating to a task record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Actions menu, select Close.
    • If are using the Task Console tab, from the Actions menu, select Close.
    • If you are using the Tasks tab (Salesforce page layout), click Close.
    • If you are using the Visualforce page layout UI, from the menu, select Close.

      • The Close Task form appears with the Closed Date field filled with the current date and time. The Category field displays the category that is entered in the task form.

      Note: This procedure describes the out-of-the-box fields only. If your system administrator has added or removed fields, complete all the fields on the form.

  3. In the Close Task form, select the staff member who has worked on the task from the Closed By list.

    4. By default, this field displays the Staff ID of the currently logged-on staff member. If you are closing the task on behalf of another staff member, select the staff member who has worked on the task.

  4. From the Status list, select a close status.

    5. By default, this field is filled with the default close status that is configured by the system administrator. If required, you can select a different close status. All statuses with the state of Close are displayed in the Status list.

  5. If the original category is incorrect or no longer applies, select the required category from the Category field.
  6. In the Resolution field, type the resolution of the task.

    7. By default, this field is filled with the value of the Resolution field in the task form. You can modify this field to include the confirmed resolution of the task or any new steps taken to complete the task.

  7. To add the contents of the task form Description field as a new standard description for the category of this task, select the Update Standard Description check box.

    8. You might need to add a new standard description for a task that you must complete on a periodic basis, such as installing operating system patches.

  8. To add the contents of the task form Resolution field as a new standard resolution for the standard description of the issue, select the Update Standard Resolution check box.

    9. You might need to add this resolution for the standard description of this issue because it provides the standard set of steps taken to complete the task.

  9. Click Save.

    10. If the system administrator has activated the Notify the client when a task is closed workflow, an email message is sent to the client of the task when you close the task. If you select a lead as the client of the task, no email message is sent.

    To send an email message to the owner of a change request or the staff member or queue to whom an incident is assigned, notifying that the last task associated with the change request or incident is closed, contact your system administrator to activate the Notify change request owner when final task linked to change request is closed or Notify incident owner when final task linked to incident is closed workflow.

    To send an email message to the incident, problem, or release owner when each linked task is closed, contact your system administrator to activate the required workflow from the following list:

    • Notify incident owner when each linked task is closed
    • Notify problem owner when each linked task is closed
    • Notify release owner when each linked task is closed
