You can select the data sources on which you want to search when you use Global Search. For each data source, you can configure the data fields that display the results of the search. The data fields that are configured in Global Search settings are used to display information in the Search Clients window when you are searching clients while creating an incident or a task.

Note: Only if you select Contacts or Leads in the Client Users Include section on the Application Settings page, the Search Clients window is displayed. If you do not select Contacts or Leads in the Client Users Include section on the Application Settings page, the Select From Client window is displayed.

To configure Global Search settings

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Global Search Settings.

    4. By default, Email, Company, and Mobile Phone fields are displayed for Leads; Account ID, Email, and Mobile Phone fields are displayed for Contacts; and Username, Email, and Account ID fields are displayed for Users.

  4. To enable searching the knowledge articles available in BMC Remedyforce, select Search Remedyforce knowledge articles.
  5. To enable a staff member to rate the knowledge article and provide feedback, select Display the Rating and Feedback fields on the knowledge article page.

    6. To add a custom rating, navigate to Setup > Create > Objects > Knowledge Feedback. In the Custom Fields & Relationship section, click Rating. To add a custom rating, in the Picklist Values section, click New.

  6. To make it required for the user to provide rating to a knowledge article, select Make rating of knowledge articles required.
  7. To set the default use of the knowledge article in resolving an incident, select the required option from the Default use of knowledge articles field.

    8. When you open the Global Search page on the Incident form and access a knowledge article, the option you select in this field is selected on the knowledge article page by default.

    If you select Copy article text, the article text is copied to the Resolution field. However, if you select Copy article link, the link to the knowledge article is copied to the Resolution field.

    Note: To create a link of the knowledge article that can be accessed by Self Service users also, enter the Self Service URL in the Site URL of Self Service field that is available on the Self Service Settings page.
    In the tabbed view of BMC Remedyforce, Self Service Settings is available in the Remedyforce Administration tab.

  8. To enable searching the knowledge articles of Salesforce, select Search Salesforce knowledge articles.

    9. To search Salesforce knowledge articles, you must have Salesforce Knowledge license and users must have Salesforce Knowledge feature license. To give access to a user to Salesforce Knowledge:

    1. Navigate to Setup > Manage Users > Users.
    2. In the Action column, click Edit next to the name of the user.
    3. Select the Knowledge User check box.
  9. (Optional) If you are using Salesforce Knowledge in multiple languages and you have more than one active language, you must set the SF Knowledge Multi Language property to TRUE. To set the SF Knowledge Multi Language property:
    1. Navigate to Setup > Develop > Custom Settings.
    2. In the Action column, click Manage next to Remedyforce Settings.
    3. In the Action column, click Edit next to SF Knowledge Multi Language.
    4. In the value field, enter TRUE.
  10. To get the article types that are available in your Salesforce organization, click Refresh Article Types.

    11. The article types available in your Salesforce Knowledge are displayed. Getting the article types from Salesforce helps BMC Remedyforce to create hyperlinks to these articles when such articles appear in a search result. You must click Refresh Article Types every time a new article type is added in your Salesforce organization.

  11. To enable searching the knowledge articles of RightAnswers, select Search RightAnswers knowledge articles.

    12. This check box is displayed if RightAnswers is installed on the same Salesforce organization where BMC Remedyforce is installed.

    Note: RightAnswers knowledge articles are not searched when you search from the Incident form using the icon. However, in the BMC Remedyforce tabbed view, RightAnswers knowledge articles are searched when you use the Knowledge Search button on the Incident Salesforce page layout.

  12. To view RightAnswers knowledge articles in BMC Remedyforce, perform the following actions:
    1. Navigate to Setup > Develop > Custom Settings.
    2. In the Action column beside Remedyforce Settings, click Manage.
    3. In the Action column beside RightAnswers URL, click Edit.
    4. In the value field, enter https://<your organization instance>.salesforce.com/apex/RAInlineSearchRF.

      5. The instance of your organization is provided in your Salesforce organization URL after https://, for example, na14.

    5. Click Save.
  13. To search the lookup fields available on the selected data sources, select Search lookup fields of the selected data sources.

    14. For example, if you are searching for “Hardware” and you have a category “Hardware”, the incidents with category “Hardware” are displayed in the search result.

  14. To add a new data source on which you want to search when you use Global Search, click .

    15. You can select a data source and click to remove the data source being displayed in the search results. You can also use the up and down arrow buttons to move the location of the data source (Contacts, Leads, and Users) on the search result page.

    The following records are required and you cannot remove these records:

    Data Source
    Data Field
    User
    Account ID
    Contact
    Account ID
    Lead
    Company

    Note: Search is not performed on the basis of the data fields that you configure in the ‘Global Search Settings’. These data fields are displayed only in the search results.

  15. From the Data Source list, select the required data source.
  16. For the data source that you have added, select the data field that displays the results of the search.
  17. To add all the required data sources and data fields, repeat step 14 through step 16.

    18. For example, you can select Chatter in the Data Source column and its fields, ChatterComment or ChatterPost in the Data Field column to enable searching chatter posts or comments when you use Global Search.

    Note: The Chatter object is displayed in the Data Source column only if Chatter is enabled in your Salesforce organization.

  18. Click Save.
