Priority records allow you to define how to process and escalate incidents, tasks, broadcasts, change requests, and problems. When you assign an impact and urgency to an incident, task, broadcast, change request, or problem record, BMC Remedyforce assigns the priority that corresponds to the unique combination of the impact and urgency values of the record. For example, if you select HIGH as the Urgency ID and MEDIUM as the Impact ID of the record, BMC Remedyforce assigns 2 to the Priority ID of the record.

When you assign an impact and urgency to an incident, task, broadcast, change request, or problem record, BMC Remedyforce uses the values to calculate the priority based on this formula:

Urgency + Impact = Priority.

If you create or delete urgency or impact records, you must update the Priority Matrix so that the Priority ID and due date calculation are updated appropriately.

Note: The due date of the incident, task, or problem is calculated using the duration defined in the priority record.

To create a priority record

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Priority.
  4. In the Priorities List view, click New.
  5. In the Priority ID field, type a unique alphanumeric identifier that you want to assign to the current priority record.
  6. In the Description field, type a description of this priority record.
  7. In the Duration field, type the amount of time that must be used to calculate the due date.

    8. The value must be typed in this format: HH:MM. For example, 08:00 means that the incident that is assigned this priority is expected to be resolved within 8 hours.

  8. To calculate the due date by using the time zone of the assigned staff member, select the Adjust due date based on the assigned staff member's time zone check box.

    9. For example, if the duration for the priority is two hours and a staff member is assigned an incident with the current priority at 1:00 p.m., the due date of the incident is set to 3:00 p.m. of the current working day. However, if you have a staff member who is working in a time zone that is three hours behind and you select this check box, the due date of the incident is set to 12:00 p.m. of the working day for that staff member.

  9. From the Impacts list, select the applicable impact.

    10. The selected impact must reflect the level of impact that is associated with all records that are assigned with the current priority.

    Note: The description of the selected impact is filled in the Impact Description field.

  10. From the Urgency list, select the required urgency.

    11. The selected urgency must reflect the speed of response that is associated with all records that are assigned with the current priority.

    Note: The description of the selected urgency is filled in the Urgency Description field.

  11. Click Save.

    12. Note: When you open a new form to create a record and you save the record, you cannot navigate to the next or previous record by using the Next Record and Previous Record buttons in the upper right section of the form. These buttons are disabled in a new form.

  12. (Optional) To designate the current record as inactive, select the Inactive check box.

    13. Note: Records marked as inactive are not included in any search performed on the database.
