If a problem is not assigned to you, you can edit it only if you meet any of the following conditions:

    • The problem is assigned to a staff member who is lower in the role hierarchy than you are and you have read access to the Problem custom object.
    • The assigned staff member of this problem has shared the problem with you.
    • The system administrator of the Salesforce.com organization has changed the default sharing access of problems from the out-of-the-box access level of Public Read Only to Public Read/Write.

      • Note: To erase the updates that you have done to a record and revert the record to the earlier saved state, click Refresh.

Editing multiple problems

  1. Click the Remedyforce Console tab.
  2. From the View menu, select Problems.
  3. Select the check boxes for the records that you want to edit, and click Update.

    4. You can update only 200 records at the same time.

  4. From the Field list, select the field that you want to update for the selected records.

    5. Only the fields that are shown on the list view are available in the Field list. Fields of type Auto Number and Formula are not available in the Field list. If you do not have permission to edit any of the fields in the list view, the field is not available in the Field list.

  5. Enter or select a value for the selected field.
  6. Click Apply.

    7. You can update one field of multiple records at the same time.

  7. To update another field of multiple records, repeat step 4 to step 6.
  8. Click OK.

Due date calculation options

The due date calculation options are available when you are editing a record in the Due Date Calculation window. The Due Date Calculation window is displayed if your system administrator has made the following settings:

    • The When the Priority changes, recalculate the Due Date without user input check box is clear. This check box is provided on the General Application Settings page.

      • If check box is selected, the Due Date Calculation window is not displayed and the due date is recalculated based on the Use Opened Date instead of Current Date to recalculate Due Date setting.

    • The Open popup dialog for recalculating due date when priority of incident changes workflow is active.

When you are editing a problem and you update the impact or urgency that leads to a change in the priority of the problem, the following due date calculation options appear:

    • Do not recalculate—The existing due date is retained for the problem.
    • Recalculate based on Open Date—The due date is recalculated by using the open date. For example, if the priority is changed to 1 and the duration assigned to a priority of 1 is 2 hours, the recalculated value of the due date is 2 hours added to the date and time when the problem was created.
    • Recalculate based on current date—The due date is recalculated by using the current date. For example, if the priority is changed to 1 and the duration assigned to a priority of 1 is 2 hours, the recalculated value of due date is 2 hours added to the current date and time.

Tracking changes

When you edit a problem, the changes are tracked in the Problem History section. In the Visualforce page layout UI, the changes are tracked in the Auditing tab in the Supporting Information section. When you modify a field in a problem, the following action appears in the Problem History section:

Changed Record Update Count from <x> to <y>.

In this message, x and y are numbers.

In the Visualforce page layout UI, the information about the changes is displayed in the form of a table (Created Date, Created By, Changed Field, Old Value, New Value).

By default, Salesforce allows you to track 20 fields on a form. BMC recommends that you always track the Record Update Count field. For all the fields of the object, this field helps you in tracking the user who makes the change and the time when the record is changed.

If your system administrator is not tracking the Record Update Count field, the following message does not appear in the Problem History section: 

Changed Record Update Count from <x> to <y>.

Contact your system administrator to change the tracked fields.
