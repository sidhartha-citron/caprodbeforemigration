The Alignability Process Model describes release management processes in a simple structure. The model provides the easiest and fastest possible access to detailed information that tells the users what needs to be done in the relevant situation. This structure ensures that detailed instructions are always only clicks away from the model itself. By default, if the Alignability tab is not visible, contact your system administrator to configure the Alignability tab.

The Release Management process of the Alignability Process Model describes how to work on releases in a prioritized manner.

To view the Alignability Process Model for Release Management

  1. Navigate to the Alignability tab.
  2. In the Alignability Process Model, click the Release Management process.
  3. If you are using the Visualforce page layout UI, perform the following actions:
    1. Click Workspaces.
    2. From the list of Workspaces, expand Incident Management.
    3. In the Incident Management list, click Incident Management Alignability Process Model Instructions.

      4. The Release Management process of the Alignability Process Model is displayed.

  4. Perform any of the following actions:
    1. You can access one of procedures of the Release Management process by clicking one of the boxes that make up the process.
    2. To access the work instructions for a step within a procedure, click the action box or decision diamond that symbolizes the step.

      3. The action box symbolizes an action that needs to be taken. One arrow always points to an action box. One arrow points from an action box when further action is required after it has been completed.

      The decision diamond symbolizes a decision that is to be taken. One arrow always points to a decision diamond. Two arrows always point from a decision diamond.

      The event circle symbolizes an event or a situation that has occurred. It is not possible for an arrow to point to an event circle. One arrow always points from an event circle.

    3. In each procedure, click Role to view a description of the roles and responsibilities of the current process.
