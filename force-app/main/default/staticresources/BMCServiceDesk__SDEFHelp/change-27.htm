You can create knowledge articles from a change request. The knowledge articles that you create from a change request are displayed in the Linked Knowledge Articles section.

To create a knowledge article from a change request

  1. Navigate to the required change request.

    2. For more information, see Navigating to a change request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Agent Tools menu, select Create Knowledge Article.

      • If this option is not available in the Agent Tools menu, contact your system administrator to add it to the menu.

    • If you are using the Change Requests tab (Salesforce page layout), click Create Knowledge Article.

      • If this button is not available on the form, contact your system administrator to customize the incident layout and add it on the form.

  3. In the Title field, enter a title for the knowledge article.

    4. The first 30 characters of the Description field of the change request appear in the Title field. You can choose to enter a different title. A maximum of 255 characters are allowed in this field.

  4. From the Type field, select the required type of the knowledge article, and click Create.

    5. The article is saved in the Draft status.

    Based on the type of knowledge article you are creating, the data of some of the fields of the change request is saved in the appropriate fields of the knowledge article. The following table provides such fields of change request and the corresponding fields of knowledge article where data is saved:

  5. In the Knowledge Article window, click Edit.
  6. Enter the required details for the knowledge article.

    7. For more information, see Creating a knowledge article.

  7. Click Save.
