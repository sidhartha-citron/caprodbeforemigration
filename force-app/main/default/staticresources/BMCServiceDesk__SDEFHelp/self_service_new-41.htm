Note the following important points about the Remedyforce Chat feature:

    • A staff member can use Remedyforce chat only from the Remedyforce Console tab and they must set the status as Available to chat with clients.
    • At a given time, one staff member can chat with only one client.
    • If required, staff members can create an incident and save the transcript of the chat. The chat transcript is copied to the Description field and the Incident Source is set to Chat if they choose to create an incident from the chat. In case the chat is more than 32000 characters, the first 32000 characters of the chat are copied to the Description field.
    • If you have added custom fields that are required or configured validation rules for fields on the Incident form, staff members have to create an incident manually and copy the chat content to the Description field and set the Incident Source field to Chat.
    • If you are upgrading to BMC Remedyforce 20.14.02, you must add the Chat value to the Incident Source picklist. For more information, see the Adding new options to the Incident Source field in the BMC Remedyforce Upgrade Guide.
    • If a client initiates chat when multiple staff members have set their status as Available from the Remedyforce Console, the chat is assigned randomly to one of the staff members.
