A service request definition (SRD) is a service request template that is available to clients through Self Service. SRDs provide the options required to facilitate the selection and execution of delivery for a service request. These options can include attributes such as categorization, effective start and end dates, or pricing information.

Note: Only system administrators can create, copy, and delete service request definitions. Staff members can view existing service request definitions.

To create a service request definition

  1. Click the Remedyforce Administration tab.
  2. On the Home page, click the Configure Application tile, and from the menu select Request Definitions.

    3. Alternatively, you can also navigate to Remedyforce Workspaces > Service Request Management > Request Definition.

  3. If you are using the Visualforce page layout user interface, perform the following actions:
    1. In the BMC Remedyforce home page, click Workspaces.
    2. From the list of Workspaces, expand Service Request Management.
    3. In the Service Request Management list, click Request Definitions.
  4. In the Request Definitions list view, click .
  5. In the Service Request Title field, type the title of the SRD, such as New Hire Request.
  6. In the Description field, type a description of the SRD.

    7. To add any information in the Rich Text Format (RTF), click Rich Text Description. You can view the embedded RTF text (for example, images, different fonts, comments) in the preview section of the service request in Self Service.

  7. In the Service field, select the service that you have added for the SRD.

    8. Note: Select a service that has defined service offerings. If you select a service that does not have defined service offerings, you cannot select a service offering for the SRD in the Service Offering list and you cannot save the SRD because the Service Offering is a mandatory field on the Request Definition form.

  8. In the Service Offering field, select the service offering for which you are creating the SRD.

    9. When you select a service offering, the Price, Cost, Start Date, and End Date of the service offering are populated in the Price, Cost, Start Date, and End Date fields of the SRD. However, you can change the values in these fields.

    You can use the Price field, Cost field, and the option lists to define the level, cost, and price of the SRD.

    By default, the currency is displayed in U.S. dollars. You can configure the currency used in Service Request Management on the Remedyforce Administration > Application Settings > General Application Setting page.

  9. In the Category field, select the category of the SRD.

    10. The Select From Categories window displays Service Request categories and categories that are available to both Incident Management and Service Request Management. Incident categories appear in this window if they are parent categories to Service Request categories.

  10. In the Service Request Template field, select the Request Details template that you have created.
  11. In the Price Option 1 list, select the first price option for which you have typed a price.
  12. In the Price Option 2 list, select the second price option for which you have typed a price.

    13. For example, if an SRD of providing email services for an Enterprise is sold at $500 a month to a customer, you must enter the following values:

    • Price: 500
    • Price Option 1: Enterprise
    • Price Option 2: Month
  13. In the Cost Option 1 list, select the first cost option for which you have typed a cost.
  14. In the Cost Option 2 list, select the second cost option for which you have typed a cost.

    15. For example, if an SRD of delivering email services for an Enterprise costs $250 a month, you must enter the following values:

    • Cost: 250
    • Cost Option 1: Enterprise
    • Cost Option 2: Month
  15. To define the turnaround time of the request, in the Days and Hours fields, enter the number of days and hours by which the service request must be completed.
  16. In the Start Date field, select the date when the SRD is available to clients.

    17. Note: The Start Date and End Date values of the SRD must be between the Start Date and End Date of the service offering.

  17. In the End Date field, select the date when the SRD is no longer available to clients.
  18. In the URL for More Information field, enter the URL that provides more information about the service request definition.
  19. To add images that represent the type of service request being offered to clients in Self Service, select the appropriate option.

    20. For example, if the service request allows clients to request for a mobile phone, you can add an image of a mobile phone to the SRD. In Self Service, clients can easily identify the service request by viewing the mobile phone image in the Top Requests panel.

    Note: After you have added an image to the SRD, you cannot delete the image.

    Tip: BMC recommends that you use an image with the length and width of 32*32.

    To select an image that is uploaded to the BMC Service Request Icons folder of the Documents tab of your Salesforce.com organization, click Select From Salesforce and perform the following actions:

  20. In the Status list, select the status of the service.
  21. To make the SRD available for Self Service clients and staff members to create service requests, select the Online check box.
  22. To submit the SRD for approval when matching criteria is met, select the Automatically Submit for Approval (when matching criteria is met) check box.

    23. After the service request is approved, the fulfillment records are created.

  23. To make the SRD available for the Self Service mobile application, select the Mobile check box.
  24. Click .
  25. To embed a direct link to the service request definition, use the link in the Self Service Link field. This field is populated automatically only after you save the SRD.

    26. You can embed this direct link on your intranet or send it by using an email message. When the user clicks the link, the corresponding form opens in Self Service.

    Note: The Self Service Link field is displayed only if you are using Self Service 2.0.

    For more information, see Providing direct links for Self Service components.

The request definition is saved. However, you must perform the following actions to complete the request definition:

    • Getting more information about the request
    • Enabling request definitions for accounts or profiles
    • Configuring the service request information to be shown to clients

Also consider if any of the following use cases fit your requirements to create an SRD:

    • Adding a picklist response type
    • Adding radio button options to the service request
    • Defining conditions to display a field
    • Defining conditions to invoke a template

For Self Service clients and staff members to create service requests from the service request definition, you must ensure that the following properties are configured for the service request definition:

    • Clear the Inactive check box to make the service request definition active.
    • Select the Online check box to make the service request definition available.
    • The End Date of the service request definition should be greater than the current date. Otherwise, the service request definition is not available.
    • The accounts and profiles of the Self Service clients and staff members should be entitled to the service request definition. You can configure the entitled accounts and profiles in the Entitlement tab of the service request definition.
