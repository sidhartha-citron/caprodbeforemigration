System administrators can create action records to allow users to open specified URLs on the following object forms:

    • Change Request
    • Incident
    • Problem
    • Task

Users can access these custom actions on these object forms in the Actions menu. For example, you can create a custom action, Launch Wikipedia, to enable your users to launch http://en.wikipedia.org/w/index.php on the Incident form.

You can create a custom action to launch the URL in one of the following ways:

    • Remedyforce tab
    • browser tab
    • browser window

You can add the following types of parameters to the custom action URLs:

    • Static parameters: The value of such parameters remains unchanged while launching the URL. For example, to enable your users to search for Hardware on the Wikipedia website, you can add "hardware" as a parameter to the custom action URL. When you create a new record of an object, the custom actions with only static parameters are enabled.
    • Dynamic parameters: The value of such parameters is replaced with actual values when the URL is launched. On the object form, the custom actions with dynamic parameters are enabled only after the custom action record is saved. Following types of dynamic parameters are available:
      • Global objects - User and Profile: You can use these parameters to provide values to the URL based on fields that are available on these objects at runtime. For example, User.Email, Profile.Name, and so on. These parameters are replaced with the associated fields of the logged-on user and profile who is using the object form on which the custom action is defined.
      • Formâ€™s Objects (objects include Incident, Task, Change Request, and Problem): You can use the fields from these objects in the URL. For example, you can use search=<Category~>, where Category~ is a field that is available in the Incident object. The values are replaced at runtime based on the record which is opened in the corresponding form.

When a custom action is launched, the launch parameters are appended to the URL, for example, http://en.wikipedia.org/w/index.php?search=Hardware.

You can add both absolute and relative URLs. Absolute URL, such as http://www.yahoo.com/, is a URL or resource to access the external system that is not part of the current application. Relative URL, such as .../setup/Personal Setup/, is a URL or resource that is available in the current application.

To create a custom action

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Custom Actions.
  4. In the Custom Actions List view, click New.
  5. In the Action Name field, enter the name of the custom action.

    6. This is the name of the action that is displayed to the user in the Action menu of the selected object form. Action name must be unique in every object. However, you can create a custom action with same name for more than one object. For example, you can have Launch Wikipedia custom action in both the Incident and Change Request objects.

    Note: BMC recommends that you enter the name of the action in the <action> <URL or website name> format. For example, Launch Wikipedia.

  6. From the Form list, select the object on which you want to make the custom action URL available.

    7. By default, Change Request is selected.

  7. In the Launch URL field, enter the URL that you want to access.
  8. In the Launch Parameters section, enter the name of the launch parameter.

    9. You can either select a field on the object form and the value of the field would be used when the URL is launched. Alternatively, you can enter a constant value to be used as the value of the parameter.

  9. To select a field of the object as the value of the parameter, select the field from the Select Field list. Alternatively, to enter a constant value, select the Enter Constant option button and enter the value in the text box.

    10. For example, To enter a dynamic parameter, enter search as parameter name and select required field name from the option list, such as Profile.Name. To enter a static parameter, enter search as parameter name, select the Enter Constant option button, and enter hardware in the text box.

  10. Click Add.

    11. To remove a parameter, select it and click Remove. To modify a parameter, select it and click Update.

    The preview of the URL is displayed in the Preview field. If you enter more than one parameter in the URL, parameters are concatenated with & in the URL.

  11. From the Launch In field, select an appropriate option to launch the URL.

    12. You can select to launch the URL in one of the following ways:

      • Remedyforce tab - when the user launches the custom action, the URL opens in a new tab in BMC Remedyforce.
      • Browser tab - when the user launches the custom action, the URL opens in a new browser tab.
      • Browser window - when the user launches the custom action, the URL opens in a new browser window.

        • Note: To launch the custom actions in Internet Explorer 7 or 8 in the way that you have selected in the Launch In field, select Tools > Internet Options > Tabs > Settings. In the When a pop-up is encountered section of the Tabbed Browsing Settings window, select the Let Internet Explorer decide how pop-ups should open option.

  12. In the Action For Use By section, perform the following actions:
    1. If required, clear the Everyone check box to allow the selected profiles to use the custom action.

      2. By default, the Everyone check box is selected and the custom action is available to all the users.

    2. In the Available Profiles list, select the profiles that can use the custom action.
    3. To move the selected profiles from the Available Profiles list to the Selected Profiles list, click the right arrow.
    4. (Optional) To move the selected profiles from the Selected Profiles list to the Available Profiles list to prevent the selected profiles from using the custom action, click the left arrow.
  13. Click Save.

The Form list is disabled after you save the custom action record. When you copy a custom action record, the Form list is disabled.

If you modify a custom action record, the modified custom action would be available to users when they open a new object form.
