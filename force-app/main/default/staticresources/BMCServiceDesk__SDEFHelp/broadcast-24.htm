BMC recommends that you do not edit the fields on the broadcast form of a closed record. You can edit the linked records of linked objects. However, you cannot link another record by using the New Action or New Incident button, or create and link another record by using the Create and Link to Broadcast button.

When closing a broadcast, you can also choose to close all the linked incidents because they are no longer required. If you have selected a closure category for the broadcast, that category is also applied in the Closure Category field of the linked incidents that are closed. Applying the same closure category to the linked incidents ensures that the reason specified for closing the broadcast and all linked incidents is consistent.

To close a broadcast

  1. Navigate to the broadcast that you want to close.

    2. For more information, see Navigating to a broadcast record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Actions menu, select Close.
    • If you are using the Broadcasts tab (Salesforce page layout), click Close.
    • If you are using the Visualforce page layout user interface, from the Actions menu, select Close.

      • The Close Broadcast form appears with the Closed Date field filled with the current date and time. The Category field displays the category that is entered in the Broadcast form.

      Note: This procedure describes the out-of-the-box fields only. If your system administrator has added or removed fields, complete all the fields on the form.

  3. In the Close Broadcast form, from the Closed By list, select the staff member who has worked on the broadcast.

    4. By default, this field displays the Staff ID of the currently logged-on staff member. If you are closing the broadcast for another staff member, select the required staff member.

  4. From the Status list, select the appropriate closed status.

    5. By default, this field is filled in with the default closed status that is configured by the system administrator. If required, you can use this field to select a different closed status. All status records with the state of closed are displayed in the Status list.

  5. From the Closure Category list, select the category that indicates the reason for closing the broadcast.
  6. In the Expiration Date field, select the date and time when you want to stop displaying the broadcast message.
  7. If the original category was incorrect or no longer applies, assign a new category to the broadcast.
  8. In the Resolution field, type the resolution of the broadcast.

    9. By default, this field displays the value of the Resolution field in the Broadcast form. You can modify this field to include the confirmed resolution or any new steps taken to resolve the broadcast.

  9. To add the contents of the broadcast form Description field as a new standard description for the category of this broadcast, select the Update Standard Description check box.

    10. You might need to add a new standard description for a frequently occurring issue or outage that belongs to the category of the current broadcast.

  10. To add the contents of the broadcast form Resolution field as a new standard resolution for the standard description of the issue, select the Update Standard Resolution check box.

    11. You might need to add this resolution for the standard description of this issue because it provides the resolution information or an alternative method of resolving the issue.

  11. To close all incidents associated to this broadcast, select the Close Incident check box.

    12. When you have resolved the broadcast, you might want to close all linked incidents because these incidents are no longer required.

    If any linked incident has open tasks, and your system administrator has selected the Closing Records with Linked Open Tasks check box for the Incident object, you cannot close the broadcast and linked incidents. You must close the open tasks and then close the broadcast and linked incidents.

    The closure category of the broadcast is also applied to the Closure Category field of the linked incidents that are closed. If the broadcast closure category is not available in the Closure Category field on the Incident object, the default value is applied to the linked incidents. If your system administrator has not configured a default value, the Closure Category field is not populated for the linked incidents.

    Note: If linked records, such as linked incidents, are assigned to a staff member who is higher in the role hierarchy than the staff member who is closing the record, these linked records might not be closed if the Private sharing model is applied on the linked object. By default, BMC Remedyforce has configured the Private sharing model for the Incidents and Tasks objects. However, administrators can also configure the Private sharing model for Change, Broadcast, and Problems objects.
