System administrators can configure BMC Remedyforce to calculate the total work time for an incident, based on the business hours of your organization.

To enable you to track the total work time for incident records, your system administrator must configure the appropriate out-of-the-box or custom status records as work status. The work clock calculates only the time for which an open incident is assigned to a status that is configured as a work status. For example, the Opened or In Progress status records can be configured as work status. In this case, the time spent on an incident in the Opened or In Progress status is considered in the total work time. However, the work clock does not calculate the time spent on that incident in any other status, such as Waiting for Reply or Approval Requested.

To view the total time spent working on an incident, you can create a custom view for incident and service request records and add the following fields to the custom view. For information about creating a custom view on the Remedyforce Console tab or the Incident Console tab, see Creating or editing a custom view for incident or service request records.

    • Work Clock Status — Displays the status of the work clock that is used to calculate the actual work time for an incident.

      • Work clock status
      Description
      Running
      The incident status is assigned to the Opened state and has the Work Status check box selected.
      Paused
      The incident status is assigned to the Opened state but does not have the Work Status check box selected.
      Stopped
      The incident status is assigned to the Closed state.

    • Total Work Time — Displays the total time for which staff members are actively working on an incident during your organization’s business hours. This field calculates only the time spent on an incident when the work clock is in the Running status. Also, the total work time is updated only when you change the incident status.

      • The Total Work Time field can be used in reports to track the total time for which staff members are actively working on resolving incidents.

    • Time Spent in Current Status — Displays the total time for which an incident is assigned to the current status. This field does not consider the business hours configured for an organization. Also, the value of this field is reset to 0 (zero) each time you change the status of an incident.
