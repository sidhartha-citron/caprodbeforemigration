You can find similar incidents that are reported for an issue. If required, you can link such incidents with an incident. All linked incidents are treated as children incidents of the incident with which these incidents are linked. The incident with which you link other incidents is treated as the parent incident.

You can specify the fields whose values you want to match in other incident records. Your system administrator configures such fields. If you want to match value of a field that is not available, contact your system administrator. Your system administrator must add that field to the Incident Matching field set.

To search for matching incidents

  1. Navigate to the incident record whose matching records you want to search.

    2. For more information, see Navigating to an incident or a service request record.

  2. Perform one of the following actions:
    • If you are using Remedyforce Console tab, from the Agent Tools menu, select Incident Matching.
    • If you are using the Incident Console tab, from the Agent Tools menu, select Incident Matching.
    • If you are using the Incidents tab (Salesforce page layout), click Incident Matching.

      • The Incident Matching button is not provided by default. Contact your administrator to customize the Incident layout and add the Incident Matching button.

  3. In the Incident Matching window, click the Match By list, select the required fields whose data you want to match in other records, and click Apply.

    4. The values in the specified fields of the opened incident record are matched with the values in other incident records.

  4. (Optional) If you know that the incident records that are searching have some common alphanumeric text, enter that text in the text box and click Search.

    5. You can search for text that appears in all the incident fields except fields of type Lookup. The records that match the fields selected in the Match By list and the text that you are searching are displayed.

  5. To link records that are matching the current incident, perform the following actions:
    1. From the list of matching records, select the check boxes for the records that you want to link.

      2. If you have linked some incident records, those records are not displayed in the list.

    2. From the Set Selections To field, select the status that you want to apply to the incidents that you are linking.

      3. For example, if the value in the Set Selections To field is set to Closed, the matching records that you link to the incident have the Status field value set to Close.

      While creating a status or by editing a status record, system administrator can configure a status value as a default linking status in the Status menu. If the system administrator has not configured any default linking status, the default status of the Closed state is displayed in the Set Selections To field by default.

    3. Click Link.

      4. You can view the linked incidents in the Linked Incidents related list. In Salesforce page layout, this related list is not displayed on the incident details form by default. Contact your system administrator to customize the incident form layout and display the Linked Incidents related list on your form.
