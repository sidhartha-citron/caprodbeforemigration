You need object-level permission to edit a release record. You cannot edit a closed release record. If a release is not assigned to you, you can edit it only if you meet one of the following conditions:

    • The release is assigned to a release owner who is lower in the Role Hierarchy than you are and you have Read Access to the Release custom object.
    • The release is assigned to a queue, and you are a member of this queue.

Tracking changes

When you edit a release, the changes are tracked in the Release History section. All fields on the release form are not tracked.

By default, Salesforce allows you to track 20 fields on a form. The following fields on the release form are tacked by default:

Budget Available
Due Date
Release Type
Category
Impact
Status
Completion Date
Priority
Urgency
Deployment Type
Record Update Count
 

To apply the date and time to updated records, BMC recommends that you always track the Record Update Count field. For all the fields of the object, this field helps you in tracking the user who makes the change and the date and time when the record is changed.

If you are not tracking the Record Update Count field, the following message does not appear in the Release History section: 

Changed Record Update Count from <x> to <y>.

In this message, x and y are numbers.

If you are tracking the Record Update Count field and you modify a tracked field, two records are created in the Release History section. One record specifies the change and the other record displays the number of times the release is modified. For example, you have changed the status of a release from Opened to Pending. The following messages appear:

Changed Status from Opened to Pending.
Changed Record Update Count from <x> to <y>.

If you are tracking the Record Update Count field and you modify a field that is not being tracked, the following message appears in the Release History section:

Changed Record Update Count from <x> to <y>.

In this message, x and y are numbers.

To change the fields that you want to track, contact your system administrators.

Editing multiple releases

  1. Click the Remedyforce Console tab.
  2. From the View menu, select Releases.
  3. Select the check boxes for the records that you want to edit, and click Update.

    4. You can update only 200 records at the same time.

  4. From the Field list, select the field that you want to update for the selected records.

    5. Only the fields that are shown on the list view are available in the Field list. Fields of type Auto Number and Formula are not available in the Field list. If you do not have permission to edit any of the fields in the list view, the field is not available in the Field list.

  5. Enter or select a value for the selected field.
  6. Click Apply.
  7. To update more values, repeat step 4 to step 6.
  8. Click OK.
