The Remedyforce Console tab is an integrated console that provides access to the Incident/Service Requests, Tasks, Problems, Change Requests, Releases, and Broadcasts modules. The main benefit of the Remedyforce Console tab is that it enables you to manage all records (incident, problem, task, change request, release, and broadcast) from a single console.

For information about working with records on the Remedyforce Console tab, see the following topics:

    • Managing incidents
    • Managing tasks
    • Managing problems
    • Managing changes
    • Managing releases
    • Managing broadcasts

Note: On the Remedyforce Console tab, you can open a maximum of 15 tabs. Each tab uses 30 to 50 MB of RAM, and your computer must have at least 1 GB of available RAM to open 15 tabs. To ensure that the performance of the BMC Remedyforce application is not impacted, close the tabs that you do not require.

The following table describes the key features that the Remedyforce Console tab provides:

Feature
Description
View
Use the View list to access the records of different modules (Incident/Service Requests, Tasks, Problems, Change Requests, Releases, and Broadcasts) from a single console. System administrators can configure the modules that are available in the View list based on profiles.
The view that is selected when you log off from the BMC Remedyforce application is displayed the next time you log on and access the Remedyforce Console tab.
For example, you select Problems from the View list and log off from the BMC Remedyforce application. The next time you log on to the application and access the Remedyforce Console tab, the Problems module is displayed in the View list.
Tabs
Each record is opened in a different tab within the Remedyforce Console tab. To identify the module for which a new record is being created, the following prefixes are added to the tab name:
  • Incidents/Service Requests – IN: New
  • Tasks – TS: New
  • Problems – PR: New
  • Change Requests – CR: New
  • Releases – RM: New
  • Broadcasts – BR: New
Record IDs
You can identify the module of a record that is opened in a tab based on the prefixes that are added to the record ID:
  • Incidents/Service Requests – IN
  • Tasks – TS
  • Problems – PR
  • Change Requests – CR
  • Releases – RM
  • Broadcasts – BR
A few examples of the record IDs are IN0009757 (incident record), TS0009768 (task record), and CR00009878 (change request record).
Keyboard shortcuts
You can use keyboard shortcuts to perform a few common actions on the Remedyforce Console body. For more information, see Keyboard shortcuts.
Related lists
From the Related lists, linked records are opened in a separate tab within the Remedyforce Console tab.
Chatter
If Chatter is enabled for your Salesforce organization, the Chatter feed is displayed above the related lists for a record.
After selecting a record, click Related Lists. To display or hide Chatter feed, click Display Feed or Hide Feed.
Broadcasts
Broadcasts on the Remedyforce Console tab have the following features:
  • The broadcast ticker is available and enables you to follow or unfollow broadcasts on Chatter by clicking .
  • You can link multiple incidents to a broadcast by clicking Link to Broadcast.
  • You can view the active broadcast as a running ticker beside the icon.
List view
By default, all records are displayed in the list view. You can also create a view that suits your requirements. For more information, see https://help.salesforce.com/htviewhelpdoc?id=customviews.htm&siteLang=en_US.

Note:  The record list displays a maximum of 10,000 records.

The list view for incidents and service requests includes icons to provide additional information about the records. For more information, see Icons in the list view for incidents and service requests.
System administrators can configure the columns in the list view and the autorefresh time for the list view.
You can resize column widths in the list views. The configured column widths are saved for the browser and the computer from which you log on. If another user uses the same browser on the same computer, the saved column widths are shown. However, if the system administrator reconfigures the list view, you must resize the columns.
Custom actions
Any custom action that the system administrator has added is displayed as a button above the related lists.
Search
In the list view, in lookup windows, in Link <object> windows, and in the Matching Incidents window, you can search fields of type Name, Email, Phone, Text, Text Area, Long Text Area, and Rich Text Area. However, fields of type Formula, Lookup, Rollup Summary, Number, and Date and Time are not searched. Also, the search operation matches the beginning characters of the value in a field. Therefore, ensure that you always provide the correct beginning characters for the field value for which you are searching.
For example, you are searching for John but you enter ohn. The records that contain John are not displayed. However, if you search for John, all records that contain John (including, for example, Johns or Johnson) are displayed.

Note:  On the Remedyforce Console tab, if you perform a search, a maximum of 2000 records are displayed as results.
Change Owner
Users can change the owner while creating a record in the Remedyforce Console tab.
Additional work space
On the Remedyforce Console tab, you can increase the space that is available for working with records. To hide the tabs, click the icon on the top below the tabs and use the additional space for working with the records.
If you use the icon to hide the tabs and you log off from the BMC Remedyforce application, the additional space is available the next time you log on and access the Remedyforce Console tab.
Chat Status
If your system administrator has enabled the Remedyforce Chat - BETA, you must set your status as Available to accept client chat request. Note the following important points:
  • If you have set the status as available, you receive the notifications only if the Remedyforce Console tab is active.
  • When you close the chat, you have an option to create an incident from the chat. If you choose to create an incident, an incident is created automatically and the chat text is saved in the Description field and the value of the Incident Source field is set as Chat.

    • If your system administrator has added custom fields that are required or configured validation rules for fields on the Incident form, you must create an incident manually and copy the chat content to the Description field and set the Incident Source field to Chat.

Icons in the list view for incidents and service requests

In the list view, the Due Date Progress column uses icons to display the status of incidents and service requests. The following table describes the icons that are used in the Due Date Progress column:

Icon
Description

The due date of a record has passed.

More than 50% of the available time remains before the due date.

Less than or equal to 50% of the available time remains before the due date.

The due date of a record is met and the record is closed.

The due date of a record is not met and the record is closed

The New Incident and VIP columns also use icons to provide additional information about the records, as described in the following table:

Column
Icon
Description
New Incident

If a record is new, the icon is displayed in the New Incident column. A record is not considered new if a staff member performs an action from the Actions menu, changes the status or owner of the record, or updates the Responded Date field of the record.
VIP

If a record is created for a VIP client, the icon is displayed in the VIP column. To mark a client as a VIP, contact your system administrator to add the VIP field in the Main Panel field set of the User object (Setup > Customize > Users > Field Sets). When the system administrator adds the VIP field to the Main Panel field set, the VIP field is displayed on the Client form (Remedyforce Administration > Manage Users > Clients).
