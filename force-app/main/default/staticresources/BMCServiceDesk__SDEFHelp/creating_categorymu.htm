Categories allow you to create classifications for the data you want to group or track for reporting purposes. You can organize your data by using category types and defining parent and child relationships between categories. Plan your categories so that the data in your system is consistent.

To create a category

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Category.
  4. In the Categories List view, click New.
  5. In the Category field, type a unique identifier that you want to assign to the current category.
  6. In the Abbreviation field, type a short label that you want to assign to the category.

    7. When you create charts or reports, you can use this field to represent the current action record in the chart or report. The field uses minimal screen space, which allows you to display more data in a chart or report.

  7. From the Category Type list, select the type of category.
  8. From the Parent Category list, select the category that should be designated as the parent category.

    9. When a parent category is selected, the current category becomes a child category to the selected parent category. Assigning a parent category to a category creates the category tree.

    Note: BMC recommends that you assign a parent category that is of the same type as the current category. For example, if you select the Available for Incidents check box for the current category, assign an Incident category as the parent category.

  9. To designate the category as available to Incident Management, select the Available for Incidents check box.

    10. Note: If you select the Available for Service Requests and the Available for Incidents check boxes for a category, the category is available to both Incident Management and Service Request Management. This category appears in the Select From Categories window, is part of the AutoComplete feature, appears in the Category tree of Self Service, and is available for defining the Conditional and Invoke Template properties of Input fields.

    When you select this check box, the following sections are affected:

    • In the Incident form, when you click the Category list to open the Select From Categories window, the current category appears by default in this window. If you want to select a Service Request category for the incident, select Filter Options > Service Request Categories to view the Service Request categories.
    • In the Incident form, when you type the current category in the Category list, the current category appears in the list of matching entries. Service Request categories do not appear in the list of matching entries.
    • In Self Service, when you click the Request Help button and then click the Category list to open the Select From Categories window, the current category appears in this window. The Display in Self Service check box must be selected for the current category to appear in Self Service.
    • In Self Service, when you click Browse in the upper right section of the Self Service Home page and then click the Request a Service tab, the current category does not appear in the Category tree. Incident categories appear in the Category tree if they are parent categories to Service Request categories. The Display in Self Service check box must be selected for the current category to appear in Self Service.
  10. To designate the category as available to Service Request Management, select the Available for Service Requests check box.

    11. When you select the Available for Service Requests check box, BMC Remedyforce automatically selects the Display in Self Service check box. In most cases, you want the Service Request categories to be displayed in Self Service. Self Service clients can use these categories when they are creating service requests. You can clear the Display in Self Service check box if you do not want the category to appear in Self Service.

    Note: If you select the Available for Service Requests and the Available for Incidents check boxes for a category, the category is available to both Incident Management and Service Request Management. This category appears in the Select From Categories window, is part of the AutoComplete feature, appears in the Category tree of Self Service, and is available for defining the Conditional and Invoke Template properties of Input fields.

    When you select this check box, the following sections are affected:

    • In the Request Definition form, when you click the Category list to open the Select From Categories window, the current category appears by default in this window. Incident categories appear in this window if they are parent categories to Service Request categories.
    • In the Fulfillment tab of the Request Definition form, only Service Request categories are available for defining the Conditional and Invoke Template properties of Input fields. For more information, see Creating service request definitions.
    • In Self Service, when you click the Request Help button and then click the Category list to open the Select From Categories window, Service Request categories are not displayed in this window.
    • In Self Service, when you click Browse in the upper right section of the Self Service Home page and then click the Request a Service tab, the current category appears in the Category tree. The Display in Self Service check box must be selected for the current category to appear in Self Service.
    • While submitting a service request in Self Service or BMC Remedyforce, if you click an Input field that is a lookup to the Category field, the current category appears in the Select From Categories window. The Display in Self Service check box must be selected for the current category to appear in Self Service.
  11. To display the category to all the users of BMC Remedyforce when they log on to Self Service, select the Display in Self Service check box.

    12. If the Display in Self Service check box is selected for child categories of current category and you clear this check box for the current category, an error message appears when you save the current category. The Display in Self Service check box must remain selected for the current category because child categories of the current category are displayed in Self Service and the current category is required to create the Category tree.

    When you select the Display in Self Service check box for the current category and save the category, this check box is selected for the parent categories of the current category. For example, you have the following categories:

    • Human Resource
    • Recruitment
    • Interview

      • Human Resource is the parent category of Recruitment, and Recruitment is the parent category of Interview. When you select the Display in Self Service check box for the Interview category, this check box is selected for the Human Resource and Recruitment parent categories.

      Note: All categories are available in the Change Request, Broadcast, Problem, Template, and Task forms. The Filter options menu does not appear in these forms when you are selecting a category. Selecting the Available for Incidents, Available for Service Requests, or Display in Self Service check boxes does not affect the availability of categories in these forms.

  12. In the Incident Handling Information section, enter the following information:
    1. Follow Up—Select this check box to designate incidents for follow-up when the current category is assigned to an incident.

      2. These incidents are followed up when the incident is closed. You can define the follow-up time in the After Hours field.

    2. After Hours—Type the number of hours after an incident is closed when a follow-up should begin.

      3. This information is added to the closing data of an incident assigned with the current category.

    3. Urgency ID—Select the urgency you want to assign when creating an incident with the current category.

      4. You can assign a level of urgency to the category depending on the importance of the category.

  13. In the Supporting Information section, click the Child Categories tab.
    1. In the Child Categories tab, click Actions > Add.
    2. In the Categories window, repeat step 5 through step 16 to create the category that you want to add as a child category to this category.
    3. Click Save.

      4. The Categories window closes, and the child category appears in the Child Categories tab.

  14. In the Supporting Information section, click the Standard Descriptions tab.
    1. In the Standard Descriptions tab, click Actions > Add.
    2. In the Standard Description window, create the standard description that you want to add to this category.

      3. For more information, see Creating standard description records.

  15. In the Supporting Information section, click the Support Experts tab.

    16. In the Supporting Information section at the lower section of the form, you can assign support experts and add standard descriptions, action records, and child categories.

    1. In the Support Experts tab, click Actions > Assign Support Expert.
    2. In the Select From Staff window, select the staff member who is an expert on this category of incidents, tasks, and broadcasts.

      3. When a staff member creates an incident, task, or broadcast of this category and selects Assign To > Suggested Expert, the expert staff member appears in the list of staff members available as suggested experts.

    3. In the Support Experts tab, click Actions > Assign Support Queue.
    4. In the Select From Queue window, select the queue that comprises the staff members who are experts on this category of incidents, tasks, and broadcasts.

      5. When a staff member creates an incident, task, or broadcast of this category and selects Assign To > Suggested Expert > Views > Queue, the queues that are created for the selected object (incident, task, or broadcast) and category are displayed as suggested experts.

      In Salesforce, if you edit the name of a queue that is assigned to a category, the modified name of the queue is not displayed in BMC Remedyforce. You must unassign the queue by using the Actions > Unassign Support Experts option and reassign the queue. Also, if you delete a queue that is assigned to a category, you must unassign the queue by using the Actions > Unassign Support Experts option.

    5. (Optional) If an expert staff member or members of queues move to another category, click Actions > Unassign Support Experts to remove the experts from this category.
  16. Click Save.
  17. (Optional) To designate the current record as inactive, select the Inactive check box.

    18. For example, the category for an outdated operating system is no longer applicable because computers in the account do not use the outdated operating system.

    Note: Records marked as inactive are not included in any search performed on the database.
