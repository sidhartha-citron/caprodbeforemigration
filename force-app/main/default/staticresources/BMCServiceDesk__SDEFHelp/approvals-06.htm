You can configure BMC Remedyforce to allow creators of incidents and change requests to select an approver from a custom lookup field on the User object. You must create this custom user lookup field for the Incident and Change Request object and add it to the Incident and Change Request forms.

After you create the custom user lookup field, you must configure your approval process to automatically assign all incidents and change requests to the related users specified in the custom user lookup fields that you have created.

To assign approvers by using custom user lookup fields

  1. To create a custom user lookup field for the Incident object, perform the following actions:
    1. Navigate to Setup > Create > Objects.
    2. Click Incident in the Label column.
    3. In the Custom Fields & Relationships section, click New.
    4. In the Step 1. Choose the field type section, select Lookup Relationship in the Data Type column.
    5. Click Next.
    6. In the Step 2. Choose the related object section, select User from the Related To list.
    7. Click Next.
    8. In the Step 3. Enter the label and name for lookup field section, type a label for the custom user lookup field in the Field Label field.

      9. For example, type Incident Approver in the Field Label field.

    9. Press the Tab key to populate the Field Name field with the value of the Field Label field.

      10. The spaces in the label are replaced with underscore characters in the Field Name field. If required, edit the value of the Field Name field so that it adheres to the following validation rules:

      • Can contain only alphanumeric characters
      • Must begin with a letter
      • Cannot end with an underscore or contain two consecutive underscore characters
      • Must be unique across all Incident fields
    10. Click Next.
    11. In the Step 4. Establish field-level security for reference field section, select the profiles to which you want to grant edit access to the custom user lookup field.

      12. Note: Ensure that the ServiceDesk Change Manager, ServiceDesk Staff, and System Administrator profiles are selected.

    12. Click Next.
    13. In the Step 5. Add reference field to Page Layouts section, ensure that the Remedyforce Incident check box is selected.
    14. Click Save.
    15. Repeat step a through step n to create multiple custom user lookup fields for the Incident object.
  2. To create a custom user lookup field for the Change Request object, perform the following actions:
    1. Navigate to Setup > Create > Objects.
    2. Click Change Request in the Label column.
    3. In the Custom Fields & Relationships section, click New.
    4. In the Step 1. Choose the field type section, select Lookup Relationship in the Data Type column.
    5. Click Next.
    6. In the Step 2. Choose the related object section, select User from the Related To list.
    7. Click Next.
    8. In the Step 3. Enter the label and name for lookup field section, type a label for the custom user lookup field in the Field Label field.

      9. For example, type Change Request Approver in the Field Label field.

    9. Press the Tab key to populate the Field Name field with the value of the Field Label field.

      10. The spaces in the label are replaced with underscore characters in the Field Name field. If required, edit the value of the Field Name field so that it adheres to the following validation rules:

      • Can contain only alphanumeric characters
      • Must begin with a letter
      • Cannot end with an underscore or contain two consecutive underscore characters
      • Must be unique across all Change Request fields
    10. Click Next.
    11. In the Step 4. Establish field-level security for reference field section, select the profiles to which you want to grant edit access to the custom user lookup field.

      12. Note: Ensure that the ServiceDesk Change Manager, ServiceDesk Staff, and System Administrator profiles are selected.

    12. Click Next.
    13. In the Step 5. Add reference field to Page Layouts section, ensure that the Remedyforce Change Request check box is selected.
    14. Click Save.
    15. Repeat step a through step n to create multiple custom user lookup fields for the Change Request object.
  3. To edit an existing Incident approval process to assign incidents to the staff members who are selected from the custom user lookup fields, perform the following actions:
    1. Click the Remedyforce Administration tab.
    2. Click the Manage Workflow and Approvals tile, and from the menu select Approval Processes.
    3. If you are using the Visualforce page layout user interface (UI), navigate to Setup > Create > Workflow & Approvals > Approval Processes.
    4. From the Manage Approval Processes For list, select the Incident object.
    5. Click <Incident Process Name> in the Approval Process Name column.
    6. In the Approval Steps section, in the Action column, click Edit.
    7. In Step 1. Enter Name and Description, click Next.
    8. In Step 2. Specify Step Criteria, click Next.
    9. In the Select Approver section, select Automatically assign to approver.
    10. Select Related User from the first list that appears.
    11. In the second list, select the custom user lookup field that you have created for the Incident object.

      12. In the Incident form, when the incident creator selects a staff member from the custom user lookup field and submits the incident for approval, the approval is assigned to the selected staff member.

    12. Click the Add Row link to add another approver to the list of approvers.
    13. Repeat step j through step l to enable incident creators to assign multiple staff members as approvers in the custom user lookup fields that are added to the Incident form.
  4. To edit an existing Change Request approval process that assigns change requests to the staff members who are selected from the custom user lookup fields, perform the following actions:
    1. Click the Remedyforce Administration tab.
    2. Click the Manage Workflow and Approvals tile, and from the menu, select Approval Processes.
    3. If you are using the Visualforce page layout UI, navigate to Setup > Create > Workflow & Approvals > Approval Processes.
    4. From the Manage Approval Processes For list, select the Change Request object.
    5. Click <Change Request Process Name> in the Approval Process Name column.
    6. In the Approval Steps section, in the Action column, click Edit.
    7. In Step 1. Enter Name and Description, click Next.
    8. In Step 2. Specify Step Criteria, click Next.
    9. In the Select Approver section, select Automatically assign to approver.
    10. Select Related User from the first list that appears.
    11. In the second list, select the custom user lookup field that you have created for the Change Request object.

      12. In the Change Request form, when the change request creator selects a staff member from the custom user lookup field and submits the change request for approval, the approval is assigned to the selected staff member.

    12. Click the Add Row link to add another approver to the list of approvers.
    13. Repeat step j through step l to enable change creators to assign multiple staff members as approvers in the custom user lookup fields that are added to the Change Request form.
  5. If you are using the Salesforce page layout, edit the form layout and add the required field on the form.

    6. For more information, see Customizing forms in Salesforce page layout.

  6. If you are using the console layout UI, add the required fields to the field sets provided for the console layout.

    7. For more information, see Customizing forms for the Remedyforce Console tab.

  7. If you are using Visualforce page layout UI and the forms cannot be customized, perform the following actions:
    1. Navigate to Configuration > Configuration > Application Settings > Custom Fields tab.
    2. Select Incident from the Custom Fields to Display For list.
    3. In the Available Fields list, select the custom user lookup field that you want to add to the Incident form.

      4. For example, select Incident Approver in the Available Fields list.

      Note: You have configured this field as the related user, who is an approver in the approval process.

    4. Click the right arrow to move the selected field from the Available Fields list to the Selected Fields list.

      5. Incident Approver appears in the Selected Fields list.

    5. Click Save.
    6. Select Change Request from the Custom Fields to Display For list.
    7. In the Available Fields list, select the custom user lookup field that you want to add to the Change Request form.

      8. For example, select Change Request Approver in the Available Fields list.

      Note: You have configured this field as the related user, who is an approver in the approval process.

    8. Click the right arrow to move the selected field from the Available Fields list to the Selected Fields list.

      9. Change Request Approver appears in the Selected Fields list.

    9. Click Save.
  8. If you are using the Visualforce page layout UI and the forms can be customized, add the required fields to the field set, and assign the custom form to the required profiles.

    9. For more information, see Customizing forms.

  9. In the BMC Remedyforce home page, click Workspaces.
  10. To submit an approval for an incident by using custom user lookup fields, perform the following actions:
    1. From the list of Workspaces, expand Incident Management.
    2. In the Incident Management list, click Incidents.
    3. In the Incidents list view, select the incident that you want to submit for approval by using one of the following options:
      • Select a field from the Group By list, navigate to the required group, and select the required incident.
      • Type a search string in the Search field, click Search, and select the required incident from the search results.
      • Scroll through the list of incidents, and select the required incident.
    4. Double-click the selected incident.
    5. In the Custom Fields section, select the staff members who must approve the incident from the custom user lookup fields.

      6. You must select staff members for all the custom user lookup fields; otherwise, you receive the Manager undefined error message when you submit the incident for approval.

    6. Click Save.
    7. Click the Approvals tab in the Supporting Information section.
    8. Click Actions > Submit For Approval.

      9. You can create only one approval at a time for the incident. After you click the Submit For Approval command in the Actions menu, this command is disabled until the approver approves or rejects the incident, or the submitter recalls the approval. If you require approvals from multiple approvers, you must add multiple custom user lookup fields to the Incident form by performing step a through step n and configure the approval process to include these approvers by performing step a through step l.

    9. In the Comments text editor, type the comments that are included in the approval.
    10. Click OK.
  11. To submit an approval for a change request by using custom user lookup fields, perform the following actions:
    1. From the list of Workspaces, expand Change Management.
    2. In the Change Management list, click Change Requests.
    3. In the Change Requests List view, select the change request that you want to submit for approval by using one of the following options:
      • Select a field from the Group By list, navigate to the required group, and select the required change request.
      • Type a search string in the Search field, click Search, and select the required change request from the search results.
      • Scroll through the list of change requests, and select the required change request.
    4. Double-click the selected change request.
    5. In the Custom Fields section, select the staff members who must approve the change request from the custom user lookup fields.

      6. You must select staff members for all the custom user lookup fields; otherwise, you receive the Manager undefined error message when you submit the change request for approval.

    6. Click Save.
    7. Click the Approvals tab in the Supporting Information section.
    8. Click Actions > Submit For Approval.

      9. You can create only one approval at a time for the change request. After you click the Submit For Approval command in the Actions menu, this command is disabled until the approver approves or rejects the change request, or the submitter recalls the approval. If you require approvals from multiple approvers, you must add multiple custom user lookup fields to the Change Request form by performing step a through step n and configure the approval process to include these approvers by performing step a through step l.

    9. In the Comments text editor, type the comments that are included in the approval.
    10. Click OK.
