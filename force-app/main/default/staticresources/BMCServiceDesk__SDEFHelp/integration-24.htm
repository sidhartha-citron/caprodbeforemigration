BMC Remedyforce uses email services to enable users to create or update incidents. To create or update incidents through email, you must create an email service and add the GenericEmailListener Apex class to the email service.

To create incidents in BMC Remedyforce from BMC Server Automation and BMC End User Experience Management, you must add the EmailCreateIncident out-of-the-box Apex class to the email service.

To create change requests in BMC Remedyforce from BMC Server Automation, you must add the ChangeRequestEmailListener out-of-the-box Apex class to the email service.

After adding the Apex class, you must configure an email address for the email service. The Apex class processes the email that is sent to the email address that you have configured.

For more information about email services, see https://na7.salesforce.com/help/doc/user_ed.jsp?loc=help&target=code_email_services.htm&section=integrate.

For information about the email limits of your Salesforce organization, see http://www.salesforce.com/us/developer/docs/apexcode/Content/apex_gov_limits.htm.

To create an email service

  1. Click the Remedyforce Administration tab.
  2. On the Home page, click the Configure Email tile, and from the menu select Email Services.
  3. If you are using the Visualforce page layout user interface, navigate to Setup > Develop > Email Services.
  4. Click New Email Service.
  5. In the Email Service Name field, type the name of your email service.

    6. Note: BMC recommends that you add BMCRF at the beginning of the name of the email service to enable you to differentiate BMC Remedyforce email services from the other email services that you create.

  6. Click the Apex Class lookup.
  7. In the Lookup window, click the required Apex class.

    8. If you are integrating BMC ProactiveNet Performance Management with BMC Remedyforce, select GenericEmailListener.

  8. From the Accept Attachments list, select All.
  9. To ensure that attachments are not lost or deleted from the email, select the Convert Text Attachments to Binary Attachments check box.
  10. To enable the email service, select the Active check box.
  11. Depending on your organization's requirements, configure the remaining fields.

    12. For more information, see https://na11.salesforce.com/help/doc/user_ed.jsp?section=help&target=code_email_services_editing.htm&loc=help&hash=topic-title.

  12. To save the email service and configure an email address for this email service, click Save and New Email Address.
  13. In the Email Address Information section of the Email Service Address page, in the Email Address field, type the user name of the email address.

    14. Salesforce.com creates and assigns a unique domain name to the email address.

    For more information, see https://na11.salesforce.com/help/doc/user_ed.jsp?section=help&target=code_email_service_address.htm&loc=help&hash=topic-title.

  14. To activate the email address, select the Active check box.
  15. Click the Context User lookup.
  16. In the Lookup window, click the required user.

    17. When an incident is created through email, the incident is assigned to the user configured as the Context User in the email service by default. This user does not receive a notification email when the incident is created. You must create workflows for the Context User to receive email notifications when an incident is assigned to the Context User.

  17. (Optional) In the Accept Email From field, type the valid email addresses.

    18. The email service address accepts emails from these email addresses only.

  18. Click Save.

    19. To create or update incidents, an email must be sent to the email address that is displayed in the Email Addresses section of the Email Service page.
