This appendix lists the out-of-the-box field sets that are available in the Base Element object.

Table C-1:  Field Sets in CMDB 2.0 (Sheet 1 of 9)
Field Set
Description
Access Point
Stores information about communication endpoints in your IT environment.
Account Configuration Item
Stores information about accounts in your IT environment that are used to access specific resources. This field set also stores information about the credentials used to log on to a system.
Activity
Stores information about a function or step in a business process definition.
Admin Domain
Stores information about the following groups:
  • System components, such as software (for example, servers, databases, and applications)
  • Operating system (for example, files, processes, and threads)
  • Devices (for example, disk drives, controllers, processors, and printers)
  • Physical components (for example, chips and cards)
Each group of components is viewed as a single entity, reflecting that all of its components are administered similarly by the same user, group of users, or policy.
Application
Stores information about an instance of an end-user application that supports a particular business function and that can be managed as an independent unit.
Application Infrastructure
Stores information about the framework (for example, SAPÂ® or J2EE) that supports applications in a distributed or composite system.
Application Service
Stores information about the services that represent low-level modules of an application; for example, the components deployed within an application server.
Application System
Stores information about an application or a software system that supports a particular business function and that can be managed as an independent unit.
Base Element
Stores the core attributes of all of the configuration items (CIs); for example Name or Instance ID.
 
The fields of this field set are displayed on the General tab of the CMDB Instance Editor. To customize the fields that must be available for all of the CIs, you must add or delete fields in this field set.
BIOS Element
Stores information about low-level, built-in software used to start a computer's operating system without accessing programs from a disk.
Business Process
Stores information about a structured sequence of activities that forms a business process or an IT process. A business process is performed by a combination of people, computers, and equipment to provide a business service or to deliver a product. An IT process is a business process, such as ITSM Change Management or Incident Management process that is performed by the IT department.
Business Service
Stores information about a structured process:
  • A business service is provided by one business for another, or by one organization for another within the same business (for example, customer support, order processing, and payroll).
  • A technical service is provided by an IT organization to support business services or IT's own operations (for example, employee provisioning, backup and recovery, and self-service help desk).
Card
Stores information about printed circuit boards, such as hosting boards or motherboards, that you can insert into a computer to give it added capabilities.
CD ROM Drive
Stores information about the capabilities and management of CD-ROM drives.
Chassis
Stores information about frames that serve as the structural support for a computer system's electronic components. This field set can store information about physical elements that enclose other elements and provide definable functionality. Examples include desktop computer, a processing node, an uninterrupted power supply (UPS), disk or tape storage, or a combination of such elements.
Cluster
Stores information about sets of two or more computer systems or application systems that operate together as a functional whole, to increase performance, reliability, or both.
Collection
Stores information about collections of CIs related to the people and organizational characteristics of your IT environment.
Column Headers
Stores fields that are displayed in the popup window whenever users open a configuration item lookup. To change the fields that you want to display in the lookup, add or delete the fields in this field set.
Communication Endpoint
Stores information about communication protocol endpoints (communication points from which data can be sent or received) that are dedicated to TCP or user datagram protocol (UDP).
Computer System
Stores information about computer systems. Computer systems can be of the following types:
  • General-purpose computing devices such as servers and workstations.
  • Specific-purpose devices such as network and storage devices.
  • Both physical and virtual systems. For virtual systems, the isVirtual attribute is set to Yes.
Concrete Collection
Stores information about generic and instantiable collection, such as a pool of hosts available for running jobs.
Connectivity Collection
Stores information about sets of protocol endpoints (of the same type) that can communicate with each other.
Connectivity Segment
Stores information about endpoints of the same type (for example, Ethernet, fiber channel, or token ring) that can communicate without the assistance of bridging or routing services.
Contract
Stores information about line items that establish a specific agreement between a provider and a customer.
Contract Line
Stores information about contract line items that establish specific agreements between the provider and the customer.
Cost
Stores information about the cost incurred for providing a service.
Database
Stores information about a logical database (a collection of interrelated data organized into one or more schemas and treated as a unit).
Database Storage
Stores information about a collection of logical storage areas that hold and retain data.
Disk Drive
Stores information about the capabilities and management of disk drives.
Disk Partition
Stores information about disk partitions. A disk partition is a contiguous range of logical blocks (groups of records) that the operating system can identify by the partition's type and subtype fields.
Document
Stores information about all documents related to an IT infrastructure in the CMDB. This field set categorizes any type of documented information in your IT environment, and includes attributes that specify various properties of the documentation. It also includes an attachment field for storing documentation.
Equipment
Stores information about noncomputing equipment, such as vehicles and maintenance tools.
File System
Stores information about systems that operating systems or programs use to organize and keep track of files and datasets. The file system can be local (such as an instance of BMC_ComputerSystem or BMC_ApplicationSystem) or remotely mounted from a file server.
Financial Element
Stores information about the cost and price information associated with a service offering.
Floppy Drive
Stores information about the capabilities and management of floppy drives.
Hardware Package
Stores information about hardware components that are made up of other hardware components. Use this field set to represent hardware components that are not represented by other field sets. For example, chassis and monitors are represented by the fields in the BMC_Chassis Details and BMC_Monitor Details field sets, but no field sets represents RAID disk arrays.
Hardware System Component
Stores information about physical objects, such as disks, disk drives, display screens, keyboards, printers, motherboards, and chips.
IP Connectivity Subnet
Stores information about groups of related IP protocol endpoints that can communicate with each other as members of a subnet.
IP Endpoint
Stores information about an IP address that belongs to a system.
IPX Connectivity Network
Stores information about networks or subnets that use the IPX protocol.
Keyboard
Stores information about keyboards attached to computer systems.
LAN
Stores information about collections of CIs related to IP subnets in a LAN.
LAN Endpoint
Stores information about the addressing information of communication endpoints. When the related interface device of communication endpoints is connected to a LAN, these endpoints can send and receive data frames. Addresses are primarily Ethernet-style MAC addresses.
LNs Collection
Stores information about groups of logical networks (LNs) that are, in turn, collections of endpoints. For example, you can use this field set to group IP subnets that are deployed across the same physical LAN or WAN.
Local File System
Stores information about the organization of files that reside on a computer system (BMC_ComputerSystem Details field set). Such files are directly managed by the computer system without the need for another computer to act as a file server.
In a clustered file system, each file system is stored in the BMC_LocalFileSystem Details field set. To represent the cluster, use the BMC_Cluster Details field set.
Logical Disk
Stores information about a contiguous range of logical blocks that is identifiable by applications such as file systems by using the Name field. For example, on Windows, the Name field might specify the hard disk drive name. On UNIXÂ®, the field might contain the access path /dev/....
Logical Entity
Stores information about all of the components in a system that represent logical system components. For example, a logical entity might be Payroll Processes or North American Sales. These CI types are used to define accounts, business processes, databases, and so on.
Logical System Component
Stores information about logical entities within a system, such as file systems and installed software.
Mainframe
Stores information about mainframe computer systems.
Media
Stores information about devices on which data can be stored, such as hard disks, floppy disks, CD-ROMs, and tapes.
Memory
Stores information about internal storage areas in a computer.
MF Coupling Facility
Stores information about a Mainframe Coupling Facility.
Monitor
Stores information about display screens attached to computer systems.
Network Port
Stores information about a physical or virtual network interface card installed in a computer, including Ethernet interfaces for general purpose networking and Fibre Channel interfaces for connection to SAN storage.
NT Domain
Stores information about sets of computers and devices that are administered as a unit with common rules and procedures on a Microsoft Windows NT network.
Offering
Stores information about service offerings that are part of a high-level service.
Service offerings are container objects that combine the utility (business or technical service) and the warranty (defined by Service Level Targets) to bring value to customers. For example, a business service offering may have an SLT that requires that IT Support deliver the service to customers in no more than two hours. This service offering might be named ERP Gold, based on a service called ERP that is defined in the service catalog.
Business customers do not subscribe to services, they subscribe to service offerings.
Operating System
Stores information about software or firmware that controls the operation of a computer and directs the processing of programs.
Organization
Stores information about business organizations, including business units. An organization is composed of employees and contractors who are providers or consumers of business services. A business unit can be composed of multiple organizations.
Package
Stores information about containers or collections of related software, for example, packages installed by using the platform's package management system.
Patch
Stores information about fixes (service patches) for program defects. For example, IT administrators can use the ServicePackInEffect field to define a specific service pack that a given patch applies to, to ensure that the patch is not propagated to incorrect service pack versions.
Person
Stores information about the people who provide, consume, use, maintain, own, or have some other relationship with the services, systems, and other CIs in your organization.
Physical Location
Stores information about the position and address of physical elements.
Pointing Device
Stores information about devices that you use to control the movement of a pointer to select items on a display screen. Pointing devices include a mouse, digitizing pad, track ball, and touch pad.
Printer
Stores information about local and network printers.
Processor
Stores information about devices that interpret a sequence of machine instructions on a computer.
Product
Stores information about model software products installed on a computer system in your organization.
Protocol End Point
Stores information about communication points from which data can be sent or received. Protocol endpoints link system or computer interfaces to logical networks. The available links are defined by the subclasses of protocol endpoint, communication, IP, and LAN.
Rack
Stores information about hardware rack components, which hold other hardware components within a system.
Remote File System
Stores information about file systems that are not directly connected to a computer system. Instead, these file systems are hosted by a computer that acts as a file server.
Resource Pool
Stores information about a logical entity (with associated controls) provided by the host system to allocate and assign resources.
A given resource pool may be used to allocate resources of a specific type. Hierarchies of resource pools may be created to provide administrative control over allocations.
In cases where resources are subdivided, multiple resource pools may exist (for example, nodal boundaries in NUMA-like systems).
In systems that support over-commitment, pools represent the reserved capacity, not an upper bound or limit on the maximum amount that can be allocated.
Admission control during power-on may detect and prevent systems from powering due to resource exhaustion.
For example, over-commitment on a resource pool with Resource Type=Memory would require that sufficient space be available in a backing store that might be managed through a storage resource pool.
Role
Stores information about the position or responsibilities that a person or system has in an IT organization, or in any other aspect of IT or a business process. One or more people or systems, or a combination of both, can be assigned to a role, and a person or system can have multiple roles.
Service Level Target
Stores information about service level targets (SLTs). SLTs define the warranty of a service and the key performance indicators (KPIs) used to measure the performance of a service offering.
Share
Stores information about components, such as files, folders, or printers, that are shared by multiple computer systems or users.
Software
Stores information about installed software elements for a particular platform.
Software Server
Stores information about a single piece of software directly running (or otherwise deployed) on a single computer.
Storage Extent
Stores information about a contiguous range of logical blocks on a storage medium that supports storing and retrieving data, for example, an external hard disk.
Storage Subsystem
Stores information about a storage subsystem.
Each storage system can be either a direct access storage device (DASD) subsystem type (for example, a disk), or a tape subsystem type (a tape).
Storage Volume
Stores information about a raw storage extent that a virtual or physical machine can access. The storage extent has a volume manager to manage the storage directly, from the host machine or from the network storage. For example, in a LinuxÂ® environment, one or more physical volumes (PVs) are created from a disk drive, and logical volumes are created from these PVs.
System Component
Stores information about elements that compose a system. These elements can consist of hardware or logical system components.
System
Stores information about the systems in your organization.
System Resource
Stores information about an entity managed by a BIOS, an operating system, or other software that is available for use by application software, logical devices, or both.
System resources are individually identified and allocated entities that can be assigned, reserved, counted, tracked, released, reset, and so on.
Examples of software resources are message queues, shared memory segments (identified by a key value), and named pipes. Examples of hardware resources in an x86 environment are IRQs, DMA channels, and memory-mapped I/O.
System Service
Stores information necessary to represent, configure, and manage the functionality provided by a device or software feature. A service is a general-purpose object to configure and manage the implementation of functionality.
System Software
Stores information about operating systems and other software that manages computer resources at a low level.
Tag
Stores information about a label that can be associated to any CI. This label is used to categorize CIs to create a logical grouping. For example, the administrator of a blueprint deployment can tag the resource set as public, private, secure, and highly secure. These tags can be used by the placement engine in performing the deployment in appropriate network zones.
Tape Drive
Stores information about the capabilities and management of tape drives. A tape drive is a device that reads data from, and writes it to, a tape.
Transaction
Stores information about a single transaction initiated by an end user or system. For example, a user selecting a web page from a user interface is a transaction, just as a computer program calling a web service to perform a function in another computer program is also a transaction.
UPS
Stores information about UPS systems, which maintain power for your organization and its systems during power outages.
User Community
Stores information about a collection of users who consume business services. Typically, the users are external to the organization or they cross organizational boundaries.
Virtual System Enabler
Stores information about software that enables a collection of virtual computer systems to run on a single physical computer system (for example, VMware).
WAN
Stores information about collections of CIs related to IP subnets in a WAN.
