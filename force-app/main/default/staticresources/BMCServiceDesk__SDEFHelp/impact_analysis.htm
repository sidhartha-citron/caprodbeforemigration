You can view the configuration items (CIs) that can be impacted when a change request is implemented on CIs. For example, you can identify which devices and applications in the network are affected if you take a specific server offline due to scheduled maintenance or lease returns.

While configuring relationships among CIs, you must select Yes in the HasImpact field for the CIs that are affected due to a change on a parent CI. For example, while configuring a relationship between a server and a server application, select Yes in the HasImpact field.

Assume that C1, C2, C3, C4, C5, C6, and C7 are CIs and C1 is source for C2 and C3 (the HasImpact field value is Yes); C2 is source for C4 and C5 (the HasImpact field value is Yes); and C3 is source for C6 and C7 (the HasImpact field value is No). If a change request is implemented on C1, the CIs C2, C3, C4, and C5 could be impacted. The following figure shows the relationship among CIs and the CIs that are impacted by changes on C1.

To view the impact of change requests on parent CIs

  1. Click the Remedyforce Console tab.
  2. From the View menu, select Change Requests.
  3. From the list of change requests, click the change request whose impact on CIs you want to view.
  4. From the Agent Tools menu, select Impact Analysis.
  5. In the Impact Analysis window, from the list of CIs linked to the change request, select the CI whose impact you want to view.

    6. To view a CI in the Instance Editor, click the name of the CI in the Instance Name column.

  6. Click Analyze Impact.

    7. If you do not select a CI that is linked to the change request, the Analyze Impact button is disabled.

    All of the CIs that can be impacted if a change request is implemented on the selected CI are shown. For all of the impacted CIs, the value selected in the HasImpact field of relationships must be Yes.

    To view business services only, click Business Services Only.

    To view different fields of a CI in the list views of the Impact Analysis window, contact your system administrator.

  7. To get a list of impacted CIs, perform the following actions:
    1. Select the CI whose impacted CIs you want to view in a report.
    2. Click the Attach impacted CIs to change request button.
    3. Navigate back to the change request.
    4. Click Record Details.
    5. In the Notes & Attachments section, in the Action column, click View.

      6. The list of CIs impacted by the selected CI is available in CSV format.

  8. To view a CI and its relationship in a graphical manner, select the CI, and click Launch CI Explorer.
