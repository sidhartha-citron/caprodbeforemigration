Viewing incidents of selected client

You can view all the incidents that are reported by the client that you have selected for the incident.

To view incidents of the selected client

  1. Navigate to the incident for which you want to see the incidents of the selected client.

    2. For more information, see Navigating to an incident or a service request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console or Incident Console tab, perform the following actions:
      1. From the Agent Tools menu, select Incident Matching.
      2. In the Match By list, select Client ID, and clear the check boxes for Account and Category.

        3. You can perform these steps in the Salesforce page layout also by clicking Incident Matching.

    • If you are using the Incidents tab (Salesforce page layout), click Incidents for Client.
    • If you are using the Visualforce page layout UI, from the View Open Incidents For menu, select Incidents for Client.

In a separate window, a list of all the incidents that are reported by the client of the the incident is displayed.

Viewing incidents of selected category

You can view all the incidents that are logged for the category that you have selected for the incident.

To view incidents of the selected category

  1. Navigate to the incident for which you want to see the incidents of the selected category.

    2. For more information, see Navigating to an incident or a service request record.

  2. If you are using the Remedyforce Console tab or the Incident Console tab, perform the following actions:
    1. From the Agent Tools menu, select Incident Matching.
    2. In the Match By list, select Category and clear the check boxes for Account and Client ID.
    3. Click Apply.

      4. You can perform these steps in the Salesforce page layout also by clicking Incident Matching.

  3. If you are using the Incidents tab (Salesforce page layout), click Incidents for Category.
  4. If you are using the Visualforce page layout UI, from the View Open Incidents For menu, select Incidents for Category.

In a separate window, a list of all the incidents that are reported by the category selected for the incident is displayed.

Viewing linked tasks in the Visualforce page layout UI

You can manage the tasks linked with the incident in the Supporting Information section.

To view linked tasks in the Visualforce page layout UI

  1. Click the record whose linked tasks you want to view.
  2. In the Supporting Information section, click the Tasks tab.
  3. Click .
  4. To view all the linked tasks, select All.
  5. To view the linked tasks that are in the Open stage, select Open.
  6. To view the linked tasks that are in the Acknowledged stage, select Acknowledged.
  7. To view the linked tasks that are in the In Process stage, select In Process.
  8. To view the linked tasks that are in the Closed stage, select Closed.

To send an email message to the staff member or queue to whom an incident is assigned, notifying that the last task associated with an incident is closed, contact your system administrator to activate the Notify incident owner when final task linked to incident is closed workflow.

Viewing Performance Metrics for the Incident form

In the Visualforce page layout UI, click the Performance Metrics icon, in the upper-right section of the form, to display the performance metrics for a form. The following performance metrics are displayed as a tooltip:

    • Server Time—The time taken by the Force.com server to process a request.
    • Network Time—The time taken to complete the data transfer to the browser over the network. The accuracy of this time depends on how closely your computer's time is synchronized with correct time of your time zone. If your computer's time is incorrect, contact your system administrator to correct it.
    • Page Render—The time taken by the browser to load all data after receiving a response from the Force.com server.

      • Note: The unit of measurement is milliseconds.

If you observe that it is taking a long time to open the incident form, you can use the performance metrics to identify problem area. For example, if the Network Time metric is high and the Page Load metric is low, the problem is with your Internet connection, which is slow and is taking a long time to receive a request sent by the Force.com server.

Note: You can also view performance metrics for lookup windows, such as the Category and Client lookup windows. However, the Performance Metrics icon is disabled in the list views, and it does not display performance metrics for the list views.
