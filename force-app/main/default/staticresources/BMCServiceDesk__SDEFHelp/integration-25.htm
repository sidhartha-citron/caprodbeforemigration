On the Remedyforce Administration Application Settings > Incident - Email Settings page, you must map the data in the events to the fields in the Incident object.

To configure the Application Settings

  1. Click the Remedyforce Administration tab.
  2. On the Home page, click the Configure Email tile, and from the menu select Incident - Email Settings.
  3. If you are using the Visualforce page layout user interface, perform the following actions:
    1. In the BMC Remedyforce home page, click Configuration.
    2. From the list of Configuration options, expand Configuration.
    3. In the Configuration list, click Application Settings.
    4. Click the Incident - Email Settings tab.

      5. The following sections are provided to configure email settings that are used when an incident is created from an email:

    5. Creating Incidents from Client Emails
    6. Identifying the Email Sender
    7. Mapping Email Content to Incident Fields
  4. In the Creating Incidents from Client Emails section, to allow clients to create incidents by sending an email with any subject or create an incident if an email does not have a subject, select Create an incident with any email subject.

    5. Selecting this option allows you to enter your requirements-specific text in the Subject fields of event rules on the Event Text Customization for individual Event Rule screen in BMC ProactiveNet Performance Management.

  5. Click Mapping Email Content to Incident Fields.
  6. From the Delimiter list, select the appropriate delimiter.

    7. To save data inside the email body, you need to enclose the data with a delimiter. For example, you select the hash symbol (#) as a delimiter. To get category from the email body, the category value must be enclosed between two # symbols, such as #Hardware#, in the email body.

  7. From the Incident Field list, select the field on the Incident form in which the data is saved.
  8. From the Email Content list, select Body.
  9. In the Keyword field, enter the keyword that is searched in the email.
  10. To add the mapping, click .

    11. The following mapping is mandatory:

    • Incident Field: Event ID, Category
    • Email Content: Body, Body
    • Keyword: EVENTID, CATEGORY

      • Note: You cannot map the disabled fields on the Incident form, such as Priority.

  11. (Optional) To remove a mapping, select it and click .
  12. (Optional) To update a mapping, select it and click .
  13. Click Save.
