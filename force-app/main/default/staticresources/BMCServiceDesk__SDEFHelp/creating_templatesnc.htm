To help ensure accuracy and completeness, BMC recommends that you use a template to create records. Templates help you to configure commonly used workflow, ensure of consistency in the way information is captured, increase productivity and efficiency, and reduce errors. Using templates, you can pre-populate fields on a form that reduces the need to enter commonly used data or create records to provide an automated workflow process. For more information, see Applying templates, Creating a Task template to deploy an operational rule, Creating an Incident template to deploy an operational rule, and Creating a Request Detail template to deploy an operational rule.

Note: Depending on the requirements of your staff members, you can create templates for commonly occurring incidents, tasks, change requests, service requests, and releases.

To create a template

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Templates.
  4. In the templates list view, click New.
  5. From the Template For list, select the type of template you are creating.

    6. To create a service request template, select Request Detail from the Template For list. When you select Request Detail, the Incident Template list appears. Click this list to open the Select From Templates window where you must select an Incident template that contains the Incident fields that you want to use to create the service request. BMC recommends that you select an Incident template that does not have any linked Task templates. If you select an Incident template with linked Task templates, the task records, from the linked Task templates, are created even when the submitted service request is not approved.

    A service request template is a combination of a Request Detail template and an Incident template. If you select Request Detail, the Quantity service request field is populated in the Select Field list. By default, the Quantity field is added to the template with a default value of 1. By default, this field is added to the template with a default value of 1. You can assign a numeric value for this field and add the field to the service request template to assign a default quantity of service requests that are requested. For example, you assign 5 to the Quantity field. When you submit a service request from this service request template, by default, 5 is the quantity of service requests that are requested.

  6. In the Name field, type a unique alphanumeric identifier that you want to assign to the current template.
  7. Select the System Template check box if you want to apply the template to the records that are created using web services or workflow or by importing data.

    8. If you select Request Detail in the Template For field, the System Template check box is disabled because you cannot create system templates for request details.

    System templates are used to apply templates on records that are created using web services or workflows or by importing data. The name of the template must be used as the value of the Template Name field in the workflows, web services, or while importing data.

    The system templates are not displayed in the Template lookup in the BMC Remedyforce Salesforce page layout. However, all the templates are displayed if you are using BMC Remedyforce in Visualforce pages.

  8. In the Description field, type a description of this template.

    9. The description should include the reason for creating the template. For example, you can create a template to define the New Hire process, which creates tasks associated with the incident template for the new employee.

    The Keywords text box is added to find suggested templates in Salesforce page and console layouts. If you are accessing it from Visualforce page layouts, text in this text box has no effect on the behavior of templates.

  9. From the Select Field list, select the field that must be added to the template.

    10. BMC Remedyforce supports adding fields of the following data types only:

    • Lookup Relationship
    • Master-Detail Relationship
    • Check Box
    • Currency
    • Email
    • Number
    • Percent
    • Phone
    • Picklist
    • Text
    • Text Area
    • Text Area (Long)
    • URL

      • If you add a Picklist (Multi-Select) field to a template and use the template to create a record in the BMC Remedyforce tabbed view, a single value set in Picklist (Multi-Select) field of the template is populated in the record.

      Depending on the field added to the template, the Field Value field dynamically displays the appropriate option to enter a value. For example, if you added the Priority field, the Field Value field displays a list that allows you to select a priority for the field. You can enter the required value if you do not want to make a selection, and the value is validated before it is added to the template. When you add the Description field, the Field Value field displays a text box that allows you to enter a description.

      Note: When you select Change Request, Incident, or Task in the Template For field, the corresponding change request, incident, or task fields appear in the Select Field list.

      If you have created custom fields of the Lookup Relationship data type for the Change Request, Incident, or Task custom objects, these custom fields do not appear in the Select Field list.

  10. In the Field Value field, enter the required value of the selected field.

    11. Note: If you add a lookup field to the template, select a value for the lookup field, update the template, close the template, update the selected value in its original form, and open the template, the lookup field in the template does not display the updated value. For example, you add the Urgency lookup field to a template, select HIGH as the urgency value, update the template, and close the template. In the Urgency form, you update HIGH to HIGHEST. Now, when you open the template, the Urgency lookup field in the template still displays HIGH instead of HIGHEST. Lookup fields can be Impact, Owner ID, Urgency, and so on. BMC recommends that you update your templates with the new lookup value whenever you update a lookup value.

  11. To add the field name and its value to the template, click Add.

    12. If you are accessing Self Service from a mobile device and you create an incident by using an incident top request, the incident record is created and populated with values of the following fields from the incident top request:

    • Category
    • Incident Description

      • If you are accessing Self Service from a browser and you create an incident by using an incident top request, the incident record is created and populated with values of the following fields from the incident top request:

    • Category
    • Incident Description
    • Incident Resolution
    • Impact
    • Urgency
    • Broadcast
    • Configuration Items
    • First Call Resolution

      • You can view the values populated for these fields only when you open the incident record in BMC Remedyforce. You cannot view these values in Self Service.

      Note: If the incident top request contains other fields, the values of these fields are not populated in the incident record.

  12. Repeat step 9 through step 11 until you have added all the fields required for the template.
  13. To modify the fields and their values in the template, perform any of the following actions:
    • To update the value of a field, from the table, select the field, in the Field Value field, edit the value of the field, and click Update.
    • To remove a field from the template, select the field from the table, and click Remove.
  14. If you are using BMC Remedyforce in Visualforce forms, in the For Use By section, perform the following actions:

    15. Note: This setting is supported for Visualforce forms only. If you are using the Salesforce page layout, all the templates for an object are displayed in the template lookup. In the Salesforce page layout, the templates that are displayed may not be assigned to the profile of the logged on user.

    1. In the Available Profiles list, select the profiles that can use the template.

      2. Note: Select the Everyone check box to allow all profiles to use the template.

    2. To move the selected profiles from the Available Profiles list to the Selected Profiles list, click the right arrow.
    3. (Optional) To move the selected profiles from the Selected Profiles list to the Available Profiles list to prevent the selected profiles from using the template, click the left arrow.

      4. Note: If you are using the Salesforce page layout for incidents and creating an incident using a template, all the templates for incidents are displayed in the template lookup. The templates that are displayed may not be assigned to the profile of the logged on user.

  15. If you are creating an incident or change request template, you can add tasks to the incident or change request template that you must complete before you can close the incident or change request. If you create a blank task template and link it to an incident template, only the client from the incident template is populated in the task record when you use the incident template. Other values from the incident template are not populated in the task record, such as category, status, and so on. If you have populated values in the task template, these values are populated in the task record. To add tasks to the incident or change request template, perform the following actions:
    1. In the Linked Templates section at the lower section of the Template form, click Actions > Add.

      2. A new Template form opens with the Template For field filled with Task as the type of template. This field is disabled and cannot be modified.

    2. Repeat step 6 through step 14 to create the task template.
    3. Click Save.

      4. The Template form of the task closes and the task template appears in the Linked Templates section at the lower section of the Template form.

    4. Repeat step a through step c for each task that you want to add to the template.
    5. To configure the order in which the tasks are completed, click Set Task Execution Order.

      6. Note: To execute all linked tasks at the same time, select the Ignore Execution Order check box. By default, this check box is selected and BMC Remedyforce executes all tasks at the same time. You can clear this check box and configure the order in which the tasks are completed.

      You can assign same execution order number to more than one task. For example, you have four tasks, Task1, Task2, Task3, and Task4. You can assign tasks Task1 and Task2 to be executed first, then Task3, and then Task4. When Task1 and Task2 are running, only these tasks are displayed in the Supporting Information of the record. After Task1 and Task2 are closed, Task3 is displayed, and after the closing of Task3, Task4 is displayed. If an incident or change request is closed before all the tasks are closed, the tasks are not created if they are not yet scheduled.

      You can apply a template on an existing incident, change request, task, or release. If you apply a template on an existing record, the tasks associated with the existing record are saved and the tasks associated with the applied template are added to the record.

    6. In the Order Number column of the Set Task Execution Order window, select the order number that you want to assign to the tasks.

      7. You can assign same execution order number to more than one task.

    7. (Optional) To ignore the execution order, clear the Ignore Execution Order check box.
    8. Click Save.

      9. Note: If you have an existing task template, you can add this task template by clicking Actions > Link Templates and selecting the required task template.

  16. Click Template Display Order.

    17. When a staff member clicks Template on the Incident or Task form, the Select from Templates window displays the available templates in the display order that you have configured. You can configure the most frequently used templates to appear higher in the display order.

  17. In the Template Display Order window, from the Template For list, select the type of templates whose display order you want to configure.
  18. Select the template that you want to appear first in the Select from Templates window and keep clicking the up arrow until the template moves to the first position.
  19. Select the template that you want to appear second in the Select from Templates window and keep clicking the up arrow until the template moves to the second position.
  20. Similarly, position the rest of the template in the order in which you want to them to appear in the Select from Templates window.
  21. Click Save.

    22. Note: When you open a new form to create a record and you save the record, you cannot navigate to the next or previous record by using the Next Record and Previous Record buttons in the upper right section of the form. These buttons are disabled in a new form.

  22. (Optional) To designate the current record as inactive, select the Inactive check box.

    23. For example, if you have an incident template to trouble shoot issues with an outdated operating system, this template is no longer be valid because your account does not use the outdated operating system.

    Note: Records marked as inactive are not included in any search performed on the database.
