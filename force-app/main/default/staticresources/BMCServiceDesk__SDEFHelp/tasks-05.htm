To help ensure accuracy and completeness, BMC recommends that you use a template to create a record. A template ensures consistency in the way information is captured. Your administrator can define templates for common tasks.

To create tasks by applying a template

  1. Navigate to one of the following tabs and click New:
    • Remedyforce Console > View > Tasks
    • Task Console
    • Tasks (Salesforce page layout)
  2. In the Client ID field, select the client.
  3. In the Template field, click .
  4. In the Templates window, select the required template.

    5. If you are using the Remedyforce Console or Task Console tab, after you select a template, the fields from the template are populated without clicking Save. However, if you have a Text Area (Rich) field on the form, template fields are not populated until you click Save.

    To create recurring records, select a template on which recurrence is defined. All the recurring records are created at the same time using the same template information. The recurrence information is stored in the Recurrence field. You cannot apply a recurring template for a record where the Recurrence field already contains information.

    Note: Recurring records are created only when you apply the recurring template to at least one record.

    You cannot associate recurring tasks to recurring incidents or change requests.

  5. If you are working in the Tasks tab, to apply the template, click Save.

    6. If the system administrator has selected the Replace form field values with template field values check box, the values that you enter on the form are replaced with the values specified in the selected template (if available).

  6. Enter other information about the task, and click Save.

    7. For more information, see Task fields.

    After you save a task, the task details page is displayed. The following additional fields are displayed:

    • Status Details section that displays the Task Progression Bar.
    • First Name and Last Name fields in the Client Details section that display the first name and last name of the client that you select for the task. The values in these fields depend upon the client that you select. You cannot select values in these fields. When you edit the client of the record, the values in these fields change automatically.
    • Client Account field in the Client Details section that displays the account of the selected client.

      • If the system administrator has activated the Notify the client when a task is created workflow, an email message is sent to the client of the task when you create the task. If you select a lead as the client of the task, no email message is sent.

When a record is created, the following action is recorded in the Task History section that tracks the date and time when a record is created and the user details:

Changed Status to Opened
Created
