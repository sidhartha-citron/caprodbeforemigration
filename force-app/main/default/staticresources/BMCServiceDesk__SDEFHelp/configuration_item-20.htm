The CI Chatter Feed displays posts, comments, and field updates that are made to the CI. For a CI, the CI Chatter Feed tracks the updates that are made to only those fields that are inherited from the Base Element object.

For more information about enabling feed tracking for Salesforce Chatter, see https://na7.salesforce.com/help/doc/user_ed.jsp?loc=help&target=collab_feed_tracking.htm&section=Collaboration.

In a new Instance Editor form, the Chatter Feed tab is disabled for editing. It is enabled for editing after you save the CI record.

Note: If feed tracking is not enabled for the Base Element custom object, the Chatter Feed tab does not appear in the Instance Editor form.

In the CI Chatter Feed, you can perform the following actions:

    • View updates that are made to the Base Element fields that are enabled for feed tracking
    • Comment on the updates that are made to the Base Element fields that are enabled for feed tracking
    • Create posts that you want to share with followers of the record
    • Follow and unfollow the CI record. You can follow a maximum of 500 records.

To track the CI updates by using the Chatter Feed

  1. Click the Remedyforce CMDB tab.
  2. If you are using the Visualforce page layout user interface, perform the following actions:
    1. Go to BMC Remedyforce Home and click Workspaces.
    2. From the list of Workspaces, expand Configuration Management.
    3. In the Configuration Management list, click Configuration Items.
  3. In the right pane of the CMDB Manager, select the CI that you own or follow.

    4. To search for a CI, type initial characters of the name of the CI or the full name of the class name in the Search text box and click . For example, to search for Email Service CI, you can type Email or Email Service in the Search text box and click . You can also search using data in the Instance ID, Instance Name, Description, Class Name, Asset #, Token ID, and Serial # fields. If you select a class in the left pane, only the CIs of that class are searched. To search all classes, select the Base Element class in the left pane.

    By default, when you create or own a record, you automatically follow it.

  4. Click .
  5. In the Instance Editor, click the Chatter Feed tab.

    6. When the CI is linked to an incident, change request, or problem, this update appears in the CI Chatter Feed in the following format:

    <User Name> linked <Object Name> <Record Number> to this Configuration Item

    When the CI is unlinked to an incident, change request, or problem, this update appears in the CI Chatter Feed in the following format:

    <User Name> unlinked <Object Name> <Record Number> from this Configuration Item

    Note: The updates made to the Base Element fields that are enabled for feed tracking are visible in the Chatter Feed only after you save the CI record.

  6. To create a post that you want to share with followers of the CI record, perform the following actions:
    1. Type the post that you want to add to the Chatter Feed.
    2. If you want to share files or links, attach it to the post.
    3. Click Share.

      4. The post appears in the CI Chatter Feed and in your Chatter Feed. You can view your Chatter Feed by navigating to Workspaces > Chatter > Chatter Feed.

  7. To comment on the updates that are made to the Base Element fields, perform the following actions:
    1. Click the comment link below the update that you want to comment.
    2. Type the comment.
    3. Click the Add Comment link.

      4. The comment appears below the update in the CI Chatter Feed and in your Chatter Feed.

  8. To follow or unfollow the CI record, perform one of the following actions:
    • To stop seeing updates to the CI record in your Chatter Feed, in the CI Chatter Feed, click the Unfollow link.

      • The contents of the CI Chatter Feed are deleted.

    • To restart following the CI record and viewing updates to the CI record in your Chatter Feed, in the CI Chatter Feed, click the Follow link.

      • The updates that are made after clicking the Follow link are displayed in the CI Chatter Feed and in your Chatter Feed.

  9. Click .

Automatically following records

By default, you follow records that meet the following conditions:

    • You have created the record or you own the record.
    • Feed tracking for the object of the record is enabled. For more information about enabling the feed tracking for the object, contact your system administrator.
    • Another user has assigned the record to you and the system administrator has enabled the Owner field on the object of the record for feed tracking.

You can stop automatically following records by navigating to Setup > My Chatter Settings > My Feeds and selecting the Stop automatically following records check box.

Note: If you have enabled the improved Setup user interface in your Salesforce organization, navigate to My Settings > Personal > Personal Information. For more information about the improved Setup user interface, see http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_US.
