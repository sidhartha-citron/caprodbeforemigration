You must consider the following points while configuring Remedyforce mobile application on Salesforce1 platform:

    • The fields Name, Status, and Priority are static fields in the Salesforce1 - List View field and BMC recommends that you add only five other fields in the Salesforce1 - List View field set. If you add more than five fields, apart from the static fields (Name, Status, and Priority) only the first five fields are displayed in list view.
    • BMC recommends that you add only 20 fields in the Salesforce1 - Incident Details field set. If you add more than 20 fields, only the first 20 fields are displayed on the form.
    • The Text Area (Rich) and dependent picklist type fields are not supported on the list view.
    • The Text Area (Rich) type field is not supported in the Incident form.
    • If you do not add the Status and Resolution fields to the Salesforce1 - Incident Details field set, these fields are not displayed on the Close form.
    • The record list view displays a maximum of 500 active records in the Remedyforce mobile application.
    • If a user applies a template to create an incident, the template values might not be applied based on the settings configured. If the Allow applying of templates on an existing record and Replace form field values with template field values check boxes (Remedyforce Administration > Application Settings > General Application Settings > Template Settings) are not selected, then the template values are not applied. In this case, the values that the user enters in the incident fields are retained in the record.
    • A lookup field displays a maximum of 20 records on the lookup page. For example, if you have 25 categories, the Category lookup page displays a maximum 20 categories.
    • You can customize the landing page for Salesforce1 platform to display your custom branding. For more information about customizing the landing page, see https://help.salesforce.com/HTViewHelpDoc?id=customize_sf1_branding.htm&language=en_US.
