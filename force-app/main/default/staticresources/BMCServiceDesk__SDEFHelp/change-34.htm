When you are closing a change request that is linked to a problem, an email message can be sent to the problem owner (user) that the change request is closed. If the problem is assigned to a queue, no email notification is sent. To enable sending the email notification, contact your system administrators to activate the Notify problem owner when change request linked to problem is closed workflow.

To close a change request

  1. Navigate to the change request that you want to close.

    2. For more information, see Navigating to a change request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Actions menu, select Close.
    • If you are using the Change Requests tab (Salesforce page layout), click Close.
    • If you are using the Visualforce page layout UI, from the menu, select Close.

      • The Close Change Request form appears with the Closed Date field filled with the current date and time. The Category field is filled with the category that is entered in the Change Request form.

      Note: This procedure describes the out-of-the-box fields only. If your system administrator has added or removed fields, complete all the fields on the form.

  3. From the Closed By list, select the staff member who has worked on the change request.

    4. By default, this field displays the Staff ID of the currently logged-on staff member. If you are closing the change request on behalf of another user, select the staff member who has worked on the change request.

  4. From the Status list, select a close status.

    5. By default, this field is filled with the default close status that is configured by the system administrator. If required, select a different close status. All status records with the state of Close are displayed in the list.

  5. From the Closure Category list, select the category that suggests the reason for closing the record.
  6. If the original category is incorrect or no longer applies, select the required category from the Category field.
  7. In the Close Comments field, type the reasons for closing the change request.
  8. To close all tasks linked with the change request, select the Close Tasks check box.

    9. When you have completed the change request, you might want to close all linked tasks because these tasks are no longer required.

  9. To close all the incidents linked with this change request, select the Close Incidents check box.

    10. When you have completed the change request, you might want to close all linked incidents because these incidents are no longer required.

    If linked records, such as linked tasks and incidents, are assigned to a staff member who is higher in the role hierarchy than the staff member who is closing the record, these linked records might not be closed if the Private sharing model is applied on the linked objects. By default, BMC Remedyforce has configured the Private sharing model for the Incidents and Tasks objects. However, system administrators can configure the Private sharing model for Change Request, Broadcast, and Problem objects also.

  10. Click Save.

    11. When you have change requests that are linked to a problem, you can an email notification is sent to the problem owner when a change request linked to the problem is closed. Contact your system administrator to activate the Notify problem owner when change request linked to problem is closed workflow. If the problem record is assigned to a queue, no email notification is sent.
