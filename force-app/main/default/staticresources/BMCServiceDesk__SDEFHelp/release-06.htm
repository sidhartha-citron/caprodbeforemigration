By default, a release is assigned to the user who creates it. After saving the release, you can change the assignment to another user, or a queue.

To change a release assignment

  1. If you are using the Remedyforce Console tab, perform the following actions:
    1. From the releases list, select the check boxes for the releases that you want to assign to another user or queue.
    2. Click Change Owner.
    3. To assign the selected records to a staff member, click the required staff member record.
    4. To assign the selected records to a queue, select the Queue option and click the required queue.
    5. To assign the selected records to a staff member who belongs to a queue, first click a queue record and from the list of staff members belonging to the queue, select the required staff member.
  2. If you are using the Releases tab (Salesforce page layout), perform the following actions:
    1. Click the Releases tab.
    2. From the list of releases, click the release that you want to assign to another user or queue.
    3. In the Staff Assignment Details section, click Change beside the user name to whom the release is currently assigned.
    4. From the Owner list, select the appropriate option.
    5. To assign the release to a user, select User.
    6. To assign the release to a queue, select Queue.
    7. The Customer Portal User option is not required for releases.
    8. Click and select the required user or queue.
    9. Click Save.

      10. To use the staff and queue assignment feature, contact your system administrator to edit the release layout to display the StaffAssignmentPage Visualforce page.
