You can order tasks and change requests linked to a release by using the release manifest. The manifest is available after you save a record. The manifest provides a tabular view of all the tasks and change requests that are linked to a record. You can order linked tasks and change requests and also associate a release status with those tasks and change requests.

The following sections are provided:

    • Understanding the manifest
    • Ordering linked tasks and change requests and associating release status

Understanding the manifest

You can use the manifest to establish a sequence for how the linked tasks and change requests must be carried out during the lifecycle of a record.

You can also associate a release status with the linked tasks and change requests to ensure that before the release moves to another status, all the tasks and change requests with which the current status is associated are closed. For example, with tasks T1 and T2 and change request C1, Opened release status is associated. In this case, you cannot change the status of the release to In Progress or any other status until T1, T2, and C1 are closed.

The tasks that are defined in the template but are not yet created are displayed with Pending Creation status (referred to as future tasks). You can create such tasks if you create a release by using a template. The order number of future tasks must be greater than the order number of the already-created tasks of the template.

If the linked tasks and change requests are ordered in the template that is used to create the release and you set different order numbers for those tasks and change requests in the manifest, the ordering done in the manifest is followed.

The following data about the tasks, change requests, and releases are displayed in the manifest:

    • Release Item: Release ID
    • Item ID: Task or change request ID
    • Description: Description of task or change request
    • Status: Status of the task or change request
    • Due Date: Due date of the task or change request
    • Priority: Priority of the task or change request
    • Assigned To: The user (staff user or change initiator) to whom the task or change request is assigned
    • Release Status: Status of the release
    • Order: Sequence number of a task or change request for a particular release status

Ordering linked tasks and change requests and associating release status

This section provides steps to order linked tasks and change requests and associate release status with required tasks and change requests.

To order linked tasks and change requests and associate release status

  1. Navigate to the record for which you want to order linked tasks and change request and associate release status.

    2. For more information, see Navigating to a release record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Agent Tools menu, select Release Manifest.
    • If you are using the Releases tab (Salesforce page layout), click Release Manifest.

      • If the Release Manifest button is not available, contact your system administrator to customize the layout of the release form and add the button.

      All the linked tasks and change requests are displayed in a tabular format.

  3. To set the order number of a task or change request, click the cell of the task or change request in the Order Number column and select the order number.

    4. The selected order number defines the sequence number in which the task or change request is performed.

  4. To associate a status with tasks and change requests, click the cell of the required task or change request in the Release Status column and select the required status.

    5. Remember the following points while setting up order number of a task or change request and associating a status:

      • When you update the status of a release, all the tasks and change requests from the current status must be completed. For example, if you changed the status of a record from Assigned to In progress, then all the tasks of the Assigned status must be complete.
      • The order number of future tasks must be greater than the order number of the already-created tasks.
      • If you are ordering one future task, ensure that you order all the future tasks.
      • It is mandatory to order a task and change request and also select a release status.
      • A task or change request must be performed or implemented when the selected release status is active.
  5. (Optional) To export the list of linked tasks and change requests to a CSV file format, click Export.
  6. (Optional) To print the list of linked tasks and change requests, click Print.
  7. Click Save.
