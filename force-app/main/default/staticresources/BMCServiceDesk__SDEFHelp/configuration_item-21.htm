If you want to add a custom attribute to an out-of-the-box class, such as Mainframe, you must add the field in the Base Element object and then add the field to the Mainframe field set. The field is shown on the Specifications tab in the Instance Editor. However, if you add a custom attribute (field) to the Base Element field set, the field is shown on the General tab in the Instance Editor.

To add a custom attribute in an existing class or field set in CMDB 2.0

  1. Click the Remedyforce Administration tab.
  2. On the Home page, click the Manage Objects tile, and from the menu, select Create and Edit Objects.
  3. On the Create and Edit Objects page, from the Objects list, select Base Element.
  4. Click Custom Fields and Relationships.
  5. In the Custom Fields & Relationships section, click New.
  6. Select the data type of the field that you want to add, and click Next.
  7. In the Field Label field, enter the field name and fill in other required details.
  8. Click Next.
  9. As per your service desk requirements, select the Visible or Read-Only check boxes for the required profiles, and click Next.
  10. Click Save.
  11. On the Custom Object Base Element page, navigate to the Field Sets section. For the field set corresponding to the class to which you want to add the field, in the Action column, click Edit.

    12. In the example, click Edit for Mainframe.

  12. Drag the field that you have added to the In the Field Set list, and click Save.
