To get a list of colliding change requests, create a Salesforce report on the Change Collision object.

To create a report on colliding change requests

  1. To create a report on the Change Collision object, perform the following actions:
    1. Click the Remedyforce Administration tab.
    2. On the Home page, click the Manage Objects tile, and from the menu, select Create and Edit Objects.
    3. On the Create and Edit Objects page, from the Objects list, select All Custom Objects.
    4. In the Action column next to Change Collision, click Edit.
    5. In the Optional Features section, select the Allow Reports check box.
    6. Click Save.
  2. Click the Reports tab.
  3. Click New Report.
  4. From the Select Report Type list, click the plus sign (+) next to Other Reports, and select Change Collisions.
  5. Click Create.
  6. Drag the columns that you want to see in the report, and click Save.

    7. The following table lists the fields that you can use in the report as columns:

    Field
    Description
    Change Collision: ID
    Unique ID of the colliding change requests record.
    Change Collision: ChangeCollisions Name
    Autonumber value generated by Salesforce.
    Change Request - 1
    Change request ID of one of the colliding change request.
    Change Request - 2
    Change request ID of the other colliding change request.
    Collision State
    Value of the collision state between the two colliding change requests.
    ConfigurationItem
    Configuration item that is linked to both the colliding change requests.
