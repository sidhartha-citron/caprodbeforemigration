A task is a unit of work that must be completed as a step in resolving an incident or service request. If the solution to an incident involves more than one action, procedure, or process, consider dividing the solution into separate tasks. Dividing the solution into separate tasks can help you to manage and monitor the incident as it moves toward resolution.

You can create a task from an existing incident or directly using the task page. When you create a task from an incident, all the associated details of the incident such as client name, category, and description are filled in to the task form. The incident number is also copied to the task so that a staff member can see the incident that is related to the task.

For example, one of your clients calls a staff member to report that the common printer is not working. The staff member checks the printer and finds that the printer cartridge needs to be replaced. Additionally, the printer paper is getting over and would need to be purchased. The staff member creates an incident to record the issue of the printer not working. Next, the staff member creates the following tasks that must be completed to resolve the incident:

    • Task 1: Purchase the printer cartridge.
    • Task 2: Replace the faulty printer cartridge with the new printer cartridge.
    • Task 3: Purchase the printer paper.
    • Task 4: Refill the printer paper in the printer.

You can populate the list and lookup fields on this form by typing the first few characters of the required value and pressing the Tab key. BMC Remedyforce searches for the matching value and populates the field with this value. For example, if you type Net in the Category field and press the Tab key, the Network category is populated in the Category field.

If there are multiple matching values, BMC Remedyforce displays a lookup window that allows you to select the required value from the matching records.

To create a task in the Visualforce page layout UI

  1. In the BMC Remedyforce home page, click Workspaces.
  2. From the list of Workspaces, expand Incident Management.
  3. In the Incident Management list, click Tasks.
  4. To open a new task form, in the tasks list view, click .

    5. The Supporting Information section is not loaded in a new form. When you save the record, the Show Supporting Information link appears. Click this link to view the Supporting Information section. You can configure the Supporting Information section to always display by selecting the Always show Supporting Information for existing records check box. For more information, see Managing supporting information for tasks.

    The Incident # field is set to the incident number of the incident where the task was created or associated. If the task is not associated to an incident, this field is blank.

  5. Click Template.

    6. To help ensure accuracy and completeness, BMC recommends that you use a template to help create the task record. A template ensures consistency in the way information about the task is captured. Your administrator can define templates for commonly occurring tasks.

  6. In the Select From Templates window, double-click an applicable template.

    7. The values of the fields in the template are filled in the corresponding fields of the task form.

  7. Click the Last Name list.

    8. Depending on how BMC Remedyforce is configured for your organization, Select From Clients or Search Clients window is displayed. If your organization is configured such that you can create tasks for contacts and leads, the Search Clients window is displayed where you can search for a user, contact, or lead and a list of matching entries is displayed under separate tabs of Users, Contacts, and Leads.

    If your administrator has configured settings such that you cannot create tasks on behalf of contacts and leads, the Select From Clients window is displayed with a list of users. You can double-click a record to select it.

  8. In the Search Clients window, enter the search keywords in the Keywords field, and press Enter or click .

    9. The list of matching records is categorized under Users, Contacts, and Leads (if applicable). You can change the number of records displayed in the list in the Maximum Records field. The maximum of 200 records per object can be displayed.

    The data fields, such as Account, Mobile Phone, Status, and so on, that display data of the matching records, can be configured on the Global Search Settings page. Contact your system administrator to configure the fields that you want to view in the search results.

  9. Double-click the appropriate record to select it.

    10. The First Name, Client ID, and Account fields are filled with the first name, client ID, and account (if exists) of the selected client. You can change the account of the client.

    When you move your mouse over the icon, the same fields that are displayed in the Search Clients window, such as account ID, Mobile Phone, and so on of the selected record are displayed. Contact details are the mandatory fields that are displayed. The fields that are displayed in the icon can be configured on the Global Search Settings page. Contact your system administrator to configure the fields for your organization. This icon is displayed on custom Task form only.

  10. Click the Category list.
  11. In the Select From Categories window, perform the following actions:
    1. Select the category of the task.

      2. Categories allow you to classify the task using standard classifications, which can be tracked for reporting purposes.

    2. Select the Standard Description check box to use the standard description of the selected category.

      3. The standard description of the selected category is appended to the Description field of the task form.

    3. Select the Standard Resolution check box to use the standard resolution of the selected category.

      4. The standard resolution of the selected category is appended to the Resolution field of the task form.

    4. Click OK.
  12. In the Description field, type the activities that must be performed to complete the task.

    13. Some of the activities are ordering hardware, configuring a test environment, performing tests, and so on.

  13. In the Resolution field, type the solution that must be implemented for this task.

    14. Provide the detailed steps taken to complete the task or describe why the task cannot be completed.

  14. In the Note field, type notes about the task.

    15. Staff members can type updates about their tasks in the Note field, and click Add. These updates are added to the Supporting Information section as history records in the History tab. The Note field allows staff members to add notes about the task without having to navigate to the Supporting Information section and create a history record.

    Note: The Note and Duration fields are enabled only after you save the record.

  15. In the Duration field, type the amount of time that you spent working on the task.
  16. Click Add.

    17. The contents of the Note field and the Duration field are added to the Supporting Information section as a history record in the History tab and the Supporting Information section with the history record is displayed.

  17. In the Details section, enter the following information:

    18. Note: Some of the fields are filled with values, such as the Opened Date field.

    1. Status—Select the status you want to assign to the current task.

      2. You can track the progress of a task from its stages of opening to closure by using this field.

      The Task Progression Bar in the header of the task form is updated depending on the stage that is linked to the status you have selected. For example, if the Working status is linked to the In Process stage and you select Working as the status of the task, then the In Process stage is selected in the Task Progression Bar.

    2. Impact—Select the impact that you want to assign to the current task.
    3. Urgency—Select the urgency that you want to assign to the current task.

      4. The urgency and the level of impact are used to calculate the priority of the task.

    4. Priority—When you select an urgency and impact and save the record, the Priority field is filled in. The priority is calculated depending on the urgency and impact values that you have selected.
    5. Opened Date—This field is filled with the date and time at which the task is created.
    6. Due Date—When you select an urgency and impact and save the record, the Due Date field is filled in. The due date is calculated based on the duration of the priority.

      7. If required, you can select a different due date from the calculated due date.

      When you are editing a task and you update the impact or urgency that leads to a change in the priority of the task, the following Due Date Calculation options appear:

      • Do not Recalculate—The existing due date is retained for the task.
      • Recalculate Based on Open Date—The due date is recalculated by using the open date. For example, if the priority is changed to 1 and the duration assigned to a priority of 1 is 2 hours, the recalculated value of the due date is 2 hours added to the date and time when the task was created.
      • Recalculate Based on Current Date—The due date is recalculated by using the current date. For example, if the priority is changed to 1 and the duration assigned to a priority of 1 is 2 hours, the recalculated value of due date is 2 hours added to the current date and time.

        • Note: If your system administrator has enabled the When the Priority changes, recalculate the Due Date without user input check box in Application Settings, the Due Date Calculation window is not displayed.

        If your system administrator has enabled the Use the Organization's default business hours to calculate the Due Date check box in Application Settings, the due date is calculated by using the default business hours, time zone associated with the business hours, and holidays that are configured for your organization. For example, the system administrator has configured the default business hours as 10:00 a.m. to 6:00 p.m. for your organization. If you create a task at 5:00 p.m. on a Monday and assign it a priority of 1, the due date of the task is calculated as 11:00 a.m. of Tuesday, the next working day. Now, if the system administrator has configured Tuesday as a holiday, the due date of the task is calculated as 11:00 a.m. of Wednesday, the next working day.

    7. Closed Date—This field is filled with the date and time at which the task is closed.
    8. Incident #/Change #/Problem #/Service Request #—This field displays the record number of the associated incident, change request, problem, or service request, if any. The record number is displayed as a hyperlink to the appropriate record. When you click the record number, the record is displayed in the corresponding tab. This field is displayed only if the task is associated with an incident, change request, problem, or service request. You can also copy the record number of the associated incident, change request, problem, or service request.

      9. Note: A task can be linked to only one object (Incident, Change Request, or Problem) at a time.

      If you are using the custom form and your system administrator has added fields from the out-of-the-box field sets, enter values for the fields present on the custom form.

  18. Click Save.

    19. Note: When you open a new form to create a record and you save the record, you cannot navigate to the next or previous record by using the Next Record and Previous Record buttons in the upper-right section of the form. These buttons are disabled in a new form.

  19. After you have created the task, you can add supporting information for the task.

    20. For more information, see Managing supporting information for tasks.

  20. (Optional) To designate the current record as inactive, select the Inactive check box.

    21. You might need to make a record inactive because the record is no longer valid, but must be retained for compliance with the records retention policy. For example, if a client has left the account, all tasks associated with the client are invalid.

    Note: Records marked as inactive are not included in any search performed on the database.

If the system administrator has activated the Notify the client when a task is created workflow, an email message is sent to the client of the task when you create the task. For a custom task form, if you select a lead as the client of the task, no email message is sent.

Managing supporting information for tasks

The Supporting Information tabs at the lower section of the task form allow you to:

    • Track the history of the task
    • Link services and CIs
    • Add documentation
    • View audit information about the task

The task form contains the following Supporting Information tabs:

    • History—Allows you to track the history of all actions that have been performed on the task by viewing the existing user-defined and system-defined actions or adding new user-defined actions.
    • Services and CIs—Allows you to link the services and CIs that are affected when the task is completed.
    • Documentation—Allows you to attach documents and URLs to the attachments that are required to close the task.
    • Auditing—Displays the history of changes made to fields in the form. The system administrator enables field history tracking for required fields.

      • Note: The Supporting Information section is not loaded in a new form. When you save the record, the Show Supporting Information link appears. Click this link to view the Supporting Information section. You can configure the Supporting Information section to always display by selecting the Always show Supporting Information for existing records check box. After displaying the Supporting Information section, click the Hide link to hide the Supporting Information section.
