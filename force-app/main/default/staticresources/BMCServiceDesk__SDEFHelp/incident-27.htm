You can create knowledge articles from incidents or service requests to share information about the resolution of the incident or service request. The knowledge articles that you create are displayed in the Linked Knowledge Articles section. If you do not see the Linked Knowledge Articles section in your record details, contact your system administrator to add it to the layout.

To create a knowledge article from an incident or service request

  1. Navigate to the required record.

    2. For more information, see Navigating to an incident or a service request record.

  2. Perform one of the following actions:
    • If you are using the Remedyforce Console tab, from the Agent Tools menu, select Create Knowledge Article.

      • If this option is not available in the Agent Tools menu, contact your system administrator to add it to the menu.

    • If you are using the Incident Console tab, from the Agent Tools menu, select Create Knowledge Article.
    • If you are using the Incidents tab (Salesforce page layout), click Create Knowledge Article.

      • If this button is not available on the form, contact your system administrator to customize the incident layout and add it on the form.

  3. In the New Knowledge Article window, in the Title field, enter a title for the knowledge article.

    4. The first 30 characters of the Description field of the record appear in the Title field by default. You can enter a different title. A maximum of 255 characters are allowed in this field.

  4. From the Type field, select the required type of the knowledge article, and click Create.

    5. The article is created and saved in the Draft status.

    Based on the type of knowledge article that you are creating, data from some of the fields of the incident or service request is saved in the appropriate fields of the knowledge article.   5. In the Knowledge Article window, click Edit.
  6. Enter the required details for the knowledge article.

    7. For more information, see Creating a knowledge article

  7. Click Save.
