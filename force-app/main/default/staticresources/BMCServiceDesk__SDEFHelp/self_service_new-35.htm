Users who are lower in the role hierarchy cannot view the records created by a user who is higher in the hierarchy. To view a record created by other users, one of the following conditions must be met:

    • The user viewing the record must be higher in the role hierarchy
    • The user must have the View All permission on the Incident object

If users have the View All permission, on the Tickets and Requests tab in Self Service, they can view the records created by other users by selecting the Others option from the View list.

By default, a user can view the records created by all users. You can restrict the users from viewing records that are created by other users.

To restrict users from viewing records created by other users

  1. Click the Remedyforce Administration tab.
  2. On the Home page, click the Application Settings tile, and from the menu select Custom Settings.
  3. In the Action column of Remedyforce Settings, click Manage.
  4. In the Action column of the propShowOthersIncident_in_SelfService, click Edit.
  5. In the Value field, enter False.
  6. Click Save.

    7. Note: To enable users to again view records created by other users, you must change the value of the propShowOthersIncident_in_SelfService custom setting to True.
