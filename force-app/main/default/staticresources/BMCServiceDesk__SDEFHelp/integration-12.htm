For commonly occurring service requests that require operational-rule deployment to complete the service request, you can create a Request Detail template that contains the appropriate Incident template. In the Incident template, link the Task template that deploys the required operational rule.

For example, your clients require Microsoft Office to be installed on their computers. You can create a service request definition (SRD) to address this request. In the SRD, select the Request Detail template that contains the Incident template that describes the service request being offered to clients. In the Incident template, link the Task template that deploys the operational rule to install Microsoft Office.

For information about creating Incident templates that are linked to Request Detail templates, see Creating an Incident template to deploy an operational rule. For information about creating Task templates that are linked to Incident templates, see Creating a Task template to deploy an operational rule.

To create a Request Detail template to deploy an operational rule

  1. Click the Remedyforce Workspaces tab.
  2. In the Configuration list, select Templates.
  3. If you are using the Visualforce page layout user interface, perform the following actions:
    1. In the BMC Remedyforce home page, click Configuration.
    2. From the list of Configuration options, expand Configuration.
    3. In the Configuration list, click Templates.
  4. In the templates list view, click New.
  5. From the Template For list, select Request Detail.

    6. When you select Request Detail, the Incident Template list appears.

  6. In the Name field, type a unique alphanumeric identifier that to assign to the current template.
  7. Click the Incident Template list.
  8. In the Select From Templates window, select an Incident template that contains the linked Task template that deploys the required operational rule.
  9. In the Description field, type a description that includes the reason for creating the template.
  10. Click Save.
