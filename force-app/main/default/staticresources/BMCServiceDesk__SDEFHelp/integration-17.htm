This section provides instructions for configuring BMC Server Automation for creating an incident:

  1. Create an email listener in BMC Remedyforce. For more information about creating email listener, see Creating email services.
  2. Add the EmailCreateIncident out-of-the-box Apex class to the email service.

    3. Note: To create incidents without using the template, skip to step 8.

  3. (Optional) Create a template in BMC Remedyforce to prepopulate commonly used data on an Incident form when a new incident is created from an email message.
  4. (Optional) In the newly created template, select the BSA Notification out-of-the-box category.
  5. (Optional) On the Remedyforce Administration > Application Settings > General Application Settings page, select Replace form field values with template field values option to use templates.
  6. (Optional) If you are using the Visualforce page layout user interface, navigate to Configuration > Application Settings > General tab, and select the Replace form field values with template field value check box.
  7. (Optional) Navigate to Remedyforce Administration > Configure Workflow & Approvals > Workflow Rules and clone the Apply template to BMC Server Automation created incident workflow rule and activate it.

    8. By default, the workflow is deactivated.

  8. (Optional) If you are using the Visualforce page layout user interface, navigate to Setup > Create > Workflows & Approvals > Workflow Rules, clone the Apply template to BMC Server Automation created incident workflow rule, and activate it.
  9. (Optional) Edit the cloned workflow, as follows:
    1. In the Workflow Actions section, from the Add Workflow Action list, select the New Field Update type of action.
    2. In the Field to Update list, select Template Name.
    3. In the Specify New Field Value section, select Use a formula to set the new value.
    4. Type the name of the newly created template enclosed in double quotes. For example, if the template name is CreateBSAIncident, type “CreateBSAIncident“.
  10. Create a job in BMC Server Automation Console. For more information, see Configuring jobs in BMC Server Automation Console.
  11. Submit the job in the BMC Server Automation Console.

Configuration results

Email messages are generated in BMC Server Automation and sent to BMC Remedyforce. From the received email message, an incident is created in BMC Remedyforce. The body of the email content is copied to the Description field of the incident.

Note: If the email address of a received email message does not match any of the client email address, you must select the Client for Unidentified Sender field. The Client for Unidentified Sender field is available on Remedyforce Administration > Configure Email > Incident - Email Settings page in the Identifying Email Sender section. If you are using the Visualforce page layout user interface, navigate to Configuration > Application Settings > Incident - Email Settings page. However, BMC recommends that you create a client for your monitoring system with the appropriate email address.
