You can import CIs from external sources to your CMDB. CMDB 2.0 is a flat table in which data is imported directly.

The Pentaho Data Integration tool is an open source solution that enables importing of data from external data sources into the BMC Remedyforce CMDB. The Pentaho packages, provided on the BMC Communities website, provide mapping between the external data sources and the CMDB objects. On the BMC Communities website, BMC provides you job files that are created in the Pentaho Data Integration tool to import data from SCCM, BMC Atrium Discovery, BMC BladeLogic Client Automation, BMC Client Management, Dell KACE, and BMC Atrium CMDB.

You can import data from BMC Atrium CMDB if you are using CMDB 2.0.

If you have upgraded to CMDB 2.0 and you were using CMDB 1.0 packages, you can continue using the CMDB 1.0 packages. However, BMC recommends that you use packages for CMDB 2.0 for faster imports. If you had created custom fields in the CMDB Staging table object but did not create corresponding fields in the Base Element object, delete those fields to avoid errors while importing data.

Data is fetched by using an SQL query (such as to import data from SCCM and BMC BladeLogic Client Automation) or an HTTP request (such as to import data from BMC Atrium Discovery).

Field mapping in CMDB 2.0

Mappings between the SCCM, BMC Atrium Discovery, BMC BladeLogic Client Automation, BMC Client Management, Dell KACE, and BMC Atrium CMDB database fields and the Remedyforce CMDB fields are predefined in the corresponding job files. If required, you can change the mapping while importing data.

You can also map the SCCM, BMC Atrium Discovery, BMC BladeLogic Client Automation, BMC Client Management, Dell KACE, and BMC Atrium CMDB database fields to custom attributes of a configuration item (CI). For more information, see Importing data into custom attributes.

If you are changing mapping, the following fields are mandatory:

    • BMCServiceDesk__CMDB_Class__c: Stores the Salesforce record ID of the class for which you are importing data.
    • BMCServiceDesk__UniqueCISourceID__c: Stores the unique ID of the CI that you import from data source. BMC recommends that you do not change the value of this field.
    • BMCServiceDesk__BE_Name__c: If there is no value specified for the BMCServiceDesk__BE_Name__c field, instance ID is used as instance name. If there is a value specified for BMCServiceDesk__BE_Name__c, the instance name is set in the <name>-<InstanceID> format.

The BMCServiceDesk__AssemblyId__c field is required to create a relationship between the computer system and various CIs. A relationship helps you to identify the source and destination of data. The source is the computer system and the destination can be operating system, processor, IP end points, LAN end points, software server, product, or persons.

Note: Do not map the BMCServiceDesk_AssemblyId__c field for BMC Atrium CMDB because you must import relationships from BMC Atrium CMDB by using the Pentaho packages available on the BMC Communities website.

You must map the fields of the SCCM, BMC Atrium Discovery, BMC BladeLogic Client Automation, BMC Client Management, Dell KACE, and BMC Atrium CMDB database fields to only one CI type and its hierarchy at a time.
