Action records are used to record details of the steps performed when working on an incident, task, broadcast, change request, or problem. BMC Remedyforce provides the following types of action records:

    • System action records—These action records are available out-of-the-box and cannot be deleted. They usually contain logon ID and duration information. For example, when an incident is created, BMC Remedyforce generates the following system action records:
      • The first action record identifies that the incident is open.
      • The second action record indicates that the clock has started
      • The third action record identifies the staff member who opened the incident.

        • These action records can be viewed in the Supporting Information section at the bottom of a form.

        Note: You cannot edit or delete system action records.

    • User-generated action records—These action records are manually created by a staff member or a system administrator. You can modify and delete them based on your requirements. You can create action records for actions that are performed frequently in your account. For example, you can create the action record of installing the latest operating system for staff members in the Network department.

To create an action

  1. In the BMC Remedyforce home page, click Configuration.
  2. From the list of Configuration options, expand Configuration.
  3. In the Configuration list, click Actions.
  4. In the actions list view, click New.
  5. In the Action field, type a unique identifier that you want to assign to the current action.
  6. In the Abbreviation field, type a short label that you want to assign to the action.

    7. When you create charts or reports, you can use this field to represent the current action record in the chart or report. The field uses minimal screen space, which allows you to display more data in a chart or report.

  7. In the Applies To section, select the required values:
    1. Broadcasts—Select this check box to designate the current action as one of the default actions that can be assigned when you add a broadcast supporting information record to a broadcast.
    2. Incidents—Select this check box to designate the current action as one of the default actions that can be assigned when you add an incident supporting information record to an incident.
    3. Tasks—Select this check box to designate the current action as one of the default actions that can be assigned when you add a task supporting information record to a task.
    4. Change Requests—Select this check box to designate the current action as one of the default actions that can be assigned when you add a change request supporting information record to a change request.
    5. Problems—Select this check box to designate the current action as one of the default actions that can be assigned when you add a problem supporting information record to a problem.
    6. Agreements—Select this check box to designate the current action as one of the default actions that can be assigned when you set up milestones for a service target in an agreement. For more information, see Creating service targets, milestones, and email templates.
    7. Configuration Items—Select this check box to designate the current action as one of the default actions that can be assigned when you add a configuration item supporting information record to a configuration item.
  8. In the Details section, select the required values:
    • Enable Edit—Select this check box to automatically open the action record for editing whenever you are adding an action.

      • Note: Even if you select this check box for the Client Note action, staff members cannot edit and save the Client Note action. Client Note actions are for the exclusive use of Self Service clients. This restriction is added to prevent staff members from editing the notes added by Self Service clients.

    • Display in Self Service—Select this check box to designate the current action as one of the default actions that can be assigned when you add, modify, or delete supporting information records in Self Service.
    • Track Time—Select this check box to track the time to complete the current action.

      • The time is recorded in the Duration column of the supporting information record of an incident, task, broadcast, change request, or problem that is assigned to the current action.

  9. Click Save.

    10. Note: When you open a new form to create a record and you save the record, you cannot navigate to the next or previous record by using the Next Record and Previous Record buttons in the upper right section of the form. These buttons are disabled in a new form.

  10. (Optional) To designate the current record as inactive, select the Inactive check box.

    11. For example, the action of replacing a CD drive is no longer applicable because all CD drives in the account were replaced with DVD drives.

    Note: Records marked as inactive are not included in any search performed on the database.
