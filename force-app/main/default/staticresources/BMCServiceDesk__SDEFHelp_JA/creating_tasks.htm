タスクとは、インシデントやサービスリクエストの解決手順として完了する必要がある作業単位です。インシデントの解決に複数のアクション、手順、プロセスが含まれている場合は、解決を個別のタスクに分割することを検討してください。解決に向けて進捗を管理するとき、解決を個別のタスクに分割することは、インシデントの管理と監視に役立ちます。

既存のインシデントからタスクを作成するか、［タスク］ページを使用して直接作成することができます。インシデントからタスクを作成する場合は、クライアント名、カテゴリ、説明などのインシデントに関するすべての詳細が、［タスク］フォームに入力されます。また、インシデント番号もタスクにコピーされるため、スタッフメンバーはタスクに関連したインシデントを参照できます。

たとえば、クライアントの1人がスタッフメンバーに連絡をとり、共有プリンタが動作しないことを報告したとします。スタッフメンバーがプリンタを確認し、プリンタカートリッジの交換が必要であることがわかりました。また、プリンタ用紙が残り少なくなっていて、購入の必要があることがわかりました。スタッフメンバーは、プリンタが動作しないという問題を記録するインシデントを作成します。次に、スタッフメンバーは、インシデントを解決するために完了する必要がある次のタスクを作成します。

    • タスク1： プリンタカートリッジを購入する。
    • タスク2： 障害のあるプリンタカートリッジを新しいプリンタカートリッジに交換する。
    • タスク3： プリンタ用紙を購入する。
    • タスク4： プリンタにプリンタ用紙を補給する。
パフォーマンスを向上させるためのベストプラクティス

目的の値の最初の数文字を入力して［Tab］キーを押すことで、リストへの入力と、このフォームのフィールドの検索を実行することができます。BMC Remedyforceでは一致する値が検索され、フィールドにその値が入力されます。たとえば、［カテゴリ］フィールドに「Net」と入力して［Tab］キーを押すと、「Network」カテゴリが［カテゴリ］フィールドに入力されます。

一致する値が複数見つかった場合、BMC Remedyforceにはルックアップウィンドウが表示され、そこで一致するレコードから必要な値を選択できます。

VisualforceページレイアウトUI でタスクを作成するには

  1. BMC Remedyforceホームページの［ワークスペース］をクリックします。
  2. ［ワークスペース］のリストから［インシデント管理］を展開します。
  3. ［インシデント管理］ドロップダウンリストで、［タスク］をクリックします。
  4. 新しい［タスク］フォームを開くには、［タスク］リストビューでをクリックします。

    5. ［補足情報］セクションは新規フォームに読み込まれません。レコードを保存すると、［補足情報を表示］リンクが表示されます。このリンクをクリックすると、［補足情報］セクションが表示されます。［補足情報］セクションが常に表示されるよう設定するには、［既存のレコードの補足情報を常に表示］チェックボックスをオンにします。詳細は、タスクの補足情報の管理を参照してください。

    ［インシデント番号］フィールドには、タスクが作成されたインシデントまたは関連付けられたインシデントのインシデント番号が設定されます。タスクがインシデントと関連付けられていない場合は、このフィールドが空白になります。

  5. ［テンプレート］をクリックします。

    6. 正確性と完全性を期するため、BMCではテンプレートを使用してタスクレコードを作成することをお勧めしています。テンプレートを使用すると、一貫した方法でタスクに関する情報が取得されます。管理者は、よく発生するタスクのテンプレートを定義できます。

  6. ［テンプレートから選択］ウィンドウで、該当するテンプレートをダブルクリックします。

    7. テンプレート内のフィールドの値が、［タスク］フォームの対応するフィールドに入力されます。

  7. ［名字］ドロップダウンリストをクリックします。

    8. 組織でのBMC Remedyforceの設定に応じて、［クライアントから選択］または［クライアントの検索］ウィンドウが表示されます。組織で連絡先やリードに対してタスクを作成できるように設定されている場合は、［クライアントの検索］ウィンドウが表示されます。このウィンドウでは、ユーザー、連絡先、リードを検索できます。また、［ユーザー］、［連絡先］、［リード］の個別のタブの下に、一致するエントリのリストが表示されます。

    管理者がおこなった設定により、連絡先やリードに対してタスクを作成できない場合は、［クライアントから選択］ウィンドウが開いてユーザーのリストが表示されます。レコードをダブルクリックして、選択することができます。

  8. ［クライアントの検索］ウィンドウでは、［キーワード］フィールドに検索キーワードを入力し、［Enter］キーを押すかをクリックします。

    9. 一致するレコードのリストは、［ユーザー］、［連絡先］、［リード］（該当する場合）の下に分類されます。リストに表示するレコードの数は、［最大レコード数］フィールドで変更できます。オブジェクトあたり、最大で200件のレコードを表示可能です。

    ［アカウント］、［携帯電話］、［ステータス］など、一致するレコードのデータを表示するデータフィールドは、［グローバル検索の設定］ページで設定できます。検索結果で表示するフィールドを設定するには、システム管理者に問い合わせてください。

  9. 適切なレコードをダブルクリックして選択します。

    10. ［名前］、［クライアントID］、［アカウント］フィールドに、選択したクライアントの名前、クライアントID、アカウント（存在する場合）が入力されます。クライアントのアカウントは変更できます。

    アイコンの上にマウスを置くと、選択したレコードの［アカウントID］、［携帯電話］など、［クライアントの検索］ウィンドウと同じフィールドが表示されます。［連絡先の詳細］は、表示される必須フィールドです。アイコンに表示されるフィールドは、［グローバル検索の設定］ページで設定できます。組織のフィールドを設定するには、システム管理者に問い合わせてください。このアイコンは、カスタムの［タスク］フォームにのみ表示されます。

  10. ［カテゴリ］ドロップダウンリストをクリックします。
  11. ［カテゴリから選択］ウィンドウで、次のアクションを実行します。
    1. タスクのカテゴリを選択します。

      2. カテゴリを使用すると、標準の分類方法でタスクを分類できます。分類によって報告を目的とした追跡が可能になります。

    2. 選択したカテゴリの標準説明を使用するには、［標準説明］チェックボックスをオンにします。

      3. 選択したカテゴリの標準説明が、タスクフォームの［説明］フィールドの末尾に追加されます。

    3. 選択したカテゴリの標準解決を使用するには、［標準解決］チェックボックスをオンにします。

      4. 選択したカテゴリの標準解決が、タスクフォームの［解決］フィールドの末尾に追加されます。

    4. ［OK］をクリックします。
  12. ［説明］フィールドに、そのタスクの完了に必要な作業内容を入力します。

    13. 作業内容とは、ハードウェアの配置、テスト環境の構成、テストの実施などです。

  13. ［解決］フィールドに、そのタスクに対して実施する必要がある解決を入力します。

    14. タスクの完了に必要な手順を詳細に入力するか、タスクを完了できない理由を入力します。

  14. ［メモ］フィールドに、タスクに関するメモを入力します。

    15. スタッフメンバーはタスクに関する更新情報を［メモ］フィールドに入力し、［追加］をクリックできます。これらの更新情報は、［補足情報］セクションに［履歴］タブの履歴レコードとして追加されます。スタッフメンバーは［メモ］フィールドを使用することで、［補足情報］セクションに移動して履歴レコードを作成しなくても、このタスクに関するメモを追加できます。

    注意: ［メモ］フィールドと［所要時間］フィールドは、レコードを保存した後でのみ使用可能になります。

  15. ［所要時間］フィールドに、タスクの処理にかかった時間を入力します。
  16. ［追加］をクリックします。

    17. ［メモ］フィールドと［所要時間］フィールドの内容が［補足情報］セクションに［履歴］タブの履歴レコードとして追加され、［補足情報］セクションの履歴レコードが表示されます。

  17. ［詳細］セクションで、次の情報を入力します。

    18. 注意: ［開始日］フィールドなど、一部のフィールドには値が自動的に入力されます。

    1. ステータス： 現在のタスクに割り当てるステータスを選択します。

      2. このフィールドを使用して、タスクの進捗状況を開始ステージから終了ステージまで追跡できます。

      タスクフォームのヘッダーにあるタスク進捗状況バーは、選択したステータスにリンクされているステージに応じて更新されます。たとえば、「作業中」ステータスが「処理中」ステージにリンクされている場合に、タスクのステータスとして［作業中］を選択した場合は、タスク進捗状況バーで［処理中］ステージが選択されます。

    2. インパクト： 現在のタスクに割り当てるインパクトを選択します。
    3. 緊急度： 現在のタスクに割り当てる緊急度を選択します。

      4. 緊急度とインパクトレベルは、タスクの優先度の計算に使用されます。

    4. 優先度： 緊急度とインパクトを選択してレコードを保存すると、［優先度］フィールドに値が入力されます。優先度は、選択した緊急度とインパクトの値によって計算されます。
    5. 開始日： このフィールドには、タスクが作成された日時が入力されます。
    6. 期限日： 緊急度とインパクトを選択してレコードを保存すると、［期限日］フィールドに値が入力されます。この期限日は優先度の期間に基づいて計算されます。

      7. 必要に応じて、計算された期限日とは異なる日時を選択できます。

      タスクの編集中にインパクトまたは緊急度を変更した結果、タスクの優先度が変更された場合は、次の［期限日の計算］オプションが表示されます。

      • 再計算しない： タスクの既存の期限日が保持されます。
      • 開始日に基づいて再計算： 開始日を使用して期限日が再計算されます。たとえば、優先度が1に変更され、優先度1に割り当てられた期間が2時間の場合は、再計算された期限日が、タスクの作成日時に2時間を加えた値になります。
      • 現在の日付に基づいて再計算： 現在の日付を使用して期限日が再計算されます。たとえば、優先度が1に変更され、優先度1に割り当てられた期間が2時間の場合は、再計算された期限日が、現在の日時に2時間を加えた値になります。

        • 注意: システム管理者が［アプリケーション設定］で［優先度を変更すると、ユーザー入力なしで期限日時が再計算されます］チェックボックスをオンにしている場合、［期限日の計算］ウィンドウは表示されません。

        システム管理者が［アプリケーションの設定］で［組織のデフォルトの営業時間を使用して期限日を再計算］チェックボックスをオンにしている場合は、期限日の計算時にデフォルトの営業時間、営業時間に関連付けられたタイムゾーン、組織に設定された休業日が使用されます。たとえば、システム管理者が組織のデフォルトの営業時間を午前10時～ 午後6時 に設定しているとします。この場合は、月曜日の午後5時にタスクを作成して 優先度を1に設定すると、タスクの期限日の計算結果は翌営業日の 火曜日の午前11時になります。ここで、システム管理者が火曜日を休業日として設定している場合は、タスクの期限日の計算結果が 翌営業日の水曜日の午前11時になります。

    7. 終了日： このフィールドには、タスクの終了日時が入力されます。
    8. インシデント番号/変更番号/問題番号/サービスリクエストID： このフィールドには、関連付けられたインシデント、変更リクエスト、問題、サービスリクエストのレコード番号が表示されます（これらが存在する場合）。レコード番号は、該当するレコードへのハイパーリングとして表示されます。レコード番号をクリックすると、そのレコードが対応するタブに表示されます。このフィールドは、タスクがインシデント、変更リクエスト、問題、サービスリクエストに関連付けられている場合にのみ表示されます。また、関連付けられたインシデント、変更リクエスト、問題、サービスリクエストのレコード番号をコピーすることもできます。

      9. 注意: タスクは、一度に1個のオブジェクト（インシデント、変更リクエスト、問題）だけにリンクできます。

      カスタムフォームを使用しており、システム管理者によって事前定義済みフィールドセットからフィールドが追加されている場合は、カスタムフィールド上のフォームに値を入力します。

  18. ［保存］をクリックします。

    19. 注意: 新規フォームを開いてレコードを作成した後、レコードを保存した場合は、フォームの右上セクションにある［次のレコード］および［前のレコード］ボタンを使用して、次または前のレコードに移動することはできません。これらのボタンは新規フォームで使用できません。

  19. タスクを作成したら、その補足情報を追加できます。

    20. 詳細は、タスクの補足情報の管理を参照してください。

  20. （オプション）現在のレコードをインアクティブとして指定するには、［インアクティブ］チェックボックスをオンにします。

    21. レコードはすでに無効になっているものの、レコード保持ポリシーを順守するためにレコードを保持する必要があるという理由で、レコードをインアクティブにしなければならない場合があります。たとえば、クライアントがアカウントを失うと、そのクライアントに関連付けられた全タスクが無効になります。

    注意: インアクティブとしてマークされたレコードは、データベースで実行される検索の対象に含まれません。

システム管理者が「タスクが作成された場合にクライアントに通知」ワークフローをアクティブにすると、タスクの作成時に、電子メールメッセージがタスクのクライアントへ送信されます。カスタムタスクフォームの場合、タスクのクライアントとしてリードを選択すると、電子メールメッセージは送信されません。

タスクの補足情報の管理

タスクフォームの下部セクションにある［補足情報］タブでは、次のアクションを実行できます。

    • タスクの履歴の追跡
    • サービスとCIのリンク
    • ドキュメンテーションの追加
    • タスクに関する監査情報の表示

タスクフォームには、次の［補足情報］タブが含まれています。

    • 履歴： 既存のユーザー定義アクションやシステム定義アクションを表示したり、新しいユーザー定義アクションを追加したりすることで、タスクに対して実行されたすべてのアクションの履歴を追跡できます。
    • ［サービスおよびCI］： タスクを完了するときに影響を受けるサービスとCIをリンクできます。
    • ドキュメンテーション： タスクを終了するために必要なドキュメントと添付ファイルへのURLを添付できます。
    • ［監査］： このフォームのフィールドに加えられた変更の履歴が表示されます。システム管理者が、目的のフィールドのフィールド履歴追跡を有効にします。

      • 注意: ［補足情報］セクションは新規フォームに読み込まれません。レコードを保存すると、［補足情報を表示］リンクが表示されます。このリンクをクリックすると、［補足情報］セクションが表示されます。［補足情報］セクションが常に表示されるよう設定するには、［既存のレコードの補足情報を常に表示］チェックボックスをオンにします。［補足情報］セクションを表示した後で［補足情報］セクションを非表示にする場合は、［非表示］リンクをクリックします。
