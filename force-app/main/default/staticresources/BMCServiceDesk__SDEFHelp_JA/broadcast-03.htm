メッセージは、アカウント全体、アカウント内の選択したプロファイル、クライアントに対して送信可能です。進行中の作業、システムステータス、計画された作業などに関して、メッセージを送信できます。また、アカウント内の別グループから自分にブロードキャストされたメッセージを表示することも可能です。

Salesforceページレイアウトでブロードキャストを作成するには

  1. Salesforceホームページで［ブロードキャスト］タブをクリックします。
  2. ブロードキャストのホームページで［新規作成］をクリックします。

    3. フォームに一般的なアイテムが表示されます。詳細は、Salesforceページレイアウトのフォーム上の共通のアイテムを参照してください。次の表に、［新規ブロードキャスト］フォームで使用可能なフィールドのリストを示します。

    フィールド                   
    説明
    ブロードキャストの詳細
    ブロードキャストID
    現在のブロードキャストに割り当てる固有の識別子を英数字で入力します。
    カテゴリ
    ブロードキャストに割り当てるカテゴリを選択します。
    レコードを保存すると、カテゴリの値の横にアイコンが表示されます。をクリックすると、カテゴリツリーが表示されます。マウスをカテゴリの上に移動すると、カテゴリの説明がツールチップとして表示されます。レコードの別のカテゴリを選択するには、そのカテゴリを選択して、［OK］をクリックします。
    メッセージ
    停止または問題の簡単な説明を入力します。
    メッセージのスクロール
    インシデント、問題、ブロードキャスト、タスク、変更リクエストの各詳細ページの［ステータスの詳細］セクションに［メッセージ］フィールドの内容を表示するには、このチェックボックスをオンにします。
    ブロードキャストメッセージは、スクロールの開始に最大5分かかることがあります。これはブロードキャストのスクロールのメカニズムが5分ごとに更新されるためです。複数のユーザーがBMC Remedyforceにログオンしている場合は、各ユーザーのブロードキャストのスクロールメカニズムが最後に更新された時期に応じて、ユーザーごとに新しいブロードキャストメッセージを表示できます。たとえば、スタッフメンバーのブロードキャストのスクロールメカニズムが2分前に更新された場合は、このスタッフメンバーはこれから3分以内にブロードキャストメッセージの表示を開始できます。
    説明
    停止または問題の詳細な説明を入力します。
    この説明は、ブロードキャストに関連付けられているインシデントの［説明］フィールドに追加されます。
    解決方法
    停止または問題を解決するために実施した手順を入力します。
    インパクト
    ブロードキャストに割り当てるインパクトを選択します。
    インパクトレベルと緊急度は、ブロードキャストの優先度の計算に使用されます。
    緊急度
    ブロードキャストに割り当てる緊急度を選択します。
    インパクトレベルと緊急度は、ブロードキャストの優先度の計算に使用されます。
    優先度
    緊急度とインパクトの値を選択してレコードを保存すると、［優先度］フィールドに値が入力されます。優先度は、選択した緊急度とインパクトの値によって計算されます。
    ステータス
    ブロードキャストに割り当てるステータスを選択します。
    このフィールドを使用して、ブロードキャストの進捗状況をステージを通して追跡できます。
    ［ブロードキャスト］フォームのヘッダーにあるブロードキャスト進捗状況バーは、選択したステータスにリンクされているステージに応じて更新されます。たとえば、「作業中」ステータスが「処理中」ステージにリンクされている場合に、ブロードキャストのステータスとして［作業中］を選択した場合は、ブロードキャスト進捗状況バーで［処理中］ステージが選択されます。
    完了カテゴリ
    ブロードキャストの終了時に完了カテゴリを選択した場合、選択した値がこのフィールドに表示されます。ブロードキャストの作成時には、完了カテゴリを選択しないでください。
    デフォルトでは、［ブロードキャスト］フォームに［完了カテゴリ］フィールドが表示されません。ページレイアウトを編集してフォームにこのフィールドを表示するには、システム管理者に問い合わせてください。
    無効
    ブロードキャストをインアクティブに表示するには、このチェックボックスをオンにします。
    レコードはすでに無効になっているものの、レコード保持ポリシーを順守するためにレコードを保持する必要があるという理由で、レコードをインアクティブにしなければならない場合があります。たとえば、問題や停止が、すでに存在しないルーターやネットワークから削除されたルーターに関連している場合などです。

    メモ:  インアクティブとしてマークされたレコードは、データベースで実行される検索の対象に含まれません。
    ブロードキャスト日時の詳細
    投稿日
    このフィールドには、ブロードキャストを保存したシステム日付が入力されます。
    期限日
    ブロードキャストメッセージの表示を停止する日付を選択します。
    終了日
    このフィールドには、［終了］をクリックした後、システム日付が表示されます。［終了］ボタンは、ブロードキャストの保存後に、ブロードキャストの詳細ページに表示されます。
    スタッフ割り当ての詳細
    オーナー
    ブロードキャストの保存後、このフィールドには、ブロードキャストを作成したログオンユーザーの名前が表示されます。ブロードキャストのオーナーは、ブロードキャストの保存後に変更できます。ブロードキャスト割り当ての変更についての詳細は、ブロードキャストオーナーの変更を参照してください。

  3. ［保存］をクリックします。

    4. ブロードキャストを保存すると、ブロードキャストの詳細ページが表示されます。ブロードキャストの詳細ページの［ブロードキャストの表示］セクションに、次の追加フィールドが表示されます。

    • すべてのスタッフ： このブロードキャストをすべてのスタッフメンバーに表示する場合は、このチェックボックスをオンにします。
    • 表示先プロファイル： ブロードキャストメッセージを表示するユーザーのプロファイルを選択します。このフィールドを有効にするには、［すべてのスタッフ］チェックボックスをオフにします。
    • セルフサービスに表示： ブロードキャストをBMC Remedyforceセルフサービスのすべてのユーザーに表示する場合は、このチェックボックスをオンにします。［セルフサービスに表示］チェックボックスをオンにする場合は、次のうち適切なオプションを選択します。
      • すべてのアカウントのユーザーに表示: このオプションを選択すると、すべてのアカウントのすべてのユーザーにブロードキャストが表示されます。
      • これらのアカウントのユーザーのみに表示: このオプションを選択すると、選択したアカウントのユーザーにのみブロードキャストが表示されます。選択したアカウントのリストが、このオプションの下に表示されます。これらのアカウントの横にあるボタンをクリックすると、［セルフサービスのブロードキャスト表示のアカウントを選択してください］ウィンドウが開きます。このウィンドウでは、［使用可能なアカウント］リストで目的のアカウントを選択して右矢印をクリックし、選択したアカウントを［使用可能なアカウント］リストから［選択したアカウント］リストに移動します。一度に多数のアカウントがブロードキャストを使用できるようにすることができます。

        • また、［検索］フィールドに検索条件を入力して［検索］をクリックすることにより、特定のアカウントを検索することもできます。［検索］フィールドの検索条件をクリアして［使用可能なアカウント］リストのアカウントリストを更新するには、［元に戻す］をクリックします。検索条件の最後にワイルドカード文字*を使用して1つまたは複数の文字をマッチングすることができますが、検索条件の最初や中間では使用できません。アカウントの次のページや前のページに移動するには、［元に戻す］の横にある［次のページ］および［前のページ］を使用します。目的のアカウントを選択したら、［OK］をクリックします。

        注意: 非ポータルユーザーにブロードキャストを表示するには、［セルフサービスに表示］チェックボックス、［すべてのアカウントのユーザーに表示］オプションを必ずオンにします。そうしないと、非ポータルユーザーのアカウントは［これらのアカウントのユーザーのみに表示］フィールドに追加されます。
        ［使用可能なアカウント］リストにアカウントが表示されない場合は、Salesforceで顧客ポータル機能が有効になっていることを確認します。顧客ポータルの有効化についての詳細は、 https://na12.salesforce.com/help/doc/user_ed.jsp?loc=help&target=customer_portal_setting_up.htm&section=CustomerPortalを参照してください。
