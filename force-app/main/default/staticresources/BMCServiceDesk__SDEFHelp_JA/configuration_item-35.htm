コンソールを起動するには、Oracle Java Runtime Environment（JRE）の最新バージョンをインストールする必要があります。BMC FootPrints Asset Coreとの統合はJRE 6以降のバージョンで機能します。BMCではJRE 7の使用をお勧めします。JREは http://www.java.com/getjavaからダウンロードできます。詳細は、JREのインストールを参照してください。JREをインストールすると、Java Web Startがコンピューターにインストールされます。BMC Remedyforceでは、Java Web Startを使用して［Asset Core アクション］メニューのコマンドを実行する必要があります。BMC FootPrints Asset Coreデバイスを現在のインシデントまたはサービスリクエストにリンクする場合は、［CIアクション］をクリックし、［Asset Coreアクション］メニューからコマンドを選択して、［実行］をクリックすると、コマンドが実行されます。

ブラウザのダウンロード設定によっては、Salesforce組織からCMDBManager.JNLPファイルをダウンロードするように求められる場合があります。CMDBManager.JNLPファイルがすでにキャッシュに存在する場合、ファイル名は1ずつ加算され、CMDBManager1.JNLP、CMDBManager2.JNLPなどになります。メッセージが表示されたら、Java Web Startを使用してJNLPファイルを開き、メニューコマンドを実行する必要があります。Java Web Startについての詳細は、 http://docs.oracle.com/javase/7/docs/technotes/gUI des/javawsを参照してください。

［Asset Coreアクション］メニューのコマンドを実行するときにJNLPファイルのダウンロードを求められないようにするには、ブラウザを次のように設定します。

注意: 一部の設定はブラウザのバージョンによって異なる場合があります。

  1. Microsoft Internet Explorerを使用している場合は、次のように設定します。
    1. ［ツール］メニューで、［インターネットオプション］をクリックします。
    2. ［セキュリティ］タブをクリックします。
    3. ［このゾーンのセキュリティのレベル］セクションで、［レベルのカスタマイズ］をクリックします。
    4. ［ダウンロード］オプションで、［ファイルのダウンロード時に自動的にダイアログを表示］オプションと［ファイルのダウンロード］オプションの［有効にする］を選択します。
    5. ［OK］をクリックします。
    6. ［詳細設定］タブをクリックします。
    7. ［Java (Oracle)］オプションで、［<アプレット>にJRE <バージョン番号>を使用する(再起動が必要)］チェックボックスをオンにします。
    8. ［セキュリティ］オプションで、［暗号化されたページをディスクに保存しない］チェックボックスをオフにします。
    9. ［インターネットオプション］ウィンドウを閉じるには、［OK］をクリックします。
  2. Mozilla Firefoxを使用している場合は、次のように設定します。
    1. ［ツール］メニューで、［オプション］をクリックします。
    2. ［ファイルをダウンロードするときに［ダウンロード］ウィンドウを表示］チェックボックスをオフにします。
    3. ［アプリケーション］タブをクリックします。
    4. JNLPファイルについて、［Java™ Web Start Launcherを使用する］を選択します。
    5. ［OK］をクリックします。
    6. ［スタート］>［コントロールパネル］>［Java］の順に選択します。
    7. ［詳細設定］タブをクリックします。
    8. ［Javaコンソール］の設定を展開して、［コンソールの非表示］オプションを選択します。
    9. ［JNLPファイル/MIMEの関連付け］の設定を展開して、［常に許可］オプションを選択します。
    10. ［OK］をクリックします。

      11. 注意: 有効なBMC FootPrints Asset Core管理者が、BMC Remedyforceから初めてBMC FootPrints Asset Coreに接続する場合、［BMC FootPrints Asset Core認証］ダイアログボックスが表示されます。BMC FootPrints Asset Core資格情報をこのダイアログボックスに入力する必要があります。BMC FootPrints Asset Coreユーザー名が保存され、以降は接続のたびに資格情報を入力する必要はありません。

［Asset Coreアクション］メニューを使用して、BMC FootPrints Asset Coreデバイスで次のアクションを実行できます。

    • インベントリの概要の表示
    • リモート制御セッションの開始
    • デバイスのファイルシステムの表示
    • デバイスのレジストリの表示
    • デバイスにログオンしているWindowsイベントの表示
    • デバイスで実行されているプログラムとプロセスの表示
    • デバイスで実行されているサービスの開始、停止、再開
    • デバイスでの監査の実行
    • デバイスと、BMC Remedyforceへのアクセスに使用するコンピューター間でのファイルの転送
    • デバイスの接続を確認して、デバイスが接続可能かどうかを確認
    • デバイスの再起動
    • デバイスのシャットダウン
    • デバイスの再開

      • 注意: コマンドを実行しているデバイスのタイプ、デバイスに設定されたアクション、およびBMC FootPrints Asset Coreアカウントに設定されたアクセス権によっては、メニューコマンドの一部が無効になっている場合があります。

次のフォームで［Asset Coreアクション］メニューが表示されます。

    • アセットサマリー
    • 構成アイテムリストビュー
    • インスタンスエディタ
    • BMC Remedyforceタブ形式ビューのインシデントフォーム

［Asset Coreアクション］メニューは、次の条件が満たされた場合にだけ利用できます。

    • システム管理者がBMC RemedyforceとBMC FootPrints Asset Coreとの統合を有効にしている。
    • BMC FootPrints Asset Coreからデバイスをインポートしていて、そのデバイスを構成アイテムリストビューで選択しているか、またはインスタンスエディタでそのデバイスを開いている。
    • CIを選択しているか、またはBMC FootPrints Asset Coreでデバイスを表すCIを開いている。
    • デバイス情報は、BMC FootPrints Asset CoreからBMC Remedyforce CMDBにインポートされる。

次の条件が満たされた場合、［Asset Coreアクション］メニューが無効になるか、または［Asset Coreアクション］メニューのコマンドを使用できません。

    • CIを選択しているか、またはBMC FootPrints Asset CoreにあるデバイスではないCIを開いている。
    • CMDBクラスにCIが含まれていない。
    • 選択したBMC FootPrints Asset Coreデバイスが接続できない、または削除されている。

次の理由により、デバイスで［Asset Coreアクション］メニューコマンドを実行できないことがあります。

    • デバイスが見つからない。
    • BMC FootPrints Asset Coreデータベースでデバイスが削除されている。
    • BMC Remedyforceでデバイスを表示する権限がない。
    • アクションを実行する権限がない。
    • BMC FootPrints Asset Coreの認証に失敗した。
    • そのアクションがデバイスでは使用できない。

      • 注意: 選択したデバイスに接続できない場合、［Asset Coreアクション］メニューのコマンドは使用できません。デバイスを選択して［Asset Coreアクション］メニューをクリックすると、次のエラーメッセージが表示されます。
      Error occurred while trying to connect to BMC FootPrints Asset Core.（BMC FootPrints Asset Coreへの接続の試行中にエラーが発生しました。） Access denied.（アクセスが拒否されました。） Please verify your access rights.（アクセス権を確認してください。）

表11-3に、［Asset Coreアクション］メニューのコマンドの説明を示します。

表11-3：  ［Asset Coreアクション］メニューのコマンド
［Asset Coreアクション］メニューのコマンド                   
説明
Inventory Summary（インベントリの概要）
選択したデバイスのインベントリの概要が表示されます。概要にはハードウェア、ソフトウェア、ライセンス、セキュリティ、財務、運用規則アクション履歴の詳細が含まれます。詳細は、インベントリの概要の表示を参照してください。
Remote Control（リモート制御）
BMC FootPrints Asset Coreのリモート制御ノードを起動します。このノードでは、選択したデバイスを使用して、リモート制御セッションを開始できます。デバイス上で必要な機能を実行することも、ローカルユーザーを支援するために制御することもできます。
ファイルシステム
選択したデバイスのディレクトリ構造とファイルを表示します。ディレクトリとファイルはコピー、移動、名前の変更、削除ができます。また、選択したデバイスのユーザーや管理者によって設定されたプライバシー制限内でテキストファイルを編集することもできます。
レジストリ
BMC FootPrints Asset Coreのレジストリノードを起動します。このノードから、Microsoftレジストリにアクセスしたり、レジストリを更新することができます。レジストリキーと接続値の追加、変更、削除ができます。
Windowsイベント
BMC FootPrints Asset Coreのイベントノードを起動します。このノードから、Windowsオペレーティングシステムを実行するデバイスにログオンしているすべてのイベントを表示したり調査することができます。
プロセス管理
BMC FootPrints Asset Coreのプロセス管理ノードを起動します。このノードから、選択したデバイスで実行されているプログラムとプロセスを表示できます。このノードを使用して、コンピューターのパフォーマンスの主要インジケータを監視できます。また、実行中のプログラムの状態を表示したり、応答が停止したプログラムを終了したりできます。さらに、15個のパラメータを使用して実行中のプロセスのアクティビティを評価したり、CPU上のデータやメモリの使用状況を確認したりできます。
サービス
BMC FootPrints Asset Coreのサービスノードを起動します。このノードでは、選択したデバイスで実行されているサービスを開始、停止、再開できます。選択したデバイスで実行されているサービスのプロパティも表示できます。
今すぐ監査
選択したデバイスに監査リクエストを送信します。監査リクエストが正常に送信されると、確認ダイアログボックスが表示されます。確認ダイアログボックスを閉じるには、［OK］をクリックします。
監査リクエストが終了すると、最新のデバイス情報が生成され、BMC FootPrints Asset Coreデータベースにアップロードされます。
ファイル転送
BMC FootPrints Asset Coreのファイル転送ノードを起動します。このノードから、選択したデバイスと、BMC Remedyforceへのアクセスに使用するコンピューターとの間でファイルを転送できます。また、ファイルとディレクトリの作成、削除、名前の変更ができます。
Check Connection（接続の確認）
選択したデバイスに、そのデバイスが接続可能であるかどうかを確認するリクエストを送信します。選択したデバイスが接続可能な場合は、BMC FootPrints Asset Coreの［Ping］ダイアログボックスに次のメッセージが表示されます。
Successful connection to the agent.（エージェントに正常に接続しました。）
［閉じる］をクリックして［Ping］ダイアログボックスを閉じ、BMC Remedyforceに戻ります。
再起動
選択したデバイスを再起動するリクエストを送信します。BMC FootPrints Asset Coreの［再起動］ダイアログボックスに確認ダイアログボックスが表示されます。確認ダイアログボックスで［Yes］をクリックします。選択したデバイスのシャットダウンと再起動のステータスが［再起動］ダイアログボックスに表示されます。
シャットダウン
選択したデバイスをシャットダウンするリクエストを送信します。BMC FootPrints Asset Coreの［シャットダウン］ダイアログボックスに確認ダイアログボックスが表示されます。確認ダイアログボックスで［Yes］をクリックします。選択したデバイスのシャットダウンのステータスが［シャットダウン］ダイアログボックスに表示されます。
Wake Up（再開）
選択したデバイスを再開するリクエストを送信します。BMC FootPrints Asset Coreの［再開］ダイアログボックスに確認ダイアログボックスが表示されます。確認ダイアログボックスで［Yes］をクリックします。リクエストが終了し、デバイスが再開されると、［再開］ダイアログボックスに確認ダイアログボックスが表示されます。［閉じる］をクリックして［再開］ダイアログボックスを閉じ、BMC Remedyforceに戻ります。
