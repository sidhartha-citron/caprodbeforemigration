Keith CarltonはLockStock社という証券会社のネットワーク管理者として、千台のサーバーを管理しています。

Keithはサーバーを監視するようにBMC Server Automationを設定しました。さらに、ターゲットサーバーの設定を分析し、必要なパッチを判別するためのパッチ適用ジョブを作成しました。また、パッチ適用ジョブに含まれるパッチ修正ジョブを使用して、パッチが必要なターゲットサーバーにパッチを導入するスケジュールを指定しました。

Keithは、関連するパッチ導入ジョブが完了した場合にアラートを通知するように、デフォルトの電子メール通知を設定しました。同様に、パッチ導入ジョブが完了した場合に完了ステータスをターゲットサーバーにアラートするように、デフォルトのSNMP通知を設定しました。このようにすると、適切なサーバーソフトウェアがSNMPメッセージを受信および解釈して、適切に対応できます。ただし、着信SNMP通信を解釈するためのソフトウェアがインストールされていないサーバーがあるため、Keithは電子メール通知を利用する必要があります。

KeithはBMC Remedyforceに登録して、電子メール通知の代わりにBMC Remedyforceを介したインシデントを使用しようとしています。Keithの狙いは、既存のアラートが発生した場合、インシデントが開き、適切なサーバーおよびサポートグループにインシデントが直接送信されるようにすることです。インシデント追跡システムを使用すると、サポートグループは電子メール通知を受信するだけでなく、インシデントをアクティブに、かつ経験に基づいて監視することもできます。

BMC Remedyforce管理者のBob Heartは、BMC Remedyforceでインシデントを作成するための統合環境を設定できるようKeithを支援しています。


手順1
BobはBMC Remedyforceで電子メールリスナーを作成し、定義済みのEmailCreateIncident Apexクラスを電子メールサービスに追加しました。


手順2
BobはBMC RemedyforceでCreateBSAIncidentという名前のテンプレートを作成しました。このテンプレートにはインシデントフォームの［インパクト］、［クライアントID］、［カテゴリ］、［説明］、［ステータス］フィールドが含まれていて、電子メールメッセージから新しいインシデントを作成することができます。BobはBMC Remedyforceでインシデントを作成するためのBMC Server Automationの設定 の手順4～7に従って、複製されたワークフロー内でCreateBSAIncidentテンプレートを指定しました。


手順3
KeithはBMC Server Automationコンソールでのジョブの設定の手順を実行して、BMC Server Automationコンソールでジョブを作成しました。


手順4
電子メールメッセージが生成され、BMC Remedyforceに送信され、BMC Remedyforceでは受信した電子メールメッセージからインシデントが作成されました。
