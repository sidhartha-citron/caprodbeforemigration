BMC FootPrints Asset Coreで作業するために、または［Asset Coreアクション］メニューで使用できるアクション以外の追加アクションを実行するために、CMDB ManagerからBMC FootPrints Asset Coreコンソールを起動できます。構成アイテムリストビューで、［FootPrintsコンソールの起動］ツールバーボタンをクリックして、BMC FootPrints Asset Coreコンソールを起動します。

注意: このツールバーボタンは、システム管理者がBMC RemedyforceとBMC FootPrints Asset Coreとの統合を有効にした場合にだけ、構成アイテムリストビューに表示されます。

コンソールを起動するには、Oracle® Java Runtime Environment（JRE™）の最新バージョンをインストールする必要があります。BMC FootPrints Asset Coreとの統合はJRE 6以降のバージョンで機能します。BMCではJRE 7の使用をお勧めします。JREは http://www.java.com/getjavaからダウンロードできます。詳細は、JREのインストールを参照してください。JREをインストールすると、Java Web Startがコンピューターにインストールされます。BMC Remedyforceでは、Java Web Startを使用して、コンソールが起動されます。［FootPrintsコンソールの起動］ツールバーボタンをクリックすると、Salesforce組織からCMDBManager.JNLPファイルをダウンロードするように求められます。CMDBManager.JNLPファイルがすでにキャッシュに存在する場合、ファイル名は1ずつ加算され、CMDBManager1.JNLP、CMDBManager2.JNLPなどになります。コンソールを起動するには、Java Web Startを使用してJNLPファイルを開く必要があります。Java Web Startについての詳細は、 http://docs.oracle.com/javase/7/docs/technotes/gUI des/javawsを参照してください。

注意: 有効なBMC FootPrints Asset Core管理者が、BMC Remedyforceから初めてBMC FootPrints Asset Coreに接続する場合、［BMC FootPrints Asset Core認証］ダイアログボックスが表示されます。BMC FootPrints Asset Core資格情報をこのダイアログボックスに入力する必要があります。BMC FootPrints Asset Coreユーザー名が保存され、以降は接続のたびに資格情報を入力する必要はありません。

Microsoft Internet Explorer 8.0または9.0を使用している場合は、ブラウザでBMC FootPrints Asset Coreコンソールを正常に起動できるように、次のように設定します。

注意: 一部の設定はブラウザのバージョンによって異なる場合があります。

  1. Microsoft Internet Explorerブラウザウィンドウで、［ツール］メニューの［インターネットオプション］をクリックします。
  2. ［セキュリティ］タブをクリックします。
  3. ゾーンを選択します。
  4. ［このゾーンのセキュリティのレベル］セクションで、［レベルのカスタマイズ］をクリックします。
  5. ［ダウンロード］オプションで、［ファイルのダウンロード時に自動的にダイアログを表示］オプションと［ファイルのダウンロード］オプションの［有効にする］を選択します。
  6. ［OK］をクリックします。
  7. ［詳細設定］タブをクリックします。
  8. ［セキュリティ］オプションで、［暗号化されたページをディスクに保存しない］チェックボックスをオフにします。
  9. ［インターネットオプション］ウィンドウを閉じるには、［OK］をクリックします。
