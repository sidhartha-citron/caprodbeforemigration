SCCMからデータをインポートする前に、SCCMからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

Pentahoを使用してSCCMからデータをインポートするには

  1. 使用しているCMDBバージョンに応じて、次のいずれかのリンクを使用して、BMC Communitiesページからジョブファイルをダウンロードします。

    2. CMDBバージョン                   
    Pentahoパッケージをダウンロードするためのリンク
    1.0
    https://communities.bmc.com/docs/DOC-14522
    2.0
    https://communities.bmc.com/docs/DOC-28432

    注意: CMDB 2.0にアップグレードしていて、以前はCMDB 1.0パッケージを使用していた場合、引き続きCMDB 1.0パッケージを使用できます。ただし、BMCは、迅速なインポートのために、CMDB 2.0用のパッケージを使用することをお勧めします。

  2. ジョブファイルは、Pentaho Data Integrationツールを解凍したフォルダにダウンロードすることをお勧めします。

    3. 注意: フォルダ構造は変更しないでください。CombinedJobs（統合ジョブ）フォルダ内のバッチファイルの実行が中断される場合があります。

  3. Pentaho Data Integrationツールを起動するには、\pdi-ce-4.x.0-stable\data-integrationまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダでSpoon.batファイルをダブルクリックして目的のKJBファイルを開きます。
  4. KTRファイルを開くには、使用しているCMDBおよびPentahoのバージョンに応じて、次のいずれかのアクションを実行します。

    5. CMDBバージョン                   
    Pentahoバージョン
    アクション
    1.0
    4.4
    ［ステージリストをSCCM<CIタイプ名>情報で更新］手順を右クリックし、［データ変換を開く］を選択します。
    1.0
    5.0.1
    ［ステージリストをSCCM<CIタイプ名>情報で更新］手順を右クリックし、［Open referenced object（参照オブジェクトを開く）］を選択してから、［Transformation（データ変換）］を選択します。
    2.0
    4.4
    ［Update BE Table With SCCM <CI type name> Information（BEテーブルをSCCM<CIタイプ名>情報で更新）］手順を右クリックし、［データ変換を開く］を選択します。
    2.0
    5.0.1
    ［Update BE table with SCCM <CI type name> information（BEテーブルをSCCM<CIタイプ名>情報で更新）］手順を右クリックし、［Open referenced object（参照オブジェクトを開く）］を選択してから、［Transformation（データ変換）］を選択します。

  5. SCCMへのアクセスに必要な情報を入力するには、次のアクションを実行します。
    1. KTRファイルで、［SCCM入力］手順をダブルクリックします。
    2. ［接続］リストの横にある［編集］をクリックします。
    3. ［データベース接続］ウィンドウで、ホスト名、データベース名、ユーザー名、およびパスワードを入力します。

      4. ホスト名は、SCCMがインストールされているコンピュータの名前です。

      注意: SCCMデータベースにWindows認証を設定している場合は、ユーザー名およびパスワードは必要ありません。

    4. （オプション）SCCMサーバーとPentaho Data Integrationツールとの間の接続を検証するには、［テスト］をクリックします。
    5. ［OK］をクリックします。
  6. Salesforce組織へのアクセスに必要な情報を入力するには、次のアクションを実行します。
    1. ［クラス<CIタイプ名>にアップサート］手順をダブルクリックします。
    2. ［Salesforceのアップサート］ウィンドウで、Salesforce組織のユーザー名とパスワードを入力します。
    3. （オプション）SCCMフィールドとBMC Remedyforce CMDB間のマッピングは事前に定義されています。マッピングを更新するには、［マッピングの編集］をクリックします。マッピングについての詳細は、CMDB 1.0でのフィールドのマッピングを参照してください。

      4. 注意: Pentaho 4.1や4.2などの以前のバージョンのPentaho Data Integrationツールを使用している場合は、Salesforce WebサービスのURLをhttps://www.salesforce.com/services/Soap/u/20.0に変更します。

    4. （オプション）接続をテストするには、［接続テスト］をクリックします。
    5. ［OK］をクリックします。
  7. （CMDB 2.0の場合のみ）［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］手順でSalesforceログオン詳細情報を入力する必要があります。Salesforce組織へのアクセスに必要な情報を入力するには、次の手順を実行します。
    1. ［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］をダブルクリックします。
    2. ［接続］セクションで、Salesforce組織のアクセス詳細を入力します。
    3. （オプション）接続をテストするには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。
  8. （オプション）識別されていないネットワークからデータをインポートする場合、SalesforceとPentaho Data Integrationツールの間の通信を有効にするには、次のアクションを実行します。
    1. Salesforceで、［設定］>［自分のセキュリティトークンのリセット］の順に選択します。

      2. 注意: Salesforce組織で、改善された［設定］ユーザーインターフェイスが有効になっている場合は、［My Settings（私の設定）］>［Personal（個人）］>［自分のセキュリティトークンのリセット］に移動します。改善された［設定］ユーザーインターフェイスの詳細については、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。

    2. ［セキュリティトークンのリセット］をクリックすると、Salesforceに保存されている電子メールアドレスに電子メールが送信されます。
    3. ［クラス<CIタイプ名>にアップサート］手順の［パスワード］フィールドで、パスワードの後ろにセキュリティトークンを追加します。

      4. たとえば、パスワードがmypasswordで、セキュリティトークンがXXXXXXXXXであれば、［パスワード］フィールドに「mypasswordXXXXXXXXX」と入力する必要があります。

  9. KJBファイルとKTRファイルを保存するには、をクリックします。
  10. KJBファイルで、をクリックします。
  11. ［ジョブの実行］ウィンドウで、［起動］をクリックします。

    12. 完了した変換は、で示されます。は現在実行中の変換を示し、は失敗した変換を示します。

  12. （オプション）ログを表示するには、［実行の結果］セクションで［ログ］タブをクリックします。

    13. すべてのエラーは赤で表示されます。

    コンピュータシステム（ソース）とソフトウェアサーバー（宛先）との関係は、構成アイテムエクスプローラには表示されません。これは、（プラットフォームの制限により）構成アイテムエクスプローラは一部の関係しか表示できないためです。コンピュータシステムとソフトウェアサーバー間の関係を表示すると、コンピュータシステムと他のCI間の重要な関係が非表示になる場合があります。

  13. （オプション）ジョブのインポートをスケジュールするには、インポートしたジョブのスケジュール作成を参照してください。

    14. 注意: BMC Communitiesページからダウンロードしたフォルダで、サブフォルダを1つずつダブルクリックし、ComputerSystem、IPEndPoints、LANEndPoints、OS、Processor、およびSoftwareServerフォルダからKTRファイルとKJBファイルをすべてコピーして、それらのファイルをSCCM\フォルダに貼り付けます。

コンピュータシステムと各種CIとの間に関係を作成する場合は、BMCServiceDesk__AssemblyId__cフィールドが必要です。ソースはコンピュータシステムであり、宛先にはオペレーティングシステム、プロセッサ、IPエンドポイント、LANエンドポイント、ソフトウェアサーバーなどがあります。

たとえば、コンピュータシステムレコードに対して［BMCServiceDesk__AssemblyId__c］フィールドの値が1で、プロセッサレコードのアセンブリID値が1の場合、コンピュータシステム（ソース）とプロセッサ（ターゲット）の間に関係が自動的に作成されます。
