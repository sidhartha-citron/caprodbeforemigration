問題を保存後、［アクション履歴］セクション（VisualforceページレイアウトUI の場合は［補足情報］セクションの［履歴］タブ）内の問題にユーザー定義アクションとシステム定義アクションを追加できます。

アクション履歴レコードを追加するには

  1. アクションを追加する問題に移動します。

    2. 詳細は、問題レコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［関連リスト］をクリックし、［アクション履歴］セクションで［新規アクション］をクリックます。
    • ［問題］タブ（Salesforceページレイアウト）を使用している場合は、［アクション履歴］セクションの［新規アクション］をクリックします。
    • VisualforceページレイアウトUI を使用している場合は、［補足情報］セクションで、［履歴］タブをクリックして、の［追加］を選択します。
  3. ［アクション履歴］セクションに追加するアクションを選択します。

    4. ［メモ］オプションを選択すると、追加メモを入力するための新しいブラウザウィンドウが開きます。メモは［アクション履歴］セクションに表示されます。

    VisualforceページレイアウトUI では、このメモは［履歴］タブに表示されます。システム管理者は、ある特定のアクションが［履歴］タブに追加されたときにクライアントに電子メールを送信するワークフローを作成できます。

    注意: このアクションを追加した後は、［Client Note（クライアントメモ）］アクションを保存できません。［Client Note（クライアントメモ）］アクションは、セルフサービスクライアントが排他的に使用するものです。この制限は、セルフサービスクライアントが［Client Note（クライアントメモ）］アクションを編集できるように追加されました。

問題に関連付けられたアクションの表示

［アクション履歴の表示］オプションを使用して、問題に関連付けられたアクションを表示できます。［アクション履歴の表示］オプションでは、新規アクションの作成、タイプに基づくアクションのフィルタリング、およびアクションの印刷を行えます。

注意: ［問題の履歴］ウィンドウに、表示専用モードでアクションが表示されます。［問題の履歴］ウィンドウに表示されるアクションの編集や削除はできません。

問題に関連付けられたアクションを表示するには

  1. アクションを表示する問題に移動します。

    2. 詳細は、問題レコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［アクション］メニューをクリックし、［アクション履歴の表示］を選択します。

      • または、［関連リスト］をクリックし、［アクション履歴］セクションで、［アクション履歴の表示］をクリックします。

    • ［問題］タブ（Salesforceページレイアウト）を使用している場合は、［アクション履歴］セクションの［アクション履歴の表示］をクリックします。

      • ［問題の履歴］ウィンドウに、表示専用モードでアクションが表示されます。［問題の履歴］ウィンドウに表示されるアクションの編集や削除はできません。

  3. （オプション）アクションタイプに基づいてレコードをフィルタリングするには、リストから目的のアクションを選択します。
  4. （オプション）問題に関連付けられたアクションを印刷するには、［印刷］をクリックします。
