Alanはセルフサービスを使用しているクライアントです。BMC FootPrints Asset Coreのデバイスである新しいラップトップにソフトウェアをインストールするようにリクエストしています。Davidは、セルフサービスで送信されたソフトウェアインストールのサービスリクエストを担当しているスタッフメンバーです。次の手順は、Alanのサービスリクエストを実行して、Microsoft OfficeおよびMicrosoft Visioをセルフサービスでインストールする場合に発生する作業の流れを示します。


手順1
DavidはBMC FootPrints Asset Coreで、Microsoft Officeをインストールする運用規則と、Microsoft Visioをインストールする運用規則を作成しました。


手順2
Microsoft Officeをインストールする運用規則では、［外部統合からの導入］プロパティの値を［統合ソリューションで使用可能］に設定しました。この運用規則をBMC Remedyforceで使用すると、この運用規則を導入するタスクテンプレートを作成できます。


手順3
Microsoft Visioをインストールする運用規則では、［外部統合からの導入］プロパティの値を［追加ワークフローが必要］に設定して、この運用規則が承認された後でないと運用規則が導入されないようにしました。この運用規則をBMC Remedyforceで使用すると、この運用規則を導入するタスクテンプレートを作成できます。


手順4
次に、Davidは運用規則を導入するための承認プロセスをBMC Remedyforceで作成しました。詳細は、運用規則を導入するための承認プロセスの作成を参照してください。


手順5
さらにDavidはBMC Remedyforceで、Microsoft OfficeやMicrosoft Visioに関するソフトウェアインストールリクエストなど、よく発生するサービスリクエストのサービスリクエスト定義を作成しました。［Microsoft Officeのインストール］サービスリクエストの定義で、必要なインシデントテンプレートが格納されたリクエストの詳細テンプレートを選択しました。また、インシデントテンプレートで、Microsoft Officeをインストールするための運用規則を導入するタスクテンプレートをリンクしました。


手順6
［Microsoft Visioのインストール］サービスリクエストの定義で、必要なインシデントテンプレートが格納されたリクエストの詳細テンプレートを選択しました。また、インシデントテンプレートで、Microsoft Visioをインストールするための運用規則を導入するタスクテンプレートをリンクしました。


手順7
Alanはセルフサービスにログオンし、［Microsoft Officeのインストール］サービスリクエストをセルフサービスで使用して、Microsoft Officeをインストールするように求めるサービスリクエストを送信しました。


手順8
Davidがこのサービスリクエストの担当者に割り当てられました。DavidはBMC Remedyforceでサービスリクエストを開き、AlanのBMC FootPrints Asset Coreデバイスをサービスリクエストにリンクしました。そのための手順として、［CIのリンク］をクリックし、［Remedyforce CMDB］タブ（またはVisualforceページレイアウトユーザーインターフェイスの［ワークスペース］>［構成管理］>［構成アイテム］ページ）にこのデバイスが表示されている場合は、ダブルクリックしました。必要なデバイスが［Remedyforce CMDB］タブに表示されていない場合、Davidは［アセットコア］タブをクリックし、デバイスを検索し、必要なデバイスをダブルクリックしてサービスリクエストにリンクします。詳細は、デバイスの検索、インポート、インシデントやサービスリクエストへのリンクを参照してください。


手順9
BMC Remedyforceによって運用規則に対応するタスクが作成され、現在のサービスリクエストにリンクされます。運用規則の導入の進行状況は、タスクのステータスで追跡できます。運用規則の導入に成功するか、または失敗した場合は、BMC Remedyforceによってタスクレコードおよびリンクしたデバイスの［アクション履歴］タブに導入の詳細が記録されます。運用規則の導入に関するリクエストが正常に完了した場合、運用規則のステータスは［リクエストが成功しました］に更新され、タスクのステータスは［リクエストが成功しました］に割り当てられたステータス（［終了済み］など）に更新されます。タスクのステータスが、［リクエストが成功しました］に割り当てられたステータスに更新されている場合は（［終了済み］など）、AlanのデバイスにMicrosoft Officeがインストールされています。


手順10
次にAlanはセルフサービスにログオンし、［Microsoft Visioのインストール］サービスリクエストをセルフサービスで使用して、Microsoft Visioをインストールするように求めるサービスリクエストを送信しました。


手順11
Davidがこのサービスリクエストの担当者に割り当てられました。DavidはBMC Remedyforceでサービスリクエストを開き、AlanのBMC FootPrints Asset Coreデバイスをサービスリクエストにリンクしました。そのための手順として、［CIのリンク］をクリックし、［Remedyforce CMDB］タブ（またはVisualforceページレイアウトユーザーインターフェイスの［ワークスペース］>［構成管理］>［構成アイテム］ページ）にこのデバイスが表示されている場合は、ダブルクリックしました。必要なデバイスが［Remedyforce CMDB］タブに表示されていない場合、Davidは［アセットコア］タブをクリックし、デバイスを検索し、必要なデバイスをダブルクリックしてサービスリクエストにリンクします。詳細は、デバイスの検索、インポート、インシデントやサービスリクエストへのリンクを参照してください。


手順12
BMC Remedyforceによって運用規則に対応するタスクが作成され、現在のサービスリクエストにリンクされます。


手順13
Microsoft Visioをインストールするための運用規則を導入する前に、承認を受ける必要があります。承認者が承認するか、承認を却下するまで、現在のサービスリクエストはロックされたままになります。また、現在のサービスリクエストの［アセットコアの承認ステータス］フィールドが［承認待ち］に更新されます。承認されると、［アセットコアの承認ステータス］フィールドは［承認済み］に更新されます。


手順14
承認されると、サービスリクエストはロックされ、運用規則を導入するためのタスクが実行されます。


手順15
運用規則の導入の進行状況は、タスクのステータスで追跡できます。運用規則の導入に成功するか、または失敗した場合は、BMC Remedyforceによってタスクレコードおよびリンクしたデバイスの［アクション履歴］タブに導入の詳細が記録されます。運用規則の導入に関するリクエストが正常に完了した場合、運用規則のステータスは［リクエストが成功しました］に更新され、タスクのステータスは［リクエストが成功しました］に割り当てられたステータス（［終了済み］など）に更新されます。タスクのステータスが、［リクエストが成功しました］に割り当てられたステータスに変更されている場合は（［終了済み］など）、Microsoft VisioがAlanのデバイスにインストールされています。
