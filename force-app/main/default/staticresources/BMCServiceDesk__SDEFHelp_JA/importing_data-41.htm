BMC FootPrints Asset Coreからデータをインポートする前に、BMC FootPrints Asset Coreからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

Pentahoを使用してBMC FootPrints Asset Coreからデータをインポートするには

  1. ジョブファイルをBMC CommunitiesのWebサイトからダウンロードするには、次のいずれかのアクションを実行します。
    • CMDB 1.0を使用している場合は、 https://communities.bmc.com/docs/DOC-21630をクリックします。
    • CMDB 2.0を使用している場合は、 https://communities.bmc.com/docs/DOC-28436をクリックします。

      • 注意: CMDB 2.0にアップグレードしていて、以前はCMDB 1.0パッケージを使用していた場合、引き続きCMDB 1.0パッケージを使用できます。ただし、BMCは、迅速なインポートのために、CMDB 2.0用のパッケージを使用することをお勧めします。

      ジョブファイルは、Pentaho Data Integrationツールを解凍したフォルダにダウンロードすることをお勧めします。

      注意: フォルダ構造は変更しないでください。CombinedJobs（統合ジョブ）フォルダ内のバッチファイルの実行が中断される場合があります。

  2. Pentaho Data Integrationツールを起動するには、\pdi-ce-4.x.0-stable\data-integrationまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダでSpoon.batファイルをダブルクリックして目的のKJBファイルを開きます。

    3. 注意: 統合ジョブを実行する場合は、必要なBMC FootPrints Asset CoreデータベースおよびSalesforce組織の資格情報で各ジョブのKTRファイルを更新します。

  3. KTRファイルを開くには、使用しているCMDBおよびPentahoのバージョンに応じて、次のいずれかのアクションを実行します。

    4. CMDBバージョン

  4. BMC FootPrints Asset Coreデータベースにアクセスするための詳細情報を設定するには、次のアクションを実行します。
    1. ［Explorer］ペインでデータベース接続を展開し、［BMC Remedyforce FootPrints Asset Core Source］をクリックします。
    2. ［データベース接続］ウィンドウで、ホスト名、BMC FootPrints Asset Coreデータベース名、ユーザー名、パスワードを入力します。

      3. ホスト名は、BMC FootPrints Asset Coreデータベースがインストールされているコンピュータの名前です。

      BMC FootPrints Asset CoreデータベースにWindows認証を設定している場合、次の条件を満たすときにはユーザー名およびパスワードは必要ありません。

      • ログオンユーザーは、BMC FootPrints Asset CoreデータベースサーバーにインストールされているPentaho Data Integrationツールからジョブを実行する必要があります。
      • ログオンユーザーには、BMC FootPrints Asset Coreデータベースへのアクセス権が必要です。
    3. （オプション）接続を確認するには、［テスト］をクリックします。
    4. ［OK］をクリックします。
  5. Salesforce組織のアクセス詳細情報を設定するには、次のアクションを実行します。
    1. ［クラス<CIタイプ名>にアップサート］手順をダブルクリックします。
    2. ［Salesforceのアップサート］ウィンドウで、Salesforce組織のユーザー名とパスワードを入力します。
    3. （オプション）マッピングを更新するには、［マッピングの編集］をクリックします。
    4. （オプション）BMC FootPrints Asset CoreフィールドとBMC Remedyforce CMDBと間のマッピングは事前に定義されています。マッピングについての詳細は、CMDB 1.0で構成アイテムをインポートする方法の概要を参照してください。

      5. 注意: Pentaho 4.1や4.2などの以前のバージョンのPentaho Data Integrationツールを使用している場合は、Salesforce WebサービスのURLをhttps://www.salesforce.com/services/Soap/u/20.0に変更します。

    5. （オプション）接続を確認するには、［接続テスト］をクリックします。
    6. ［OK］をクリックします。
  6. （CMDB 2.0の場合のみ）［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］手順でSalesforceアクセス詳細情報を入力する必要があります。Salesforce組織へのアクセスに必要な情報を入力するには、次のアクションを実行します。
    1. ［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］をダブルクリックします。
    2. ［接続］セクションで、Salesforce組織のアクセス詳細を入力します。
    3. （オプション）接続をテストするには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。
  7. （オプション）Salesforce組織にアクセスできるIPアドレスを限定するように設定されている場合、SalesforceとPentaho Data Integrationツールの間での通信を有効にするには、次のアクションを実行します。
    1. Salesforceで、［設定］>［自分のセキュリティトークンのリセット］の順に選択します。

      2. 注意: Salesforce組織で、改善された［設定］ユーザーインターフェイスが有効になっている場合は、［My Settings（私の設定）］>［Personal（個人）］>［自分のセキュリティトークンのリセット］に移動します。改善された［設定］ユーザーインターフェイスの詳細については、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。

    2. ［セキュリティトークンのリセット］をクリックすると、Salesforceに保存されている電子メールアドレスに電子メールが送信されます。
    3. ［クラス<CIタイプ名>にアップサート］手順の［パスワード］フィールドで、パスワードの後ろにセキュリティトークンを追加します。

      4. たとえば、パスワードがmypasswordで、セキュリティトークンがXXXXXXXXXであれば、［パスワード］フィールドに「mypasswordXXXXXXXXX」と入力する必要があります。

  8. KTRファイルとKJBファイルを保存するには、をクリックします。
  9. ［KJBファイル］タブで、をクリックします。
  10. ［ジョブの実行］ウィンドウで、［起動］をクリックします。

    11. データが適切なCMDBクラスにインポートされます。

    完了した変換は、で示されます。は現在実行中の変換を示し、は失敗した変換を示します。

  11. （オプション）ログを表示するには、［実行の結果］セクションで［ログ］タブをクリックします。

    12. すべてのエラーは赤で表示されます。

    コンピュータシステム（ソース）と製品（宛先）との関係は、構成アイテムエクスプローラには表示されません。これは、（プラットフォームの制限により）構成アイテムエクスプローラは一部の関係しか表示できないためです。コンピュータシステムと製品間の関係を表示すると、コンピュータシステムと他のCI間の重要な関係が非表示になる場合があります。

  12. （オプション）ジョブのインポートをスケジュールするには、インポートしたジョブのスケジュール作成を参照してください。

コンピュータシステムと各種CIとの間に関係を作成する場合は、BMCServiceDesk__AssemblyId__cフィールドが必要です。ソースはコンピュータシステムであり、宛先にはオペレーティングシステム、プロセッサ、LANエンドポイント、製品などがあります。

たとえば、コンピュータシステムレコードに対して［BMCServiceDesk__AssemblyId__c］フィールドの値が1で、プロセッサレコードのアセンブリID値が1の場合、コンピュータシステム（ソース）とプロセッサ（ターゲット）の間に関係が自動的に作成されます。
