変更リクエストの保存後に、［アクション履歴］セクションで、ユーザー定義アクションおよびシステム定義アクションを変更リクエストに追加できます。

変更リクエストにアクション履歴レコードを追加するには

  1. アクションを追加する変更リクエストに移動します。

    2. 詳細は、変更リクエストレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［アクション］メニューをクリックし、［新規アクション］を選択します。

      • または、［関連リスト］をクリックし、［アクション履歴］セクションで、［新規アクション］をクリックします。

    • ［変更リクエスト］タブ（Salesforceページレイアウト）を使用している場合は、［アクション履歴］セクションの［新規アクション］をクリックします。
  3. ［アクション履歴］セクションで［新規アクション］をクリックします。
  4. ［アクション履歴］セクションに追加するアクションを選択します。

    5. ［メモ］オプションを選択すると、追加メモを入力するための新しいブラウザウィンドウが開きます。メモは［アクション履歴］セクションに表示されます。

  5. VisualforceページレイアウトUI を使用している場合は、次のアクションを実行します。
    1. ［補足情報］セクションで、［履歴］タブをクリックします。
    2. メニューから、［追加］を選択します。

      3. VisualforceページレイアウトUI では、このメモは［履歴］タブに表示されます。ある特定のアクションが［履歴］タブに追加されたときにクライアントに電子メールメッセージを送信するワークフローを作成できます。

      注意: このアクションを追加した後は、［Client Note（クライアントメモ）］アクションを保存できません。［Client Note（クライアントメモ）］アクションは、セルフサービスクライアントが排他的に使用するものです。この制限は、セルフサービスクライアントが［Client Note（クライアントメモ）］アクションを編集できるように追加されました。

  6. 必要な詳細を追加して、［保存］をクリックします。

変更リクエストに関連付けられたアクションの表示

［アクション履歴の表示］オプションを使用して、変更リクエストに関連付けられたアクションを表示できます。［アクション履歴の表示］オプションでは、新規アクションの作成、タイプに基づくアクションのフィルタリング、およびアクションの印刷を行えます。

変更リクエストに関連付けられたアクションを表示するには

  1. アクションを表示する変更リクエストに移動します。

    2. 詳細は、変更リクエストレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［アクション］メニューをクリックし、［アクション履歴の表示］を選択します。

      • または、［関連リスト］をクリックし、［アクション履歴］セクションで、［アクション履歴の表示］をクリックします。

    • ［変更リクエスト］タブ（Salesforceページレイアウト）を使用している場合は、［アクション履歴］セクションの［アクション履歴の表示］をクリックします。

      • ［変更リクエストの履歴］ウィンドウには表示専用モードでアクションが表示されます。［変更リクエストの履歴］ウィンドウに表示されたアクションを編集したり、削除したりすることはできません。

  3. （オプション）タイプに基づいてアクションをフィルタリングするには、リストから目的のアクションを選択します。
  4. （オプション）変更リクエストに関連付けられたアクションを印刷するには、［印刷］をクリックします。
