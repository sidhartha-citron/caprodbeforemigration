BMC Atrium CMDBから構成アイテムをインポートする前に、BBMC Atrium CMDBからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

Pentahoパッケージを使用してBMC Atrium CMDBから構成アイテムをインポートするには

  1. BMC CommunitiesページからPentahoパッケージをダウンロードします。 https://communities.bmc.com/docs/DOC-28437をクリックして、ファイルをコンピュータに保存します。
  2. BMC Remedy AR Systemにログオンして、Atrium Integrator Consoleにアクセスします。
  3. Atrium Integrator Spoonを起動して、をクリックします。
  4. Atrium Integratorで新しいフォルダを作成してKTRファイルおよびKJBファイルを保存するには、次のアクションを実行します。
    1. ［ツール］>［Repository（リポジトリ）］>［Explore（探索）］の順に選択します。
    2. ウィンドウを右クリックして、［New Folder（新規フォルダ）］を選択します。
    3. BMC CommunitiesのWebサイトからダウンロードしたKTRファイルとKJBファイルを保存するフォルダの名前を入力し、［OK］をクリックします。

      4. たとえば、Remedyforceという名前のフォルダを作成します。

  5. KTRファイルとKJBファイルをこのフォルダに保存するには、各ファイルに対して次の手順を実行します。
    1. ［ファイル］>［Open URL（URLを開く）］をクリックして、BMC Communitiesからダウンロードしたジョブファイルを保存したフォルダを参照します。
    2. 目的のCIタイプ（コンピュータシステムなど）のKJBファイルとKTRファイルを開きます。
    3. ［ファイル］>［名前を付けて保存］をクリックします。
    4. ［Directory（ディレクトリ）］フィールドで、をクリックし、作成したフォルダを選択します。
    5. ［OK］をクリックします。
  6. KJBファイルで、［AtriumtoCMDB_<CIタイプ>］手順をダブルクリックします。

    7. たとえば、［AtriumtoCMDB_ComputerSystem］手順をダブルクリックします。

  7. 現在の変換ウィンドウのジョブエントリ詳細で、［Specify by name and directory（名前とディレクトリを指定）］オプションのをクリックし、作成したフォルダから開いているKJBファイルに対するKTRを選択します。

    8. たとえば、［Remedyforce］>［AtriumToCMDB_ComputerSystem.ktr］を選択します。

  8. ［ファイル］>［開く］をクリックし、目的のKTRファイルを開きます。
  9. KTRファイルからAR Systemにアクセスするために必要な情報を入力するには、次の手順を実行します。
    1. KTRファイルで、［ARInput（AR入力）］をダブルクリックします。
    2. ［接続］フィールドで［編集］をクリックします。
    3. ［設定］セクションで、AR Systemサーバーのアクセス詳細情報を入力します。
    4. （オプション）サーバーとの接続をテストするには、［テスト］をクリックします。
    5. ［OK］をクリックします。
  10. BMC Atrium CMDBが使用するSQL Serverへのアクセスに必要な情報を入力するには、次の手順を実行します。
    1. ［Table Source（テーブルソース）］をダブルクリックします。
    2. ［接続］フィールドで［編集］をクリックします。
    3. 別の種類のデータベースを使用している場合は、［データベース接続］ウィンドウの［接続の種類］リストで、目的の接続の種類を選択します。

      4. 使用しているデータベースの種類に応じて、［Table Source（テーブルソース）］手順で作成されるクエリが異なる場合があります。この手順で、Oracleデータベースクエリーがコメント付きフォームで入力されることもあります。Oracleクエリーを使用するには、クエリーからコメントマークを削除し、MS SQLクエリーにコメントを入力します。

    4. ［設定］セクションで、BMC Atrium CMDBデータベースが使用するSQL Serverのログオン情報を入力します。
    5. （オプション）データベースとの接続をテストするには、［テスト］をクリックします。
    6. ［OK］をクリックします。
  11. Salesforce組織へのアクセスに必要な情報を入力するには、次の手順を実行します。
    1. ［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］をダブルクリックします。
    2. ［接続］セクションで、Salesforce組織のアクセス詳細を入力します。
    3. （オプション）接続をテストするには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。
    5. ［Salesforceのアップサート］をダブルクリックします。
    6. ［接続］セクションで、同じSalesforce組織のアクセス詳細を入力します。
    7. （オプション）接続をテストするには、［接続テスト］をクリックします。
    8. ［OK］をクリックします。
  12. KTRファイルとKJBファイルを保存するには、をクリックします。
  13. Atrium Integrator Consoleに戻ります。
  14. 設定したジョブを選択して、をクリックします。

データのインポート時に、次のログファイルが生成されます。

    • ジョブレベルのログ： Atrium Integrator Consoleで作成されます。
    • 成功ログ、処理ログ、エラーログ： BBMC Atrium CMDBからデータをインポートする前にで作成したLog FilesフォルダにCSVファイルとして作成されます。失敗ログは、BMC Remedyforceでのレコードの挿入時にエラーが発生した場合にのみ作成されます。delta_valueログには、データを最後にインポートした日時のタイムスタンプが保存されます。固有識別子値が空白のレコードが存在する場合、拒否ログも作成されます。
