BMC FootPrints Asset Coreからデータをインポートする前に、Dell KACEからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

Pentahoを使用してDell KACEからデータをインポートするには

  1. ジョブファイルをBMC CommunitiesのWebサイトからダウンロードするには、次のいずれかのアクションを実行します。
    • CMDB 1.0を使用している場合は、 https://communities.bmc.com/docs/DOC-21752をクリックします。
    • CMDB 2.0を使用している場合は、 https://communities.bmc.com/docs/DOC-28433をクリックします。

      • ジョブファイルがすでにダウンロードされている場合は、この手順を省略します。

      注意: CMDB 2.0にアップグレードしていて、以前はCMDB 1.0パッケージを使用していた場合、引き続きCMDB 1.0パッケージを使用できます。ただし、BMCは、迅速なインポートのために、CMDB 2.0用のパッケージを使用することをお勧めします。

      ジョブファイルは、Pentaho Data Integrationツールを解凍したフォルダにダウンロードすることをお勧めします。ジョブファイルについての詳細は、Dell KACEでのKJBファイルとKTRファイルの概要を参照してください。

      注意: フォルダ構造は変更しないでください。CombinedJobs（統合ジョブ）フォルダ内のバッチファイルの実行が中断される場合があります。

  2. Pentaho Data Integrationツールを起動するには、\pdi-ce-4.x.0-stable\data-integrationまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダでSpoon.batファイルをダブルクリックします。
  3. Pentahoツールのメニューバーで、［ファイル］>［開く］の順にクリックします。
  4. 目的のKJBファイルを選択して、［開く］をクリックします。

    5. たとえば、\ComputerSystem\TransferDELLKACEComputerSystemInfotoCMDB.kjbファイルを開きます。

  5. KTRファイルを開くには、使用しているCMDBおよびPentahoのバージョンに応じて、次のいずれかのアクションを実行します。

    6. CMDBバージョン                   
    Pentahoバージョン
    アクション
    1.0
    4.4
    ［Update staging table with Dell KACE <CI type name> information（ステージリストをDell KACE<CIタイプ名>で更新）］手順を右クリックし、［データ変換を開く］を選択します。
    1.0
    5.0.1
    ［Update staging table with Dell KACE <CI type name> information（ステージリストをDell KACE<CIタイプ名>で更新）］手順を右クリックし、［Open referenced object（参照オブジェクトを開く）］を選択してから、［Transformation（データ変換）］を選択します。
    2.0
    4.4
    ［Update BE table with Dell KACE <CI type name> Information（BEテーブルをDell KACE<CIタイプ名>情報で更新）］手順を右クリックし、［データ変換を開く］を選択します。
    2.0
    5.0.1
    ［Update BE table with Dell KACE <CI type name> Information（BEテーブルをDell KACE<CIタイプ名>情報で更新）］手順を右クリックし、［Open referenced object（参照オブジェクトを開く）］を選択してから、［Transformation（データ変換）］を選択します。

  6. Dell KACEデータベースにアクセスするための詳細情報を入力するには、次のアクションを実行します。
    1. ［KTRファイル］タブで、［Dell KACE入力］手順をダブルクリックします。
    2. ［接続］リストの横にある［編集］をクリックします。
    3. ［データベース接続］ウィンドウで、ホスト名、データベース名、ユーザー名、およびパスワードを入力します。

      4. ホスト名は、Dell KACEデータベースがインストールされているコンピュータの名前です。接続の種類としてMYSQLデータベースが選択され、アクセス値としてネイティブ（JDBC）が選択されていることを確認します。

    4. （オプション）接続を確認するには、［テスト］をクリックします。
    5. ［OK］をクリックします。
  7. Salesforce組織にアクセスするための詳細情報を入力するには、次のアクションを実行します。
    1. ［クラス<CIタイプ名>にアップサート］手順をダブルクリックします。
    2. ［Salesforceのアップサート］ウィンドウで、Salesforce組織のユーザー名とパスワードを入力します。

      3. 注意: Pentaho 4.1や4.2などの以前のバージョンのPentaho Data Integrationツールを使用している場合は、Salesforce WebサービスのURLを https://www.salesforce.com/services/Soap/u/20.0に変更します。

      サンドボックス環境でパッケージを実行する場合は、Salesforce WebサービスのURLをhttps://www.salesforce.com/services/Soap/u/20.0 から https://test.salesforce.com/services/Soap/u/20.0に変更します。

    3. （オプション）マッピングを更新するには、［マッピングの編集］をクリックします。Dell KACEフィールドとSalesforce CMDBステージオブジェクト間のマッピングは事前に定義されています。

      4. マッピングについての詳細は、CMDB 1.0で構成アイテムをインポートする方法の概要を参照してください。

    4. （オプション）接続を確認するには、［接続テスト］をクリックします。
    5. ［OK］をクリックします。
  8. （CMDB 2.0の場合のみ）［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］手順でSalesforceアクセス詳細情報を入力する必要があります。Salesforce組織へのアクセスに必要な情報を入力するには、次のアクションを実行します。
    1. ［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］をダブルクリックします。
    2. ［接続］セクションで、Salesforce組織のアクセス詳細を入力します。
    3. （オプション）接続をテストするには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。
  9. （オプション）Salesforce組織にアクセスできるIPアドレスを限定するように設定されている場合、SalesforceとPentaho Data Integrationツールの間での通信を有効にするには、次のアクションを実行します。
    1. Salesforceで、［設定］>［自分のセキュリティトークンのリセット］の順に選択します。

      2. 注意: Salesforce組織で、改善された［設定］ユーザーインターフェイスが有効になっている場合は、［My Settings（私の設定）］>［Personal（個人）］>［自分のセキュリティトークンのリセット］に移動します。改善された［設定］ユーザーインターフェイスの詳細については、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。

    2. ［セキュリティトークンのリセット］をクリックすると、Salesforceに保存されている電子メールアドレスに電子メールが送信されます。
    3. ［クラス<CIタイプ名>にアップサート］手順の［パスワード］フィールドで、パスワードの後ろにセキュリティトークンを追加します。

      4. たとえば、パスワードがmypasswordで、セキュリティトークンがXXXXXXXXXであれば、［パスワード］フィールドに「mypasswordXXXXXXXXX」と入力する必要があります。

  10. KTRファイルとKJBファイルを保存するには、をクリックします。
  11. ［KJBファイル］タブで、をクリックします。
  12. ［ジョブの実行］ウィンドウで、［起動］をクリックします。

    13. データが適切なCMDBクラスにインポートされます。

    完了した変換は、で示されます。は現在実行中の変換を示し、は失敗した変換を示します。

  13. 適切なCMDBクラスの関係エディタで、エントリが正しいことを確認します。
  14. （オプション）ログを表示するには、［実行の結果］セクションで［ログ］タブをクリックします。

    15. すべてのエラーは赤で表示されます。

    注意: コンピュータシステム（ソース）とソフトウェアサーバー（宛先）との関係は、構成アイテムエクスプローラには表示されません。これは、（プラットフォームの制限により）構成アイテムエクスプローラは一部の関係しか表示できないためです。コンピュータシステムとソフトウェアサーバー間の関係を表示すると、コンピュータシステムと他のCI間の重要な関係が非表示になる場合があります。

  15. （オプション）ジョブのインポートをスケジュールするには、インポートしたジョブのスケジュール作成を参照してください。

    16. 警告: 2台の異なるコンピュータを使用している場合、同じユーザーに対してCMDBの個人パッケージから2つのレコードがインポートされます。

Dell KACEコンソールを起動して、Dell KACEからインポートされたレコードの詳細を表示できます。

コンピュータシステムと各種CIとの間に関係を作成する場合は、BMCServiceDesk__AssemblyId__cフィールドが必要です。ソースはコンピュータシステムであり、宛先にはオペレーティングシステム、プロセッサ、LANエンドポイント、ソフトウェアサーバー、スタッフなどがあります。

たとえば、コンピュータシステムレコードに対して［BMCServiceDesk__AssemblyId__c］フィールドの値が1で、プロセッサレコードのアセンブリID値が1の場合、コンピュータシステム（ソース）とプロセッサ（ターゲット）の間に関係が自動的に作成されます。
