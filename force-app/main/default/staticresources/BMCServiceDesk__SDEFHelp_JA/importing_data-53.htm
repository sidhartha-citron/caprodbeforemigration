KTRファイルには1つの変換が含まれています。BMC Atrium CMDBからデータをインポートするために提供されるKTRファイルには、次の手順が格納されています。

手順                   
説明
ARInput（AR入力）
このパッケージのタイムスタンプを取得し、BMC Remedy AR Systemサーバーの［NGIE:Delta（NGIE:デルタ）］フォームに保存します。
Table Source（テーブルソース）
BMC Remedy AR Systemが使用するMicrosoft SQL Serverから、保存されているタイムスタンプで示された日時より後に変更されたCIに関する情報が取得されます。初めてデータをインポートする場合は、すべてのCIがインポートされます。
Read Response Variable（応答変数の読み取り）
フィールド値の追加や削除など、JavaScriptの変換が容易になります。たとえば、BMC Remedy AR Systemでは［削除済みとしてマーク］フィールドの値はブーリアン値（0または1）として保存されますが、BMC RemedyforceではTrueまたはFalseとして保存されます。この手順では、0と1はそれぞれTrueとFalseに変換されます。
Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）
Salesforce組織のアクセス詳細情報を受け入れ、BMC Remedyforce CMDBクラスの詳細情報を取得します。
Get CMDB Class ID（CMDBクラスIDの取得）
ソースレコードのBMC Remedyforce CMDBクラスのクラスIDを取得します。たとえば、ソースがコンピューターシステムレコードの場合、BMC Remedyforce CMDBからBMC_ComputerSystemクラスのクラスIDが取得されます。
Unique Identify Filter（固有識別子フィルタ）
固有識別子フィールドが空白であるレコードがすべて削除され、拒否されたレコードで定義されたCSVファイルにレコードが参照ログとして保存されます。
Salesforceのアップサート
インポートされたデータの保存先となるSalesforce組織のアクセス詳細情報を受け入れます。この手順では、BMC RemedyforceとBMC Atrium CMDBフィールドの間のフィールドマッピングも表示されます。
Success logging（成功のログ）
正常にインポートされた行が保存されます。
Error logging（エラーのログ）
エラーのコード、エラーの説明、エラーが発生したフィールド、正常にインポートされなかった行のフィールド値が格納されます。
Identify last row in a stream（ストリーム内の最終行の識別）
正常にインポートされたレコードから、（変更日時順で）最後のレコードを取得します。
AROutput（AR出力）
［Identify last row in a stream（ストリーム内の最終行の識別）］手順で取得したレコードの変更日時を、実行中のKTRに対応する［NGIE:Delta（NGIE:デルタ）］フォームのレコードに保存します。

KJBファイル（ジョブファイル）には一連の変換処理が格納されています。BMC Atrium CMDBからデータをインポートするためのKJBファイルには、次の手順が格納されています。

    • 起動
    • AtriumToCMDB_<CIタイプ>： この手順には、データをBMC Atrium CMDBからデータにインポートするための変換が含まれています。
