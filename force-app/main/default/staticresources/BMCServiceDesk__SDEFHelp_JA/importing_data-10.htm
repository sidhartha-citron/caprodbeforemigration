Pentaho Data IntegrationツールとWindows認証済みMicrosoft SQL Server（SCCMまたはBMC BladeLogic Client Automation）との間の通信を可能にするには、両方を接続する必要があります。

Windows認証済みMS SQL Serverに接続するには

  1. Microsoft SQL Server Management Studioにログオンします。
  2. Securityフォルダの横にあるプラス記号（+）をクリックします。
  3. Securityフォルダ内のLoginsフォルダを右クリックし、［新規ログイン］を選択します。
  4. ［ログイン： 新規］ウィザードの［ログイン名］テキストボックスで、Windowsのユーザー名を入力します。

    5. 形式は、<ドメイン名>\<ユーザー名>です。Pentaho Data Integrationツールを実行しているコンピュータのドメインユーザーのログオンIDを入力します。

  5. ［サーバーの役割］オプションをクリックして、［パブリック］チェックボックスがオンになっていることを確認します。
  6. ［ユーザーマッピング］オプションをクリックして、SCCMまたはBMC BladeLogic Client Automationデータベース名がある行を選択し、［マップ］チェックボックスをオンにします。

    7. <データベース名> セクションのデータベースの役割とメンバーシップが表示されます。

  7. ［db_datareader］チェックボックスをオンにして、［パブリック］チェックボックスがオンになっていることを確認します。
  8. ［OK］をクリックします。

    9. 注意: デフォルトでは、SCCM 2007はWindows認証済みモードでインストールされます。このセクションで説明されている手順を設定して、Pentaho Data IntegrationツールとSCCM 2007との間の通信を可能にする必要があります。
