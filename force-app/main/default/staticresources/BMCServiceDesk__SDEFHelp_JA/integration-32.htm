BMC Remedyforceでは、電子メールサービスを使用して、ユーザーがインシデントを作成または更新できます。電子メールを介してインシデントを作成または更新するには、電子メールサービスを作成し、GenericEmailListener Apexクラスを電子メールサービスに追加する必要があります。

BMC Server AutomationおよびBMC End User Experience ManagementからBMC Remedyforceでインシデントを作成するには、定義済みのEmailCreateIncident Apexクラスを電子メールサービスに追加する必要があります。

BMC Server AutomationからBMC Remedyforceで変更リクエストを作成するには、定義済みのChangeRequestEmailListener Apexクラスを電子メールサービスに追加する必要があります。

Apexクラスを追加したら、電子メールサービス用の電子メールアドレスを設定する必要があります。Apexクラスは、設定した電子メールアドレスに送信される電子メールを処理します。

電子メールサービスについての詳細は、 https://na7.salesforce.com/help/doc/user_ed.jsp?loc=help&target=code_email_services.htm&section=integrateを参照してください。

Salesforce組織の電子メール制限についての詳細は、 http://www.salesforce.com/us/developer/docs/apexcode/Content/apex_gov_limits.htmを参照してください。

電子メールサービスを作成するには

  1. ［Remedyforce管理］タブをクリックします。
  2. ［ホーム］ページで、［電子メールの設定］タイルをクリックし、メニューから［電子メールサービス］を選択します。
  3. Visualforceページレイアウトユーザーインターフェイスを使用している場合は、［設定］>［開発］>［電子メールサービス］の順に選択します。
  4. ［新規電子メールサービス］をクリックします。
  5. ［電子メールサービス名］フィールドに、電子メールサービスの名前を入力します。

    6. 注意: BMC Remedyforceの電子メールサービスを、ユーザーが作成する他の電子メールサービスと区別するため、電子メールサービス名の最初に「BMCSD」を付けることをお勧めします。

  6. ［Apexクラス］ルックアップをクリックします。
  7. ［ルックアップ］ウィンドウで、必要なApexクラスをクリックします。

    8. BMC ProactiveNet Performance ManagementとBMC Remedyforceを統合している場合は、GenericEmailListenerを選択します。

  8. ［添付ファイルを許可］ドロップダウンリストから、［すべて］を選択します。
  9. 添付ファイルが失われたり、電子メールから削除されたりしないように、［テキスト添付ファイルをバイナリ添付ファイルに変換］チェックボックスをオンにします。
  10. 電子メールサービスを有効にするには、［アクティブ］チェックボックスをオンにします。
  11. 組織の要件に応じて、その他のフィールドを設定します。

    12. 詳細は、 https://na11.salesforce.com/help/doc/user_ed.jsp?section=help&target=code_email_services_editing.htm&loc=help&hash=topic-titleを参照してください。

  12. 電子メールサービスを保存し、この電子メールサービスの電子メールアドレスを設定するには、［保存および新規電子メールアドレス］をクリックします。
  13. ［電子メールサービスのアドレス］ページの［電子メールアドレス情報］セクションで、電子メールアドレスのユーザー名を［電子メールアドレス］フィールドに入力します。

    14. Salesforce.comによって固有のドメイン名が作成され、電子メールアドレスに割り当てられます。

    詳細は、 https://na11.salesforce.com/help/doc/user_ed.jsp?section=help&target=code_email_service_address.htm&loc=help&hash=topic-titleを参照してください。

  14. 電子メールアドレスを有効にするには、［アクティブ］チェックボックスをオンにします。
  15. ［コンテキストユーザー］ルックアップをクリックします。
  16. ［ルックアップ］ウィンドウで、必要なユーザーをクリックします。

    17. 電子メールを介してインシデントが作成されると、そのインシデントは電子メールサービスのコンテキストユーザーとして設定されているユーザーにデフォルトで割り当てられます。このユーザーには、インシデントが作成されても通知が送信されません。インシデントがコンテキストユーザーに割り当てられた場合に、コンテキストユーザーに通知を送信するワークフローを作成する必要があります。

  17. （オプション）［次の送信者の電子メールを許可］フィールドに、有効な電子メールアドレスを入力します。

    18. 電子メールサービスは、これらの電子メールアドレスから送信された電子メールだけを受け入れます。

  18. ［保存］をクリックします。

    19. インシデントを作成または更新するには、［電子メールサービス］ページの［電子メールアドレス］セクションに表示される電子メールアドレスに電子メールが届く必要があります。
