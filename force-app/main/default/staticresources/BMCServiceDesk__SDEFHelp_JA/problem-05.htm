レコードへの移動は、使用しているUI によって異なります。使用するUI を決定するには、さまざまなユーザーインターフェイスのさまざまな機能へのアクセスを参照してください。ここでは、問題を作成する手順を説明します。

問題を作成するには

  1. 次のいずれかのタブに移動し、［新規］をクリックします。
    • ［Remedyforceコンソール］>［ビュー］>［問題］
    • ［問題］（Salesforceページレイアウト）

      • フォームに一般的なアイテムが表示されます。詳細は、Salesforceページレイアウトのフォーム上の共通のアイテムを参照してください。

  2. ［問題］フォームで必要な詳細情報を入力します。問題フィールドの詳細については、問題フィールドを参照してください。
  3. ［保存］をクリックします。

レコードが作成されると、レコードが作成された日時とユーザーの詳細を追跡する［問題の履歴］セクションに次のアクションが記録されます。

問題フィールド

システム管理者は［問題］フォームのルックアップフィールドにフィルタを適用できます。ここで、ルックアップフィルタを理解するために役立ついくつかの重要なポイントを説明します。

    • ルックアップフィールドにフィルタが適用されている場合、フィルタ条件と一致しているレコードのみがポップアップウィンドウに表示されます。設定されたフィルタがオプションの場合、［すべての結果を表示］オプションを使用してフィルタを削除できます。
    • ルックアップタイプフィールドの先行入力は、フィルタ結果の一部である結果に対してのみ機能します。たとえば、［カテゴリ］ルックアップフィールドで、結果として［ハードウェア］および［アプリケーション］が表示されるようにフィルタが適用される場合、先行入力はこれらの2つのカテゴリのみに対して動作します。また、ルックアップアイコンをクリックして、検索フィールドに入力することによって他のカテゴリを検索する場合、［すべての結果を表示］オプションを使用してフィルタを削除しない限り、検索で何も結果が返されません。

次の表に、［新規問題］フォームで使用可能なフィールドのリストを示します。

フィールド                   
説明
問題の詳細
問題の発生源
問題の発生源を選択します。
カテゴリ
問題のカテゴリを選択します。
レコードを保存すると、カテゴリの値の横にアイコンが表示されます。をクリックすると、カテゴリツリーが表示されます。マウスをカテゴリの上に移動すると、カテゴリの説明がツールチップとして表示されます。
説明
解決する問題の説明を入力します。
システム管理者に連絡して、レイアウトを編集することで問題レイアウトに［標準説明および解決］ボタンを追加することができます。
詳細ページに［標準説明および解決］ボタンが配置されている場合は、これをクリックして、［説明の定型文］および［解決方法の定型文］カラムで必要なオプションを選択します。オプションを選択しようとしている目的の［説明の定型文］または［解決方法の定型文］カラム内でチェックボックスをオンにして、［OK］をクリックします。選択したオプションが詳細ページの［説明］および［解決方法］フィールドに表示されます。
解決方法
その問題に対して実施する必要がある解決策を入力します。
既知エラー
問題を既知のエラーとして表示するには、このチェックボックスをオンにします。その問題が既知のエラーである場合は、［問題の対処］フィールドに回避策を入力します（回避策がある場合）。
回避策
既知エラーの回避策を入力します。
ステータスと優先度の詳細
インパクト
問題に割り当てるインパクトを選択します。
緊急度とインパクトレベルは、問題の優先度の計算に使用されます。
緊急度
問題に割り当てる緊急度を選択します。
緊急度とインパクトレベルは、問題の優先度の計算に使用されます。
優先度
緊急度とインパクトを選択してレコードを保存すると、［優先度］フィールドに値が入力されます。優先度は、選択した緊急度とインパクトの値によって計算されます。
ステータス
問題に割り当てるステータスを選択します。
このフィールドを使用して、問題の進捗状況をステージを通して追跡できます。
選択したステータスにリンクされたステージに応じて、［問題］フォームの見出しにある問題進捗状況バーが更新されます。たとえば、「作業中」ステータスが「処理中」ステージにリンクされている場合に、ブロードキャストのステータスとして［作業中］を選択した場合は、問題進捗状況バーで［処理中］ステージが選択されます。
無効
現在のレコードをインアクティブとして表示するには、このチェックボックスをオンにします。
レコードはすでに無効になっているものの、レコード保持ポリシーを順守するためにレコードを保持する必要があるという理由で、レコードをインアクティブにしなければならない場合があります。

メモ:
 インアクティブとしてマークされたレコードは、データベースで実行される検索の対象に含まれません。
日時の詳細
開始
このフィールドには、問題が作成された日時が入力されます。
期限日
緊急度とインパクトを選択してレコードを保存すると、［期限日］フィールドに値が入力されます。この期限日は選択された優先度に関連付けられている期間に基づいて計算されます。
必要に応じて、計算された期限日とは異なる日時を選択できます。詳細は、期限日の計算オプションを参照してください。
終了日
このフィールドには、［終了］をクリックした後、システム日付が表示されます。［終了］ボタンは、問題の保存後に、問題の詳細ページに表示されます。
割り当ての詳細
オーナー
このフィールドには、問題を作成したログオンユーザーの名前が表示されます。問題のオーナーは、問題の保存後に変更できます。問題の割り当ての変更についての詳細は、問題オーナーの変更を参照してください。
