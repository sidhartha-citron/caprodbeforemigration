BMC Remedyforceと、ソフトフォンレイアウトをベースとするSalesforce用のCTIアダプタを統合できます。ここに記載された手順では、BMC Remedyforceとの連携についてテスト済みのCTI Demo Adapter 4.03の統合をテストします。この手順を使用すると、CTI Demo Adapterバージョン4.03を使用してSalesforce組織をテストして、使用しているCTIアダプタとBMC Remedyforceを統合することができます。

CTIアダプタをテストする前に、CTIアダプタがコンピュータにインストールされていることを確認してください。詳細は、 http://wiki.developerforce.com/page/CTI_Toolkit#Installing_The_Demo_Adapterを参照してください。CTI Demo Adapter 4.03を設定すると、システムトレイにアイコン（）が表示されます。

注意: クライアントレコードの［電話番号］フィールドが更新済みで、クライアントの既存のインシデントの［クライアントの電話］フィールドに値を追加する場合、UpdateIncidentClientPhone Pentahoパッケージ（https://communities.bmc.com/community/bmcdn/bmc_remedyforce/content?filterID=contentstatus[published]~objecttype~objecttype[document]で入手可能）を使用します。

統合設定をテストするには

  1. Demo CTIアダプタを起動するには、［スタート］>［すべてのプログラム］>［salesforce.com］>［CTI Demo Adapter 4.03］>［Salesforce.com Demo Adapter 4.03］の順に選択します。
  2. Demo CTIアダプタが設定されているSalesforce組織にログオンします。
  3. ［Remedyforceコンソール］タブをクリックします。
  4. （オプション）ページの左側にユーザーの資格情報を入力して、［ログイン］をクリックします。

  5. システムトレイでを右クリックし、［<電話番号>からの通話］をクリックします。

    6. <電話番号>はCTI Demo Adapter 4.03の設定時に入力した電話番号です。CTIの統合をテストするには、<電話番号>がクライアントレコードの［電話番号］フィールドに追加されていることを確認します。

アダプタが受信するようになります。アダプタは発信者の電話番号と一致するクライアントレコードの電話番号を検索します。その結果によって、次のいずれかが発生します。

    • 発信者の電話番号がどのクライアントレコードの電話番号とも一致しない場合は、新しいインシデントレコードが開かれます。
    • 発信者の電話番号がクライアントレコードの［電話番号］フィールドの電話番号と一致する場合は、新しいインシデントレコードが開かれます。［クライアントID］および［アカウント］フィールドには、一致するクライアントレコードの情報が入力されます。



      • 発信者の電話番号が複数のクライアントレコードの［電話番号］フィールド内の電話番号と一致する場合は、最初に一致したレコードの［クライアントID］および［アカウント］フィールドの値が取得されます。

    • 電話番号がクライアントレコードの［電話番号］フィールドの電話番号と一致し、そのクライアントに対して1件のインシデントが登録されている場合、該当するインシデントが表示されます。

      • 発信者の電話番号が複数のインシデントレコードの［クライアントの電話］フィールドの電話番号と一致する場合は、それらのインシデントがリストビューに表示されます。このリストビューには、クライアントレコードの［電話番号］フィールドが更新される前にクライアントに対して登録されたインシデントは表示されません。
