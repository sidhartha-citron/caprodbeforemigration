Pentahoでは、メタデータはファイルシステムにXML形式で保存されます。このとき、変換はKTRファイルで、ジョブはKJBファイルで保存されます。

使用しているCMDBバージョンによって、それぞれ異なるPentahoパッケージフォルダを使用する必要があります。使用できるPentahoフォルダについては、CMDB 1.0および2.0用のPentahoパッケージを参照してください。

KTRファイルには1つの変換が含まれています。提供されているKTRファイルには、すべてのCIタイプに対応する次の変換が含まれています。

手順                   
説明
デルタタイムスタンプ
保存したタイムスタンプを読み取ります。
Asset Core入力
保存したタイムスタンプに基づいてBMC FootPrints Asset Coreから情報を取得するSQLクエリー。
Asset Core変数の変換
フィールドの追加や削除など、JavaScriptの変換が容易になります。
［クラス<CIタイプ名>にアップサート］手順または［Update BE table with Asset Core <CI type name> information（BEテーブルをAsset Core<CIタイプ名>情報で更新）］
インポートしたデータの保存先とする、データおよびSalesforce組織の資格情報の宛先が受け入れられます。
（CMDB 2.0の場合のみ）Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）
BMC Remedyforce CMDBクラスの詳細を取得するためのSalesforce組織のアクセス詳細が受け入れられます。
成功行
正常にインポートされた行が保存されます。
失敗行
インポートされなかった行、エラーコード、エラーの説明、エラーフィールドが保存されます。

ジョブファイルには一連の変換処理が含まれています。提供されているKJBファイルには、次の手順が含まれています。

手順                   
説明
デルタタイムスタンプファイルの有無の確認
タイムスタンプファイルが存在するかどうかが確認されます。
Pentahoパッケージでは、タイムスタンプファイルを使用して、ジョブを最後に実行してから追加または変更されたレコードが確認されます。この手順は、インクリメンタルインポートを実行するために必要です。
タイムスタンプファイルが存在しない場合は［タイムスタンプの初期ファイルの作成］手順が実行され、それ以外の場合は［ステージリストをAsset Core<CIタイプ名>情報で更新］または［BEテーブルをAsset Core<CI タイプ名>情報で更新］手順が実行されます。
タイムスタンプの初期ファイルの作成
インポートの時間を記録するタイムスタンプファイルが作成されます。この手順は、ファイルを初めてインポートするとき、または既存のタイムスタンプファイルを削除したときにだけ実行されます。
（CMDB 1.0の場合のみ）ステージリストをAsset Core<CIタイプ名>情報で更新
ステージリストを使用してBMC FootPrints Asset CoreからBMC Remedyforce CMDBにデータが転送されます。
（CMDB 2.0の場合のみ）Update BE table with Asset Core <CI type name> Information（BEテーブルをAsset Core<CIタイプ名>情報で更新）
BMC FootPrints Asset CoreからBMC Remedyforce CMDBの基本要素オブジェクトにデータが転送されます。
現在のタイムスタンプの保存
データのインポートが成功すると、インポート時間が保存されます。

KTRファイルには、BMC FootPrints Asset Coreのデータベーステーブルのフィールドまたはビューと、CMDBステージオブジェクトのフィールド間のマッピングが含まれています。CMDBステージオブジェクトは、列であるCMDBデータベースの全フィールドがフラットな構造を持ちます。インポートされたデータは、CMDBデータベースへ保存される前に、まずCMDBステージオブジェクトへ保存されます。
