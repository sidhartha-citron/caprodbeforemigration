Alignabilityプロセスモデルは、リリース管理プロセスをシンプルな4層構造で記述します。このモデルを使用すると、該当する状況に実施すべきことを示す詳細な情報に、簡単かつ迅速にアクセスできます。また、常に、モデル自体からクリックするだけで詳細な指示へ到達できるようになっています。デフォルトでは［Alignability］タブが表示されていない場合、システム管理者に［Alignability］タブを設定するように依頼してください。

Alignabilityプロセスモデルのリリース管理プロセスは、優先度の設定でリリースを処理する方法を表します。

リリース管理のAlignabilityプロセスモデルを表示するには

  1. ［Alignability］タブに移動します。
  2. Alignabilityプロセスモデルで、［リリース管理］プロセスをクリックします。
  3. VisualforceページレイアウトUI を使用している場合は、次のアクションを実行します。
    1. ［ワークスペース］をクリックします。
    2. ［ワークスペース］のリストから［インシデント管理］を展開します。
    3. ［インシデント管理］ドロップダウンリストで、［インシデント管理Alignabilityプロセスモデルの手順］をクリックします。
  4. 次のアクションを実行します。
    1. リリース管理プロセスの手順の1つにアクセスするには、プロセスを構成するいずれかのボックスをクリックします。
    2. 手順内のステップの作業指示にアクセスするには、そのステップを表すアクションボックスまたは判断ひし形をクリックします。

      3. アクションボックスは実行する必要があるアクションを表します。常に、1本の矢印が1つのアクションボックスを指し、 そのアクションが完了した後でさらに他のアクションが必要な場合は、アクションボックスから1本の矢印が伸びます。

      判断ひし形は進んだ先の判断を表します。常に、1本の矢印が1つの判断ひし形を指し、 判断ひし形からは常に、2本の矢印が伸びます。

      イベント円は、発生したイベントまたは状況を表します。矢印がイベント円を指すことはありません。イベント円からは常に、1本の矢印が伸びます。

    3. 各手順で役割をクリックして、役割の説明と現在のプロセスの責任を表示します。
