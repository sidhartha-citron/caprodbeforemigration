Downtown Bankの運用スタッフメンバーでのDavid Martinは、100の重要なWebアプリケーションを管理しています。

アプリケーションを監視するようにBMC End User Experience Managementを設定し、アプリケーションごとに1つのウォッチポイントを設定し、さらにビジネスクリティカルアプリケーション（ログイン、送金、残高照会など）には5つのウォッチポイントを設定しました。Davidはアプリケーションサポートチームの電子メールアドレスを保持しています。

アプリケーションのウォッチポイントごとにボリューム基準を設定し、いずれかのクリティカルアプリケーション’（ログインなど）でトラフィックが増大した場合にアラートを送信するように設定しました。これは早期警戒システムです。トラフィックが増大しても、エンドユーザーに障害が発生しているとは限らないためです。ただし、この警告が通知されることにより、Davidはパフォーマンス基準に基づくアラートがいつでも発生する可能性があることを把握できます。

DavidはBMC Remedyforceに登録し、電子メールメッセージを使用してBMC Remedyforceを介してインシデントを作成しようとしています。Davidの狙いは、既存のアラートが発生した場合、インシデントが開き、適切なサポートグループにインシデントが送信されるようにすることです。

BMC Remedyforce管理者のBob Heartは、基準に従ってBMC Remedyforceでインシデントを作成するための統合環境を設定できるようDavidを支援しています。


手順1
BobはBMC Remedyforceで電子メールリスナーを作成し、定義済みのEmailCreateIncident Apexクラスを電子メールサービスに追加しました。


手順2
BobはBMC RemedyforceでCreateEUEMIncidentという名前のテンプレートを作成しました。このテンプレートには、インシデントフォームの［インパクト］、［クライアントID］、［カテゴリ］、［説明］、［ステータス］フィールドが含まれていて、電子メールメッセージから新しいインシデントを作成することができます。BobはBMC End User Experience Managementから受信した電子メールメッセージに基づいてインシデントを作成するようにBMC Remedyforceを設定するには の手順4～7に従って、複製されたワークフロー内でCreateEUEMIncidentテンプレートを指定しました。


手順3
DavidはBMC End User Experience Managementコンソールでアラートを設定するにはの手順を実行して、BMC End User Experience Management - Real User Monitorでウォッチポイントを設定しました。


手順4
ログインページのトラフィックが増大すると、電子メールメッセージが生成されて、［電子メール送信先］フィールドで指定した電子メールアドレスに送信されます。受信した電子メールメッセージに基づいてBMC Remedyforceで電子メールが作成され、アプリケーションサポートチームに通知されます。
