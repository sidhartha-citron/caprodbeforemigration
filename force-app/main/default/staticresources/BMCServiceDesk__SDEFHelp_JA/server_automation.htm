BMC Server Automation製品はデータセンターサーバーの初期プロビジョニングと継続的な自動管理をおこなう包括的なシステムです。

BMC RemedyforceおよびBMC Server Automationが統合されている場合、ユーザーはBMC Server Automationで次のアクションに関する通知を受けることができます。

    • プロビジョニング： 新しいサーバーの場合はベアメタルプロビジョニングをおこない、既存サーバーンの場合はオペレーティングシステムを再プロビジョニングします。
    • 設定： 会社のポリシーに従って、異機種環境サーバーのプロビジョニング、パッチ適用、設定、管理をおこないます。
    • 遵守： 社内および社外の標準と規制に準拠しているかを評価して、準拠していることを示します。
    • サーバー仮想化： 統合管理プラットフォームから物理環境と仮想環境を管理します。

この統合により、BMC Server Automationで実行されたジョブに基づいてインシデントおよび変更リクエストを作成できるようになります。ジョブのデータはインシデントオブジェクトおよび変更リクエストオブジェクトのフィールドにマップされます。

スタッフメンバーに注意を促す必要がある設定やコンプライアンスが追加または変更された場合は、BMC Remedyforceに電子メールメッセージを送信して、インシデントまたは変更リクエストを作成する必要があります。BMC Remedyforceに電子メールメッセージを送信するには、ジョブが実行されたときに電子メールメッセージを送信するジョブを作成する必要があります。

次のセクションで構成されています。

    • BMC Server Automationコンソールでのジョブの設定
    • BMC Remedyforceでインシデントを作成するためのBMC Server Automationの設定
    • BMC Remedyforceで変更リクエストを作成するためのBMC Server Automationの設定
    • ケースシナリオの使用： BMC Server Automationを使用したBMC Remedyforceでのインシデントの作成
    • ケースシナリオの使用： BMC Server Automationを使用したBMC Remedyforceでの変更リクエストの作成
