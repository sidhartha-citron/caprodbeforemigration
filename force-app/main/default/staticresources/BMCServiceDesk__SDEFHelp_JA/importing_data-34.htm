LDAPサーバーからデータをインポートする前に、LDAPサーバーからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

顧客ポータルマネージャのライセンスを持つユーザーをLDAPサーバーからPentahoを使用してインポートするには

  1. BMC CommunitiesのWebサイトからジョブファイルをダウンロードするには、 https://communities.bmc.com/docs/DOC-17004をクリックします。

    2. ジョブファイルは、Pentaho Data Integrationツールを解凍したフォルダにダウンロードすることをお勧めします。

  2. Pentaho Data Integrationツールを起動するには、\pdi-ce-4.x.0-stable\data-integrationまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダでSpoon.batファイルをダブルクリックして目的のKJBファイルを開きます。
  3. 該当するKTRファイルを開くには、［クライアントユーザーインポートの削除］手順を右クリックし、［データ変換を開く］を選択します。
  4. Salesforce組織にアクセスするための詳細情報を入力するには、次のアクションを実行します。
    1. KTRファイルで、［クライアントユーザーインポートレコードをすべて選択］手順をダブルクリックします。
    2. ［Salesforce入力］ウィンドウの［ユーザー名］および［パスワード］フィールドに、Salesforce組織のユーザー名とパスワードを入力します。
    3. （オプション）接続を確認するには、［接続テスト］をクリックします。
    4. ［［クライアントユーザーインポート］リストからレコードを削除］手順をダブルクリックします。
    5. ［Salesforceの削除］ウィンドウの［ユーザー名］および［パスワード］フィールドに、Salesforce組織のユーザー名とパスワードを入力します。
    6. （オプション）接続を確認するには、［接続テスト］をクリックします。
  5. LDAPサーバーにアクセスするための詳細情報を入力するには、次のアクションを実行します。
    1. KJBファイルに戻り、［クライアントユーザーインポートリストをLDAPユーザー情報で更新］手順を右クリックし、［データ変換を開く］を選択します。
    2. KTRファイルで、［LDAP入力］手順をダブルクリックします。
    3. ［LDAP入力］ウィンドウで、ホスト、ユーザー名、パスワードの詳細をそれぞれ所定のフィールドに入力します。
    4. （オプション）接続を確認するには、［接続テスト］をクリックします。
    5. （オプション）LDAPサーバーからさらにフィールドを取得するには、［フィールド］タブで［フィールドの取得］をクリックします。

      6. ［KTRファイル］で［動的アカウントおよびプロファイルの割り当て］手順を使用すると、LDAP属性で適用された条件に基づいて、カスタマイズされたLDAP属性値がSalesforce組織のオブジェクトに割り当てることができます。たとえば、userPrinicpalNameなどのLDAP属性に基づいて、アカウントとプロファイルを割り当てることができます。［動的アカウントおよびプロファイルの割り当て］手順をダブルクリックすると、［スクリプト値／変更］ウィンドウが表示されます。［スクリプト値／変更］ウィンドウに表示されるコメント付きの例は参照可能です。

    6. また、このKTRファイルではSalesforce組織のアクセス詳細情報を指定する必要があります。［Salesforceのアップサート］手順をダブルクリックし、Salesforce組織のユーザー名とパスワードを入力します。

      7. 注意: Pentaho 4.1や4.2などの以前のバージョンのPentaho Data Integrationツールを使用している場合は、Salesforce WebサービスのURLをhttps://www.salesforce.com/services/Soap/u/20.0に変更します。

    7. （オプション）LDAPフィールドとSalesforceクライアントユーザーインポートオブジェクト間のマッピングは事前に定義されています。マッピングを更新するには、［マッピングの編集］をクリックします。マッピングについての詳細は、CMDB 1.0で構成アイテムをインポートする方法の概要を参照してください。
    8. （オプション）Pentaho Data IntegrationツールとSalesforce組織との間の接続を検証するには、［接続テスト］をクリックします。
  6. （オプション）Salesforce組織にアクセスできるIPアドレスを限定するように設定されている場合、SalesforceとPentaho Data Integrationツールの間での通信を有効にするには、次のアクションを実行します。
    1. Salesforceで、［設定］>［自分のセキュリティトークンのリセット］の順に選択します。

      2. 注意: Salesforce組織で、改善された［設定］ユーザーインターフェイスが有効になっている場合は、［My Settings（私の設定）］>［Personal（個人）］>［自分のセキュリティトークンのリセット］に移動します。改善された［設定］ユーザーインターフェイスの詳細については、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。

    2. ［セキュリティトークンのリセット］をクリックすると、Salesforceに保存されている電子メールアドレスに電子メールが送信されます。
    3. ［クラス<CIタイプ名>にアップサート］手順の［パスワード］フィールドで、パスワードの後ろにセキュリティトークンを追加します。

      4. たとえば、パスワードがmypasswordで、セキュリティトークンがXXXXXXXXXであれば、［パスワード］フィールドに「mypasswordXXXXXXXXX」と入力する必要があります。

  7. KTRファイルとKJBファイルを保存するには、をクリックします。
  8. KJBファイルで、をクリックします。
  9. ［ジョブの実行］ウィンドウで、［起動］をクリックします。

    10. Salesforceの［クライアントユーザーインポート］リストにデータがインポートされます。インポートされたクライアントがSalesforce組織に存在しない場合は、新規クライアントが作成されます。インポートされたレコードにアカウントやプロファイル情報がない場合は、BMC Remedyforceの［Remedyforce管理］>［ユーザーの管理］>［ユーザー設定］ページで指定されたデフォルトのアカウントとプロファイルがレコードに追加されます。インポートされたレコードに存在するアカウントまたはプロファイル情報がSalesforce組織に存在しない場合、このようなレコードは［失敗行］手順でインポートされます。

    完了した変換は、で示されます。は現在実行中の変換を示し、は失敗した変換を示します。

  10. （オプション）ログを表示するには、［実行の結果］セクションで［ログ］タブをクリックします。

    11. すべてのエラーは赤で表示されます。

  11. （オプション）ジョブのインポートをスケジュールするには、インポートしたジョブのスケジュール作成を参照してください。
