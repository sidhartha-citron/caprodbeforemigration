変更スケジュールには、特定の日、週、または月について、スケジュール設定された変更、タスク、およびリリースが表示されます。また、変更管理者が作成したサービス停止計画やブラックアウト期間も表示されます。変更スケジュールを使用すると、変更リクエストまたはリリースの実行が予定されている日時や、変更を実施すべきではない期間（ブラックアウト期間）を確認できます。

目的の変更リクエストまたはリリースの上にマウスを移動すると、変更番号またはリリース番号、ステージ、優先度、説明、変更タイプまたはリリースタイプ、ステータス、割り当て先、イニシエータID、開始日、終了日などの情報がツールチップに表示されます。サービス停止計画またはブラックアウト期間の上にマウスカーソルを移動すると、名前、影響を受けるサービス、開始日、終了日、サービス停止計画がブラックアウト期間かどうかなどの情報がツールチップに表示されます。目的の変更リクエスト、リリース、サービス停止計画、ブラックアウト期間のいずれかをクリックすると、そのレコードがそれぞれのフォームで開きます。リリースレコードは、実装日時や配布日時に基づいて変更スケジュールに表示されます。実装日時や配布日時が変更スケジュールを表示している期間内である場合は、変更スケジュールに同じリリースレコードが2回表示されます。配布日時に表示されるリリースレコードは同じ色で示されますが、レコードの色には交差線が表示されます。

変更スケジュールには、変更リクエストとリリースが次の形式で表示されます。

    • 番号別： 変更スケジュールの凡例に変更リクエスト番号またはリリース番号のみが表示されます。このオプションは、変更リクエスト番号またはリリース番号がわかっていて、その変更リクエストの実行予定時期を確認する場合に使用します。
    • タイプ別： 変更スケジュールの凡例に変更リクエストまたはリリースタイプのみが表示されます。このオプションは、タイプに基づいて変更リクエストまたはリリースを表示する場合に使用します。の上にマウスカーソルを置くと、変更スケジュールで変更リクエストおよびリリースのタイプに使用する色の凡例が表示されます。

      • カスタム値（最大15個）を［変更タイプ］フィールドまたは［リリースタイプ］フィールドに追加すると、変更スケジュールではそれぞれのカスタム値に異なる色が割り当てられます。16個目以降のカスタム値には同じ色が使用されます。

変更スケジュールのビューの設定

デフォルトでは、現在の月に関する情報が表示されます。変更スケジュールの情報を日、週、または月別に表示するには、次のいずれかのビューを使用します。

    • 1日(12時間制)
    • 1日(24時間制)
    • 1週間
    • 1ヵ月

1日ビューを選択すると、変更スケジュールの開始時刻を定義できます。1日ビューでは12時間制と24時間制のどちらで変更およびリリースを表示するのかを指定することが可能です。選択したビューはブラウザのクッキーに保存されます。変更スケジュールを閉じて再び開くと、変更スケジュールは最後に選択したビューで表示されます。

変更スケジュールのビューを設定するには

  1. 変更スケジュールを表示するレコードに移動します。

    2. 詳細は、リリースレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［エージェントツール］メニューをクリックし、［変更スケジュール］を選択します。
    • ［リリース］タブ（Salesforceページレイアウト）を使用している場合、［変更スケジュール］をクリックします。

      • または、［Remedyforce管理］>［ワークフローと承認の管理］>［変更スケジュール］または［Remedyforceワークスペース］>［変更管理］>［変更スケジュール］の順に選択することもできます。

  3. 変更スケジュールの左セクションにある［ビュー］ドローワをクリックします。
  4. 次のいずれか、または複数のアクションを実行して、変更スケジュールのビューを設定します。

    5. ビュー

変更スケジュールの印刷

表示された変更スケジュールを印刷するには、印刷の前に背景の色と画像を含めるようブラウザを設定する必要があります。

変更スケジュールをPDF形式で印刷する場合は、doPDFをインストールします。doPDFを http://www.dopdf.com/download.phpからダウンロードしてインストールします。

変更スケジュールを印刷するには

  1. Microsoft Internet Explorerを使用している場合は、次の手順に従って、変更スケジュールの印刷時に背景の色と画像を含めるようブラウザを設定します。
    1. ［ファイル］>［ページ設定］を選択します。
    2. ［用紙オプション］セクションで、［背景の色とイメージを印刷する］をオンにします。
    3. ［OK］をクリックします。
  2. Mozilla Firefoxを使用している場合は、次の手順に従って、変更スケジュールの印刷時に背景の色と画像を含めるようブラウザを設定します。
    1. ［ファイル］>［ページ設定］を選択します。
    2. ［書式とオプション］タブで、［オプション］エリアの［背景を印刷（配色と画像）］チェックボックスをオンにします。
    3. ［OK］をクリックします。
  3. Apple Safariを使用している場合は、次の手順に従って、変更スケジュールの印刷時に背景の色と画像を含めるようブラウザを設定します。
    1. ［ファイル］>［プリント］を選択します。
    2. ［印刷部数と印刷ページ］ドロップダウンリストから［Safari］を選択します。
    3. ［背景をプリント］チェックボックスをオンにします。
  4. Salesforceホームページで［リリース］タブをクリックします。
  5. リリースのリストで、変更スケジュールを印刷するリリースをクリックします。
  6. ［変更スケジュール］をクリックします。

    7. または、［Remedyforceワークスペース］>［変更管理］>［変更スケジュール］の順に選択します。

  7. をクリックします。
  8. ［印刷］ウィンドウで、目的のプリンタを選択します。
  9. 選択したプリンタの印刷設定を設定します。
  10. ［印刷］をクリックします。
  11. Internet Explorerで変更スケジュールをPDF形式で印刷する場合は、次の手順を実行します。
    1. ［印刷］ウィンドウで、［プリンタの選択］ドロップダウンリストから［doPDF v7］を選択し、［詳細設定］をクリックします。
    2. ［用紙サイズ］セクションで［サイズを指定］を選択します。
    3. ［幅］フィールドの値を「230」に設定し、［高さ］フィールドの値を「250」に設定します。
    4. ［OK］をクリックします。
    5. ［印刷］ウィンドウで、［印刷］をクリックします。
    6. （オプション）［doPDF - PDFファイルを保存］ウィンドウで［参照］をクリックし、PDFファイルの保存先を選択します。
    7. ［OK］をクリックします。
  12. Firefoxで変更スケジュールをPDF形式で印刷する場合は、次の手順を実行します
    1. ［印刷］ウィンドウで、［名前］ドロップダウンリストから［doPDF v7］を選択し、［プロパティ］をクリックします。
    2. ［用紙サイズ］セクションで［サイズを指定］を選択します。
    3. ［幅］フィールドの値を「350」に設定し、［高さ］フィールドの値を「250」に設定します。
    4. ［OK］をクリックします。
    5. ［印刷］ウィンドウで、［印刷］をクリックします。
    6. （オプション）［doPDF - PDFファイルを保存］ウィンドウで［参照］をクリックし、PDFファイルの保存先を選択します。
    7. ［OK］をクリックします。

リンクされたタスクの表示

変更リクエストまたはリリースにリンクされたタスクは、［タスク］ペインに表示できます。

リンクされたタスクを表示するには

  1. リンクされたタスクを表示するレコードに移動します。

    2. 詳細は、リリースレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［エージェントツール］メニューをクリックし、［変更スケジュール］を選択します。
    • ［リリース］タブ（Salesforceページレイアウト）を使用している場合、［変更スケジュール］をクリックします。

      • または、［Remedyforce管理］>［ワークフローと承認の管理］>［変更スケジュール］の順に選択するか、［Remedyforceワークスペース］>［変更管理］>［変更スケジュール］の順に選択することもできます。

  3. ページの一番下にある［タスク］ペインをクリックします。
  4. ［リンク先］ドロップダウンリストで、タスクを表示する変更リクエストまたはリリースを選択します。

    5. 選択した変更リクエストまたはリリースにリンクされたタスクが表示されます。まだ変更リクエストまたはリリースに作成されていないタスクは、［Unscheduled Tasks（予定外のタスク）］ペインに表示されます。［開始予定日時］または［終了予定日時］フィールドで値が選択されていないタスクは、［予定外のタスク］ペインに表示されます。

  5. タスクの詳細を表示するには、タスクをクリックします。

変更スケジュールに表示されるレコードのレポートの作成

変更スケジュールに表示される変更リクエストおよびリリースのレポートを作成できます。

変更スケジュールに表示されるレコードのレポートを作成するには

  1. 変更スケジュールに表示されるレコードのレポートを作成するレコードに移動します。

    2. 詳細は、リリースレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［エージェントツール］メニューをクリックし、［変更スケジュール］を選択します。
    • ［リリース］タブ（Salesforceページレイアウト）を使用している場合、［変更スケジュール］をクリックします。

      • または、［Remedyforce管理］>［ワークフローと承認の管理］>［変更スケジュール］の順に選択するか、［Remedyforceワークスペース］>［変更管理］>［変更スケジュール］の順に選択することもできます。

  3. をクリックします。

    4. 変更スケジュールに表示される変更リクエストとリリースの詳細を含むレポートが、CSV形式で保存されます。

［検索］テキストボックスについて

変更スケジュールの右上隅にある［検索］テキストボックスから、変更リクエスト、リリース、タスク、およびインターネットを検索できます。システム管理者は、［フィールドの表示］セクションの［グローバル検索の設定］ページで検索結果に表示されるフィールドをカスタマイズできます。［データソース］カラムの［変更リクエスト］、［リリース］、または［タスク］、および［データフィールド］カラムの必須フィールドを選択します。
