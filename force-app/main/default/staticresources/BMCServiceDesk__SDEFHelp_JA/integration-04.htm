Davidは会社のシステム管理者です。DavidはBMC ProactiveNet Performance Managementを使用して、仮想コンピュータ（VM）のCPU使用率を監視しています。CPU使用率が85%を超えると、イベントが生成されます。DavidはBMC ProactiveNet Performance ManagementとBMC Remedyforceを統合して、イベント生成時にインシデントが作成されるようにしました。イベントはBMC Remedyforceで電子メールメッセージとして受信されます。この電子メールメッセージを使用してインシデントが作成されます。このインシデントがスタッフユーザーに割り当てられて、必要な手順がおこなわれます。

このシナリオでインシデントを作成するために、Davidは次のタスクを実行する必要があります。

    • BMC Remedyforceで電子メールサービスを作成し、この電子メールサービスにGenericEmailListener Apexクラスを追加する。このタスクはSalesforce設定で実行します（［Remedyforce管理］>［電子メールの設定］>［電子メールサービス］）。Visualforceページレイアウトユーザーインターフェイス（UI ）を使用している場合は、［設定］>［開発］>［電子メールサービス］の順に選択します。
    • BMC ProactiveNet Performance Managementでイベントを作成する。BMC ProactiveNetは、VMのCPU使用率が85%を超えるとイベントを生成します。
    • イベント作成中に、イベント内でBMC Remedyforceに設定される電子メールアドレスを指定する。BMC ProactiveNet Performance Managementのイベント中に生成されるアラートは、この電子メールアドレスに送信され、インシデントの作成に使用されます。
    • インシデントオブジェクトのフィールドにアラートのデータをマップする。このタスクは［Remedyforce管理］>［電子メールの設定］>［インシデント - 電子メールの設定］ページで実行します。VisualforceページレイアウトのUI を使用している場合は、［設定］>［アプリケーションの設定］>［インシデント - 電子メールの設定］の順に選択します。

      • 指定したオブジェクトを含む電子メールメッセージをBMC Remedyforceが受信した場合はインシデントが作成されるように、このタブで指定します。BMC ProactiveNet Performance Managementのイベント規則の［件名］フィールドで件名を指定できます。次に、電子メールメッセージの内容を［インシデント］フィールドにマップします。たとえば、電子メールメッセージの本文をインシデントの［説明］として保存することができます。
