リリースマニフェストを使用すると、リリースにリンクされたタスクおよび変更リクエストの並べ替え順序を設定できます。レコードを保存すると、マニフェストを使用できるようになります。マニフェストによって、レコードにリンクされるすべてのタスクと変更リクエストが表形式で表示されます。リンクされたタスクおよび変更リクエストの並べ替え順序を設定し、リリースのステータスをそれらのタスクおよび変更リクエストに関連付けることもできます。

マニフェストについて

マニフェストを使用して、レコードのライフサイクル中にリンクされたタスクおよび変更リクエストを実行する方法のシーケンスを確立できます。

また、リリースのステータスをリンクされたタスクおよび変更リクエストと関連付けて、リリースが別のステータスに移行する前に、現在のステータスが関連付けられているすべてのタスクおよび変更リクエストが終了するようにすることもできます。たとえば、タスクT1およびT2と変更リクエストC1と、ステータスが［未完了］のリリースが関連付けられているとします。この場合、T1、T2、およびC1が終了するまで、リリースのステータスを［進行中］やその他のステータスに変更することはできません。

テンプレートで定義されているものの、まだ作成されていないタスクは、ステータスが［作成保留中］と表示されます（将来のタスクとして参照される）。テンプレートを使用してリリースを作成する場合、このようなタスクを作成できます。今後のタスクの並べ替え順の番号は、テンプレートの作成済みタスクの並べ替え順の番号より大きくなければなりません。

リンクされたタスクおよび変更リクエストの並べ替えの番号が、リリースを作成するために使用されるテンプレートで指定されていて、マニフェストでこれらのタスクおよび変更リクエストに別の並べ替えの番号を設定した場合、マニフェストで指定された並べ替えの順序に従います。

マニフェストには、タスク、変更リクエスト、リリースに関する次のデータが表示されます。

    • リリースアイテム： リリースID
    • アイテムID： タスクまたは変更リクエストID
    • 説明： タスクまたは変更リクエストの説明
    • ステータス： タスクまたは変更リクエストのステータス
    • 期限日時： タスクまたは変更リクエストの期限日時
    • 優先順位： タスクまたは変更リクエストの優先順位
    • 割り当て先： タスクまたは変更リクエストの割り当て先ユーザー（スタッフユーザーまたは変更イニシエータ）
    • リリースステータス： リリースのステータス
    • 順序： 特定のリリースステータスについての、タスクまたは変更リクエストのシーケンス番号

リンクされたタスクおよび変更リクエストの並べ替え順序の設定とリリースステータスの関連付け

このセクションでは、リンクされたタスクおよび変更リクエストの並べ替え順序を設定し、リリースのステータスを目的のタスクおよび変更リクエストに関連付ける手順を説明します。

リンクされたタスクおよび変更リクエストの並べ替え順序を設定し、リリースステータスと関連付けるには

  1. リンクされたタスクおよび変更リストの並べ替え順序を設定してリリースステータスに関連付けるレコードに移動します。

    2. 詳細は、リリースレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［エージェントツール］メニューから［リリースマニフェスト］を選択します。
    • ［リリース］タブ（Salesforceページレイアウト）を使用している場合、［リリースマニフェスト］をクリックします。

      • ［リリースマニフェスト］ボタンを使用できない場合は、システム管理者に連絡して、リリースフォームのレイアウトをカスタマイズしてボタンを追加するように依頼してください。

      すべてのリンクされたタスクと変更リクエストが表形式で表示されます。

  3. タスクまたは変更リクエストの並べ替え順の番号を設定するには、［並べ替え順の番号］カラムでタスクまたは変更リクエストのセルをクリックして、並べ替え順の番号を選択します。

    4. 選択した並べ替え順の番号により、タスクまたは変更リクエストが実行されるシーケンス番号が定義されます。

  4. タスクおよび変更リクエストにステータスを関連付けるには、［リリースステータス］カラムで目的のタスクまたは変更リクエストのセルをクリックして、目的のステータスを選択します。

    5. タスクまたは変更リクエストの並べ替え順の番号を設定し、ステータスを関連付けるとき、次の点に注意してください。

      • リリースのステータスを更新するとき、現在のステータスのすべてのタスクと変更リクエストが完了している必要があります。たとえば、レコードのステータスを［割り当て済み］から［進行中］に変更する場合、［割り当て済み］ステータスのすべてのタスクと変更リクエストは完了している必要があります。
      • 今後のタスクの並べ替え順の番号は、作成済みタスクの並べ替え順の番号より大きくなければなりません。
      • 今後のタスクの並べ替え順の番号を設定する場合、今後のすべてのタスクの並べ替え順の番号を設定してください。
      • タスクおよび変更リクエストの並べ替え順の番号を設定し、インシデントのステータスを選択する必要があります。
      • 選択したリリースのステータスがアクティブなとき、タスクまたは変更リクエストが実行されるか、または実装される必要があります。
  5. （オプション）リンクされたタスクと変更リクエストのリストをCSVファイル形式にエクスポートするには、［エクスポート］をクリックします。
  6. （オプション）リンクされたタスクと変更リクエストのリストを印刷するには、［印刷］をクリックします。
  7. ［保存］をクリックします。
