コンソールを起動するには、Oracle Java Runtime Environment（JRE™）の最新バージョンをインストールする必要があります。BMC FootPrints Asset Coreとの統合はJRE 6以降のバージョンで機能します。BMCではJRE 7の使用をお勧めします。JREは http://www.java.com/getjavaからダウンロードできます。詳細は、JREのインストールを参照してください。JREをインストールすると、Java Web Startがコンピュータにイ]ンストールされます。BMC Remedyforceでは、Java Web Startを使用して［Asset Core アクション］メニューのコマンドを実行する必要があります。BMC FootPrints Asset Coreデバイスを現在のインシデントまたはサービスリクエストにリンクする場合は、［CIアクション］をクリックし、［Asset Coreアクション］メニューからコマンドを選択して、［実行］をクリックすると、コマンドが実行されます。

［Asset Coreアクション］メニューからの運用規則の導入

［Asset Coreアクション］メニューの［詳細アクション］コマンドをクリックすると、［アセットコア運用規則からの選択］ウィンドウが開き、インシデントまたはサービスリクエストにリンクされたデバイスに導入する運用規則を選択できます。

1つ以上のBMC FootPrints Asset Core機能が、運用規則に定義された順番でデバイスに導入されます。

運用規則の内容は次のとおりです。

    • パッケージ： 運用規則の手順を使用してインストールされるソフトウェア
    • 依存関係： 現在の運用規則の導入時期を定義する運用規則

      • 注意: ［Asset Core運用規則から選択］ウィンドウで表示されるためには、運用規則の［外部統合からの導入］プロパティに、運用規則の［統合ソリューションで利用可能］または［追加のワークフローが必要］の値が設定されている必要があります。運用規則の［外部統合からの導入］プロパティが［追加のワークフローが必要］に設定されている場合、運用規則では実装前に承認が必要です。このプロパティは、BMC FootPrints Asset Coreコンソールで設定できます。

    • 説明
    • 最終変更日時
    • 最終変更者
    • 承認が必要： 運用規則を導入する前に、承認が必要かどうかを表示します。

    • 運用規則の導入に関するリクエストが正常に完了しなかった場合、運用規則のステータスは［実行エラー］に更新され、タスクのステータスは［実行エラー］に割り当てられたステータス（［キャンセル済み］など）に更新されます。
    • 運用規則の導入に関するリクエストが正常に完了した場合、運用規則のステータスは［リクエストが成功しました］に更新され、タスクのステータスは［リクエストが成功しました］に割り当てられたステータス（［終了済み］など）に更新されます。

運用規則を導入する前に承認が必要な場合は、承認者が承認するか、承認を却下するまで、現在のインシデントまたはサービスリクエストはロックされたままになります。また、現在のインシデントまたはサービスリクエストの［アセットコアの承認ステータス］フィールドが［承認待ち］に更新されます。承認されると、［アセットコアの承認ステータス］フィールドは［承認済み］に更新されます。承認が却下されると、［アセットコアの承認ステータス］フィールドは［却下済み］に更新されます。承認プロセスに失敗した場合は、［インシデントの履歴］セクションに失敗の原因が記録されます。承認が不要な場合は、［アセットコアの承認ステータス］フィールドの値は［適用外］になります。承認に失敗した理由を修正して、インシデントまたはサービスリクエストを再送信した場合は、承認された後に運用規則が導入されます。承認プロセスが作成されていない場合、タスクは作成されますが、［アクション履歴］タブにエラーが記録されます。承認プロセスを作成して、インシデントまたはサービスリクエストを再送信した場合は、承認された後に運用規則が導入されます。

デバイスに複数の運用規則を導入できます。BMC Remedyforceによって運用規則ごとにタスクが作成されます。導入前に複数の運用規則をそれぞれ承認する必要がある場合は、必要な承認が得られた場合のみ、関連付けられた運用規則が導入されます。
