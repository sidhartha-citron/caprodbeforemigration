BMC Atrium Discoveryから構成アイテムをインポートする前に、BMC Atrium Discoveryからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

Pentahoを使用してBMC Atrium Discoveryからのデータをインポートするには

  1. \pdi-ce-4.x.0-stable\data-integrationフォルダまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダから、Kitchen.batファイルのバックアップコピーを作成します。
  2. 使用しているCMDBまたはBMC Atrium Discoveryのバージョンに応じて、BMC Communitiesページからジョブファイルをダウンロードします。

    3. CMDBバージョン              
    BMC Atrium Discoveryバージョン
    Pentahoパッケージのリンク
    1.0
    8.2
    https://communities.bmc.com/docs/DOC-14523
    1.0
    9.0、10.0
    https://communities.bmc.com/docs/DOC-21741
    2.0
    9.0、10.0
    https://communities.bmc.com/docs/DOC-28434

    注意: CMDB 2.0にアップグレードしていて、以前はCMDB 1.0パッケージを使用していた場合、引き続きCMDB 1.0パッケージを使用できます。ただし、BMCは、迅速なインポートのために、CMDB 2.0用のパッケージを使用することをお勧めします。

    ジョブファイルは、Pentaho Data Integrationツールを解凍したフォルダにダウンロードすることをお勧めします。

    注意: PentahoパッケージをBMC Communitiesページからすでにダウンロードしていて、BMC_IPEndPoint CIタイプのIPv6レコードもインポートする場合、IPEndpointsフォルダから最新の変換ファイルをダウンロードすることをお勧めします。

  3. BMC Communitiesページからダウンロードしたフォルダで、CommonフォルダをダブルクリックしてKitchen.batファイルをコピーし、\pdi-ce-4.x.0-stable\data-integrationフォルダまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダに貼り付けます。
  4. Pentaho Data Integrationツールを起動するには、\pdi-ce-4.x.0-stable\data-integrationまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダでSpoon.batファイルをダブルクリックして目的のKJBファイルを開きます。
  5. KTRファイルを開くには、使用しているCMDBおよびPentahoのバージョンに応じて、次のいずれかのアクションを実行します。

    6. CMDBバージョン            
    Pentahoバージョン
    アクション
    1.0
    4.4
    ［ステージリストをADDM<CIタイプ名>情報で更新］手順を右クリックし、［データ変換を開く］を選択します。
    1.0
    5.0.1
    ［ステージリストをADDM<CIタイプ名>情報で更新］手順を右クリックし、［Open referenced object（参照オブジェクトを開く）］を選択してから、［Transformation（データ変換）］を選択します。
    2.0
    4.4
    ［BEテーブルをADDM<CIタイプ>情報で更新］手順を右クリックし、［データ変換を開く］を選択します。
    2.0
    5.0.1
    ［BEテーブルをADDM<CIタイプ名>情報で更新］手順を右クリックし、［Open referenced object（参照オブジェクトを開く）］を選択してから、［Transformation（データ変換）］を選択します。

  6. KTRファイルで、［CSV URLの作成］手順をダブルクリックします。

  7. スクリプトで、次のアクションを実行します。
    1. <Enter HostName>には、BMC Atrium Discoveryのサーバー名またはBMC Atrium DiscoveryのプライマリIPアドレスを入力します。
    2. <Enter user name>には、BMC Atrium Discoveryのユーザー名を入力します。
    3. <Enter Password>には、BMC Atrium Discoveryのパスワードを入力します。
  8. （オプション）データのうち特定の行数をインポートするには、次のアクションを実行します。
    1. ［ADDM入力］手順を右クリックして、［手順の編集］を選択します。
    2. テキストファイル入力ウィンドウで、［内容］タブをクリックします。
    3. ［制限］フィールドで、インポートする行数を入力します。

      4. デフォルトでは、［制限］フィールドには「0」が入力されています。この場合は、クエリーを満たすすべての行がインポートされます。

      ［制限］フィールドには1行分追加した値を入力してください。これは、取得される行はこの値より1行分少なくなるためです、 たとえば、100行取得する場合は、「101」と入力します。

    4. ［OK］をクリックします。
  9. Salesforce組織のアクセス詳細情報を設定するには、次のアクションを実行します。
    1. ［クラス<CIタイプ名>にアップサート］手順をダブルクリックします。
    2. ［Salesforceのアップサート］ウィンドウで、Salesforce組織のユーザー名とパスワードを入力します。
    3. （オプション）BMC Atrium DiscoveryフィールドとRemedyforce CMDBの間のマッピングは事前に定義されています。マッピングを更新するには、［マッピングの編集］をクリックします。マッピングについての詳細は、CMDB 1.0で構成アイテムをインポートする方法の概要を参照してください。

      4. 注意: Pentaho 4.1や4.2などの以前のバージョンのPentaho Data Integrationツールを使用している場合は、Salesforce WebサービスのURLをhttps://www.salesforce.com/services/Soap/u/20.0に変更します。

    4. （オプション）接続を確認するには、［接続テスト］をクリックします。
    5. ［OK］をクリックします。
  10. （CMDB 2.0の場合のみ）［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］手順でSalesforceアクセス詳細情報を入力する必要があります。Salesforce組織へのアクセスに必要な情報を入力するには、次のアクションを実行します。
    1. ［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］をダブルクリックします。
    2. ［接続］セクションで、Salesforce組織のアクセス詳細を入力します。
    3. （オプション）接続をテストするには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。
  11. （オプション）Salesforce組織にアクセスできるIPアドレスを限定するように設定されている場合、SalesforceとPentaho Data Integrationツールの間での通信を有効にするには、次のアクションを実行します。
    1. Salesforceで、［設定］>［自分のセキュリティトークンのリセット］の順に選択します。

      2. 注意: Salesforce組織で、改善された［設定］ユーザーインターフェイスが有効になっている場合は、［My Settings（私の設定）］>［Personal（個人）］>［自分のセキュリティトークンのリセット］に移動します。改善された［設定］ユーザーインターフェイスの詳細については、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。

    2. ［セキュリティトークンのリセット］をクリックすると、Salesforceに保存されている電子メールアドレスに電子メールが送信されます。
    3. ［クラス<CIタイプ名>にアップサート］手順の［パスワード］フィールドで、パスワードの後ろにセキュリティトークンを追加します。

      4. たとえば、パスワードがmypasswordで、セキュリティトークンがXXXXXXXXXであれば、［パスワード］フィールドに「mypasswordXXXXXXXXX」と入力する必要があります。

  12. KTRファイルとKJBファイルを保存するには、をクリックします。
  13. Pentaho Data Integrationツールを閉じます。
  14. データのインポート先であるCIタイプのフォルダに含まれた.batファイルをダブルクリックします。

    15. たとえば、コンピュータシステムのデータをインポートしている場合は、SchedulingComputerSystem.batファイルを開きます。

    注意: エラーを防ぐために、.batファイルを使用する必要があります。

    コンピュータシステム（ソース）とソフトウェアサーバー（宛先）との関係は、構成アイテムエクスプローラには表示されません。これは、（プラットフォームの制限により）構成アイテムエクスプローラは一部の関係しか表示できないためです。コンピュータシステムとソフトウェアサーバー間の関係を表示すると、コンピュータシステムと他のCI間の重要な関係が非表示になる場合があります。

  15. （オプション）ジョブのインポートをスケジュールするには、インポートしたジョブのスケジュール作成を参照してください。

コンピュータシステムと各種CIとの間に関係を作成する場合は、BMCServiceDesk__AssemblyId__cフィールドが必要です。ソースはコンピュータシステムであり、宛先にはオペレーティングシステム、プロセッサ、IPエンドポイント、LANエンドポイント、ソフトウェアサーバーなどがあります。

たとえば、コンピュータシステムレコードに対して［BMCServiceDesk__AssemblyId__c］フィールドの値が1で、プロセッサレコードのアセンブリID値が1の場合、コンピュータシステム（ソース）とプロセッサ（ターゲット）の間に関係が自動的に作成されます。
