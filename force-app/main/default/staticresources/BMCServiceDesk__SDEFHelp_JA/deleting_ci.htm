使用していないCIがある場合や古いハードウェアなどのCIを破棄する場合は、これらのCIを削除すると、IT環境にあるCIを正確に把握できるようになります。データベースからCIを削除すると、CIに関連付けられたすべての関係も削除されます。

注意: CIのクラスに対して作成、編集、削除の各権限を所有していない場合は、CIを作成、編集、削除できません。これらの権限を所有していない場合、［新規］および［保存］ツールバーボタンが無効になります。

構成アイテムを削除するには

  1. ［Remedyforce CMDB］タブをクリックします。
  2. Visualforceページレイアウトユーザーインターフェイスを使用している場合は、次のアクションを実行します。
    1. BMC Remedyforceホームに移動し、［ワークスペース］をクリックします。
    2. ［ワークスペース］のリストから［構成管理］を展開します。
    3. ［構成管理］ドロップダウンリストで、［構成アイテム］をクリックします。
  3. CMDB Managerの右のペインで、削除するCIを選択します。

    4. CIを検索するには、CIの名前の最初の数文字またはクラス名の完全な名前を［検索］テキストボックスに入力して、 をクリックします。たとえば、Email Service CIを検索するには、「Email」または「Email Service」と［検索］テキストボックスに入力して、 をクリックします。また、［インスタンスID］、［インスタンス名］、［説明］、［クラス名］、［アセット番号］、［トークンID］、［シリアル番号］の各フィールドのデータを使用して検索することもできます。左のペインでクラスを選択すると、そのクラスのCIだけが検索されます。すべてのクラスを検索するには、左のペインで［基本要素］クラスを選択します。

  4. をクリックします。
  5. 確認ダイアログボックスで［Yes］をクリックします。

Salesforce Object Query Language（SOQL）のクエリー、Salesforceページレイアウト、データローダ、またはPentaho Data Integrationツールを使用して、CMDBの［基本要素］オブジェクトから1つまたは複数のCIを削除できます。

たとえば、［基本要素］オブジェクトからコンピューターシステムレコードを削除する場合、CMDB階層にしたがって［コンピューターシステム］オブジェクト、［システム］オブジェクト、および［基本要素］オブジェクトからレコードが削除されます。複数のレコードを同時に削除できます。

CIを削除すると、CI関係、CIクライアントリンク、およびCIタスクリンクも削除されます。

開発者向けコンソールを使用して構成アイテムを削除するには

  1. ［Developer Console（開発者向けコンソール）］に移動します。
  2. ［実行］ボタンの横にあるテキストボックスをクリックします。
  3. スクリプトを入力して、CIを削除します。クエリーのバッチサイズは50にする必要があります。

    4. たとえば、特定のクラスの［基本要素］からCIを削除するには、次のスクリプトを使用します。

    Delete [Select ID From BMCServiceDesk__BMC_BaseElement__c where BMCServiceDesk__className__c ='BMC_ComputerSystem' Limit 50]

    削除するようにマークされた［基本要素］クラスからCIを削除するには、次のスクリプトを使用します。

    Delete [Select ID From BMCServiceDesk__BMC_BaseElement__c where BMCServiceDesk__MarkAsDeleted__c =TRUE Limit 50]

  4. ［実行］をクリックします。
