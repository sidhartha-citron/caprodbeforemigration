次の表は、BMC Atrium CMDBからデータをインポートする場合に発生する可能性があるエラーや問題と、それらのエラーや問題を解決するために適用する必要のある解決策の一覧です。

エラーまたは問題                   
解決策
Atrium Integratorにジョブが表示されない。
Atrium IntegratorのフォルダにKTRファイルとKJBファイルを保存したことを確認します。
Error setting value #2 [ARDelta_1 Integer] on prepared statement (Integer)（準備されたステートメントの設定値 #2 [ARDelta_1 Integer] （整数）のエラー）
このエラーは、変換を実行するためのBMC Remedy AR Systemの［NGIE:Delta（NGIE:デルタ）］フォームに複数のレコードが作成された場合に発生します。レコードを削除して、エラーが発生した変換ファイルに対して作成された余分なARDeltaエントリを削除します。
変換ファイルで重複しているレコードを検索するには、BMC Remedy AR Systemで［NGIE:Delta（NGIE:デルタ）］フォームを開きます。［TransName（変換名）］フィールドに変換ファイル名を入力し、［検索］をクリックします。
Salesforceのクエリーを実行できない
［Salesforceのアップサート］手順と［Salesforce Input[CMDB_Class]（Salesforce入力[CMDB_Class]）］手順で、Salesforce組織の資格情報を確認します。
Did not find Remedy Application Service password for server <server name> in UDM:RAppPassword Form on server <server name>（サーバー<サーバー名>の［UDM:RAppPassword］フォームで、サーバー<サーバー名>のRemedyアプリケーションサービスのパスワードが見つからない）
［ARInput（AR入力）］手順で、Atriumサーバー接続の資格情報を確認します。
Failed in writeToSalesForce: java.lang.IllegalArgumentException: The char '0x0' after 'Print' is not a valid XML character（writeToSalesForceで失敗する： java.lang.IllegalArgumentException： 「Print」の後の文字「0x0」が有効なXML文字列でない）
XMLパーサーで解析できないUnicode文字が、マッピングされているBMC Atrium CMDBフィールドのいずれかに存在します。データからこれらの文字を削除するか、当該のBMC Atrium CMDBフィールドのマッピングを削除します。
‘Oracle Database Server 10g Release 2’ is not valid for the type xsd（「Oracle Database Server 10g Release 2」はxsdタイプでは有効ではありません）: <データ型>
［Salesforceのアップサート］手順でマッピングされたフィールドのデータ型を確認します。
