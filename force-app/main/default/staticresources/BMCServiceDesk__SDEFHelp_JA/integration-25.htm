ここでは、インシデントを作成するためのBMC Server Automationの設定手順を示します。

  1. BMC Remedyforceで電子メールリスナーを作成します。電子メールリスナーの作成についての詳細は、電子メールサービスの作成を参照してください。
  2. 定義済みのEmailConversationListener Apex クラスを電子メールサービスに追加します。

    3. 注意: テンプレートを使用しないでインシデントを作成する場合は、手順8を省略します。

  3. （オプション）新しいインシデントが電子メールから作成されている場合に、よく使用されるデータを［インシデント］フォームに入力するには、BMC Remedyforceでテンプレートを作成します。
  4. （オプション）新規作成されたテンプレートで、定義済みの［BSA通知］カテゴリを選択します。
  5. （オプション）［Remedyforce管理］>［アプリケーションの設定］>［アプリケーションの全般設定］ページで、［フォームのフィールド値をテンプレートフィールド値に置換］オプションを選択してテンプレートを使用します。
  6. （オプション）Visualforceページレイアウトユーザーインターフェイスを使用している場合は、［設定］>［アプリケーションの設定］>［一般］タブの順に選択し、［フォームフィールド値をテンプレートフィールド値と置換］チェックボックスをオンにします。
  7. （オプション）［Remedyforce管理］>［ワークフローと承認の設定］>［ワークフロー規則］の順に移動して、［BMC Server Automationで作成されたインシデントにテンプレートを適用］ワークフロー規則を複製して有効にします。

    8. デフォルトでは、ワークフローは非アクティブです。

  8. （オプション）Visualforceページレイアウトユーザーインターフェイスを使用している場合は、［設定］>［作成］>［ワークフローと承認］>［ワークフロー規則］の順に移動して、［BMC Server Automationで作成されたインシデントにテンプレートを適用］ワークフロー規則を複製して有効にします。
  9. （オプション）複製されたワークフローを次のように編集します。
    1. ［ワークフローアクション］セクションの［ワークフローのアクションの追加］ドロップダウンリストで、［新しいフィールドの更新］アクションタイプを選択します。
    2. ［更新するフィールド］ドロップダウンリストで、［テンプレート名］を選択します。
    3. ［新しいフィールド値の指定］セクションで、［式を使用して新しい値を設定する］を選択します。
    4. 新規作成したテンプレートの名前を、二重引用符で囲んで入力します。たとえば、テンプレート名がCreateBSAIncidentの場合は、「“CreateBSAIncident“」と入力します。
  10. BMC Server Automationコンソールでジョブを作成します。詳細は、BMC Server Automationコンソールでのジョブの設定を参照してください。
  11. BMC Server Automationコンソールでジョブを送信します。

設定の結果

BMC Server Automationで電子メールメッセージが生成され、BMC Remedyforceに送信されます。受信した電子メールメッセージに基づいて、BMC Remedyforceでインシデントが作成されます。電子メールの内容の本文がインシデントの［説明］フィールドにコピーされます。

注意: 受信した電子メールメッセージの電子メールアドレスがどのクライアントの電子メールアドレスとも一致しない場合は、［識別されていない送信者のクライアント］フィールドを選択する必要があります。［識別されていない送信者のクライアント］フィールドは、［Remedyforce管理］>［電子メールの設定］> ［インシデント - 電子メールの設定］ページの［電子メール送信者の識別］セクションにあります。Visualforceページレイアウトユーザーインターフェイスを使用している場合は、［設定］>［アプリケーションの設定］>［インシデント - 電子メールの設定］ページの順に選択します。ただし、監視システムのクライアントを作成する場合は、適切な電子メールアドレスを指定することをお勧めします。
