BMC Atrium CMDBから関係をインポートする前に、BBMC Atrium CMDBからデータをインポートする前にに記載されているすべてのアクションを実行し、BMC Atrium CMDBからすべてのCIをインポートします。

BMC Atrium CMDBから関係をインポートするには

  1. BMC Atrium Action Request Systemにログイオンして、Atrium Integrator Consoleにアクセスします。
  2. Atrium Integrator Spoonを起動して、をクリックします。
  3. KTRファイルとKJBファイルをAtrium Integratorのフォルダに保存するには、各ファイルに対して次の手順を実行します。
    1. ［ファイル］>［Open URL（URLを開く）］をクリックして、BMC Communitiesからダウンロードしたジョブファイルを保存したフォルダを参照します。
    2. AtriumtoCMDB_BaseRelationship.kjbファイルとAtriumtoCMDB_BaseRelationship.ktrファイルを開きます。
    3. ［ファイル］>［名前を付けて保存］をクリックします。
    4. ［Directory（ディレクトリ）］フィールドで、をクリックし、CIのKTRファイルとKJBファイルを保存したフォルダを選択します。
    5. ［OK］をクリックします。
  4. KJBファイルで、［AtriumtoCMDB_BaseRelationship］手順をダブルクリックします。
  5. 現在の変換ウィンドウのジョブエントリ詳細で、［Specify by name and directory（名前とディレクトリを指定）］オプションのをクリックし、KTRファイルを保存したフォルダからAtriumtoCMDB_BaseRelationship.ktrを選択します。
  6. ［ファイル］>［開く］をクリックし、AtriumtoCMDB_BaseRelationship.ktrファイルを開きます。
  7. KTRファイルからBMC Remedy AR Systemにアクセスするために必要な情報を入力するには、次のアクションを実行します。
    1. KTRファイルで、［ARInput（AR入力）］をダブルクリックします。
    2. ［接続］フィールドで［編集］をクリックします。
    3. ［設定］セクションで、BMC Remedy AR Systemのアクセス詳細情報を入力します。
    4. （オプション）サーバーとの接続をテストするには、［テスト］をクリックします。
    5. ［OK］をクリックします。
  8. BMC Remedy AR Systemが使用するSQL Serverへのアクセスに必要な情報を入力するには、次の手順を実行します。
    1. ［Table Source（テーブルソース）］をダブルクリックします。
    2. ［接続］フィールドで［編集］をクリックします。
    3. ［データベース接続］ウィンドウの［接続の種類］リストで、［MS SQL Server］を選択します。

      4. 別の種類のデータベースを使用している場合は、目的の接続の種類を選択し、［設定］セクションに詳細情報を入力します。

    4. ［設定］セクションで、SQLサーバーのアクセス詳細情報を入力します。
    5. （オプション）データベースとの接続をテストするには、［テスト］をクリックします。
    6. ［OK］をクリックします。
  9. Salesforce組織へのアクセスに必要な情報を入力するには、次のアクションを実行します。
    1. ［Salesforceのアップサート］をダブルクリックします。
    2. ［接続］セクションで、Salesforce組織のアクセス詳細を入力します。
    3. （オプション）接続をテストするには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。

      5. 注意: BMC Remedyforceで関係を作成できるようにするには、UniqueCIRelationshipID__c、UniqueDestinationCIID__c、UniqueSourceCIID__cの各フィールドのマッピングを変更しないでください。

  10. BMC Atrium CMDBで使用するデータベースにカスタム関係を作成した場合は、次のアクションを実行します。
    1. Salesforceで、［設定］>［作成］>［オブジェクト］の順に選択します。
    2. ［ラベル］カラムの［基本関係］をクリックします。
    3. ［カスタムフィールドおよび関係］セクションで、［タイプ］をクリックします。
    4. ［Base Relationship Custom Field Type（基本関係カスタムフィールドタイプ）］ページの［ピックリストの値］セクションで、［新規］をクリックします。
    5. BMC Atrium CMDBカスタム関係の名前を入力し、［保存］をクリックします。
    6. インポートするすべてのカスタム関係について、手順dと手順eを繰り返します。
    7. Atrium Integrator Spoonに移動します。
    8. AtriumtoCMDB_BaseRelationship.ktrファイルで、［Read Response Variable（応答変数の読み取り）］手順をダブルクリックします。
    9. 最後の「else if」ステートメントをコピーして貼り付けます。

      10. 例： else if(endsWith(ClassIdUpper,"BMC_SETTINGOF")){TypeScript="Setting Of";}

    10. 最初のクラス文字列をカスタム関係のクラスIDに、2番目のテキスト文字列を［タイプ］フィールドで追加した値に置き換えて、手順を保存します。

      11. たとえば、BMC Atrium CMDBからモバイルカスタム関係タイプをインポートし、［タイプ］フィールドにAtrium Mobile値を追加したとします。［Read Response Variable（応答変数の読み取り）］手順で、テキスト文字列を次のテキストに置き換えます。

      else if(endsWith(ClassIdUpper,"MOBILE")){TypeScript="Atrium Mobile";}

      最初のテキスト文字列は大文字にしてください。

      このステートメントで、ClassIdUpperはBMC Atrium CMDBのカスタム関係の名前で、TypeScriptは［タイプ］フィールドに入力した値です。

    11. インポートするすべてのカスタム関係について、手順iと手順jを繰り返します。
  11. KTRファイルとKJBファイルを保存するには、をクリックします。
  12. Atrium Integrator Consoleに戻ります。
  13. 設定したジョブを選択して、をクリックします。

データのインポート時に、次のログファイルが生成されます。

    • ジョブレベルのログ： Atrium Integrator Consoleで作成されます。
    • 成功ログ、処理ログ、エラーログ： BBMC Atrium CMDBからデータをインポートする前にで作成したLog FilesフォルダにCSVファイルとして作成されます。エラーログは、BMC Remedyforceでのレコードの挿入時にエラーが発生した場合にのみ作成されます。delta_valueログには、データを最後にインポートした日時のタイムスタンプが保存されます。固有識別子値が空白のレコードが存在する場合、拒否ログも作成されます。
