Chatterフィードは、Salesforce Chatterアプリケーションのユーザー用ホームページです。Chatterフィードには、ユーザーの写真とユーザーの最新のステータス更新が表示されます。ここではユーザーの更新をすべて1か所で確認できます。Chatterフィードでは、更新を投稿して、自分をフォローするユーザーとの間でその更新を共有できます。また、自分がフォローするレコードとユーザーについての投稿にコメントできるほか、自分をフォローするユーザーとリンクを共有できます。Chatterフィードを使用することで、自分がフォローする次の種類のレコードに対するすべての更新を追跡できるようになります。

    • インシデントレコード
    • ブロードキャストレコード
    • 変更リクエストレコード
    • 構成アイテムレコード

これらの更新は次のフォーマットで表示されます。

<オブジェクト名> <レコード番号> <ユーザー名> <レコードに対して実行されたアクション>

注意: <オブジェクト名>では、更新で記述されているレコードの種類を確認できます。

インシデント、変更リクエスト、問題のいずれかに構成アイテムがリンクしている場合は、Chatterフィードではこの更新が次のフォーマットで表示されます。

<ユーザー名>は、この構成アイテムの<オブジェクト名> <レコード番号>にリンクしています。

デフォルトでは、レコードを作成または所有すると、そのレコードが自動的にフォローの対象となります。インシデント、ブロードキャスト、変更リクエストのいずれかが閉じられた場合は、Chatterフィードでそのフォローを解除できます。

また、Chatterフィードを使用して、次のようにチームメンバーとコラボレーションすることが可能です。

    • インシデントの解決策に追加するコメントを使用していて、そのインシデントを閉じた場合は、そのコメントをChatterフィードで検索し、類似のインシデントに割り当てられているチームメンバーとの間でそのコメントを共有できる。
    • Chatterグループのすべてのメンバーにとって有用なリンクを見つけた場合は、Chatterフィードでそのリンクをメンバーと共有できる。
    • チームで完了する必要があるタスクレコードに対するコメントを使用することにより、Chatterフィードでディスカッションを開始できる。
    • 重要な問題に取り組んでいる場合は、その問題の進捗に関する定期的な更新をChatterフィードでチームと共有できる。進捗に関する更新が発生するたびに電子メールアプリケーションを開いて電子メールを送信する必要はありません。
    • インシデントに関する助言を早急に必要としているチームメンバーがそれぞれのChatterフィードにクエリーを投稿すると、そのクエリーが他のチームメンバーのChatterフィードに表示される。そのクエリーを見たチームメンバーは、それぞれの解決策をクエリーに対するコメントとして追加し、チームと共有できます。別のチームメンバーは、クエリーに対する別の解決策を提供できます。これにより、解決策のリポジトリをチームで使用できるようになります。

Chatterフィードを使用するには

  1. BMC Remedyforceホームページの［ワークスペース］をクリックします。
  2. ［ワークスペース］のリストから［Chatter］を展開します。
  3. ［Chatter］ドロップダウンリストの［Chatterフィード］をクリックします。
  4. 自分のフォロワーと共有する投稿を作成するには、次のアクションを実行します。
    1. Chatterフィードに追加する投稿を入力します。
    2. ［共有］をクリックします。

      3. 自分のChatterフィードと自分のフォロワーのChatterフィードに、入力した投稿が表示されます。

  5. 自分がフォローしているレコードに対する更新にコメントするには、次のアクションを実行します。
    1. コメントする更新の下にあるコメントリンクをクリックします。
    2. コメントを入力します。
    3. ［コメントの追加］リンクをクリックします。

      4. Chatterフィードに表示された更新の下に、入力したコメントが表示されます。

  6. Chatterフィードでレコードのフォローを中止し、自分のChatterフィードでそのレコードの更新を表示しないようにするには、［フォローを解除］リンクをクリックします。

    7. そのレコードの更新がChatterフィードから削除されます。

  7. 自分のChatterフィードにリンクを追加するには、次のアクションを実行します。
    1. Chatterフィードの画面上部にある［リンク］をクリックします。
    2. ［リンクを添付］セクションで、［URLのリンク］フィールドに目的のリンクのURLを入力します。
    3. ［リンク名］フィールドに、リンクの名前を入力します。
    4. ［共有］をクリックします。

      5. ［名前のリンク］フィールドにリンクの名前を入力せず、［URLのリンク］フィールドにURLだけを入力した場合は、ChatterフィードにリンクのURLが表示されます。たとえば、［URLのリンク］フィールドに「bmc.com」と入力すると、Chatterフィードには［bmc.com］と表示されます。一方、［名前のリンク］フィールドにリンクの名前を入力し、［URLのリンク］フィールドにURLを入力した場合は、Chatterフィードにリンクの名前が表示されます。たとえば、［名前のリンク］フィールドに「BMC Website」と入力し、［URLのリンク］フィールドに「bmc.com」と入力すると、Chatterフィードには［BMC Website］と表示されます。

自動的にフォローの対象となるレコード

デフォルトでは、次の条件を満たすレコードは自動的にフォローの対象となります。

    • 自分でレコードを作成した場合、またはレコードを所有している場合。
    • システム管理者がそのレコードのオブジェクトに対してフィード追跡を有効にしている場合。
    • システム管理者がレコードのオブジェクト上の［所有者］フィールドでフィード追跡を有効にしている場合に、別のユーザーが自分にレコードを割り当てたとき。

レコードが自動的にフォローの対象とならないようにするには、［設定］>［私のChatter設定］>［私のフィード］の順に移動し、［レコードの自動登録を停止］チェックボックスをオンにします。

注意: Salesforce組織の設定で、改善された［設定］ユーザーインターフェイスを有効化した場合は、［私の設定］>［Chatter］>［私のフィード］に移動します。改善された［設定］ユーザーインターフェイスについての詳細は、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。

［レコードの自動登録を停止］チェックボックスをオンにしていても、インシデントのステータスを［承認済み］の状態に変更すると、インシデントレコードは自動的にフォローされます。
