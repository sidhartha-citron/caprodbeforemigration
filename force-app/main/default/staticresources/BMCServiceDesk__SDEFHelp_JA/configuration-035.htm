リストビューに表示されるレコードは、［ビュー］メニュー、［分類基準］ドロップダウンリスト、特定のレコードの検索のいずれかを使用して、フィルタリングすることができます。［ビュー］メニューは、ログオンしているユーザーに割り当てられているレコードや未完了のレコード、完了済みのレコード、アクティブなレコード、インアクティブなレコードだけを表示する場合に使用します。［分類基準］ドロップダウンリストは、このリストで選択したフィールドに基づいてレコードをグループ化します。必要なレコードのキーワードがわかっている場合は、そのキーワードを入力して検索を実行すると、それに一致するレコードだけが表示されます。リストビューでは、最後のレコードを開いているときに［次のレコード］をクリックすると、リストビューの最初のレコードが表示されます。最初のレコードを開いているときに［前のレコード］をクリックすると、リストビューの最後のレコードが表示されます。

注意: カラム見出しをダブルクリックすると、そのカムの並び順が昇順または降順に切り替わります。

リストビューでレコードをフィルタリングするには

  1. リストビューで次のアクションを実行し、［ビュー］メニューを使用してレコードをフィルタリングします。
    1. ［ビュー］をクリックします。
    2. ［ビュー］メニューから、目的のコマンドを選択します。次の表に、［ビュー］メニューの各コマンドの説明を示します。

      3. メニューのコマンド                   
      説明
      自分に割り当て
      ログオンしているユーザーに割り当てられているすべてのレコードを表示します。
      アイテムの開始
      ［未完了］ステータスのレコードをすべて表示します。
      終了済みのアイテム
      ［完了］ステータスのレコードをすべて表示します。
      有効アイテム
      アクティブなアイテムをすべて表示します。
      無効アイテム
      インアクティブなアイテムをすべて表示します。
      リフレッシュ（分）
      リストビューをリフレッシュする間隔を入力します。
      デフォルトの間隔は10分で、最小間隔は1分です。必要に応じて、この値を変更することができます。アクションを実行するとリストビューがリフレッシュされるため、リフレッシュする間隔を短くするとアクションにインパクトを与えます。リフレッシュする間隔は、最小間隔よりも大きくすることをお勧めします。
      新たに設定したリフレッシュ間隔は［適用］をクリックすると適用され、その値はブラウザのcookieに保存されます。ログオンすると、リストビューは新しいリフレッシュ間隔でリフレッシュされます。スタッフメンバーがブラウザのcookieを削除すると、リストビューのリフレッシュ間隔は10分にリセットされます。リストビューの自動リフレッシュは、リフレッシュ間隔として「0」を入力すると無効にできます。

      メモ:
       ［前回のリフレッシュ<時刻>］ラベルには、リストビューが前回リフレッシュされた時刻が表示されます。

    3. ［適用］をクリックします。

      4. ［ビュー］メニューで選択したコマンドに応じてリストビューがリフレッシュされ、一致するレコードが表示されます。

  2. ［分類基準］ドロップダウンリストから、フィールドを選択します。

    3. リストビューがリフレッシュされ、［分類基準］ドロップダウンリストで選択したフィールドに基づいて、一致するレコードのグループが作成されます。

  3. 特定のレコードを検索してレコードをフィルタリングするには、次のアクションを実行します。
    1. ［検索］フィールドに検索文字列を入力します。
    2. ［検索］をクリックします。

      3. 検索結果として、検索文字列に一致するレコードが表示されます。

    3. 検索結果をクリアするには、［元に戻す］をクリックします。
