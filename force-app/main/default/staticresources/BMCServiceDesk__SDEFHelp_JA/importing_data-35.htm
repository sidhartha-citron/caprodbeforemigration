LDAPサーバーからデータをインポートする前に、LDAPサーバーからデータをインポートする前にセクションに記載されているアクションをすべて実行してください。

Salesforceプラットフォームのライセンスを持つユーザーをLDAPサーバーからPentahoを使用してインポートするには

  1. Pentaho Data Integrationツールで、データをインポートするためのジョブファイルを作成します。

    2. サンプルジョブファイルが、BMC Communities BMC Remedyforceページに用意されています。BMC Communitiesページからサンプルジョブファイルをダウンロードするには、 https://communities.bmc.com/docs/DOC-22547をクリックします。

    ジョブファイルは、Pentaho Data Integrationツールを解凍したフォルダにダウンロードすることをお勧めします。

  2. Pentaho Data Integrationツールを起動するには、\pdi-ce-4.x.0-stable\data-integrationまたは\pdi-ce-5.0.1.A-stable\data-integrationフォルダでSpoon.batファイルをダブルクリックして目的のKJBファイルを開きます。
  3. KTRファイルを開くには、［Transfer LookUpExcelExport（LookUpExcelExportの転送）］手順を右クリックし、［データ変換を開く］を選択します。
  4. Salesforce組織にアクセスするための詳細情報を入力するには、次のアクションを実行します。
    1. ［Transfer LookUpExcelExport（LookUpExcelExportの転送）］タブで、［Salesforce Input [For Profile]（Salesforceの入力［プロファイル］）］手順をダブルクリックします。
    2. 該当するフィールドにユーザー名、パスワードの詳細を入力します。
    3. （オプション）接続を確認するには、［接続テスト］をクリックします。
    4. ［Salesforce Input [From Account]（Salesforceの入力［アカウント］）］、［Salesforce Input [From UserRole]（Salesforceの入力［ユーザーの役割］）］、［Salesforce Input [From User Account Link]（Salesforceの入力［ユーザーアカウントリンク］）］手順のために、手順a～手順cを繰り返します。
  5. LDAPサーバーにアクセスするための詳細情報を入力するには、次のアクションを実行します。
    1. KJBファイルで［クライアントユーザーインポートリストをLDAPユーザー情報で更新］手順を右クリックし、［データ変換を開く］を選択します。
    2. ［LDAP入力］手順をダブルクリックします。
    3. ［LDAP入力］ウィンドウで、ホスト、ユーザー名、パスワードの詳細をそれぞれ所定のフィールドに入力します。
    4. （オプション）接続を確認するには、［接続テスト］をクリックします。
    5. （オプション）LDAPサーバーからさらにフィールドを取得するには、［フィールド］タブで［フィールドの取得］をクリックします。
    6. （オプション）［動的アカウントおよびプロファイルの割り当て］手順では、アカウント、プロファイル、役割のデフォルト値、およびスクリプト内の特定の条件のカスタム値を変更したり、割り当てたりできます。また、［ロケール］、［言語］、［タイムゾーン］、［EmailEncoding］フィールドのデフォルト値を変更することもできます。［動的アカウントおよびプロファイルの割り当て］手順をダブルクリックすると、［スクリプト値／変更］ウィンドウが表示されます。［スクリプト値／変更］ウィンドウに表示されるコメント付きの例は参照可能です。
    7. また、このKTRファイルではSalesforce組織のアクセス詳細情報を指定する必要があります。［Salesforce Upsert [For User]（Salesforceのアップサート［ユーザー］）］手順をダブルクリックし、Salesforce組織のユーザー名とパスワードを入力します。

      8. 注意: Pentaho 4.1や4.2などの以前のバージョンのPentaho Data Integrationツールを使用している場合は、Salesforce WebサービスのURLをhttps://www.salesforce.com/services/Soap/u/20.0に変更します。

    8. （オプション）LDAPフィールドとSalesforceクライアントユーザーインポートオブジェクト間のマッピングは事前に定義されています。マッピングを更新するには、［マッピングの編集］をクリックします。マッピングについての詳細は、CMDB 1.0で構成アイテムをインポートする方法の概要を参照してください。
    9. （オプション）接続を確認するには、［接続テスト］をクリックします。
  6. また、KJBファイルで指定された手順でSalesforce組織にアクセスするための詳細情報を指定する必要があります。KJBファイルでSalesforce組織のアクセス詳細情報を指定するには、次のアクションを実行します。
    1. ［Salesforce Insert [For User Account Link]（Salesforceの挿入［ユーザーアカウントリンク］）］手順をダブルクリックします。
    2. 該当するフィールドにユーザー名、パスワードの詳細を入力します。
    3. （オプション）接続を確認するには、［接続テスト］をクリックします。
    4. ［OK］をクリックします。
    5. ［Salesforce Update（Salesforceの更新）］手順をダブルクリックします。
    6. 該当するフィールドにユーザー名、パスワードの詳細を入力します。
    7. （オプション）接続を確認するには、［接続テスト］をクリックします。
    8. ［OK］をクリックします。
  7. （オプション）Salesforce組織にアクセスできるIPアドレスを限定するように設定されている場合、SalesforceとPentaho Data Integrationツールの間での通信を有効にするには、次のアクションを実行します。
    1. Salesforceで、［設定］>［自分のセキュリティトークンのリセット］の順に選択します。

      2. 注意: Salesforce組織で、改善された［設定］ユーザーインターフェイスが有効になっている場合は、［My Settings（私の設定）］>［Personal（個人）］>［自分のセキュリティトークンのリセット］に移動します。改善された［設定］ユーザーインターフェイスについての詳細は、 http://help.salesforce.com/apex/HTViewHelpDoc?id=admin_setup_improved.htm&language=en_USを参照してください。［セキュリティトークンのリセット］をクリックすると、Salesforceに保存されている電子メールアドレスに電子メールが送信されます。

    2. ［クラス<CIタイプ名>にアップサート］手順の［パスワード］フィールドで、パスワードの後ろにセキュリティトークンを追加します。

      3. たとえば、パスワードがmypasswordで、セキュリティトークンがXXXXXXXXXであれば、［パスワード］フィールドに「mypasswordXXXXXXXXX」と入力する必要があります。

  8. KTRファイルとKJBファイルを保存するには、をクリックします。
  9. KJBファイルで、をクリックします。
  10. ［ジョブの実行］ウィンドウで、［起動］をクリックします。

    11. 完了した変換は、で示されます。は現在実行中の変換を示し、は失敗した変換を示します。

  11. （オプション）ログを表示するには、［実行の結果］セクションで［ログ］タブをクリックします。

    12. すべてのエラーは赤で表示されます。

  12. （オプション）ジョブのインポートをスケジュールするには、インポートしたジョブのスケジュール作成を参照してください。
