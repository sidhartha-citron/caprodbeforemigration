正確性と完全性を期するため、BMCではテンプレートを使用してレコードを作成することをお勧めしています。テンプレートは、よく使用するワークフローの設定や、情報を取得する方法の一貫性の維持、生産性・効率性の向上、エラーの低減に役立ちます。テンプレートを使用することで、フォーム上のフィールドにあらかじめ値を入力しておくことができるため、よく使用するデータの入力やレコードの作成が不要になり、自動化されたワークフロープロセスが提供されます。詳細は、テンプレートの適用、タスクテンプレートを作成して運用規則を実装する、インシデントテンプレートを作成して運用規則を実装する、リクエストの詳細テンプレートを作成して運用規則を実装するを参照してください。

注意: スタッフメンバーの要件に応じて、頻繁に発生するインシデント、タスク、変更リクエスト、サービスリクエスト、リリースの各テンプレートを作成できます。

テンプレートを作成するには

  1. BMC Remedyforceホームページの［設定］をクリックします。
  2. ［設定］オプションのリストから［設定］を展開します。
  3. ［設定］ドロップダウンリストで、［テンプレート］をクリックします。
  4. ［テンプレート］リストビューで、［新規］をクリックします。
  5. ［テンプレートの用途］ドロップダウンリストから、作成しているテンプレートのタイプを選択します。

    6. サービスリクエストテンプレートを作成するには、［テンプレートの用途］ドロップダウンリストから［リクエストの詳細］を選択します。［リクエストの詳細］を選択すると、［インシデントテンプレート］ドロップダウンリストが表示されます。このリストをクリックすると［テンプレートから選択］ウィンドウが開きます。このウィンドウでは、サービスリクエストの作成に使用するインシデントフィールドが含まれるインシデントテンプレートを選択する必要があります。リンクされたタスクテンプレートが設定されていないインシデントテンプレートを選択することをお勧めします。リンクされたタスクテンプレートを持つインシデントテンプレートを選択すると、登録されたサービスリクエストが承認されていない場合でも、リンクされたタスクテンプレートからタスクレコードが作成されます。

    サービスリクエストテンプレートは、リクエストの詳細テンプレートとインシデントテンプレートの組み合わせです。リクエストの詳細を選択すると、［選択フィールド］ドロップダウンリストで［数量］サービスリクエストフィールドが入力されます。デフォルトでは、［数量］フィールドにデフォルト値の1が入力された状態でテンプレートに追加されます。デフォルトでは、このフィールドにデフォルト値の1が入力された状態でテンプレートに追加されます。このフィールドに数値を割り当ててサービスリクエストテンプレートに追加することで、必要なサービスリクエストのデフォルト数を割り当てることができます。たとえば、［数量］フィールドに5を割り当てます。このサービスリクエストテンプレートからサービスリクエストを登録すると、デフォルトでは、必要なサービスリクエストの数量が5になります。

  6. ［名前］フィールドで、現在のテンプレートに割り当てる固有の英数字識別子を入力します。
  7. Webサービスまたはワークフローを使用して、またはデータをインポートして作成されたレコードにテンプレートを適用する場合は、［システムテンプレート］チェックボックスをオンにします。

    8. ［テンプレートの用途］フィールドで［リクエストの詳細］を選択すると、リクエストの詳細にシステムテンプレートを選択できないため、［システムテンプレート］チェックボックスはオフになります。

    システムテンプレートは、Webサービスまたはワークフローを使用して、またはデータをインポートして作成されたレコードで、テンプレートを適用するのに使用します。このテンプレートの名前は、ワークフローやWebサービスまたはデータのインポート中に、［テンプレート名］フィールドの値として使用する必要があります。

    システムテンプレートは、BMC Remedyforce Salesforceページレイアウトの［テンプレート］ルックアップに表示されません。ただし、VisualforceページでBMC Remedyforceを使用している場合は、すべてのテンプレートが表示されます。

  8. ［説明］フィールドで、そのテンプレートの説明を入力します。

    9. 説明には、そのテンプレートを作成した理由を記述してください。たとえば、新入社員用のインシデントテンプレートに関連付けられているタスクを作成するような、新規採用プロセスを定義するテンプレートを作成できます。

    Salesforceページレイアウトおよびコンソールレイアウトで提案されたテンプレートを検索するために、［キーワード］テキストボックスが追加されます。Visualforceページレイアウトからアクセスしている場合、このテキストボックス内のテキストはテンプレートの動作に影響しません。

  9. ［選択フィールド］ドロップダウンリストから、そのテンプレートに追加する必要のあるフィールドを選択します。

    10. BMC Remedyforceでは、次のデータタイプのフィールドの追加だけがサポートされます。

    • ルックアップ関係
    • マスター-詳細関係
    • チェックボックス
    • 通貨
    • 電子メール
    • 番号
    • パーセント
    • 電話番号
    • ピックリスト
    • テキスト
    • テキスト領域
    • テキスト領域（ロング）
    • URL

      • ピックリスト（複数選択）フィールドをテンプレートに追加して、テンプレートを使用してBMC Remedyforceタブ形式ビューでレコードを作成する場合、テンプレートのピックリスト（複数選択）フィールドで設定した1つの値がレコードに入力されます。

      テンプレートに追加したフィールドに応じて、［フィールド値］フィールドには、該当する入力値のオプションが動的に表示されます。たとえば、［優先度］フィールドを追加した場合、［フィールド値］フィールドには、選択可能な優先度のリストが表示されます。選択したい値がない場合は、必要な値を入力できます。入力した値はテンプレートに追加される前に検証されます。［説明］フィールドを追加すると、［フィールド値］フィールドに、説明を入力できるテキストボックスが表示されます。

      注意: ［テンプレートの用途］フィールドで［変更リクエスト］、［インシデント］、［タスク］のいずれかを選択すると、それに対応する変更リクエスト、インシデント、タスクのいずれかのフィールドが［フィールドの選択］ドロップダウンリストに表示されます。

      ［変更リクエスト］、［インシデント］、［タスク］のいずれかのカスタムオブジェクトに対して、ルックアップ関係データタイプのカスタムフィールドを作成した場合は、これらのカスタムフィールドが［フィールドの選択］ドロップダウンリストに表示されません。

  10. ［フィールド値］フィールドで、選択したフィールドの目的の値を入力します。

    11. 注意: テンプレートにルックアップフィールドを追加して、ルックアップフィールドの値を選択し、テンプレートの更新後にテンプレートを閉じて、元のフォームで選択された値を更新した場合、テンプレートを開いても、テンプレートのルックアップフィールドには、更新された値が表示されません。たとえば、［緊急度］ルックアップフィールドをテンプレートに追加して、緊急度の値として［高］を選択し、テンプレートを更新してテンプレートを閉じます。［緊急度］フォームで、［高］を［最高］に更新します。この時点で、テンプレートを開くと、テンプレートの［緊急度］ルックアップフィールドには、［最高］ではなく［高］が表示されます。ルックアップフィールドには、［インパクト］、［オーナーのID］、［緊急度］などを使用できます。ルックアップ値を更新する場合、必ず新しいルックアップ値でテンプレートを更新することをお勧めします。

  11. テンプレートにフィールド名とその値を追加するには、［追加］をクリックします。

    12. モバイルデバイスからセルフサービスにアクセスし、インシデントの上位のリクエストを使用してインシデントを作成する場合は、次のフィールドの値についてインシデントの上位のリクエストの値が入力された状態で、インシデントレコードが作成されます。

    • カテゴリ
    • インシデントの説明

      • ブラウザからセルフサービスにアクセスし、インシデントの上位のリクエストを使用してインシデントを作成する場合は、次のフィールドの値についてインシデントの上位のリクエストの値が入力された状態で、インシデントレコードが作成されます。

    • カテゴリ
    • インシデントの説明
    • インシデントの解決
    • インパクト
    • 緊急度
    • ブロードキャスト
    • 構成アイテム
    • 最初の問合せ時に解決

      • これらのフィールドに入力された値は、BMC Remedyforceでインシデントレコードを開いた場合のみ表示できます。セルフサービスでは表示できません。

      注意: インシデントの上位のリクエストに他のフィールドが含まれている場合は、それらのフィールドの値がインシデントレコードに入力されません。

  12. 手順9～手順11を繰り返して、テンプレートに必要なすべてのフィールドを追加します。
  13. テンプレートのフィールドとその値を変更するには、次のアクションを実行します。
    • フィールドの値を更新するには、表からフィールドを選択し、［フィールド値］フィールドでフィールドの値を編集して［更新］をクリックします。
    • テンプレートからフィールドを削除するには、表から目的のフィールドを選択して［削除］をクリックします。
  14. VisualforceフォームでBMC Remedyforceを使用している場合は、［For Use By（使用者）］セクションで次のアクションを実行します。

    15. 注意: この設定は、Visualforceフォームだけでサポートされています。Salesforceページレイアウトを使用している場合、オブジェクトの全テンプレートがテンプレートのルックアップに表示されます。Salesforceページレイアウトでは、表示されるテンプレートをログオンユーザーのプロファイルに割り当てることはできません。

    1. ［使用可能なプロファイル］リストで、テンプレートの使用を許可するプロファイルを選択します。

      2. 注意: すべてのプロファイルに対してテンプレートの使用を許可する場合は、［全ユーザー］チェックボックスをオンにします。

    2. 選択したプロファイルを［使用可能なプロファイル］リストから［選択したプロファイル］リストに移動するには、右矢印をクリックします。
    3. （オプション）選択したプロファイルを［選択したプロファイル］リストから［使用可能なプロファイル］リストに移動し、選択したプロファイルによってテンプレートが使用されないようにするには、左矢印をクリックします。

      4. 注意: インシデントにSalesforceページレイアウトを使用している場合、1つのテンプレートを使用してインシデントを作成しようとすると、インシデントのすべてのテンプレートがテンプレートのルックアップに表示されます。表示されるテンプレートをログオンユーザーのプロファイルに割り当てることはできません。

  15. インシデントまたは変更リクエストのテンプレートを作成する場合は、インシデントまたは変更リクエストを終了するために完了しなければならないタスクを、インシデントまたは変更リクエストのテンプレートに追加することができます。空のテンプレートを作成し、そのテンプレートをインシデントテンプレートにリンクしている場合は、そのインシデントテンプレートを使用したときにタスクレコードに入力されるのは、インシデントテンプレートから取得されたクライアントだけです。カテゴリやステータスなど、インシデントテンプレートのその他の値は、タスクレコードに入力されません。タスクテンプレートに値を入力している場合は、それらの値がタスクレコードに入力されます。インシデントまたは変更リクエストテンプレートにタスクを追加するには、次のアクションを実行します。
    1. ［テンプレート］フォームの下部にある［リンク済みテンプレート］セクションで、［アクション］>［追加］の順にクリックします。

      2. テンプレートのタイプに応じたタスクが［テンプレートの用途］フィールドに入力された状態で、新しい［テンプレート］フォームが開きます。このフィールドは無効になっており、変更することはできません。

    2. 手順6～手順14を繰り返して、タスクテンプレートを作成します。
    3. ［保存］をクリックします。

      4. タスクの［テンプレート］フォームが閉じ、［テンプレート］フォームの下部にある［リンク済みテンプレート］セクションに、タスクテンプレートが表示されます。

    4. テンプレートに追加する各タスクについて、手順a～手順cを繰り返します。
    5. タスクを完了する順序を設定するには、［タスクの実行順序の設定］をクリックします。

      6. 注意: すべてのリンク済みタスクを同時に実行するには、［実行順序を無視］チェックボックスをオンにします。デフォルトでは、このチェックボックスはオンになっていて、BMC Remedyforceですべてのタスクが同時に実行されます。このチェックボックスをオフにすると、タスクを完了する順序を設定することができます。

      同じ実行順序番号を複数のタスクに割り当てることができます。たとえば、Task1、Task2、Task3、Task4という4つのタスクがあるとします。この場合、Task1およびTask2の各タスクを最初に実行し、その後でTask3、Task4の順に実行するように割り当てることができます。Task1およびTask2の実行時は、これらのタスクがレコードの［補足情報］に表示されます。Task1およびTask2が完了するとTask3が表示され、Task3の完了後にTask4が表示されます。すべてのタスクが完了する前にインシデントまたは変更リクエストが終了すると、スケジュールが設定されていないタスクは作成されません。

      テンプレートは、既存のインシデント、変更リクエスト、タスク、リリースのいずれかに適用できます。既存のレコードにテンプレートを適用すると、既存のレコードに関連付けられたタスクが保存され、適用したテンプレートに関連付けられたタスクがレコードに追加されます。

    6. ［タスクの実行順序の設定］ウィンドウの［順序番号］カラムで、タスクに割り当てる順序番号を選択します。

      7. 同じ実行順序番号を複数のタスクに割り当てることができます。

    7. （オプション）実行順序を無視するには、［実行順序を無視］チェックボックスをオンにします。
    8. ［保存］をクリックします。

      9. 注意: 既存のタスクテンプレートを作成済みの場合は、［アクション］>［テンプレートのリンク］の順にクリックして目的のタスクテンプレートを選択することで、そのタスクテンプレートを追加することができます。

  16. ［テンプレート表示順序］をクリックします。

    17. スタッフメンバーが、［インシデント］または［タスク］フォームで［テンプレート］をクリックすると［テンプレートから選択］ウィンドウが開き、使用可能なテンプレートが設定した順序で表示されます。最も頻繁に使用されるテンプレートが上位に表示されるように設定することができます。

  17. ［テンプレート表示順序］ウィンドウの［テンプレートの用途］ドロップダウンリストから、表示順序を設定するテンプレートのタイプを選択します。
  18. ［テンプレートから選択］ウィンドウで最初に表示するテンプレートを選択し、そのテンプレートが一番上になるまで上矢印をクリックし続けます。
  19. ［テンプレートから選択］ウィンドウで2番目に表示するテンプレートを選択し、そのテンプレートが2番目の位置になるまで上矢印をクリックし続けます。
  20. 同様の方法で、［テンプレートから選択］ウィンドウでの表示順序になるように、その他のテンプレートを配置します。
  21. ［保存］をクリックします。

    22. 注意: 新規フォームを開いてレコードを作成した後、レコードを保存した場合は、フォームの右上セクションにある［次のレコード］および［前のレコード］ボタンを使用して、次または前のレコードに移動することはできません。これらのボタンは新規フォームで使用できません。

  22. （オプション）現在のレコードをインアクティブとして指定するには、［インアクティブ］チェックボックスをオンにします。

    23. たとえば、古いオペレーティングシステムの問題のトラブルシューティングに使用するインシデントテンプレートがあり、アカウント内で古いオペレーティングシステムがすでに使用されていない場合は、今後そのテンプレートが必要になることはありません。

    注意: インアクティブとしてマークされたレコードは、データベースで実行される検索の対象に含まれません。
