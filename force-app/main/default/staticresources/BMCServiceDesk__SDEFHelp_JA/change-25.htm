承認のための変更リクエストを送信するように承認プロセスが設定されている場合は、承認のための変更リクエストを送信します。

承認のために変更リクエストを送信するには

  1. 承認のために送信する変更リクエストに移動します。

    2. 詳細は、変更リクエストレコードへの移動を参照してください。

  2. 次のいずれかのアクションを実行します。
    • ［Remedyforceコンソール］タブを使用している場合は、［関連リスト］をクリックし、［承認履歴］セクションで［承認のために送信］をクリックします。
    • ［変更リクエスト］タブ（Salesforceページレイアウト）を使用している場合は、［承認履歴］セクションの［承認のために送信］をクリックします。
  3. 確認ダイアログボックスで、［OK］をクリックします。
  4. VisualforceページレイアウトUI を使用している場合は、次のアクションを実行します。
    1. ［補足情報］セクションで、［変更承認］タブをクリックします。
    2. メニューから［承認のために送信］を選択します。
    3. ［コメント］テキストボックスで、変更承認に含めるコメントを入力します。

      4. ［コメント］カラムにある吹き出しアイコンの上にマウスを置くと、コメントが表示されます。

    4. ［OK］をクリックします。
    5. （オプション）変更リクエストの承認リクエストを取り消すには、メニューから［承認リクエストの取り消し］をクリックします。

      6. 変更リクエストの承認リクエストを取り消すことができるのは、自分自身またはシステム管理者のみです。

      変更リクエストの承認または却下の全体的ステータスが、［補足情報］セクションの右上セクションにある［全体のステータス］フィールドに表示されます。
