ここでは、問題を作成する手順を説明します。

VisualforceページレイアウトUI で問題を作成するには

  1. BMC Remedyforceホームページの［ワークスペース］をクリックします。
  2. ［ワークスペース］のリストから［問題管理］を展開します。
  3. ［変更管理］ドロップダウンリストで、［問題］をクリックします。
  4. ［問題］リストビューで、［新規］をクリックします。
  5. ［問題の発生源］ドロップダウンリストから、問題の発生源を選択します。
  6. ［カテゴリ］ドロップダウンリストをクリックします。
  7. ［カテゴリから選択］ウィンドウで、次のアクションを実行します。
    1. 問題のカテゴリを選択します。

      2. カテゴリを使用すると、標準の分類方法で問題を分類できます。分類によって報告を目的とした追跡が可能になります。

    2. 選択したカテゴリの標準説明を使用するには、［標準説明］チェックボックスをオンにします。

      3. 選択したカテゴリの標準説明が、［問題］フォームの［説明］フィールドに入力されます。［説明］フィールドに手動で入力した説明は、この標準説明によって置き換えられます。

    3. 選択したカテゴリの標準解決を使用するには、［標準解決］チェックボックスをオンにします。

      4. 選択したカテゴリの標準解決が、［問題］フォームの［解決］フィールドの末尾に追加されます。［解決］フィールドに手動で入力した解決は、この標準解決によって上書きされます。

    4. ［OK］をクリックします。
  8. 問題が既知エラーの場合は、次のアクションを実行します。
    1. ［既知エラー］チェックボックスをオンにします。

      2. 注意: このチェックボックスをオンにすると［回避策］ボタンが使用可能になり、既知エラーの回避策を入力できます。

    2. ［回避策］をクリックします。
    3. 既知エラーの回避策を入力します。
  9. ［説明］フィールドに、解決する問題の説明を入力します。

    10. レコードを保存すると、［補足情報］セクションの［ドキュメンテーション］タブを使用して、解決する問題の詳細情報を提供するファイルやドキュメントを添付できます。

  10. ［解決］をクリックします。
  11. ［解決］フィールドに、その問題に対して実施する必要がある解決を入力します。
  12. ［メモ］フィールドに、問題に関するメモを入力します。

    13. スタッフメンバーは問題に関する更新情報を［メモ］フィールドに入力し、［追加］アイコンをクリックできます。これらの更新情報は、［補足情報］セクションに［履歴］タブの履歴レコードとして追加されます。スタッフメンバーは［メモ］フィールドを使用することで、［補足情報］セクションに移動して履歴レコードを作成しなくても、この問題に関するメモを追加できます。

    注意: ［メモ］フィールドと［所要時間］フィールドは、レコードを保存した後でのみ使用可能になります。

  13. ［所要時間］フィールドに、問題の処理にかかった時間を入力します。
  14. ［追加］をクリックします。

    15. ［補足情報］セクションが表示されます。［メモ］フィールドと［所要時間］フィールドの内容が［補足情報］セクションに［履歴］タブの履歴レコードとして追加され、［補足情報］セクションの履歴レコードが表示されます。

  15. ［詳細］セクションで、次の情報を入力します。
    1. ステータス： 現在の問題に割り当てるステータスを選択します。

      2. このフィールドを使用して、問題の進捗状況を開始ステージから終了ステージまで追跡できます。

      選択したステータスにリンクされたステージに応じて、［問題］フォームの見出しにある問題進捗状況バーが更新されます。たとえば、「作業中」ステータスが「処理中」ステージにリンクされている場合に、問題のステータスとして「作業中」を選択すると、問題進捗状況バーで［処理中］ステージが選択されます。

    2. インパクト： 現在のタスクに割り当てるインパクトを選択します。

      3. インパクトレベルと緊急度は、問題の優先度の計算に使用されます。

    3. 緊急度： 現在の問題に割り当てる緊急度を選択します。

      4. 緊急度とインパクトレベルは、問題の優先度の計算に使用されます。

    4. 優先度： 緊急度とインパクトを選択してレコードを保存すると、［優先度］フィールドに値が入力されます。優先度は、選択した緊急度とインパクトの値によって計算されます。
    5. 開始： このフィールドには、問題が作成された日時が入力されます。
    6. 期限日： 緊急度とインパクトを選択してレコードを保存すると、［期限日］フィールドに値が入力されます。この期限日は優先度の期間に基づいて計算されます。

      7. 必要に応じて、計算された期限日とは異なる日時を選択できます。

      問題の編集中にインパクトまたは緊急度を変更した結果、問題の優先度が変更された場合は、次の［期限日の計算］オプションが表示されます。

      • 再計算しない： 問題の既存の期限日が保持されます。
      • 開始日に基づいて再計算： 開始日を使用して期限日が再計算されます。たとえば、優先度が1に変更され、優先度1に割り当てられた期間が2時間の場合、再計算された期限日は、問題の作成日時に2時間を加えた値になります。
      • 現在の日付に基づいて再計算： 現在の日付を使用して期限日が再計算されます。たとえば、優先度が1に変更され、優先度1に割り当てられた期間が2時間の場合は、再計算された期限日が、現在の日時に2時間を加えた値になります。

        • 注意: システム管理者が［アプリケーション設定］で［優先度を変更すると、ユーザー入力のない現在の日付に基づいて、期限日時が再計算されます］チェックボックスをオンにしている場合、［期限日の計算］ウィンドウは表示されず、現在の日付に基づいて期限日が再計算されます。

        システム管理者が［アプリケーションの設定］で［組織のデフォルトの営業時間を使用して期限日を再計算］チェックボックスをオンにしている場合は、期限日の計算時にデフォルトの営業時間、営業時間に関連付けられたタイムゾーン、組織に設定された休業日が使用されます。たとえば、システム管理者が組織のデフォルトの営業時間を午前10時～ 午後6時 に設定しているとします。この場合は、月曜日の午後5時に問題を作成して 優先度を1に設定すると、問題の期限日の計算結果は翌営業日の 火曜日の午前11時になります。ここで、システム管理者が火曜日を休業日として設定している場合は、問題の期限日の計算結果が 翌営業日の水曜日の午前11時になります。
        組織の営業時間と休業日を設定するには、システム管理者に問い合わせてください。

    7. 終了日： このフィールドには、問題の終了日時が入力されます。

      8. 注意: メニューコマンドを実行したとき、または関連するデータが取得されたときに、一部のフィールドには値が入力されます。

  16. ［ブロードキャスト］セクションで、次の情報を入力します。
    1. ブロードキャストID： 問題に関連付けるブロードキャストを選択します。
    2. メッセージ： このフィールドには、ブロードキャストのブロードキャストメッセージが入力されます。
  17. ［保存］をクリックします。

    18. 注意: 新規フォームを開いてレコードを作成した後、レコードを保存した場合は、フォームの右上セクションにある［次のレコード］および［前のレコード］ボタンを使用して、次または前のレコードに移動することはできません。これらのボタンは新規フォームで使用できません。

  18. 問題を作成したら、その補足情報を追加できます。

    19. 詳細は、問題の補足情報の管理を参照してください。

  19. （オプション）現在のレコードをインアクティブとして指定するには、［インアクティブ］チェックボックスをオンにします。

    20. レコードはすでに無効になっているものの、レコード保持ポリシーを順守するためにレコードを保持する必要があるという理由で、レコードをインアクティブにしなければならない場合があります。

    注意: インアクティブとしてマークされたレコードは、データベースで実行される検索の対象に含まれません。

    ［補足情報］セクションは新規フォームに読み込まれません。レコードを保存すると、［補足情報を表示］リンクが表示されます。このリンクをクリックすると、［補足情報］セクションが表示されます。［補足情報］セクションが常に表示されるよう設定するには、［既存のレコードの補足情報を常に表示］チェックボックスをオンにします。詳細は、問題の補足情報の管理を参照してください。

問題の補足情報の管理

問題フォームの下部セクションにある［補足情報］セクションでは、次のアクションを実行できます。

    • 問題の履歴の追跡
    • インシデント、サービス、構成アイテム、変更リクエストのリンク
    • 新しい変更リクエスト・タスクの追加
    • ドキュメンテーションの追加
    • 問題に関する監査情報の表示

問題フォームには、［補足情報］セクションに次のタブが含まれています。

    • 履歴： 既存のユーザー定義アクションやシステム定義アクションを表示したり、新しいユーザー定義アクションを追加したりすることで、問題に対して実行されたアクションの履歴をすべて追跡できます。
    • インシデント： 問題の源である既存のインシデントをリンクできます。
    • 変更リクエスト： 問題を解決するために使用する必要がある新しい変更リクエストの追加や既存の変更リクエストのリンクを実行できます。問題にリンクされた変更リクエストを閉じるときは、変更リクエストが閉じられたことを知らせる電子メールメッセージを問題オーナー（ユーザー）に送信できます。問題がキューに割り当てられた場合、電子メール通知は送信されません。電子メール通知の送信を有効にするには、システム管理者に連絡して［問題にリンクされた変更リクエストが終了した場合に問題オーナーに通知］ワークフローを有効にするように依頼します。
    • サービスおよびCI： 問題を解決するときに影響を受けるサービスとCIをリンクできます。
    • タスク： 問題を解決するために実行する必要がある新しいタスクを追加できます。
    • ドキュメンテーション： 問題を終了するために必要なドキュメントと添付ファイルへのURLを添付できます。
    • 監査： このフォームのフィールドに加えられた変更の履歴が表示されます。システム管理者が、目的のフィールドのフィールド履歴追跡を有効にします。
