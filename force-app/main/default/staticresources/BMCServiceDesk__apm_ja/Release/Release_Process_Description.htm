プロセスの説明

リリース管理プロセスは以下の手順で構成されています。

  • 変更リクエスト処理。 リリース管理に渡されたリクエストや問題を、リリースコーディネータが確認するために使用されます。

  • リリース定義。リリースコーディネータがCABミーティングを準備する際や、CABレビュー用に収集された変更リクエストのうち、次回リリースで完了すべきリクエストを、CABスタッフメンバーが決定する際に使用されます。リリースコーディネータはまた、リリースの要件を論理グループに分類し、変更コーディネータが効率的に処理できるようにするために、この手順を使用します。

  • 業務上の変更理由。リリースの実装のために、追加資金の取得が必要になった場合に、リリースコーディネータによって使用されます。

  • リリース調整。リリースの実装を開始し、必要な修正アクションの必要に応じて修正アクションを決定するためにリリースコーディネータによって使用されます。

これら手順の詳細を確認するには、プロセス をクリックしてグラフィック表示に戻り、該当する手順のボックスをクリックします。この手順が、右のペインで説明と共にグラフィック表示されます。
