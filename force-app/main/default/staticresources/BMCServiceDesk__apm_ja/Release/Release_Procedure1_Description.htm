手順1、変更リクエストの処理


サービスの変更コーディネータが変更リクエストをサービスのリリースコーディネータに渡した後、リリースコーディネータは要件を理解するためにリクエストをレビューします。変更リクエストにリリース管理が不要である場合は、リリースコーディネータは変更コーディネータにリクエストを返します。標準以外の変更を実装する変更リクエストは、リリース管理が処理する必要がありますが、問題の発生防止または解決のために登録された変更リクエストは例外とします。ただし次の実装に限定されます。

1名の変更コーディネータが変更リクエストを行い、

実装によってサービスの機能が異なることのない

追加資金が必要ない

変更リクエストにリリース管理が必要な場合は、リリースコーディネータがCAB認証が必要かどうかを確認します。変更リクエストが問題の発生防止または解決のために変更の実装を要求しており、変更を実装してもサービスの機能は変更されない場合、CAB は変更リクエストのレビューを行う必要はありません。CAB認証が必要な場合は、リリースコーディネータはサービスのCABの次のミーティング中でのレビューを待っているリクエストのリストに、変更リクエストを追加します。
