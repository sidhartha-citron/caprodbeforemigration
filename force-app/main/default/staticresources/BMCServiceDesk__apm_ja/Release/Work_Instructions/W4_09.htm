作業指示について

手順

リリースコーディネータへの作業指示

4.9.1

リリースの［メモ］アクションを追加して、どの変更リクエストの何の要件が満たされていないか（存在する場合）を要約します。

4.9.2

リリースの変更リクエストがリンクされている、変更リクエストを開きます。この変更リクエストから、リリースのリクエスト者と承認者を更新するタスクを開きます。このタスクは、変更作成時のテンプレートによって生成されます。

4.9.3

タスクの［期限日］フィールドに、達成可能な日時を指定します。

4.9.4

タスクの［オーナー］フィールドで自分が選択されていることを確認します。

4.9.5

リクエスト者と承認者の更新開始の準備ができていない場合は、タスクの［ステータス］フィールドを［登録済み］から［受諾済み］に変更します。準備ができた場合は、タスクの［ステータス］フィールドを［進行中］に設定します。

4.9.6

複数の変更リクエストがリンクされている、リリースのための変更リクエスト（これは、リリースのリクエスト者と承認者を更新するためのタスクがリンクされている変更リクエストでもあります）の［リンクされたインシデント］セクションで、リリースの実装によって要件がすべて満たされた各リクエストを開きます。各リクエストの［解決］フィールドに、リリースがどのように実装されたかを要約し、［完了カテゴリ］フィールドを［Solved - Root Cause Analysis not Required（解決済み - 根本原因の分析は必要なし）］に設定します。

［メモ］アクションの［所要時間(HH:MM)］フィールドに、リクエストの更新にかかった時間を入力します。

4.9.7

リクエストの［ステータス］フィールドを［終了済み］に設定すると、サービスデスクが［クライアントID］フィールドのリクエストにリンクされているクライアントに通知します。

4.9.8

複数の変更リクエストがリンクされている、リリースのための変更リクエスト（これは、リリースのリクエスト者と承認者を更新するためのタスクがリンクされている変更リクエストでもあります）の［リンクされた問題］セクションで、リリースの実装によって要件がすべて満たされた各問題を開きます。それらの問題の［アクション履歴］フィールドに、リリースがどのように実装されたかを要約し、［ステータス］フィールドを［変更完了］に設定します。

［メモ］アクションの［所要時間(HH:MM)］フィールドに、問題の更新にかかった時間を入力します。

メモ:

問題の［ステータス］フィールドを［変更完了］に設定すると、問題が担当している問題コーディネータに返されます。

4.9.9

要件が（すべて）満たされていない各リクエストを開き、すべて満たすために必要な社内従業員および長期契約企業の労力を見積もります。また、要件をすべて満たすためのサポートリクエストについては、サプライヤからの入手が必要なCI、サービス、サポートのコストを見積もります。それらリクエストに［メモ］アクションを追加して、次の内容を指定します。

  • どの要件が満たされなかったか、そしてこれらの要件が満たされなかった理由

  • 新しいコストと労力の見積もりおよびその根拠となる前提

  • サービスの次のCABミーティング中に、満たされなかった要件を再レビューすること

これらの各リクエストを、サービスの次のCABミーティング中にレビューされる変更リクエストの、プレースホルダーとして動作する変更に関連付けて、リリースにリンクされていた変更からは外します。また、各リクエストの［クライアントID］フィールドで選択されているクライアントに電子メールメッセージを送信します。電子メールには、受信者のリクエストの［メモ］アクションに追加した情報を含めます。

［メモ］アクションの［所要時間(HH:MM)］フィールドに、リクエストの更新にかかった時間を入力します。

4.9.10

リリースの実装によって要件が（すべて）満たされなかった各問題を開きます。それら問題に［メモ］アクションを追加して、要件が満たされなかった理由を指定します。それら問題の［ステータス］フィールドを［変更完了］に設定し、その問題を担当している問題コーディネータに返すようにします。リリースのための複数の変更リクエストがリンクされている変更リクエストから、それら問題のリンクを外します。

［メモ］アクションの［所要時間(HH:MM)］フィールドに、リクエストの更新にかかった時間を入力します。

4.9.11

CABスタッフメンバー（リリースがCABによって定義されている場合）と、ビジネスケースの承認者（リリースのために予算を準備しなければならなかった場合）宛ての電子メールを準備します。この電子メールには、すべての要件が満たされたかどうかを指定します。すべての要件が満たされなかった場合は、どの要件が満たされなかったか、そしてこれらの要件が満たされないまま、リリースを完了する決定がなされた理由を指定します。

4.9.12

リリースのリクエスト者と承認者の更新タスクの［ステータス］フィールドを、［終了済み］に設定します。

4.9.13

［メモ］アクションの［所要時間(HH:MM)］フィールドに、リクエスト者と承認者の更新にかかった時間を入力します。

4.9.14

リリースの［完了カテゴリ］フィールドで、適切なオプションを選択します。リリースの［ステータス］フィールドを［終了済み］に設定します。
