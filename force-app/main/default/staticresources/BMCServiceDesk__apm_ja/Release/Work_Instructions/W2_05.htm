作業指示について

手順

CABスタッフへの作業指示

2.5.1

次回リリースする変更リクエストをリリースコーディネータに通知します。

メモ:

必要に応じて、他のCABスタッフメンバーとディスカッションを行い、次にリリースする変更リクエストの数と、その実装にかかる時間との間に、現実的なバランスが保たれているか確認します。可能であれば、リリースのビジネスケースに対する承認取得に必要な時間を考慮に入れて確認します。

メモ:

変更リクエストは、CABスタッフメンバーの3分の2以上の要求があった場合に、次にリリースされます。

メモ:

却下されず、次にリリースされない変更リクエストは、CABの次のミーティング中に再度レビューされます。
