作業指示について

手順

リリースコーディネータへの作業指示

2.9.1

［リリース］タブを選択して［新規］ボタンを押し、新しいリリースを開きます。

2.9.2

［リリースのタイトル］フィールドにリリースの説明を入力し、［説明］フィールドにリリースの要件を要約します。

2.9.3

正しいカテゴリを、［カテゴリ］フィールドで選択します。正しいリリースのタイプ、サブタイプ、配布のタイプを選択します。

2.9.4

インパクトと緊急度を指定します。

2.9.5

リリースの［ステータス］フィールドを［登録済み］に設定します。

2.9.6

［期限日］、［実装開始］、［実装終了］、［配布開始］、［配布終了］についての各フィールドに達成可能な日時を指定してリリースを保存します。

2.9.7

［リンク済みサービスとCI］セクションで、リリースの実装先サービスを選択します。

2.9.8

［リンクされた変更リクエスト］セクションにある［新規］ボタンをクリックして、新しい変更を開きます。

2.9.9

［テンプレート］フィールドで該当するリリース調整のテンプレートを選択し、変更を保存します。これにより、変更の必須フィールドが入力され、次のタスクを作成するとともに保存時にリンクされます。

  • リリースの複数の変更への分割

  • リリースの業務上の変更理由

  • 変更コーディネータのための要件ドキュメントの準備

  • リリースのリクエスト者と承認者の更新

  • 実装後のミーティング

2.9.10

変更リクエストの［イニシエータのID］と［オーナー］フィールドで、自分が選択されていることを確認します。

2.9.11

［説明］フィールドの変更リクエストの説明を編集します。

2.9.12

変更リクエストの［ステータス］フィールドを［進行中］に設定し、変更を保存します。

2.9.13

［リンク済みサービスとCI］セクションで、リリースの実装先サービスを選択します。

2.9.14

［Linked Release（リンクされたリリース）］セクションで、［新しいリンク］ボタンをクリックします。

2.9.15

変更の［Linked Request（リンクされたリクエスト）］セクションで［新しいリンク］ボタンをクリックして、新しいリリースにより満たされる予定の要件を持つすべてのリクエストにリンクします。同様に、変更の［リンクされた問題］セクションで［新しいリンク］ボタンをクリックして、新しいリリースにより提案済みの構造的な解決方法が実装される予定の、すべてのリクエストにリンクします。

2.9.16

CABがレビューする予定のサービスの変更リクエストの、プレースホルダーとして動作する変更を開きます。新しいリリースの調整のために変更に関連付けたばかりの、このプレースホルダー変更リクエスト、リクエスト、問題の間のリンクをそれぞれ削除します。サポートリクエストのリンクを削除するには、プレースホルダー変更の［Linked Request（リンクされたリクエスト）］セクションで、［アクション］カラムの［削除］リンクをクリックします。問題とのリンクを削除するには、プレースホルダー変更の［リンクされた問題］セクションで、［アクション］カラムの［削除］リンクをクリックします。
