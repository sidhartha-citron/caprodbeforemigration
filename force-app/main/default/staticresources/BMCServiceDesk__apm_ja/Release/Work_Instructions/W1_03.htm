作業指示について

手順

リリースコーディネータへの作業指示

1.3.1

変更リクエストを変更管理プロセスに返します。

1.3.2

変更リクエストをサポートリクエストの形式で受領した場合は、［メモ］アクションを追加して、リリース管理プロセスを使用せずに変更を実装できることを示します。［メモ］アクションの［所要時間(HH:MM)］フィールドに、リクエストのレビューにかかった時間を入力します。

リクエストの［オーナー］フィールドの横にある［変更］オプションを使用して、自分に変更を割り当てた変更コーディネータを選択します。サポートリクエストの［ステータス］フィールドを、［割り当て済み］に設定します。

メモ:

リクエストの［ステータス］フィールドは［却下済み］に設定しないでください。

1.3.3

変更リクエストを問題の形式で受領した場合は、［メモ］アクションを追加して、リリース管理プロセスを使用せずに変更を実装できることを示します。［メモ］アクションの［所要時間(HH:MM)］フィールドに、問題のレビューにかかった時間を入力します。

リクエストの［オーナー］フィールドの横にある［変更］オプションを使用して、自分に変更を割り当てた変更コーディネータを選択します。問題の［ステータス］フィールドを［変更リクエスト済み］に設定します。

メモ:

問題の［ステータス］フィールドは［却下済み］に設定しないでください。
