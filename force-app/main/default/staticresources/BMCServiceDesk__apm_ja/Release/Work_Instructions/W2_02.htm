作業指示について

手順

リリースコーディネータへの作業指示

2.2.1

CABの次回ミーティングまでの変更リクエスト提出期限が過ぎた場合は、CABミーティングの準備、および円滑な進行のためのタスクの［ステータス］フィールドを、［進行中］に設定します。

2.2.2

CABレビュー予定の変更リクエストそれぞれの詳細を、まとめたリストを準備します。タスクと同様に、これらのリクエストはサービスの、CABの次回ミーティング中にレビューされる変更リクエストの、プレースホルダーとして動作する変更にリンクされています。

メモ:

変更リクエストのリストには、次の情報に対するカラムが必要です。

  • リクエスト番号または問題番号

  • リクエスト者の名前または組織

  • 変更リクエストの説明

  • 実装に必要な労力の見積もり

  • 実装に必要なコストの見積もり

2.2.3

次のCABミーティング中にレビュー予定の変更リクエストリストの最終版を、CABスタッフメンバー全員に電子メールで送信します。これによりメンバーは、ミーティング前に変更リクエストについてディスカッションすることができます。

2.2.4

CABミーティングの準備、および円滑な進行のためのタスクに［メモ］アクションを追加して、変更リクエストリストの最終版をCABスタッフメンバー全員に送信したこと、およびCABミーティング開始の際に引き続き処理されることを記載します。

2.2.5

ミーティングの準備、および円滑な進行のためのタスクの［ステータス］フィールドを、［待機中］に設定します。

2.2.6

［メモ］アクションの［所要時間(HH:MM)］フィールドに、変更リクエストリストの最終版作成にかかった時間を入力します。
