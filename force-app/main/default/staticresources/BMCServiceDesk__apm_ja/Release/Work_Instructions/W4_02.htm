作業指示について

手順

リリースコーディネータへの作業指示

4.2.1

リクエスト済み変更がサービス管理アプリケーションに登録され、リリースにリンクされるようにして、変更コーディネータに送った変更リクエストのフォローアップを行います。

4.2.2

変更リクエストが未登録の場合は、担当の変更コーディネータに変更の登録を通知します。

4.2.3

変更コーディネータが、変更のリリースへのリンクを忘れた場合は、［リンクされたリリース］セクションで変更を開き、リリースにリンクします。

4.2.4

要件ドキュメント準備タスクの［解決］フィールドで、変更がサービス管理アプリケーションに登録され、リリースにリンクされたことを指定します。タスクの［ステータス］を［終了済み］に設定します。

4.2.5

［メモ］アクションの［所要時間(HH:MM)］フィールドに、要件ドキュメントのフォローアップにかかった時間を入力します。

メモ:

変更コーディネータから、関連する変更の1つが完了したと通知された場合は、続けて4.3.1の作業を行います。
