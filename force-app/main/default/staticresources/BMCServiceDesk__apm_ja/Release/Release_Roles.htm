役割と責任


次の表に、リリース管理プロセスに関する各種の役割とそれぞれの責任を示します。プロファイルをレビューするには、役割をクリックします。

責任

CABメンバー

CABレビュー用に収集された変更リクエストのうち、却下するもの、次にリリースするもの、CABの次のミーティングで再度レビューするものを決定します。

リリースコーディネータ

  • リリースコーディネータとして処理を行うサービスの変更リクエストについて、変更管理から渡された後で確認します。

  • リリースコーディネータとしてユーザーが行うサービスについてCABミーティングを準備し、円滑に進めます。

  • リリースの要件を論理グループに分類し、変更コーディネータが効率的に処理できるようにします。

  • 新しいリリースの実装に追加資金が必要な場合は、そのリリースに対するビジネスケースを作成します。

  • リリースの実装を開始し、必要に応じて修正アクションを決定します。

  • 実装後のミーティングを準備および開催し、今後のリリースに対する改善案を収集します。
