作業指示について

手順

グループコーディネータへの作業指示

2.3.1

リクエストのフルフィルメントがリクエストを処理できる場合は、続けて6.1.1の作業を行います。その他の場合は、続けて2.4.1の作業を行います。

メモ:

リクエストがインシデントである場合は、スタッフは5.2.1で解決方法を決定します。インシデントについては、2.4.1に進みます。

メモ:

サービスオーナーが承認したリクエストテンプレートで定義済みのワークフローに従ったリクエストの実装が可能であり、このテンプレートを使用してリクエストを作成する場合は、リクエストのフルフィルメントがリクエストを処理できます。
リクエストの実装について、リスクおよびインパクト分析、または経理上の承認以外の承認が必要である場合は、変更管理が処理しなければなりません。
