作業指示について

手順

サービスデスクアナリストへの作業指示

2.5.1

［メモ］アクションの［所要時間(HH:MM)］ フィールドに、リクエストの登録または更新にかかった時間を入力します。

2.5.2

これが新しいリクエストである場合は、［オーナー］フィールドの横にあるルックアップアイコンを使用して、適切な割り当てグループを選択します。

2.5.3

リクエストが却下された場合（［ステータス］フィールドが［却下済み］に設定されています）、またはユーザーが解決策に満足しなかった場合（［ステータス］フィールドが［終了済み］または［ClosedMail（終了済みのメール）］に設定されています）は、リクエストのステータスを［割り当て済み］にして、［オーナー］フィールドの横にあるルックアップアイコンを使用して、適切な割り当てグループを選択します。

メモ:

リクエストを特定のスタッフメンバーに割り当てないでください。グループのグループコーディネータが、リクエストを処理する上で適切なスタッフメンバー（スキル、空き状況、アクセス権限の面で）を決定します。

2.5.4

ユーザーと通話中の場合、リクエスト番号を伝えます。この番号は［インシデント］フォームのトップにあります。リクエスト番号を伝えることで、ユーザーはリクエストが登録されたことを確認できます。

メモ:

リクエスト番号は、サポートリクエスト保存後の電子メールでもユーザーに通知されます。

2.5.5

ただちにリクエストの処理が必要な場合（たとえば、ウィルスに攻撃された場合など）は、割り当て済みグループのコーディネータに電話をかけたり連絡を取ったりして、コーディネータが緊急度を認識していることを確認します。通常の業務時間外にも同様に行います。スタッフがリクエスト処理を開始することが重要です。
