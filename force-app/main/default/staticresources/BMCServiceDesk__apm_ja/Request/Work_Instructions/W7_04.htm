サービスオーナーへの作業指示

7.4.1

インシデントの解決に長時間を要することが明確で、継続サイトでの影響を受けたサービスの復旧が、もっとも効率的で信頼性が高い場合は、続けて7.5.1の作業を行います。当てはまらない場合は、7.6.1に進みます。

メモ:

次のような方法でインシデントを解決できる場合は、継続サイトでのサービスの復旧を考慮しないでください:

  • セキュリティの向上（たとえば、ウィルス定義の更新や、セキュリティパッチの適用など）

  • バックアップまたは、前回の本番稼働サイトのソフトウェアリリースの復元。（たとえば、データ破壊や、アプリケーション変更の実装に失敗した場合など）

  • サービス提供に失敗したハードウェアを、本番稼働サイトと交換
