手順4:リクエストの追跡


サービスデスク未完了リクエストのステータスを定期的に確認しながら、完了するまで進捗状況を追跡します。リクエストが進行中になっていなければ、サービスデスクアナリストが割り当てられたグループのコーディネータに連絡を取り、リクエストが進行中であることを確認します。

リクエストが複数ユーザーに影響する新しいサービス停止である場合は、サービスデスクアナリストは次の該当するユーザーに通知します。

  • 影響を受ける ユーザー
  • 影響を受けるサービスのオーナー
  • 影響を受けるサービスの顧客担当者
  • 影響を受ける顧客組織のサービスレベルマネージャ
  • 勤務中のマネージャ

使用可能な解決時間の50%を過ぎてもサービス停止が解決されない場合には、サービスデスクアナリストが進捗に関する更新を、該当するユーザーに提供します。

期限日時を過ぎてもインシデントが解決されない場合には、サービスデスクアナリストが、別の進捗に関する更新を該当するユーザーに提供します。2回目の更新後、サービスデスクアナリストはインシデントをサービスオーナーにエスカレートします。
