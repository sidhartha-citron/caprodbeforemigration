手順2、リクエストの割り当て


リクエストが苦情または称賛の場合は、サービスデスクアナリストは、そのリクエストをサービスデスクマネージャに割り当てます。

リクエストのフルフィルメントが、リクエストを処理できる場合は、サービスデスクアナリストがそのリクエスト実装を調整します。

サービスデスクアナリストは、解決に向けて、すべてのリクエストカテゴリについて考慮します。サービスデスクアナリストがリクエストを解決できない場合（スキル、アクセス権限、時間制限の面で）は、サービスデスクアナリストがリクエストを適切なグループに割り当てます。

グループのコーディネータは、サービスデスクアナリスト、オペレータ、継続マネージャ、Webリクエストフォームを登録するユーザーおよび顧客担当者別のグループに割り当てられたすべてのリクエストを確認します。

グループコーディネータは、リクエストが間違ったグループに割り当てられていると判断した場合、または、サービスデスクアナリスト、オペレータ、または継続マネージャから提供された情報が不足しているためにリクエストを完了できないと判断した場合、グループコーディネータはリクエストを却下し、却下の理由を指定します。

却下済みリクエストは、自動的にサービスデスクに渡されます。サービスデスクアナリストは、不足している情報を収集し、このような却下されたリクエストをサービスデスクで解決できない場合は、適切なグループに割り当て直します。

リクエストを正しいグループに割り当てると、リクエストが変更管理によって変更の実装が調整されるような変更である場合、必要な情報を持つリクエストは、関連するサービスの変更コーディネータに渡されます。その他のすべてのリクエストは、グループの適切なスタッフに割り当てられます（スキル、空き状況、アクセス権限の面で）。
