役割と責任


次の表に、リクエスト管理プロセスに関する各種の役割とそれぞれの責任を示します。プロファイルをレビューするには、役割をクリックします。

役割

責任

グループコーディネータ

スキル、空き状況、アクセス権限の面で、スタッフ間グループに割り当てられるリクエストを分配します

勤務中のマネージャ

サービスオーナーの不在時に、インシデントのエスカレーション処理を担当します

サービスデスクアナリスト

  • サービスプロバイダ組織によって提供されたサービスのユーザー向けに、単一の連絡窓口を提供します

  • ユーザーから必要な情報を得て、効率的に、正確に、完全にリクエストにある情報を登録します

  • サービスデスクアナリストは、アクセス権に関する制限および時間的制約の範囲内で、受領済みリクエストのうち、可能な限り多くを解決します　

  • サービスデスクで解決できないリクエストは、解決するために、適切なグループに割り当てます

  • 影響力の大きいインシデント（たとえば、複数のユーザーに影響するサービス停止）の進捗状況を監視し、すべての該当するユーザーに通知します

  • インシデントが関連SLAに定義された解決時間内に解決されない場合、影響力の大きいインシデントを影響するサービスのサービスオーナーにエスカレートします。

  • リクエストが解決された後に、解決策をユーザーに通知して、解決策の確認を依頼します

サービスデスクマネージャ

  • ユーザーには丁重に対応します

  • 苦情と称賛カテゴリのすべてのサポートリクエストを処理し、苦情を報告したユーザーが、サービスプロバイダ組織の満足したユーザーに変わるようにします

サービスオーナー

エスカレーション済みのインシデントが緊急の変更の実装継続サイトで影響を受けるサービスの復旧、または、リクエスト管理プロセスのインシデント解決の継続により、解決する必要があるかどうかを決定します　　

スタッフ

  • リクエストとタスクを解決します

  • 関連情報とステータス変更に伴い、リクエストとタスクを更新します

  • 解決策が、変更管理プロセスによってのみ実装できるインシデントについて、影響を受けるサービスのオーナーにそのインシデントをエスカレートします
