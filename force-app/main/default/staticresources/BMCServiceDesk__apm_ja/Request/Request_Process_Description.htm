プロセスの説明

リクエスト管理プロセスはリクエストを処理するための8つの手順で構成されています。

  • リクエスト登録はユーザーからのリクエストを登録する場合にサービスデスクアナリストが使用します。

  • リクエスト割り当てはサービスデスクアナリストが使用し、苦情と称賛の場合にグループとサービスデスクマネージャにリクエストを割り当てます。サービスデスクアナリストはグループコーディネータが却下したリクエストを処理する場合にもこの手順を用います。グループコーディネータは自分のグループにリクエストが割り当てられた後にこの手順を用います。グループコーディネータはこのようなリクエストを特定の変更コーディネータまたはスタッフに割り当てて解決を求めるためにこの手順を用います。

  • 苦情と称賛処理はサービスデスクマネージャが正常に苦情または称賛カテゴリのリクエストを完了した場合に使用します。

  • 追跡のリクエストはサービスデスクアナリストがリクエストの進捗状況を追跡する場合に使用します。

  • スタッフによるリクエストの解決はスタッフが割り当てられたリクエストを完了する場合に使用します。

  • フルフィルメント処理のリクエストはサービスデスクアナリストまたはスタッフが事前定義済みワークフローを用いることで処理できるリクエストを完了するために使用します。

  • インシデントのエスカレーション処理はインシデントがエスカレートされる場合にサービスオーナーが使用します。影響を受けるサービスのサービスオーナーはこの手順を使用して効率的な解決方法を決定します。

  • リクエストの完了は解決されて終了済みとなったリクエストを完了する場合にサービスデスクアナリストが使用します。

これら手順の詳細を確認するには、このプロセスのボタンをクリックしてグラフィック表示に戻り、該当する手順のボックスをクリックします。手順が、右のペインで説明と共にグラフィック表示されます。
