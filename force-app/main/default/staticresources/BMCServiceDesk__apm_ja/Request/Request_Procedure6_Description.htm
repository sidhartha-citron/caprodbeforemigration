手順6:リクエストのフルフィルメント処理


リクエストを実行するためのワークフローを含むテンプレートによりリクエストが作成されると、サービスデスクアナリストまたはスタッフはそのテンプレートにより作成されたタスクを確認し、追加のタスクが必要か判断します。必要な場合、追加のタスクが作成されリクエストにリンクされます。

次に、サービスデスクアナリストまたはスタッフは、どのタスクが他の人を必要とせず完了できるか判断し、それらのタスクを自分に割り当てます。
サービスデスクアナリストまたはスタッフは、残りのタスクごとにタスクを完了するための適任者を決定します。他のサービスデスクアナリストまたはスタッフが完了させる必要があるタスクは以下に割り当てられます:

  • サービスデスクアナリストまたはリクエストを処理するスタッフと同じグループにいる場合の適切なスタッフ
  • 適切なスタッフが他のグループにいる場合の目標グループのグループコーディネータ
  • 適切なスタッフが他のグループにいる場合の目標グループのグループコーディネータ


タスクを割り当てる際、それらのタスクを完了する順序を考慮します。割り当てられたタスクが、他のタスクが完了するまで待つ必要がないように、順序を決めます。

リクエストの処理を担当するサービスデスクアナリストまたはスタッフは、自分に割り当てられたタスクを完了し、残りのタスクの完了を監視します。必要に応じて追加のタスクを作成し、割り当てることができます。すべてのタスクが正しく完了すると、リクエストは完了します。
