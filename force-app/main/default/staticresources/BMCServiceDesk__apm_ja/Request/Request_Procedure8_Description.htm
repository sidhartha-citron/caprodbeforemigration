手順8:リクエストの完了


サービスデスクアナリストがリクエストを解決できる場合（スキル、アクセス権限、時間制限の面で）は、サービスデスクアナリストがリクエストを解決し完了します。

リクエストが完了した後、サービスデスクアナリストは、リクエストを送信したユーザーに通知し、ユーザーに解決策の確認を依頼します。

ユーザーが、自分のリクエストが満足に解決されていない事をサービスデスクに通知した場合、サービスデスクアナリストは、再びリクエストの解決に取り組みます。

それ以外の場合、サービスデスクアナリストは、複数のユーザーに影響があるサービスの停止に関するリクエストかどうかを確認します。これに該当する場合、サービスデスクアナリストは、影響を受けるユーザーにインシデントの解決方法を知らせます。

影響を受けるユーザー

  • 影響を受けるサービスのオーナー
  • 影響を受けるサービスの顧客担当者
  • 影響を受ける顧客組織のサービスレベルマネージャ
  • 勤務中のマネージャ

ユーザーがリクエストの解決に満足していることを確認した場合、またはユーザーが28日以内に応答しない場合、リクエストを完了します。
