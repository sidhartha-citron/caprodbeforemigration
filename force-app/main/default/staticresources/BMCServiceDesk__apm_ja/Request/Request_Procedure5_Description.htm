手順5、スタッフによるリクエスト解決


リクエストがグループのスタッフに割り当てられた後、スタッフはリクエスト情報を確認し、リクエストの解決方法を決定します。

リクエストのフルフィルメントがリクエストを処理できる場合、スタッフがリクエストの実装を調整します。

解決には変更管理が必要なインシデントがリクエストに関係している場合、スタッフはリクエストをサービスオーナーにエスカレーションします。これは、解決策によって次のような結果が生じるためです。

  • サービス時間内にサービスが利用できない、またはサービスが低下する
  • サービスの機能が変更になる
  • CMDBの更新が必要になる　

まずスタッフがインシデントを調査する必要があるので、グループコーディネータは、インシデントの解決に変更管理が必要かどうかを決定できません。

変更管理が不要で、サポートするサービスのエラーが原因で発生したインシデントに対するリクエストの場合、スタッフは関連するリクエストを登録し、リクエストが完了するまで待ってから、リクエストの解決を続行します。

スタッフは、別のスタッフがリクエストを解決する必要があると判断した場合、適切なグループコーディネータに通知します。その他の場合は、スタッフがリクエストを解決し、リクエストを更新します。
