手順1、リクエストの登録


ユーザーがサービスデスクに連絡してきたときは、サービスデスクアナリストは、ユーザの記録を新規リクエストにリンクします。次に、ユーザーが電話または直接来訪した場合、サービスデスクアナリストが、ユーザーにどのようなサポートができるかを聞くことで、ユーザーのリクエストの内容を判断します。ユーザーが、書面でサービスデスクに連絡してきた場合、（たとえば、電子メール、Fax、書面でなど）サービスデスクアナリストは、それらの情報を読むことで、リクエストの内容を判断します。

ユーザーが、以前送信されたリクエストについて連絡してきたときは、サービスデスクアナリストは、そのリクエストを検索します。リクエストがすでに完了しているか、または［ClosedMail（終了済みのメール）］として設定されているが、提示された解決策でユーザーが満足していない場合、サービスデスクアナリストは状態を割り当て済みに設定します。リクエストはスタッフと元の解決策を提供したグループのTo Doリストに表示されます。ユーザーが、リクエストのステータスの更新を希望した場合、サービスデスクアナリストはユーザーに対し、進捗状況を通知します。

ユーザーが、連絡をして新規リクエストを送信してきたときは、サービスデスクアナリストはそのリクエストに対し、リクエストテンプレートが使用できるかどうかを判断します。

リクエストテンプレートが使用できる場合、サービスデスクアナリストはテンプレートを使用してリクエストを作成します。サービスデスクアナリストは、テンプレートの留意事項を読み、それに従います。留意事項には、ユーザーから得るべき情報や、その情報を得た後の処理方法（リクエストの解決方法や、割り当て先グループなど）が書かれています。

リクエストにリクエストテンプレートを使用できない場合、サービスデスクアナリストがリクエストを作成します。サービスデスクアナリストは、適用できるサービスと構成アイテムをリクエストにリンクします。

ユーザーがインシデントを報告してきた場合、サービスデスクアナリストは、次に基づいてインパクトや緊急度を判断します。

  • ユーザーによって提供された情報

  • サービスインスタンスについての構成に関する情報

  • インパクトを受けるサービスで現在未完了となっている他のインシデント

緊急度とインパクトレベルによってインシデントの解決リクエストに割り当てられる優先度が決められます。

複数のユーザーが重要なまたは重大なサービスに失敗したと報告してきた場合、サービスデスクアナリストは、これらのインシデントをサービスオーナーにエスカレートします。サービスオーナーはメジャーインシデントチームをアクティブにしてメジャーインシデントを解決します。
