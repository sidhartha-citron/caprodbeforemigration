プロセスの説明

問題管理プロセスは、4つの手順で構成されています。

  • リクエストのレビューは、問題コーディネータがリクエストをレビューして、担当サービスの問題を特定するために使用されます。

  • 根本原因分析は、スタッフが問題を分析する場合に使用されます。

  • 分析レビューは、スタッフが行った根本原因の分析の結果を、問題コーディネータがレビューする際に使用されます。

  • 問題の完了は、問題コーディネータが問題を完了するときに使用されます。

これら手順の詳細を確認するには、このプロセスのボタンをクリックしてグラフィック表示に戻り、該当する手順のボックスをクリックします。手順が、右のペインで説明と共にグラフィック表示されます。
