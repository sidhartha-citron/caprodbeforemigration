手順4、問題の完了


問題に対して構造的な解決策を提供するために変更が実装された後で、問題コーディネータが、その変更によって実際に問題を解決したかどうかを確認します。

実装が問題を解決したと考えられる場合、問題コーディネータはその問題を終了します。問題が事後対応的に特定された場合、サービスからは削除されているため、電子メールでサービスデスクに通知します。

一方で、変更が実装されなかった場合や変更によって問題が解決しなかった場合、問題コーディネータは問題に対して、別の構造的な解決策を提案できるかどうかを判断します。提案できる場合は、問題コーディネータはグループ内のスタッフに問題を割り当てなおします。

または、恒久的対策や根本原因の解決のための現実的な方法がないことが明確な場合、問題コーディネータはその問題を解決不能として更新します。現実的で構造的な解決策を提供する新しい、または別のアプローチや技術があると思われる場合、問題コーディネータは定期的に新しい可能性を確認し、別の分析を依頼します。

問題に対する解決策や対処方法が、ナレッジベースで使用可能である必要がある場合、新規ナレッジ記事が作成されます。
