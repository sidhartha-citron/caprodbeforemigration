役割と責任


次の表に、問題管理プロセスに関する各種の役割とそれぞれの責任を示します。プロファイルをレビューするには、役割をクリックします。

役割

責任

問題コーディネータ

  • 問題を特定するため、自分が問題コーディネータであるサービスに関連するリクエストをレビューします。

  • 問題が、問題管理プロセスの優先順位に従って期限内に処理されるようにします。

  • 問題で入力されている情報が正確で漏れがないことを確認します。

  • 構造的な解決策が見つかっていない問題に関して、定期的にレビューを行います。

  • 構造的な解決策を確認し、問題を終了します。

スタッフ

  • 問題の回避策を提案します。

  • 問題の特定のために根本原因をを確定します。

  • 問題に対する構造的な解決策を提案します。

  • 変更管理が不要な場合、問題に対する構造的な解決策を実装します。

  • 関連情報とステータス変更による問題の更新を行います。
