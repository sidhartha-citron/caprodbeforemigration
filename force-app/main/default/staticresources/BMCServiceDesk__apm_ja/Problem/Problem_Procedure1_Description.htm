手順1、リクエストのレビュー


問題コーディネータは、割り当てられたサービス内で問題を特定するためにリクエスト情報を使用します。問題コーディネータは、サービス管理アプリケーションの検索および報告機能を使用して、問題を特定することができます、 スタッフは、問題コーディネータに（スタッフの意見として）問題のあるリクエストを通知することができます。

問題の特定後に、問題コーディネータはサービス管理アプリケーションに問題を登録します。問題コーディネータは、適切なインパクトおよび緊急度のレベルを選択し、問題から発生したリクエストをリンクします。

問題コーディネータは、分析に最適なスタッフ（スキル、空き状況、アクセス権限の面で）に問題を割り当てます。

サービスデスクには、問題が保存された後で新しい問題が特定されたことを、電子メールで自動通知されます。

問題コーディネータが、問題にまだリンクされていないが、過去に見つかった問題によって発生しているリクエストを見つけた場合、これらのリクエストを問題にリンクします。
