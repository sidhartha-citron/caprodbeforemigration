手順3、分析のレビュー


スタッフが問題の根本原因の分析を完了した後に、問題コーディネータは結果をレビューして、構造的な解決策が提案されたか、すでに実装されたかを判断します。変更管理が構造的な解決策の実装を調整する必要がないために、スタッフがすでにこの問題を解決済みである場合、問題コーディネータは直接手順4、問題の完了に進みます。

また、スタッフは適切な分析を実装したが、実施できる構造的な解決策を提案できなかった場合にも、問題コーディネータは手順4、問題の完了に進みます。

スタッフが構造的な解決策を提案したが、変更管理が必要だと判断したために実装されなかった場合は、問題コーディネータが、本当に変更管理が必要かどうかを確認します。変更管理が不要な場合、問題コーディネータは、問題の解決に最適なスタッフに問題を割り当てます。変更管理が必要な場合、問題コーディネータは影響を受けるサービスの変更コーディネータに問題を渡します。

スタッフの分析を許容できない場合、問題コーディネータは、同じスタッフまたは別のスタッフに問題を割り当て直して、分析の改善を求めます。
