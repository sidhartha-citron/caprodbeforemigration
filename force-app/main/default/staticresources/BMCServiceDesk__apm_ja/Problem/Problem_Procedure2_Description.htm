手順2、根本原因の分析


スタッフは問題の詳細をレビューします。1つ以上のインシデントを作成した後で問題が特定された場合、スタッフは回避策の提供を試みます。回避策と回避策の実装方法に関する情報を含むナレッジ記事が作成されます。問題に記事の参照情報が追加されます。

この回避策は、構造的な解決策が見つかって実装されるまで、問題によって将来引き起こされるインシデントを解決するために使用することができます。

スタッフが一旦根本原因の分析を行ったら、説明と共にその問題を更新します。

次に、スタッフは可能な構造的解決策を検討して、その評価を行います。望ましい構造的な解決策の推奨と共に、各オプションの説明が問題に追加されます。

変更管理が、根本原因を取り除かなくてもよい場合、スタッフは望ましい構造的な解決策を実装します。

スタッフが根本原因を見つけられない場合、または構造的な解決策を提案できない場合、問題にその理由を指定します。

構造的な解決策が提案されたか、または既に実装済みであるかにかかわらず、スタッフは問題のステータスを更新して、作業が完了したことを問題コーディネータに通知します。
