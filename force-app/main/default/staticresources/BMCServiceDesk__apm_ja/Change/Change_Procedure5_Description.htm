手順5、アプリケーション変更の実装


アプリケーション変更の最初の実装タスクが割り当てられたら、1名以上のスタッフが開発環境で新規リリースの開発を行います。新しいリリースは、変更の要件を満たしていることを確認するために、スタッフによってテストされます。別のテスト環境がテストに使用可能である場合は、そこでテストを行います。その他の場合は、開発環境でテストを行います。アプリケーションに、回帰テストの（自動）スクリプトが使用できる場合、スタッフはこれを更新して使用します。スタッフは、開発環境のテストで発生したすべてのエラーを修正します。

スタッフが変更要件が満たされ、新しい問題（バグ）が発生していないことを確認したら、リリース管理者は、顧客担当者がテストするテスト環境に新規リリースを移行します。

新規リリースが顧客テストに合格しなかった場合、変更コーディネータはリリースを修正するための追加のタスクを作成して割り当てます。追加のタスクによって本番稼働環境への移行スケジュールが変更になった場合は、その変更は承認を受けなおさなければなりません。

新規リリースが顧客テストに合格したら、スタッフは運用準備（つまり、新規リリースの本番環境への準備が整っていること）を確認します。ここでの準備は、ユーザーのトレーニング、本番環境や継続環境への容量の追加、追加のライセンス発注などが含まれることがあります。

最後に、リリース管理者は新規リリースを本番環境に移行します。
