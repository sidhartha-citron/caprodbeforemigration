手順7、緊急変更の実装

サービスオーナー（不在の場合は、勤務中のマネージャ）がスタッフに緊急変更を実装してインシデントを解決するよう指示した後で、スタッフは実装を開始します。緊急変更がインフラの変更だった場合、スタッフは通常のサポートリクエストを完了する場合と同様に実装します。

緊急変更がアプリケーションの変更だった場合、新たに問題（バグ）を発生させないように開発環境で新しいリリースを構築します。そのあとで、スタッフはリリース管理者に新しいリリースをテスト環境に移行するよう依頼します。テスト環境で、スタッフは新しいリリースをテストします。

新しいリリースがテストに合格しなかった場合、スタッフは開発環境に戻り、修正を行います。新しいリリースがテストに合格した場合、スタッフはリリース管理者に、製品の本番稼働環境への移行を依頼します。
