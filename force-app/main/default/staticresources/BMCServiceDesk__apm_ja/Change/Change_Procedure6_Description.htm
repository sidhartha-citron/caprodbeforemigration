手順6、計画済み変更の完了

変更が本番稼働環境に実装された後、スタッフが（スタッフに十分なアクセス権限がない場合は、ユーザーの助けを借りて）テストをおこない、実装が成功したことを確認します。

変更の要件が満たされたとスタッフが判断した場合、必要に応じてCMDB、容量管理、継続管理情報を更新します。変更コーディネータは、変更の実装が成功したことをリクエスト者と承認者に通知します。その後に、変更コーディネータは変更のリクエストを終了します。

すべての要求を満たさなかった場合は、本番稼働試験を行ったスタッフが、変更をロールバックする必要があるかどうかを判断します。変更が、状況の改善をもたらさない場合や、セキュリティまたは整合性リスクを発生させる場合は、変更をロールバックします。

実装が成功する可能性がある場合、変更コーディネータは必要に応じて追加のタスクを作成します。ただし、変更要件の満たし方が不明である場合、変更コーディネータはリクエスト者と承認者に状況を通知し、変更リクエストを終了します。
