プロセスの説明


構成管理プロセスは以下の手順で構成されています:

  • CIの申請は、新規構成アイテム（CI）を注文する必要がある場合に、構成管理者が使用します。

  • CIの受領は、新しいCIが納品された場合に、構成管理者が使用します。

  • サプライヤ情報の保守は、新しいサプライヤを登録する場合、および以前に登録されたサプライヤの連絡先詳細を更新する場合に、構成管理者が使用します。

  • CIの更新は、CMDBで既に登録されていたCIの属性と関係を更新する場合に、構成管理者が使用します。

これら手順の詳細を確認するには、このプロセスのボタンをクリックしてグラフィック表示に戻り、該当する手順のボックスをクリックします。この手順が、右のペインで説明と共にグラフィック表示されます。
