手順1、CIの申請


新しい構成アイテム（CI）を注文する必要があるたびに、または構成管理データベース（CMDB）を更新する必要がある場合、担当構成管理者は変更管理またはリクエスト管理からタスクを受け取ります。

タスクが新規CIを要求していない場合、構成管理者は手順2、CIの受領へ直接進みます。一方、タスクが1つ以上の新しいCIを注文する場合、構成管理者は、まず必要なCIに対する購入申請を送信します。購入部署へ購入申請を送信した後は、構成管理者は定期的に進捗状況を確認します。

CIが購入により注文されている場合、構成管理者は注文されたCIをCMDBで登録します。一方、購入申請が却下された場合、構成管理者は手順4、CIの更新へ進みます。
