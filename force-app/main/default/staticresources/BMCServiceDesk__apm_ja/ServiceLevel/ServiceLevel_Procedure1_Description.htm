手順1、サービスカタログの保守


サービスレベル管理の情報更新が必要な変更には、この更新のタスクが含まれます。　 このタスクは、サービスレベル管理者に割り当てられます。

このタスクをサービスレベル管理者が受信すると、管理者は、新しいカタログアイテムが必要がどうかを設定します。新しいカタログアイテムが必要ない場合、サービスレベル管理者は、この手順の残りの部分にスキップし、手順2、サービスの有効化に進みます。新しいカタログアイテムが必要な場合、サービスレベル管理者は、カタログアイテムテンプレートを使用し、または類似のカタログアイテムをコピーして準備します。

新しいカタログアイテムが準備されると、サービスレベル管理者は、以下の者から承認を得ます。

カタログアイテムに指定されているSLTを満たす責任があるサービスオーナー　

カタログアイテムを準備して、顧客のSLRがカタログに含まれるようにする顧客組織のサービスレベルマネージャ

カタログアイテムの財務管理を求められるサービスプロバイダ組織の財務マネージャ

新しいカタログアイテムが署名される前に、承認者が調整を必要とする場合、サービスレベル管理者は、すべての承認署名が集まるまで必要に応じてカタログアイテムを調整します。承認者が合意に達しない場合、サービスレベル管理者は、手順4、サービスの終了に進み、タスクの新しいカタログアイテムの作成が失敗した理由を説明します。

すべての承認者が新しいカタログアイテムに署名をした場合、サービスレベル管理者は、その署名をサービスプロバイダ組織のWebベースのサービスカタログに追加します。ある顧客の特定のSLRに応じた新しいカタログアイテムを準備する場合、サービスレベルマネージャは、新しいカタログアイテムの公開済みバージョンが顧客からアクセスできないようにします。
