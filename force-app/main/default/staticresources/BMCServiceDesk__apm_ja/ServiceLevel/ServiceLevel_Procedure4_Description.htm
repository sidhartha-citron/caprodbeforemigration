手順4、サービスの終了


サービスレベル管理のタスクがカタログアイテムの停止をリクエストする場合、サービスレベル管理者は、サービスプロバイダ組織のWebベースサービスカタログのカタログアイテムに、顧客がアクセスできないようにします。

タスクが1つまたは複数のSLAの停止をリクエストする場合、サービスレベル管理者はこれらのSLAのステータスを更新し、SLAが終了したことを示します。さらに、サービスレベル管理者は、管理者が終了したSLAのサービスに他のリンクしているSLAのステータスを確認します。これらのサービスにリンクしているすべてのSLAが終了した場合、サービスレベル管理者はこれらのステータスを更新してサービスを終了させます。

サービスレベル管理者がサービスレベル管理情報を更新する際、これ以上のアクションを必要としない場合、管理者は、タスクを更新して、成功したか失敗したかを変更コーディネータに通知します。（たとえば、リクエスト済みのカタログアイテムが承認されていない、またはリクエスト済みのSLAに署名されていない場合）
