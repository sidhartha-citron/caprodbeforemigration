手順5、SLAのレビューとリクエストの処理


サービスレベルマネージャの顧客組織のいずれかのSLAが、評価期間を終了した後、サービスレベルマネージャは、顧客担当者とのミーティングの予定を決定します。次に、サービスレベルマネージャは、過去の期間にSLAの顧客に提供された サービスの実際のレベルを決定します。

次にサービスレベルマネージャは、SLAに指定されているサービスレベル目標（SLT）を使用して実際のサービスレベルを比較します。1つまたは複数のSLTが、評価期間中に違反した場合、サービスレベルマネージャは、担当のサービスオーナーに連絡を取り、SLTに違反した理由と再びSLTの違反が起きない方法を調査します。サービスレベルマネージャは、事前の改善提案とすべての情報を使用して、顧客担当者の詳細レポートをコンパイルします。

サービスレベルマネージャは、評価期間終了後の1ヶ月以内に顧客担当者とレポートを確認します。顧客担当者は、会議中に顧客組織が説明するあらゆるリクエトをサービスレベルマネージャに通知します。

会議の後、リクエストはサービスレベルマネージャからのリクエストとして登録され、最も適切なグループに割り当てられます。その次にサービスレベルマネージャは、ミーティングの議事録とこれらのリクエストの参照番号を記録します。議事録は、顧客担当者、関係するサービスオーナー、サービスデスクマネージャに配信されます。

また顧客担当者からのアドホックリクエストは、サービスレベルマネージャによって登録され、最も適切なグループに割り当てられます。
