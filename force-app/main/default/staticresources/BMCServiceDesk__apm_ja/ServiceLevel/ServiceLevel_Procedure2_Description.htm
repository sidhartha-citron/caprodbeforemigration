手順2、サービスの有効化


サービスレベル管理のタスクが新しいサービスインスタンスの登録を依頼する場合、サービスレベル管理者は、サービス管理アプリケーションにあるサービスフォームに記入します。

タスクが新しいSLAの準備をリクエストする場合、サービスレベル管理者はSLAテンプレートに記入し、コピーを2部取り、関連するカタログアイテムの印刷を各コピーに添付する準備をします。サービスレベル管理者は、SLAに指定されたSLTミーティングに出席する顧客担当者とサービスオーナーの両者の署名を集めます。

顧客担当者またはサービスオーナーのどちらかがSLAに署名しない場合、サービスレベル管理者は、手順4、サービスの停止に進み、タスクにある新しいSLAが設定されない理由を記入します。新しいSLAに署名されたら、サービスレベル管理者は、署名されたSLAのコピー1部をファイルし、もう１部を顧客担当者のファイル用に返却します。

新しいSLAを登録する前に、サービスレベル管理者は、顧客の連絡先が最新であるかどうかを確認するため、サービス管理アプリケーションをチェックします。SLAの顧客の登録が済んでいない場合、もしくは顧客の連絡先が更新されていない場合、サービスレベル管理者は、手順3、顧客情報保守に顧客の情報が登録され、更新されていることを確認します。

顧客の連絡先が最新である場合、サービスレベル管理者は、新しいSLAをサービス管理アプリケーションに登録し、顧客へのサービス管理アプリケーションに登録し、顧客へのサービス提供で使用されるサービスインスタンスにリンクさせます。
