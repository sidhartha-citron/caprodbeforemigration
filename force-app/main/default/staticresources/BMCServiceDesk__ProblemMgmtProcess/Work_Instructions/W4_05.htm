Work Instructions for Problem Coordinators

4.5.1

Add Note action specifying why a new analysis of the problem is requested.

4.5.2

Determine which specialist is best suited (in terms of skills, availability and access rights) to perform the analysis. This could be the same specialist who performed the previous  analysis . Use the "Assign To" option of the toolbar to select this specialist in the Staff Assignment Detail accordian  of the problem.

Note:

If the problem appears to reside in a Configuration Item that is part of a service infrastructure that supports the servoce for which you act as problem coordinator, contact the problem coordinator of the supporting service. Discuss the problem and if the other problem coordinator agrees with your reasoning, transfer the responsibility for the problem by selecting him/her in the Staff Assignment Details accordian of the problem. Add a Note action smmarizing why you passed the responsibility for the problem to the other problem coordinator.

4.5.3

Enter the time you spent on the review and assignment of the problem in the Duration (HH:MM) field.

4.5.4

Set the Status field of the problem to "Assigned".
