Work Instructions for Problem Coordinators

1.7.1

Open a new problem by selecting the "Problem" option that is located in the Problem Management section of the Navigator Bar.

1.7.2

Link the affected Configuration Item in the Configuration Items tab of Supporting Information.

1.7.3

Enter a short description of the problem in the Description field and summarize the requirements by adding a Notes action to the problem.

1.7.4

Select the appropriate category in the Category field of the problem.

1.7.5

Select the appropriate Impact and Urgency of the problem.

1.7.6

Use the "Assign To" option of the toolbar to select appropriate staff for assignment.

Note: upon save the problem will be assigned to yourself automatically.

1.7.7

Enter the time you spent on the identification and registration of the problem in the Duration (HH:MM) field, and save the new problem.

1.7.8

Link support request to the problem through Supporting Infromation Incidents tab page. Also link previously reviewed support requests that appear to have been caused by the same problem, but which you did not consider significant at the time because they signify a service disruption that you did not expect to recur.
