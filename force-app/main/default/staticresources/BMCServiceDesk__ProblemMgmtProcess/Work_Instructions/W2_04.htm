Work Instructions for Specialists

2.4.1

Find the root cause of the problem. Review for example log files or search the internet for information from organizations experiencing similar symptoms.

Note:

For problems that have been identified proactively , it is still important to determine why the service infrastructure is affected. It could be that the service is used in an inefficient or unauthorized manner.

2.4.2

When the root cause has been found, add a note action detail describing it and set the Status field to "Known Error". Alternatively, if the root cause of the problem cannot be found, summarise it through the Note action detail.

Note:

If for any reason the analysis is temporarily not able to progress (e.g. because you are waiting for information from a supplier), specify through the Notes action detail what you are waiting for and set the Status field to "Waiting for…".

2.4.3

Ensure that the Configuration Item in which the root cause of the problem resides is listed under Configuration Item tab.

2.4.4

If the root cause has been found and the problem is reactive type, review the workaround proposed in the Workaround field. If it is possible to propose a better workaround now that the root cause is known, describe (step by step) how it can be implemented in the Workaround field.

2.4.5

Enter the time you spent on the problem since you last updated in the Duration (HH:MM) field.
