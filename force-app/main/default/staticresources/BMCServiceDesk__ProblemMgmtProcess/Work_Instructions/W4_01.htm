Work Instructions for Problem Coordinators

4.1.1

Read the Note that was last added to the  History tab of the problem.

4.1.2

Verify the structural solution if one has been implemented (e.g. by reviewing the production test results).

Note:

If you do not have the access rights to verify the structural solution, work with a user who does.

Note:

If it was not possible to consistently reproduce the symptoms of incidents caused by the problem before the solution was implemented, the only way to verify the solution might be to monitor the support requests for the service to ensure that no new incidents are being caused by the problem.
