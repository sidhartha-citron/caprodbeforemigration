サービスをリクエストする場合、次のいずれかのオプションを使用できます。

    • ［一般的なサービスリクエスト］パネルにある［サービスのリクエスト］タブで、使用可能なサービスリクエストからサービスリクエストを作成する。

      • システム管理者が、インシデントテンプレートとサービスリクエストが別々に［一般的なサービスリクエスト］パネルに表示されるようにセルフサービスを設定した場合、［一般的なサービスリクエスト］パネルには、［問題の報告］と［サービスのリクエスト］タブが表示されます。［問題の報告］タブにインシデントのテンプレートが表示され、［サービスのリクエスト］タブにサービスリクエストが表示されます。システム管理者が、インシデントテンプレートとサービスリクエストの両方を［一般的なサービスリクエスト］パネルに表示するようにSelf Serviceを設定した場合、これらのタブは［一般的なサービスリクエスト］パネルに表示されません。［一般的なサービスリクエスト］パネルの［サービスのリクエスト］タブで20個のサービスリクエストを表示するには、［続きを追加表示>］リンクをクリックします。［一般的なサービスリクエスト］パネルの［サービスのリクエスト］タブに表示するサービスリクエストを少なくするには、［<表示数を減らす］リンクをクリックします。

    • サービスのリクエストに使用できるサービスリクエストのカタログを参照する。
    • 目的に合ったサービスリクエストを検索する。

      • 必要なサービスリクエストが検索結果の中にない場合は、検索結果の下にある［ヘルプのリクエスト］リンクをクリックして、新しいサービスリクエストを作成できます。

必要なサービスリクエストをクリックすると、そのサービスリクエストは［Service Request Preview（サービスリクエストのプレビュー）］ウィンドウに表示されます。このウィンドウでは、サービスリクエストを［マイカート］ポートレットに追加したり、［今すぐリクエスト］をクリックしてサービスリクエストを作成したりできます。そのサービスリクエストが［サービスリクエスト］フォームに表示されます。このフォームでは、要求するサービスリクエストの数やサービスリクエストの希望実行日付を指定することができます。

注意: システム管理者がサービスリクエストのプレビューを無効にしている場合は、必要なサービスリクエストをクリックすると、そのサービスリクエストが［サービスリクエスト］フォームに表示されます。

システム管理者が設定したサービスリクエストの定義によっては、サービスリクエストを実行するための追加情報の入力を求められることがあります。たとえば、新しいApple iPhoneデバイスに関するサービスリクエストでは、iPhoneのモデルや必要な容量の入力を求められることがあります。

サービスリクエストの値の入力を完了すると、サービスリクエストを後で確認するためにカートに保存するか、サービスリクエストを処理するために送信するかを選択できます。カートに含まれているサービスリクエストの件数とリストは、サイドバーの［マイカート］ポートレットに表示されます。カートでは、サービスリクエストの編集、削除、チェックアウトをおこなうことができます。



サービスをリクエストするには

  1. 必要なサービスリクエストが［一般的なサービスリクエスト］パネルの［サービスのリクエスト］タブに表示されない場合は、サービスリクエストのカタログを参照したり、必要なサービスリクエストを検索したりできます。
  2. （オプション）サービスリクエストのカタログを参照するには、次のアクションを実行します。
    1. Self Serviceホームページの右上のセクションにある［参照］をクリックします。
    2. 表示されたサービスリクエストをフィルタ表示するには、［カテゴリ］ツリーで必要なカテゴリをクリックします。
    3. ［サービスのリクエスト］タブをクリックします。

      4. これにより、選択したカテゴリのサービスリクエストだけが［サービスのリクエスト］タブに表示されます。

    4. 必要なサービスリクエストをクリックします。
  3. （オプション）必要なサービスリクエストを検索するには、次のアクションを実行します。
    1. ［検索］フィールドに、必要なサービスリクエストに関連するキーワードを入力します。

      2. キーワードはたとえば、ネットワーク、ハードウェアなどです。［検索］フィールドには、最低2文字を入力する必要があります。

      注意: 「All」と入力すると、いずれかのフィールドに「All」という文字列が含まれているサービスリクエストとインシデントテンプレートが検索結果に表示されます。また、すべてのプロファイルが権限を持っているサービスリクエストも検索結果に表示されます。

    2. をクリックします。
    3. ［サービスのリクエスト］タブで、必要なサービスリクエストをクリックします。
  4. ［一般的なサービスリクエスト］パネルの［サービスのリクエスト］タブに目的のサービスリクエストが表示されている場合、目的に合ったサービスリクエストをクリックします。

    5. そのサービスリクエストが［Service Request Preview（サービスリクエストのプレビュー）］ウィンドウに表示されます。デフォルトでは、このウィンドウにサービスリクエストの説明が表示されます。システム管理者が、［Price（料金）］と［Turnaround Time（納期）］フィールドが表示されるようにサービスリクエストを設定した場合、このウィンドウには［Price（料金）］と［Turnaround Time（納期）］フィールドの値が表示されます。［Service Request Preview（サービスリクエストのプレビュー）］ウィンドウでは、サービスリクエストを作成する手順を実行しないで、サービスリクエストの詳細を表示できます。［Service Request Preview（サービスリクエストのプレビュー）］ウィンドウでサービスリクエストの詳細を確認したら、サービスリクエストを［マイカート］ポートレットに追加できます。また、そのサービスリクエストの作成を続行しない場合は、このウィンドウを閉じることができます。必要なサービスリクエストが見つかるまで、［Service Request Preview（サービスリクエストのプレビュー）］ウィンドウでその他のサービスリクエストを開いて、その詳細を確認できます。見つかったら、サービスリクエストの作成を続行できます。

    注意: システム管理者がサービスリクエストのプレビューを無効にしている場合は、必要なサービスリクエストをクリックすると、そのサービスリクエストが［サービスリクエスト］フォームに表示されます。

  5. ［Service Request Preview（サービスリクエストのプレビュー）］ウィンドウで必要なサービスリクエストの詳細を確認したら、次のいずれかのアクションを実行します。
    • ［今すぐリクエスト］をクリックして、サービスリクエストを作成します。

      • この一般的なサービスリクエストのフィールドの値が、新しいサービスリクエストの対応するフィールドに入力されます。

    • ［カートに追加］をクリックしてサービスリクエストを［マイカート］ポートレットに追加します。

      • ［カートに追加］をクリックすると、step10が表示されます。

  6. ［サービスリクエスト］フォームで［入手期限］や［数量］などのサービスリクエストフィールドに値を入力します。

    7. ここで表示されるフィールドは、設定されたサービスリクエストの定義により異なります。

    サービスリクエストの［Request For（このユーザーの代理でリクエストを作成する）］フィールドには、あなたが誰の代理でそのサービスリクエストを作成しているかがデフォルトで表示されます。また、［申請者］フィールドには、サービスリクエストを作成しているユーザーの名前がデフォルトで表示されます。

    サービスリクエストに［電話］フィールドと［電子メール］フィールドが表示されるように設定している場合は、ログオンしているユーザーの電話番号と電子メールアドレスが表示されます。別のユーザーのサービスリクエストを送信する場合は、選択したユーザーの電話番号と電子メールアドレスが［電話］フィールドと［電子メール］フィールドに表示されます。選択したユーザーの電話番号と電子メールアドレスが定義されていない場合は、ログオンしているユーザーの電話番号と電子メールアドレスが［電話］フィールドと［電子メール］フィールドに表示されます。

    サービスリクエストに［予定日］フィールドが表示されるように設定している場合、サービスリクエストが送信された日付と時間に納期が加算され、［予定日］フィールドに計算済みの値が表示されます。

  7. 異なるクライアントに対して問題を報告するには、［代理］ドロップダウンリストをクリックして［クライアントから選択］ウィンドウを開きます。

    8. 次の条件のいずれかが満たされると、［代理］ドロップダウンリストが表示されます。

    • 自分の役割が<アカウント名>顧客エグゼクティブまたは<アカウント名>顧客マネージャであり、［顧客ポータルマネージャカスタム］ライセンスを使用して自分のプロファイルを作成している。自分やアカウント内の別のユーザーのサービスリクエストを作成することができる。

      • 注意: 自分の役割が<アカウント名>顧客ユーザーである場合、役割階層で最下位の役割であるか、またはユーザーレコードに役割が割り当てられておらず、［代理］ドロップダウンリストは表示されません。

    • 自分のプロファイルが［Salesforce］、［Salesforceプラットフォーム］、または［Chatterのみ］ライセンスを使用して作成されている。自分や役割階層内で自分より下の別のユーザーのサービスリクエストを作成することができる。
  8. サービスリクエストに追加された入力フィールドに応じて、それらの入力フィールドに値を入力してサービスリクエストの実行に使用される情報を提供します。

    9. サービスリクエストに［数量］フィールドが表示されるように設定している場合、［数量］フィールドの数量を増やすと、［合計価格］フィールドが更新されてサービスリクエストの新しい合計価格が表示されます。たとえば、サービスリクエストの［単価］が$100の場合、［数量］フィールドの数量を5に変更すると、［合計価格］フィールドは$500になります。

    一部の入力フィールドが非表示になっている場合があります。これらのフィールドは、現在表示されている入力フィールドに入力した値に応じて表示されることがあります。たとえば、新しい電話の調達を求めるサービスリクエストを作成する場合、目的の電話のタイプとしてiPhoneを選択すると、入手可能なiPhoneのモデルがサービスリクエストに表示されます。BlackBerryを選択すると、入手可能なRIM BlackBerryのモデルがサービスリクエストに表示されます。この例では、電話のモデルは非表示になっており、目的の電話のタイプを選択した場合にのみモデルが表示されます。

    そのサービスリクエストの入力フィールドとそのフィールドに入力した値が［その他の情報］セクションの［リクエストエントリ］タブに表示されます。［リクエストエントリ］タブでは、［プロンプト・質問］カラムに入力フィールドが表示され、［回答］カラムには入力フィールドに入力した値が表示されます。入力フィールドが［見出しセクション］のレスポンスタイプである場合は、入力フィールドの［回答］カラムには空白になります。入力フィールドの総数が［リクエストエントリ］タブのタイトルに表示されます。

    注意: カスタムフォームを使用していて、システム管理者によって事前定義済みフィールドセットからフィールドが追加されている場合は、カスタムフォーム上のフィールドに値を入力します。

  9. サービスリクエストで値を入力したら、次のいずれかのアクションを実行します。
    • 後で確認や編集を実行できるようにサービスリクエストをカートに保存するには、［Save To Cart（カートに保存）］をクリックします。

      • サイドバーの［マイカート］ポートレットにサービスリクエストが表示されます。ポートレットのサービスリクエストの総数が1増えます。

    • サービスリクエストを実行のために送信するには、［送信］をクリックします。
  10. ［Save To Cart（カートに保存)］をクリックしたら、次のアクションを実行します。
    1. 送信前にサービスリクエストをさらに更新するには、［マイカート］ポートレットでそのサービスリクエストを選択します。

      2. Self Serviceホームページにサービスリクエストが表示されます。

    2. サービスリクエストフィールドに値を入力するか、既存の値を編集します。
    3. サービスリクエストを実行のために送信するには、［送信］をクリックします。

      4. ［マイカート］ポートレットからサービスリクエストが削除され、［サービスリクエスト］セクションに表示されます。

      注意: ［マイカート］ポートレットにあるすべてのサービスリクエストを送信するには、［マイカート］ポートレットで［チェックアウト］をクリックします。サービスリクエストを処理のために送信する前に、サービスリクエストの必須フィールドに値を入力する必要があります。
