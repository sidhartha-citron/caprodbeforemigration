Etape de procédure

Instructions de travail pour les coordinateurs de problème

4.5.1

Indiquez les raisons pour lesquelles une nouvelle analyse du problème a été demandée.

4.5.2

Déterminez le spécialiste le plus apte (selon ses compétences, sa disponibilité et ses droits d'accès) à réaliser une meilleure analyse. Il peut s'agir du même spécialiste qui a réalisé l'analyse précédente du problème. Utilisez l'option Changement à côté du champ Propriétaire pour sélectionner le spécialiste.

Remarque :

si le problème est lié à un EC et que celui-ci fait partie d'une instance de service intervenant dans le support du service dont vous êtes le coordinateur de problèmes, contactez le coordinateur de problèmes de ce service de support. Discutez du problème et, si l'autre coordinateur de problèmes et vous-même êtes d'accord, transférez-lui la responsabilité du problème en modifiant le nom indiqué dans le champ Propriétaire. Résumez les raisons de ce transfert de responsabilité dans le champ Informations.

4.5.3

Dans le champ Durée (HH:MM) de l'action Remarques, saisissez le temps que vous avez consacré à l'identification et l'enregistrement du problème.

4.5.4

Définissez le champ Statut sur Assigned (Affecté).
