Etape de procédure

Instructions de travail pour les spécialistes

2.4.1

Pour trouver la cause fondamentale du problème, consultez les fichiers journaux ou recherchez des informations pertinentes sur Internet.

2.4.2

Lorsque la cause fondamentale a été trouvée, ajoutez cette information dans une action Remarques et définissez le champ Statut sur Erreur connue. Vous pouvez également ajouter une action Remarques si la cause fondamentale n'est pas trouvée.

Remarque :

si, pour une raison quelconque, l'analyse ne peut pas progresser (si vous attendez des informations d'un fournisseur, par exemple), ajoutez une action Remarques afin d'indiquer l'objet de l'attente et définissez le champ Statut sur Waiting for… (En attente).

2.4.3

Liez l'EC défini comme cause fondamentale dans la section EC et services associés. Si la cause fondamentale n'a pas été trouvée et que l'EC à l'origine du problème est inconnu, sélectionnez l'EC spécial portant le code N/A.

2.4.4

Si la cause fondamentale a été trouvée, consultez l'article de connaissance référencé dans le champ Solution de contournement. Si une meilleure solution de contournement peut être proposée, ajoutez les informations nécessaires dans l'article de connaissance.

2.4.5

Dans le champ Durée (HH:MM) de l'action Remarques, saisissez le temps que vous avez consacré au problème depuis sa dernière mise à jour.
