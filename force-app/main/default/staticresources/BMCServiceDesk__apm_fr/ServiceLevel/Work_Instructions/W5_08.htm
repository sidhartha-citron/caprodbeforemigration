Etape de procédure

Instructions de travail pour les gestionnaires de niveau de service

5.8.1

Pour ouvrir une nouvelle demande, accédez à l'onglet Incident et cliquez sur Nouveau.

5.8.2

Dans le champ ID de client, sélectionnez le représentant client. Pour effectuer cette action rapidement, choisissez la recherche par ID de client et saisissez le nom de famille ou le prénom dans le champ Rechercher. Si les caractères saisis dans le champ de recherche sont récurrents, la liste des personnes dont le nom de famille ou le prénom commence par ces caractères s'affiche. Sélectionnez le représentant client dans la liste.

5.8.3

Sélectionnez le type de demande dans le champ Request Type (Type de demande). Les options disponibles pour les demandes soumises par un représentant client à l'intention d'un gestionnaire de niveau de service sont les suivantes :

  • Demande de changement

  • Demande de service

  • Plainte

  • Eloge

Ce champ n'est pas fourni automatiquement. Vous devez personnaliser le formulaire d'incident et l'ajouter dans la section requise.

5.8.4

Dans le champ Description, saisissez une description concise mais significative de la demande.

5.8.5

Dans le champ Type de contact, sélectionnez Visite personnelle.

Ce champ n'est pas fourni automatiquement. Vous devez personnaliser le formulaire d'incident et l'ajouter dans la section requise.

5.8.6

Dans le champ Catégorie, sélectionnez la catégorie de demande appropriée.

5.8.7

Sélectionnez les niveaux d'impact et d'urgence appropriés dans les champs Impact et Urgence.

5.8.8

Définissez le champ Statut de l'incident sur Assigned (Affecté).

5.8.9

Dans les détails de Services et EC, sélectionnez le service auquel la demande est associée.

Remarque :

si la demande concerne une plainte, un éloge ou une demande de développement d'un nouveau service, sélectionnez le service N/A - Service does not (yet) exist or is out of scope (Le service n'existe pas (encore) ou est hors du champ d'application).

Ce champ n'est pas fourni automatiquement. Vous devez personnaliser le formulaire d'incident et l'ajouter dans la section requise.

5.8.10

Dans les détails de Services et EC, sélectionnez l'élément de configuration (EC) auquel la demande est associée.

Remarque :

si l'EC n'est pas enregistré dans la CMDB mais qu'il fait partie du processus de gestion des configurations, sélectionnez l'EC spécial portant le code NORECORD.

Ce champ n'est pas fourni automatiquement. Vous devez personnaliser le formulaire d'incident et l'ajouter dans la section requise.

Remarque :

si l'incident ne concerne pas un EC spécifique (par exemple, en cas de plainte ou d'éloge) ou si l'EC ne fait pas partie du processus de gestion des configurations, sélectionnez l'EC spécial portant le code N/A.

Ce champ n'est pas fourni automatiquement. Vous devez personnaliser le formulaire d'incident et l'ajouter dans la section requise.

5.8.11

Enregistrez l'incident.

5.8.12

Dans le champ Propriétaire de la demande, sélectionnez la file d'attente spécifique. Ne sélectionnez pas un membre du personnel spécifique car il incombe au coordinateur de groupe de choisir l'employé le plus à même de traiter la demande.

5.8.13

Ajoutez une action Remarques pour décrire les besoins métier du client, en utilisant le vocabulaire propre au prestataire de service afin d'éviter tout malentendu.

Remarque :

si le représentant client demande un nouveau service ou un changement du niveau de service, ajoutez les SLR dans ce champ afin d'éviter que les besoins métier du client soient mal compris par le personnel du prestataire de services.

5.8.14

Dans le champ Durée (HH:MM) de l'action Remarques, saisissez le temps que vous avez consacré à l'enregistrement de la demande.

5.8.15

Revenez à l'étape 5.8.1 jusqu'à ce qu'une demande séparée ait été enregistrée pour toutes les demandes soumises par le représentant client.
