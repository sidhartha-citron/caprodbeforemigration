d'élément de configuration


Le tableau ci-dessous répertorie les champs du formulaire d'élément de configuration (EC) et fournit des consignes de saisie pour chaque champ.

Champ

Saisie

Général

Nom d'instance

Ce champ permet de saisir un nom d'EC unique.

Description

Ce champ permet de saisir une description de l'EC.

Brève description

Ce champ permet de saisir une brève description de l'EC, affichée dans l'explorateur d'EC.

Remarques

Ce champ permet de saisir des informations supplémentaires sur l'EC.

Nom du fabricant

Ce champ permet de sélectionner le fabricant de l'EC.

Modèle

Utilisez ce champ pour saisir le nom du modèle, s'il s'agit d'un EC matériel.

Nº de version

  • Ce champ permet de saisir le numéro de version s'il s'agit d'un EC logiciel ou de documentation.

  • S'il s'agit d'un certificat de licence logicielle, il permet de saisir les versions couvertes par les licences.

Numéro de série

Ce champ permet de saisir le numéro de série de l'EC. S'il s'agit d'un certificat de licence logicielle, saisissez le numéro de certificat de licence.
S'il s'agit d'un EC de type Base de données, Logiciel, Software Distribution Package (Packages de distribution de logiciels), Software/Interface Configuration Settings (Paramètres de configuration d'interface/de logiciel), Web Site (Site Web) ou WAN Link (Lien WAN), laissez ce champ vide.

N° de bien

Ce champ permet de saisir le numéro sous lequel l'EC a été enregistré dans l'application financière (par ex. SAP).

ID de jeton

Utilisez ce champ pour saisir l'identifiant unique de l'EC.

Priorité

Ce champ permet de sélectionner la priorité de l'EC pour définir son niveau d'importance dans la CMDB.

Date de création

Ce champ est réglé sur la date et l'heure d'enregistrement de l'EC.

Dernière modification par

Le nom de connexion du membre du personnel ayant effectué la dernière mise à jour de l'EC est enregistré dans ce champ.

Date de dernière modification

Ce champ est réglé sur la date et l'heure de dernière mise à jour de l'EC.

ID classe

Ce champ est défini sur la classe de l'élément utilisé pour enregistrer l'EC.

Nom de classe

Ce champ est défini sur le nom de classe de l'élément utilisé pour enregistrer l'EC.

Pris en charge

Ce champ permet d'indiquer si un support technique est fourni pour cet EC.

Date d'expiration de la garantie

Ce champ permet d'indiquer la date d'expiration de la garantie de l'EC.

Site

  • S'il s'agit d'un EC matériel, ce champ permet de sélectionner le site où se trouve l'EC.

  • S'il s'agit d'un EC logiciel ou de type Software Distribution Package, ce champ permet de sélectionner le site où se trouve la copie maîtresse de la DML.

  • Si l'EC est un certificat de licence logicielle, il permet de sélectionner le site de la DML contenant le certificat.

  • Définissez ce champ sur N/A s'il s'agit d'un EC de type Base de données, Software/Interface Configuration Settings (Paramètres de configuration de logiciel/interface) ou Web Site (Site Web).

Etape

Ce champ permet de sélectionner l'étape appropriée pour l'EC parmi les options suivantes :

  • Exigences

  • Défini

  • Analysé

  • Approuvé

  • Cartographié

  • Conçu

  • Développé

  • Build

  • Testé

  • Libéré

  • Opérationnel

  • Publié

  • Non publié

  • Retiré

Marquer comme supprimé

Utilisez ce champ pour marquer l'EC comme supprimé de façon non définitive.

ID instance

Ce champ est défini sur l'instance de l'élément utilisé pour enregistrer l'EC.

Créateur

Le nom de connexion du membre du personnel ayant enregistré l'EC est enregistré dans ce champ.

Ce champ n'est pas fourni automatiquement. Vous devez personnaliser le formulaire pour afficher le champ dans la section Personnalisé.

Caractéristiques

Caractéristiques (plusieurs champs spécifiques à l'élément)

Chaque élément possède des champs différents dans l'onglet Caractéristiques. Ces champs ne sont donc pas décrits dans ce formulaire.

Relations

Marquer comme supprimé

Utilisez ce champ pour marquer la relation comme supprimée de façon non définitive.

Direction d'impact

Ce champ permet de sélectionner la direction d'impact appropriée parmi les options suivantes :

  • Autre

  • Inconnu

  • Source-Cible

  • Cible-Source

  • Service/Sous-service

  • Service-Offre

Listes associées ou informations de support

Historique

Ce champ permet d'enregistrer les remarques de l'historique des actions créées pour l'EC.

Clients

Cet onglet répertorie les clients associés à l'EC. Il permet de sélectionner les clients utilisant l'EC ou ceux pour lesquels l'EC est réservé.
Laissez ce champ vide si l'EC n'est pas réservé ou utilisé par un client spécifique.

Comptes

Cet onglet répertorie les comptes associés à l'EC. Il permet de sélectionner les organisations pour lesquelles l'EC est réservé.
Laissez ce champ vide si l'EC n'est pas réservé ou utilisé par une organisation spécifique.

Incidents

Cet onglet répertorie les incidents associés à l'EC. Il permet de lier les incidents à l'EC.

Demandes de changement

Cet onglet répertorie les demandes de changement associées à l'EC. Il permet de lier les demandes de changement à l'EC.

Problèmes

Cet onglet répertorie les problèmes associés à l'EC. Il permet de lier les problèmes à l'EC.

Mises en production

Cet onglet répertorie les mises en production associées à l'EC. Il permet de lier les mises en production à l'EC.

Documentation

Cet onglet répertorie les documents associés à l'EC. Il permet de joindre tout fichier apportant des informations supplémentaires concernant l'EC.

Audit

Cet onglet répertorie les modifications apportées aux informations relatives à l'EC.
