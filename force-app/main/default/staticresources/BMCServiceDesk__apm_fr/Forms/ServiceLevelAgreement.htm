Accord de niveau de service


Le tableau ci-dessous répertorie les champs du formulaire d'accord de niveau de service et propose des consignes de saisie pour chaque champ.

Champ

Saisie

Intitulé de l'accord

Ce champ permet de saisir le nom de l'accord de niveau de service (SLA). Par exemple : gestion des ressources humaines P21 (SAP) pour UKPLANT.

Description

Ce champ permet de saisir la description du SLA.

Informations générales

Type d'accord

Ce champ permet de sélectionner le type d'accord parmi les options suivantes :

  • Accord de niveau de service

  • Accord sur les niveaux opérationnels

  • Contrat de sous-traitance

Service

Ce champ permet de sélectionner le service pour lequel le SLA est créé.

Compte

Ce champ permet de sélectionner les organisations couvertes par le SLA. Les options suivantes sont disponibles :

  • L'accord s'applique à tous les comptes.

  • L'accord s'applique aux comptes sélectionnés.

Si vous sélectionnez L'accord s'applique aux comptes sélectionnés, sélectionnez les comptes appropriés.

Effectif à partir de

Utilisez ce champ pour préciser la date à laquelle le SLA prend effet.

Effectif jusque

Ce champ permet de préciser la date avant laquelle le SLA doit être renouvelé pour éviter son expiration.

Analyse planifiée

Utilisez ce champ pour préciser la date à laquelle le SLA doit être analysé.

Heures de prise en charge

Ce champ permet de sélectionner les heures de support appropriées pour le SLA. Les heures de service sélectionnées sont utilisées pour calculer le temps de réponse et l'heure de résolution.

Titulaire de l'accord

Ce champ permet de sélectionner le gestionnaire de niveau de service.

Cibles du service

Cible de niveau de service

Utilisez ce champ pour saisir le titre de la cible de service.

Description

Utilisez ce champ pour entrer la description de la cible de niveau de service.

Cible du service

S'applique à

Ce champ permet de sélectionner le module auquel la cible s'applique. Les options suivantes sont disponibles :

  • Incident

  • Demande de changement

Type de cible

Ce champ permet de sélectionner le type de cible parmi les options suivantes :

  • Heure de résolution

  • Heure de réponse

Heures de prise en charge

Utilisez ce champ pour sélectionner les heures de support adaptées à la cible du service. Les heures de service sélectionnées sont utilisées pour calculer le temps de réponse et l'heure de résolution.

Cible

Ce champ permet de saisir l'heure cible d'achèvement du service (en jours, heures et minutes). Vous pouvez également utiliser le champ Jours ouvrables pour définir le temps cible d'achèvement du service (en jours ouvrables).

Qualification

Conditions

Ce champ permet de sélectionner les champs et valeurs dont les critères doivent être définis.

Mesure

Mesure

Ce champ permet de sélectionner les statuts pour lesquels l'horloge associée au SLA doit démarrer, s'interrompre et s'arrêter.

Repères et actions

Repère

Ce champ permet de choisir le moment d'envoi des notifications automatiques par e-mail. Sélectionnez une étape clé parmi les options suivantes :

  • Pourcentage de temps écoulé

  • Pourcentage de temps restant

  • La cible du service est annulée

  • La cible du service est accomplie

  • La cible du service a échoué

  • L'avertissement de la cible du service s'ouvre

  • Temps écoulé

  • Temps restant

Action

Ce champ permet de sélectionner le modèle d'e-mail à envoyer lorsque l'étape clé est respectée. Les options suivantes sont disponibles :

  • Incident Percentage of Time Elapsed (Pourcentage de temps écoulé pour l'incident)

  • Incident Percentage of Time Remaining (Pourcentage de temps restant pour l'incident)

  • Incident Service Target is cancelled (La cible du service pour l'incident est annulée)

  • Incident Service Target is met (La cible du service pour l'incident est accomplie)

  • Incident Service Target is missed (La cible du service pour l'incident a échoué)

  • Incident Service Target warning (Avertissement de la cible du service pour l'incident)

  • Incident time elapsed (Temps écoulé pour l'incident)

  • Incident time remaining (Temps restant pour l'incident)

  • Service Request Percentage of time remaining (Pourcentage de temps restant pour la demande de service)

  • Service Request Service Target is cancelled (La cible du service est annulée pour la demande de service)

  • Incident Service Target is cancelled (La cible du service pour l'incident est annulée)

  • Service Request Service target is met (La cible du service est accomplie pour la demande de service)

  • Service Request Service target is missed (La cible du service a échoué pour la demande de service)

  • Service Request Service target warning (Avertissement de la cible du service pour la demande de service

  • Service Request time elapsed (Temps écoulé pour la demande de service)

  • Service Request time remaining (Temps restant pour la demande de service)

A

Ce champ permet de préciser les adresses destinataires de l'e-mail.

Cc

Ce champ permet de préciser les adresses qui doivent figurer dans la liste de copie conforme de l'e-mail.

Listes associées ou informations de support

Historique

Cet onglet répertorie l'historique des actions enregistrées pour l'accord.

Incidents

Cette section répertorie les demandes associées au SLA. Elle permet de lier les demandes au service.

Demandes de changement

Cette section répertorie les demandes de changement associées au SLA. Elle permet de lier les demandes de changement au service.

Problèmes

Cette section répertorie les problèmes liés au SLA. Elle permet de lier les problèmes au SLA.

Documentation

Cette section répertorie les documents liés au SLA. Elle permet de joindre tout fichier apportant des informations supplémentaires concernant l'exécution du SLA.

Audit

Cet onglet répertorie les modifications apportées aux informations relatives au SLA.
