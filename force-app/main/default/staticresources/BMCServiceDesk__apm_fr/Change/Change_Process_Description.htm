Description du processus


Le processus de gestion des changements est constitué de huit procédures : six concernent l'implémentation des changements planifiés, deux s'appliquent à l'implémentation des changements d'urgence.

  • L'enregistrement de changement est utilisé par les coordinateurs de changement lorsqu'ils gèrent des demandes de changement.

  • La planification de changement est utilisée par les coordinateurs de changement et les spécialistes pour préparer des plans d'implémentation des changements planifiés.

  • L'approbation de changement est utilisée par le gestionnaire des changements et les approbateurs de changement (représentants clients et propriétaires de service) pour approuver les changements planifiés.

  • L'implémentation de changement d'infrastructure est utilisée par les spécialistes pour implémenter les changements d'infrastructure.

  • L'implémentation de changement d'application est utilisée par les spécialistes, les administrateurs de mise en production, les représentants clients et les coordinateurs de changement pour implémenter les changements d'application.

  • La clôture de changement planifié est utilisée par les spécialistes lorsqu'ils effectuent un test de production final après l'implémentation d'un changement et par les coordinateurs de changement lorsqu'ils ferment un changement.

  • L'implémentation de changement d'urgence est utilisée par les spécialistes et les administrateurs de mise en production pour implémenter des changements destinés à résoudre des incidents.

  • La clôture de changement d'urgence est utilisée par les spécialistes et les coordinateurs de changement pour exécuter et fermer les changements d'urgence.
