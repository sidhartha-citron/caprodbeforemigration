Procédure 6 - Clôture de changement planifié

Une fois qu'un changement a été mis en production, un spécialiste (aidé d'un utilisateur s'il ne dispose pas de droits d'accès suffisants) effectue un test pour vérifier que l'implémentation s'est bien déroulée.

Si le spécialiste détermine que toutes les conditions du changement ont été remplies, la CMDB, les informations sur la gestion de la capacité et la gestion de la continuité sont mises à jour si nécessaire. Le coordinateur de changement informe alors les demandeurs et les approbateurs du changement que celui-ci a été implémenté correctement. Puis le coordinateur de changement ferme la demande de changement.

Toutefois, si toutes les conditions n'ont pas été remplies, le spécialiste ayant effectué le test de production détermine si le changement doit être annulé (retour arrière). Si le changement n'améliore pas la situation ou qu'il génère un risque pour la sécurité et pour l'intégrité des données, il est annulé.

Si l'implémentation présente néanmoins des chances de réussite, le coordinateur de changement crée les tâches supplémentaires nécessaires. Toutefois, s'il n'est pas possible de déterminer clairement comment toutes les conditions peuvent être remplies, le coordinateur de changement informe les demandeurs et les approbateurs de la situation et ferme la demande de changement.
