Etape de procédure

Instructions de travail pour les coordinateurs de changement

6.9.1

Une fois le traitement du changement terminé, définissez le champ Statut des tâches n'ayant pas besoin d'être exécutées (par exemple, si l'implémentation du changement a été refusée par les approbateurs) sur Annulé. Dans le champ Résolution spécifiez les raisons de l'annulation de la tâche.

6.9.2

Vérifiez que toutes les tâches liées à la demande de changement ont atteint leur statut final (Failed (Echec), Terminé ou Annulé). Sélectionnez ensuite la catégorie de clôture appropriée dans le champ Catégorie de clôture du changement.

Remarque :

vous ne pouvez définir le statut de la demande de changement sur Terminé que lorsque toutes les tâches liées ont atteint leur statut final (Failed (Echec), Annulé ou Terminé).

6.9.3

Définissez le champ Statut de la demande de changement sur Terminé.

6.9.4

Saisissez la date de fin réelle de l'implémentation dans le champ Date de fin réelle.
