Instructions de travail pour les spécialistes

4.3.1

Les tâches de finalisation d'un changement peuvent inclure des formations pour les opérateurs, des tests de nouveaux équipements dans un environnement de laboratoire, etc. Ces tâches permettent de garantir la disponibilité opérationnelle.

Lorsque le champ Statut de votre tâche est défini sur Assigned (Affecté), ouvrez-la et lisez les instructions dans le champ Description et les actions Remarques.

4.3.2

Si vous n'êtes pas prêt à commencer à travailler sur la tâche, définissez le champ Statut sur Accepted (Accepté).

Remarque :

si la tâche doit être affectée à un autre spécialiste, demandez au coordinateur de changement de modifier son affectation. Pour cela, définissez le champ Statut de la tâche sur Refusé et ajoutez une action Remarques pour indiquer la raison du refus.

4.3.3

Lorsque vous êtes prêt à commencer le traitement, définissez le champ Statut de la tâche sur En cours et effectuez les dernières préparations avant l'implémentation du changement.

Remarque :

si vous ne pouvez poursuivre le traitement de la tâche (par exemple parce que vous attendez la livraison d'équipement), définissez son champ Statut sur Waiting for... (En attente de...) et ajoutez une action Remarques pour spécifier l'objet de l'attente et quand vous pensez pouvoir terminer la tâche.

4.3.4

Une fois le traitement terminé, définissez le champ Statut de la tâche sur Terminé.

Si vous n'avez pas pu terminer la tâche, définissez son champ Statut de la tâche sur Failed (Echec) et, dans le champ Résolution, spécifiez la raison pour laquelle vous n'avez pas pu la finir.

4.3.5

Dans le champ Durée (HH:MM) d'une action Remarques, saisissez le temps que vous avez consacré à l'exécution de la tâche.
