Etape de procédure

Instructions de travail pour les représentants clients

5.3.1

Lorsque le champ Statut de votre tâche de test de la nouvelle mise en production passe sur Assigned (Affecté), ouvrez la tâche et lisez les instructions dans le champ Description et les actions Remarques.

5.3.2

Si vous ne pouvez pas commencer le traitement de la tâche, définissez le champ Statut de la tâche sur Accepted (Accepté).

Remarque :

si la tâche doit être affectée à une autre personne, demandez au coordinateur de changement de modifier son affectation. Pour cela, définissez le champ Statut de la tâche sur Refusé et ajoutez une action Remarques pour indiquer la raison du refus.

5.3.3

Lorsque vous êtes prêt à tester la nouvelle mise en production, définissez le champ Statut de la tâche sur En cours et effectuez les tests dans l'environnement de test.

5.3.4

Décrivez les tests et leurs résultats dans un document que vous joignez à la tâche.

Remarque :

si vous ne pouvez poursuivre le traitement de la tâche (par exemple si l'accès à l'environnement de test est temporairement impossible), définissez son champ Statut sur Waiting for... (En attente de...) et ajoutez une action Remarques pour spécifier l'objet de l'attente et quand vous pensez pouvoir terminer les tests.

5.3.5

Lorsque vous avez terminé les tests de la nouvelle mise en production et s'ils sont tous concluants, définissez le champ Statut de la tâche sur Terminé.

Si la nouvelle mise en production n'a pas réussi tous les tests, définissez le champ Statut de la tâche sur Failed (Echec) et, dans le champ Résolution, indiquez les tests non concluants.

5.3.6

Dans le champ Durée (HH:MM) de l'action Remarques, saisissez le temps que vous avez consacré à l'exécution de la tâche.
