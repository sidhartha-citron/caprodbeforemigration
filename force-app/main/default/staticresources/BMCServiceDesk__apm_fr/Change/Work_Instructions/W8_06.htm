Etape de procédure

Instructions de travail pour les coordinateurs de changement

8.6.1

Examinez les détails des demandes pour lesquelles le changement d'urgence a été implémenté.

8.6.2

Ouvrez une nouvelle demande de changement en cliquant sur Créer et lier à l'incident, dans la section Demandes de changement liées de la demande.

8.6.3

Sélectionnez-vous dans le champ ID initiateur du changement.

8.6.4

Sélectionnez le modèle de changement d'urgence dans le champ Modèle de la demande de changement.

8.6.6

Saisissez une description rapide du changement d'urgence dans le champ Description et saisissez Correction dans le champ Reason (Motif).

8.6.7

Sélectionnez Urgent dans le champ Type de changement, et la catégorie appropriée dans le champ Catégorie de changement.

8.6.8

Sélectionnez En cours dans le champ Statut du nouveau changement et enregistrez la demande de changement.

8.6.9

Ajoutez une action Remarques pour résumer le changement.

8.6.5

Dans la section EC et services associés, sélectionnez le service affecté par l'incident pour lequel le changement d'urgence a été implémenté.

8.6.10

Si la CMDB doit être mise à jour, affectez la tâche de mise à jour de la CMDB à l'administrateur de configuration de votre groupe et définissez son champ Statut sur Assigned (Affecté). S'il n'est pas nécessaire de mettre à jour la CMDB, définissez le champ Statut de cette tâche sur Annulé et, dans le champ Résolution, spécifiez la raison pour laquelle aucune mise à jour de la CMDB n'est nécessaire.

8.6.11

Assurez-vous d'être bien sélectionné dans le champ Propriétaire de la tâche visant à informer le propriétaire du service que le changement d'urgence a été implémenté.
