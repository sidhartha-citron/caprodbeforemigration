Etape de procédure

Instructions de travail pour les spécialistes

5.8.1

Dans le champ Résolution spécifiez les opérations réalisées pour exécuter la demande. Utilisez ce champ pour fournir des instructions détaillées sur une solution (par exemple si l'utilisateur doit modifier un paramètre), pour répondre à une demande d'informations ou pour expliquer pourquoi la demande ne peut être résolue s'il est impossible de fournir une solution permanente ou de contournement.

5.8.2

Sélectionnez la catégorie de clôture appropriée dans le champ Catégorie de clôture et définissez le champ Statut de la demande sur Terminé.

5.8.3

Sélectionnez l'EC approprié dans le champ EC s'il n'est pas rempli ou si l'EC renseigné n'est pas correct.

Remarque :

si l'EC n'est pas enregistré dans la CMDB mais qu'il fait partie de la gestion des configurations, sélectionnez l'EC ayant le code NORECORD.

L'EC présentant le code NORECORD n'est peut-être pas disponible par défaut.

Remarque :

si la demande ne concerne pas un EC spécifique ou que l'EC ne fait pas partie du processus de gestion des configurations, sélectionnez l'EC ayant le code N/A.

5.8.4

Si le champ Impact est défini sur Elevé ou Majeur, indiquez la date et l'heure auxquelles le service est devenu indisponible dans le champ Outage start date (Date de début de l'interruption), ainsi que la date et l'heure de reprise du service dans le champ Outage end date (Date de fin de l'interruption).

5.8.5

Si vous avez résolu un incident et constatez que d'autres incidents similaires pourraient se produire si la cause fondamentale n'est pas éliminée, vérifiez si le problème ayant entraîné l'incident est déjà enregistré. Si c'est le cas, associez la demande résolue à ce problème.

Si le problème à l'origine de l'incident n'est pas enregistré, avertissez le coordinateur de problème du service affecté par e-mail ou par téléphone et veillez à fournir le numéro de demande de l'incident et la raison pour laquelle il risque de se reproduire.

5.8.6

Dans le champ Durée (HH:MM) de l'action Remarques, saisissez le temps que vous avez consacré à la résolution de la demande.

5.8.7

Vérifiez une dernière fois que les informations de la demande sont correctes et complètes.

5.8.8

Si des demandes similaires sont susceptibles d'être soumises, créez un article de connaissance sur la solution fournie.
