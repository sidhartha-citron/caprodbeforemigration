Etape de procédure

Instructions de travail pour les analystes du centre de services

1.13.1

Sélectionnez la catégorie de demande applicable dans le champ Catégorie.

1.13.2

Si vous créez une demande de service, sélectionnez une définition
de demande à utiliser dans le champ Définition de la demande. Les définitions de demande pour
la catégorie sélectionnée s'affichent.

1.13.3

Saisissez une description brève mais significative dans le champ Description .

1.13.4

Sélectionnez le mode de soumission de la demande au centre de services dans le champ Type de contact .

1.13.5

Sélectionnez le service concerné dans le champ Service .

Remarque :

si la demande concerne une plainte, un éloge ou une demande de développement d'un nouveau service, sélectionnez le service N/A - Service does not (yet) exist or is out of scope (Le service n'existe pas (encore) ou est hors du champ d'application).

1.13.6

Si la demande concerne un élément de configuration (EC) spécifique ou s'il est probable que celui-ci en soit la cause, sélectionnez l'EC dans le champ EC.

Remarque :

si l'EC n'est pas enregistré dans la CMDB mais qu'il fait partie du processus de gestion des configurations, sélectionnez l'EC ayant le code NORECORD.

Remarque :

si la demande ne concerne pas un EC spécifique ou que l'EC ne fait pas partie du processus de gestion des configurations, sélectionnez l'EC ayant le code N/A.

1.13.7

Enregistrez la demande.

1.13.8

Si le type de demande est défini sur Demande de service, les interventions correspondantes sont générées et une fenêtre contextuelle s'affiche, dans laquelle vous devez répondre aux questions relatives à la demande de service.

1.13.9

Ajoutez une action Remarques pour saisir toute information supplémentaire pouvant être fournie par l'utilisateur et s'avérer utile pour résoudre la demande.

1.13.10

Si type de demande est défini sur Incident, vérifiez l'application de gestion des services pour savoir quels changements, implémentés au cours des dernières 24 heures, ont pu causer l'incident. Si vous trouvez un changement semblant être la cause de l'incident, associez-le dans la liste correspondante Demandes de changement liées. Ajoutez une action Remarques pour indiquer en quoi l'implémentation de ce changement a causé l'incident.
