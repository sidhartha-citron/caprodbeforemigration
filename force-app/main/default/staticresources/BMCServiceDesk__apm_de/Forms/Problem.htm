Problem


In der Tabelle unten werden die Felder des Problemformulars aufgeführt und Verwendungsrichtlinien für jedes Feld bereitgestellt.

Feld

Verwendung

Problem-Nummer

Dieses Feld enthält die eindeutige Problemnummer. Diese Nummer wird von der Anwendung automatisch erstellt.

Problemdetails

Quelle des Problems

Verwenden Sie dieses Feld, um die Quelle aus folgender Liste auszuwählen:

  • Incident

  • Event-Auslöser

  • Kunde

  • Mitarbeiter

Kategorie

Verwenden Sie dieses Feld, um die passende Kategorie auszuwählen.

Bekannter Fehler

Wählen Sie dieses Feld, um anzugeben, dass es sich bei dem Problem um einen bekannten Fehler handelt.

Beschreibung

Verwenden Sie dieses Feld, um eine kurze Beschreibung der Symptome einzugeben, die das Problem verursacht.

Lösung

Verwenden Sie dieses Feld, um kurz zu beschreiben, wie das Problem gelöst wurde, d. h., wie dessen Ursache beseitigt oder eine Umgehungslösung gefunden wurde.

Umgehungslösung

Verwenden Sie dieses Feld, um die Umgehungslösung zu beschreiben, die zur Lösung von durch dieses Problem verursachten Incidents verwendet werden soll, bis eine strukturelle Lösung implementiert wird.

Status und Priorität

Auswirkung

Verwenden Sie dieses Feld, um anzugeben, in welchem Umfang der Service beeinträchtigt ist. Die folgenden Optionen sind möglich:

  • Geringe – Service für 1 Benutzer verschlechtert

  • Mittlere – Service für 1 Benutzer ausgefallen oder für mehrere Benutzer verschlechtert

  • Hohe – Service für mehrere Benutzer verschlechtert

Beachten Sie, dass ein Service als verschlechtert gilt, wenn einige seiner Funktionen ausfallen oder die Antwortzeit des Service langsam ist. Ein Service gilt als ausgefallen, wenn keine seiner Funktionen verfügbar sind.

Dringlichkeit

Verwenden Sie dieses Feld, um die Dringlichkeit aus der folgenden Liste auszuwählen:

  • Gering – Antwort über Standardverfahren und je nach zeitlicher Verfügbarkeit

  • Mittel – Antwort über Standardverfahren und innerhalb normaler Aufsichtsstrukturen

  • Hoch – Sofortige Antwort erforderlich

Priorität

Die Anwendung wählt die Priorität, nachdem die Auswirkungs- und die Dringlichkeitsstufe ausgewählt wurden.

Status

Verwenden Sie dieses Feld, um den passenden Status für das Problem auszuwählen. Die folgenden Optionen sind möglich:

Geöffnet

Der anfängliche Status des Problems.

Abgelehnt

Das Problem muss einer anderen Warteschlange oder einem anderen Mitarbeiter zugewiesen werden.

Zugewiesen

Die Analyse des Problems kann beginnen.

Angenommen

Mit der Analyse des Problems wird begonnen, sobald der zugewiesene Mitarbeiter verfügbar ist.

In Bearbeitung

Das Problem wird analysiert.

Bekannter Fehler

Die Ursache des Problems wurde gefunden und im Falle eines reaktiv identifizierten Problems wurde eine temporäre Umgehungslösung vorgeschlagen.

Warten auf…

Derzeit ist kein Fortschritt bei der Analyse des Problems möglich.

Analysiert

Die Ursache des Problems wurde gefunden und es wurde eine strukturelle Lösung vorgeschlagen, oder es war nicht möglich, eine praktische strukturelle Lösung vorzuschlagen.

Change angefordert

Das Problem wurde an den Change Management-Prozess weitergeleitet, um die vorgeschlagene strukturelle Lösung zu implementieren.

Change ausstehend

Zur Implementierung der vorgeschlagenen strukturellen Lösung wurde ein Change registriert.

Change fertig gestellt

Der Change, der für die Implementierung der vorgeschlagenen strukturellen Lösung registriert wurde, wurde abgeschlossen.

Dead-End

Es ist nicht möglich, das Problem zu lösen, weil entweder die Ursache nicht gefunden werden kann oder es derzeit nicht möglich ist, eine praktische strukturelle Lösung vorzuschlagen.

Keine Lösung

Es wurde keine Lösung gefunden.

Gelöst

Es wurde eine strukturelle Lösung für das Problem implementiert.

Geschlossen

Das Problem ist geschlossen.

Abschlusskategorie

Verwenden Sie dieses Feld, um die Abschlusskategorie aus der folgenden Liste auszuwählen:

  • Duplikat

  • Gelöst/behoben

  • Keine Lösung möglich

  • Umgehungslösung erstellt

Nachricht

Verwenden Sie dieses Feld, um die Nachricht auszuwählen, mit der dieses Problem verknüpft werden soll.

Inaktiv

Aktivieren Sie dieses Feld, wenn das Problem nicht mehr gültig ist, sein Record jedoch aus Gründen der Konformität mit den Archivierungsrichtlinien erhalten bleiben muss.

Details zu Datum und Uhrzeit

Öffnungsdatum

In diesem Feld sind automatisch Erstellungsdatum und -uhrzeit für das Problem eingestellt.

Fälligkeitsdatum

In diesem Feld sind automatisch Datum und Uhrzeit für die Fertigstellung der Ursachenanalyse eingestellt, nachdem der Schweregrad festgelegt wurde.

Abschlussdatum

In diesem Feld ist automatisch der Zeitpunkt (Datum und Uhrzeit) eingestellt, zu dem der Status des Problems auf „Dead-End“ oder „Gelöst“ festgelegt wurde.

Mitarbeiterzuweisung – Details

Besitzer

In diesem Feld wird die Warteschlange oder der Mitarbeiter angezeigt, der/dem das Problem zugewiesen ist.

Verknüpfte Listen

Aktionsverlauf

In diesem Bereich werden die Aktionen aufgeführt, die in der Vergangenheit für das Problem registriert wurden.

Verknüpfte Incidents

In diesem Bereich werden die mit dem Problem in Verbindung stehenden Incidents aufgelistet. Verwenden Sie diesen Bereich, um Anforderungen zu verknüpfen, die mit dem Problem in Verbindung gebracht wurden, entweder weil die Anforderungen durch das Problem verursacht wurden (für die Problemkategorie Reactive - Existing Problem (Reaktiv – Bestehendes Problem)) oder weil sie als Warnhinweis auf das Problem registriert wurden (für die Problemkategorie Proactive - Anticipated Problem (Proaktiv – Erwartetes Problem)).

Verknüpfte Change Requests

In diesem Bereich werden die mit dem Problem in Verbindung stehenden Change-Anforderungen aufgelistet. Verwenden Sie diesen Bereich zum Verknüpfen von Change-Anforderungen mit dem Problem.

Verknüpfte Services und CIs

In diesem Bereich werden die mit dem Problem verknüpften Services und CIs aufgeführt. Verwenden Sie diesen Bereich zum Verknüpfen von Services oder CIs mit dem Problem.

Verknüpfte Aufgaben

In diesem Bereich werden die mit dem Problem in Verbindung stehenden Aufgaben aufgelistet. Verwenden Sie diesen Bereich zum Verknüpfen von Aufgaben mit dem Problem.

Hinweise und Anlagen

In diesem Bereich werden die mit dem Problem in Verbindung stehenden Dokumente aufgelistet. Verwenden Sie diesen Bereich, um Dateien anzuhängen, die zusätzliche Informationen zur Analyse und Lösung des Problems enthalten.

Problem-Historie

In diesem Bereich werden die an den Informationen zum Problem vorgenommenen Änderungen aufgeführt.

Verknüpfte Releases

In diesem Bereich werden die mit dem Problem in Verbindung stehenden Releases aufgelistet. Verwenden Sie diesen Bereich zum Verknüpfen von Releases mit dem Problem.
