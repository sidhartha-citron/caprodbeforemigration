Service Level Agreement


In der Tabelle unten werden die Felder des Service Level Agreement-Formulars aufgeführt und Verwendungsrichtlinien für jedes Feld bereitgestellt.

Feld

Verwendung

Vereinbarungstitel

Geben Sie in diesem Feld den Namen des Service Level Agreements (SLA) ein, z. B. Human Resource Management P21 (SAP) für UKPLANT.

Beschreibung

Verwenden Sie dieses Feld, um die Beschreibung des SLA einzugeben.

Allgemeine Informationen

Vereinbarungstyp

Verwenden Sie dieses Feld, um den Vereinbarungstyp aus der folgenden Liste auszuwählen:

  • Service Level Agreement

  • Operational Level Agreement

  • Unterstützender Vertrag

Service

Wählen Sie in diesem Feld den Service, für den Sie das SLA erstellen.

Konto

Wählen Sie in diesem Feld die Unternehmen, für die das SLA gilt. Die folgenden Optionen sind möglich:

  • Vereinbarung gilt für alle Konten.

  • Vereinbarung gilt für ausgewählte Konten.

Wenn Sie Vereinbarung gilt für ausgewählte Konten wählen, wählen Sie die benötigten Konten aus.

Gültig ab

Verwenden Sie dieses Feld, um das Datum einzugeben, ab dem das SLA gültig ist.

Gültig bis

Geben Sie in diesem Feld das Datum an, zu dem das SLA spätestens verlängert werden muss, damit es nicht abläuft.

Geplante Überprüfung

Verwenden Sie dieses Feld, um das Datum einzugeben, an dem das SLA geprüft werden muss.

Support-Zeiten

Wählen Sie in diesem Feld die entsprechenden Support-Zeiten für das SLA aus. Diese ausgewählten Servicezeiten werden verwendet, um Reaktionszeiten und Lösungszeiten zu berechnen.

Vereinbarungsbesitzer

Wählen Sie in diesem Feld den Benutzer aus, der als Service Level Manager fungiert.

Serviceziele

Servicezieltitel

Verwenden Sie dieses Feld, um den Servicezieltitel einzugeben.

Beschreibung

Verwenden Sie dieses Feld, um die Beschreibung des Serviceziels einzugeben.

Serviceziel

Anwenden für

Wählen Sie in diesem Feld das Modul aus, für das das Ziel gilt. Die folgenden Optionen sind möglich:

  • Incident

  • Change Request

Zieltyp

Verwenden Sie dieses Feld, um den Zieltyp aus der folgenden Liste auszuwählen:

  • Lösungszeit

  • Reaktionszeit

Support-Zeiten

Wählen Sie in diesem Feld die entsprechenden Support-Zeiten für das Serviceziel aus. Diese ausgewählten Servicezeiten werden verwendet, um Reaktionszeiten und Lösungszeiten zu berechnen.

Ziel

Legen Sie in diesem Feld die Zielzeit bis zum Abschluss des Service in Tagen, Stunden und Minuten fest. Sie können stattdessen auch das Feld Geschäftstage verwenden, um die Zielzeit für den Abschluss des Service in Geschäftstagen einzugeben.

Qualifikation

Bedingungen

Wählen Sie in diesem Feld die Felder und Werte, für die das Kriterium festgelegt werden soll.

Messung

Messung

Wählen Sie in diesem Feld die Statuswerte für das Starten, Anhalten und Stoppen der SLA-Uhr.

Meilensteine und Aktionen

Meilenstein

Legen Sie in diesem Feld fest, wann eine automatische E-Mail-Benachrichtigung gesendet werden muss. Die folgenden Optionen für Meilensteine stehen zur Auswahl:

  • Vergangene Zeit (Prozent)

  • Verbleibende Zeit (in Prozent)

  • Serviceziel ist storniert

  • Serviceziel ist erfüllt

  • Serviceziel ist verfehlt

  • Servicezielwarnung tritt auf

  • Vergangene Zeit

  • Verbleibende Zeit

Aktion

Wählen Sie in diesem Feld die Vorlage für die E-Mail, die beim Erreichen eines Meilensteins gesendet werden soll. Die folgenden Optionen sind möglich:

  • Incident – vergangene Zeit (Prozent)

  • Incident – verbleibende Zeit (in Prozent)

  • Incident-Serviceziel wurde storniert

  • Incident-Serviceziel ist erfüllt

  • Incident-Serviceziel ist verfehlt

  • Warnung zu Incident-Serviceziel

  • Incident – vergangene Zeit

  • Incident – verbleibende Zeit

  • Serviceanforderung – verbleibende Zeit (Prozent)

  • Serviceziel der Serviceanforderung wurde storniert

  • Incident-Serviceziel wurde storniert

  • Serviceziel der Serviceanforderung wurde erfüllt

  • Serviceziel der Serviceanforderung ist verfehlt

  • Warnung zu Serviceziel der Serviceanforderung

  • Serviceanforderung – vergangene Zeit

  • Serviceanforderung – verbleibende Zeit

An

Verwenden Sie dieses Feld, um anzugeben, an welche E-Mail-Adressen die E-Mail geschickt werden soll.

CC

Verwenden Sie dieses Feld, um anzugeben, an welche E-Mail-Adressen eine Kopie der E-Mail gehen soll.

Verknüpfte Listen oder zusätzliche Informationen

Historie

Auf dieser Registerkarte werden die in der Vergangenheit registrierten Aktionen für die Vereinbarung aufgelistet.

Incidents

In diesem Bereich werden die mit dem SLA in Verbindung stehenden Anforderungen aufgeführt. Verwenden Sie diese Registerkarte zum Verknüpfen von Anforderungen mit dem Service.

Change-Anforderungen

In diesem Bereich werden die mit dem SLA in Verbindung stehenden Change-Anforderungen aufgeführt. Verwenden Sie diese Registerkarte zum Verknüpfen von Change-Anforderungen mit dem Service.

Probleme

In diesem Bereich werden die mit dem SLA in Verbindung stehenden Probleme aufgeführt. Verwenden Sie diese Registerkarte zum Anfügen von Problemen an das SLA.

Dokumentation

In diesem Bereich werden die mit dem SLA in Verbindung stehenden Dokumente aufgeführt. Verwenden Sie diesen Bereich, um Dateien anzufügen, die zusätzliche Informationen für die Fertigstellung des SLA enthalten.

Prüfung

Auf dieser Registerkarte werden die an den Informationen zum SLA vorgenommenen Änderungen aufgelistet.
