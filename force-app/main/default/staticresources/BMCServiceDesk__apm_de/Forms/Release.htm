Release


In der Tabelle unten werden die Felder des Release-Formulars aufgeführt und Verwendungsrichtlinien für jedes Feld bereitgestellt.

Feld

Verwendung

Release-Nr.

Dieses Feld enthält die eindeutige Release-Nummer. Diese Nummer wird von der Anwendung automatisch erstellt.

Release-Details

Vorlage

Wählen Sie in diesem Feld die für das Release zu verwendende Vorlage.

Release-Titel

Verwenden Sie dieses Feld, um den Release-Titel einzugeben.

Beschreibung

Verwenden Sie dieses Feld, um eine kurze Beschreibung einzugeben, die den Hauptgrund für die Implementierung des Release zusammenfasst. Sie können den Grund für die Erstellung des Release angeben, z. B. Vermeidung oder Behebung eines Problems, Neue Geschäftsanforderung oder Neue gesetzliche Bestimmung.

Kategorie

Verwenden Sie dieses Feld , um die Release-Kategorie auszuwählen.

Release-Typ

Verwenden Sie dieses Feld , um den Release-Typ aus der folgenden Liste auszuwählen:

  • Schwerwiegend

  • Gering

  • Notfall

Subtyp

Verwenden Sie dieses Feld , um den Untertyp aus der folgenden Liste auszuwählen:

  • Rückstand

  • Delta

  • Vollständig

  • Paket

Bereitstellungstyp

Verwenden Sie dieses Feld , um den Bereitstellungstyp aus der folgenden Liste auszuwählen:

  • Mit Phasen

  • Ohne Phasen

Tests fertiggestellt

Geben Sie in diesem Feld das Datum und die Uhrzeit ein, zu dem die Tests fertiggestellt wurden.

Status und Priorität

Auswirkung

Verwenden Sie dieses Feld , um die Auswirkung aus der folgenden Liste auszuwählen:

  • Gering – Service für 1 Benutzer verschlechtert

  • Mittel – Service für 1 Benutzer ausgefallen oder für mehrere Benutzer verschlechtert

  • Hoch – Service für mehrere Benutzer ausgefallen

Dringlichkeit

Verwenden Sie dieses Feld , um die Dringlichkeit aus der folgenden Liste auszuwählen:

  • Gering – Antwort über Standardverfahren und je nach zeitlicher Verfügbarkeit

  • Mittel – Antwort über Standardverfahren und innerhalb normaler Aufsichtsstrukturen

  • Hoch – Sofortige Antwort erforderlich

Priorität

Die Anwendung wählt die Priorität, nachdem die Auswirkungs- und die Dringlichkeitsstufe ausgewählt wurden.

Status

Verwenden Sie dieses Feld , um den passenden Status für den Release aus der folgenden Liste auszuwählen:

Geöffnet

Der anfängliche Status des Release.

Registriert

Der Release wurde von seinem Koordinator registriert.

Zur Genehmigung

Ein Geschäftsfall wird für den Release vorbereitet oder genehmigt.

Genehmigt

Der Geschäftsfall oder Release wurde genehmigt.

In Bearbeitung

Die Anforderungen für die Implementierung des Release wurden an die Change-Koordinatoren gesendet.

Plan

Der Release befindet sich in der Planungsphase.

Bereitstellen

Der Release befindet sich in der Bereitstellungsphase.

Fertiggestellt

Die Arbeit an der Implementierung des Release wurde aus dem im Feld Closure Category (Schließungskategorie) angegebenen Grund beendet.

Geschlossen

Die Implementierung des Release wurde während der Implementierungsnachbesprechung geprüft.

Abschlusskategorie

Wählen Sie in diesem Feld die Schließungskategorie für den Release, nachdem dieser fertiggestellt wurde. Die folgenden Optionen sind möglich:

  • Business Case abgelehnt

  • Abgelehnt

  • Zurückgesetzt

  • Teilweise implementiert

  • Implementiert

Inaktiv

Wählen Sie dieses Feld, wenn der Release-Record nicht mehr gültig ist, jedoch zur Einhaltung der Archivierungsrichtlinien erhalten bleiben muss. Wenn beispielsweise ein Kunde nicht mehr zu einem Konto gehört, werden alle Release-Records dieses Kunden ungültig.

Details zu Release-Datum und -Uhrzeit

Fälligkeitsdatum

Legen Sie in diesem Feld das Datum und die Uhrzeit für die Fertigstellung der mit dem Release verknüpften Changes fest. Ausgenommen ist der Change, mit dem die Change-Anforderungen für den Release verknüpft sind.

Implementierungsstart

Geben Sie in diesem Feld das Datum und die Uhrzeit für den Beginn der Implementierung des Release an.

Implementierungsende

Geben Sie in diesem Feld das Datum und die Uhrzeit für den Abschluss der Implementierung des Release an.

Bereitstellungsstart

Geben Sie in diesem Feld das Datum und die Uhrzeit für den Beginn der Bereitstellung des Release an.

Bereitstellungsende

Geben Sie in diesem Feld das Datum und die Uhrzeit für den Abschluss der Bereitstellung des Release an.

Datum der Fertigstellung

Geben Sie in diesem Feld das Datum und die Uhrzeit für die Fertigstellung des Release ein.

Tatsächlicher Implementierungsstart

Geben Sie in diesem Feld das tatsächliche Datum und die Uhrzeit an, zu dem die Implementierung des Release begonnen hat.

Tatsächliches Implementierungsende

Geben Sie in diesem Feld das tatsächliche Datum und die Uhrzeit an, zu dem die Implementierung des Release abgeschlossen wurde.

Tatsächlicher Bereitstellungsstart

Geben Sie in diesem Feld das tatsächliche Datum und die Uhrzeit an, zu dem die Bereitstellung des Release begonnen hat.

Tatsächliches Bereitstellungsende

Geben Sie in diesem Feld das tatsächliche Datum und die Uhrzeit an, zu dem die Bereitstellung des Release abgeschlossen wurde.

Geschäftliche und finanzielle Details

Verfügbares Budget

Verwenden Sie dieses Feld, um anzugeben, ob ein Budget für diesen Release verfügbar ist.

Geschäftsbegründung

Wählen Sie in diesem Feld die zutreffende Geschäftsbegründung für den Release. Die folgenden Optionen sind möglich:

  • Geschäftsstrategie

  • Strategie des Geschäftsfelds

  • Wartung

  • Defekt

  • Upgrade

  • Verbesserung

  • Kundenbindung

  • Sarbanes-Oxley

  • Andere

Kommentare

Verwenden Sie dieses Feld für Kommentare zu geschäftlichen und finanziellen Aspekten.

Risikobeurteilung

Verwenden Sie dieses Feld , um die passende Risikobewertung für den Release aus folgender Liste auszuwählen:

  • Niedrig

  • Mittel

  • Hoch

Erforderliche Verträge überprüft

Verwenden Sie dieses Feld, um anzugeben, ob die erforderlichen Verträge überprüft wurden.

Erforderliche Lizenzen überprüft

Verwenden Sie dieses Feld, um anzugeben, ob die erforderlichen Lizenzen überprüft wurden.

Erforderliche CIs verfügbar

Verwenden Sie dieses Feld, um anzugeben, ob die erforderlichen Konfigurationselemente (CIs) verfügbar sind.

Mitarbeiterzuweisung – Details

Besitzer

In diesem Feld wird die Warteschlange oder der Mitarbeiter angezeigt, der/dem der Release zugewiesen ist.

Release-Koordinator

Verwenden Sie dieses Feld, um sich selbst als für diesen Release verantwortlichen Release-Koordinator festzulegen.

Verknüpfte Listen

Aktionsverlauf

In diesem Bereich werden die Aktionen aufgeführt, die in der Vergangenheit für den Release registriert wurden.

Verknüpfte Aufgaben

In diesem Bereich werden die mit dem Release verknüpften Aufgaben aufgeführt. Verwenden Sie diesen Bereich zum Verknüpfen von Aufgaben mit dem Release.

Verknüpfte Probleme

In diesem Bereich werden die mit dem Release verknüpften Probleme aufgeführt. Verwenden Sie diesen Bereich zum Verknüpfen von Problemen mit dem Release.

Verknüpfte Incidents

In diesem Bereich werden die mit dem Release verknüpften Incidents aufgeführt. Verwenden Sie diesen Bereich zum Verknüpfen von Incidents mit dem Release.

Genehmigungsverlauf

In diesem Bereich werden die für den Release erstellten Genehmigungsdatensätze aufgeführt.

Verknüpfte Change Requests

In diesem Bereich werden die mit dem Release verknüpften Change-Anforderungen aufgeführt. Verwenden Sie diesen Bereich zum Verknüpfen von Change-Anforderungen mit dem Release.

Verknüpfte Services und CIs

In diesem Bereich werden die mit dem Release verknüpften Services und CIs aufgeführt. Verwenden Sie diesen Bereich, um die CIs und Services zu verknüpfen, auf die der Release Auswirkungen hat.

Hinweise und Anlagen

In diesem Bereich werden die mit dem Release verknüpften Dokumente aufgeführt. Verwenden Sie diesen Bereich, um Dateien anzufügen, die zusätzliche Informationen für den Abschluss des Release enthalten.

Release-Verlauf

In diesem Bereich werden die an den Release-Informationen vorgenommenen Änderungen aufgeführt.
