Change Request


In der unten stehenden Tabelle werden die Felder des Change-Formulars aufgeführt und Verwendungsrichtlinien für jedes Feld bereitgestellt.

Feld

Verwendung

Change-Nummer

Dieses Feld enthält eine eindeutige Change-Nummer.
Diese Nummer wird von der Anwendung generiert, nachdem Sie den Change Request gespeichert haben.

Initiatordetails

Initiator-ID

Wählen Sie in diesem Feld sich selbst als Koordinator für den Change aus.

Change-Details

Vorlage

Wählen Sie in diesem Feld eine Vorlage aus, mit deren Hilfe Sie den Change Request und seine Aufgaben registrieren können.

Beachten Sie, dass die Aufgaben, die nach Anwendung der Vorlage mit dem Change Request verknüpft werden, erst angezeigt werden, nachdem der Change Request gespeichert wurde.

Kategorie

Verwenden Sie dieses Feld , um die korrekte Kategorie für den Change Request auszuwählen.

Beschreibung

Geben Sie in diesem Feld eine Kurzbeschreibung des Change Request ein.

Gründe

Geben Sie in diesem Feld den Grund für den Change ein.

Rolloutplan

Geben Sie in diesem Feld die Verfahren für die Implementierung des Change ein.

Backout-Plan

Geben Sie in diesem Feld den Backout-Plan ein, um den anfänglichen Zustand des Konfigurationselements vor der Implementierung des Change wiederherzustellen.

Genehmigt

Verwenden Sie dieses Feld, wenn Sie einen vorgenehmigten Change Request erstellen möchten, um die Implementierung zu beschleunigen.

Review-Kommentare

Geben Sie in diesem Feld Ihre Kommentare zur Prüfung des aktuellen Change ein. Dazu können die Ursachen für das Fehlschlagen des Change gehören und wie dies in Zukunft vermieden werden kann.

Change-Typ

Verwenden Sie dieses Feld, um den Change-Typ aus der folgenden Liste von Optionen auszuwählen:

  • Notfall

  • Normal

  • Standard

Change-Kategorie

Verwenden Sie dieses Feld, um die Change-Kategorie aus der folgenden Liste von Optionen auszuwählen:

  • Schwerwiegend

  • Gering

  • Wesentlich

Auswirkung

Wählen Sie in diesem Feld aus, in welchem Umfang der Service betroffen ist. Die folgenden Optionen sind möglich:

  • Gering – Service für 1 Benutzer verschlechtert

  • Mittel– Service für 1 Benutzer ausgefallen oder für mehrere Benutzer verschlechtert

  • Hoch – Service für mehrere Benutzer verschlechtert

Dringlichkeit

Verwenden Sie dieses Feld, um die Dringlichkeitsstufe aus folgender Liste von Optionen auszuwählen:

  • Gering – Antwort über Standardverfahren und je nach zeitlicher Verfügbarkeit

  • Mittel – Antwort über Standardverfahren und innerhalb normaler Aufsichtsstrukturen

  • Hoch – Sofortige Antwort erforderlich

Priorität

Die Anwendung weist die Priorität zu, nachdem die Auswirkungs- und Dringlichkeitsstufe ausgewählt wurde.

Status

Verwenden Sie dieses Feld, um den passenden Status für den Change Request auszuwählen. Die folgenden Optionen sind möglich:

Geöffnet

Der anfängliche Status des Change Request.

Registriert

Der Change Request ist noch nicht bereit für die Risiko- und Auswirkungsanalyse.

Risiko und Auswirkung

Die Risiko- und Auswirkungsanalyse für den Change Request wird durchgeführt und der Implementierungsplan wird vorbereitet.

Zur Genehmigung

Der Implementierungsplan wurde vorbereitet und muss durch den Change Manager geprüft werden, wonach die Genehmigung angefordert werden muss.

Genehmigung angefordert

Die Genehmigung für den Change wurde durch den Change Manager angefordert.

Genehmigung ausstehend

Die Genehmigung ist ausstehend.

Genehmigt

Der Change Request wurde genehmigt.

In Bearbeitung

Der Change Request wird derzeit implementiert.

Abgelehnt

Der Change Request wurde abgelehnt.

Fertiggestellt

Hinsichtlich dieses Change werden keine weiteren Schritte mehr unternommen, da dieser implementiert, zurückgesetzt oder gemäß den Angaben im Feld Abschlusskategorie abgelehnt wurde.

Geschlossen

Der Change Request ist geschlossen.

Abschlusskategorie

Verwenden Sie dieses Feld, um die passende Abschlusskategorie für den Change Request nach dessen Abschluss auszuwählen. Die folgenden Optionen sind möglich:

  • Von Anforderer zurückgezogen

  • Abgelehnt

  • Zurückgesetzt

  • Teilweise implementiert

  • Implementiert

Inaktiv

Wählen Sie dieses Feld, wenn der Change nicht mehr gültig ist, jedoch gemäß den Archivierungsrichtlinien aufbewahrt werden muss.

Details zu Change-Datum und -Uhrzeit

Öffnungsdatum

In diesem Feld sind Erstellungsdatum und -uhrzeit für den Change Request eingestellt.

Antwortdatum

In diesem Feld sind Datum und Uhrzeit der Antwort eingestellt. Dieses Feld enthält das Datum und die Uhrzeit, um die die Mitarbeiter mit dem Initiator über den Change gesprochen haben.

Überprüfungsdatum

Wählen Sie in diesem Feld das Datum für die Prüfung des Change Request, nachdem dieser geschlossen wurde.

Abschlussdatum

Wählen Sie in diesem Feld das Abschlussdatum des Change Request aus.

Geplanter Start

Wählen Sie in diesem Feld das Datum für die geplante Implementierung des Change Request.

Geplantes Ende

Wählen Sie in diesem Feld das Datum, an dem der Change Request fertig gestellt werden soll.

Tatsächliches Startdatum

Wählen Sie in diesem Feld das Datum, an dem Sie mit der Implementierung des Change beginnen.

Tatsächliches Enddatum

Verwenden Sie dieses Feld, um das tatsächliche Enddatum für die Implementierung auszuwählen.

Zuweisungsdetails

Besitzer

Verwenden Sie dieses Feld, um den Change Request einer Warteschlange oder einem Mitarbeiter zuzuweisen.

Verknüpfte Listen

Aktionsverlauf

In diesem Abschnitt können Sie Verlaufaktionen hinzufügen, die für den Change Request registriert wurden.

Verknüpfte Probleme

In diesem Abschnitt können Sie verknüpfte Probleme hinzufügen und verbinden. Die Probleme, für die der Change Request erstellt wurde, werden in diesem Abschnitt aufgelistet.

Verknüpfte Services und CIs

In diesem Abschnitt werden die mit dem Change Request verknüpften Konfigurationselemente (CIs) aufgelistet. In diesem Abschnitt können Sie die Services und CIs verbinden, die mit dem Change verknüpft sind.

Change-Bewertungen

In diesem Abschnitt werden die mit dem Change Request verknüpften Change-Bewertungen aufgelistet. Hier können Sie die Change-Bewertungen erstellen und verknüpfen, die für den Change registriert werden sollen.

Genehmigungsverlauf

Mithilfe dieser Liste können Sie Change Requests genehmigen oder die für den Change Request registrierte Genehmigungshistorie anzeigen.

Verknüpfte Aufgaben

In diesem Abschnitt werden die mit dem Change Request verknüpften Aufgaben aufgelistet. In diesem Abschnitt können Sie Aufgaben erstellen und verknüpfen, die für den Change Request registriert wurden.

Verknüpfte Incidents

In diesem Abschnitt werden die mit dem Change Request verknüpften Incidents aufgelistet. Hier können Sie Incidents verknüpfen, für die der Change Request erstellt wurde.

Hinweise und Anlagen

In diesem Abschnitt werden die mit dem Change Request verknüpften Dokumente aufgelistet. Hier können Sie Dateien anfügen, die weitere Informationen zur Risiko- und Auswirkungsanalyse, Genehmigung, Implementierung und zum Rollback des Change Request enthalten.

Change Request-Prüfung

In diesem Abschnitt werden die an den Change-Informationen vorgenommenen Änderungen aufgeführt.

Verknüpfte Releases

In diesem Abschnitt werden die mit dem Change Request verknüpften Releases aufgeführt. Hier können Sie die Releases hinzufügen, die für den Change Request registriert wurden.
