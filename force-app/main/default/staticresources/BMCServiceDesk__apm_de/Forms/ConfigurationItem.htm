Konfigurationselement


In der untenstehenden Tabelle werden die Felder des Konfigurationselement-Formulars aufgeführt und Verwendungsrichtlinien für jedes Feld bereitgestellt.

Feld

Verwendung

Allgemein

Instanzname

Geben Sie in diesem Feld den eindeutigen Namen für das CI ein.

Beschreibung

Geben Sie in diesem Feld die Beschreibung für das CI ein.

Kurzbeschreibung

Geben Sie in diesem Feld eine kurze Beschreibung des CI für die Anzeige im CI-Explorer ein.

Hinweise

Geben Sie in diesem Feld zusätzliche Informationen über das CI ein.

Herstellername

Verwenden Sie dieses Feld, um den Hersteller des CI auszuwählen.

Modell

Geben Sie in diesem Feld den Modellnamen ein, falls es sich um ein Hardware-Cl handelt.

Versionsnummer

  • Geben Sie in diesem Feld die Versionsnummer ein, falls es sich um ein Software- oder Dokumentations-CI handelt.

  • Für Software-Lizenzzertifikat-CIs geben Sie hier die Versionen an, für die die Lizenz gilt.

Seriennummer

Geben Sie in diesem Feld die Seriennummer des CIs ein. Für Software-Lizenzzertifikat-CIs geben Sie hier die Nummer des Lizenzzertifikats an.
Falls es sich bei dem CI um eine Datenbank, um Software, um ein Softwareverteilungspaket, um Software-/Benutzeroberflächen-Konfigurationseinstellungen, um eine Website oder eine WAN-Verknüpfung handelt, lassen Sie das Feld leer.

Asset-Nummer

Geben Sie in diesem Feld die Nummer ein, unter der das CI in der Finanzanwendung (z. B. SAP) registriert wurde.

Token-ID

Geben Sie in diesem Feld die eindeutige ID des CI ein.

Priorität

Wählen Sie in diesem Feld die Priorität des CI. Dies gibt dessen Wichtigkeit in der CMDB an.

Erstellungsdatum

In diesem Feld sind automatisch Datum und Uhrzeit der Registrierung des CI eingestellt.

Zuletzt geändert von

In diesem Feld wird der Anmeldename des Mitarbeiters gespeichert, der das CI zuletzt aktualisiert hat.

Datum der letzten Änderung

In diesem Feld sind automatisch Datum und Uhrzeit der letzten Aktualisierung des CI eingestellt.

Klassen-ID

In diesem Feld ist automatisch die Klasse des Elements eingestellt, das zu Registrierung des CI verwendet wird.

Klassenname

In diesem Feld ist automatisch der Klassenname des Elements eingestellt, das zu Registrierung des CI verwendet wird.

Unterstützt

Geben Sie in diesem Feld an, ob für dieses CI technischer Support bereitgestellt wird.

Warranty-Ablaufdatum

Geben Sie in diesem Feld das Datum an, zu dem die Garantie für das CI abläuft.

Standort

  • Wählen Sie für ein Hardware-CI den Standort, an dem sich das CI befindet.

  • Falls es sich bei dem CI um Software oder um ein Softwareverteilungspaket handelt, geben Sie in diesem Feld den Standort an, an dem sich die Masterkopie der DML befindet.

  • Für ein Software-Lizenzzertifikat-CI wählen Sie in diesem Feld den Standort der DML, in der das Zertifikat gespeichert wurde.

  • Für CIs, die Datenbanken, Software-/Benutzeroberflächen-Konfigurationseinstellungen oder Websites sind, legen Sie in diesem Feld „N/A“ fest.

Phase

Verwenden Sie dieses Feld, um die passende Phase für das CI aus der folgenden Liste auszuwählen:

  • Anforderungen

  • Definiert

  • Analysiert

  • Genehmigt

  • Diagramm erstellt

  • Design abgeschlossen

  • Entwickelt

  • Build

  • Getestet

  • Freigegeben

  • Betriebsbereit

  • Veröffentlicht

  • Unveröffentlicht

  • Stillgelegt

Als gelöscht markieren

Verwenden Sie dieses Feld, um das Cl als vorläufig gelöscht zu markieren.

Instanz-ID

In diesem Feld ist automatisch die Instanz des Elements eingestellt, das zur Registrierung des CI verwendet wird.

Übermittler

In diesem Feld wird der Anmeldename des Mitarbeiters gespeichert, der das CI registriert hat.

Dieses Feld ist nicht vordefiniert. Sie müssen das Formular anpassen, damit das Feld im Bereich „Anwenderdefiniert“ angezeigt wird.

Spezifikationen

Spezifikationen (für bestimmte Elemente spezifische Felder)

Auf der Registerkarte Spezifikationen stehen je nach Element unterschiedliche Felder zur Verfügung. Die Felder werden daher nicht in diesem Formular beschrieben.

Beziehungen

Als gelöscht markieren

Verwenden Sie dieses Feld, um die Beziehung als vorläufig gelöscht zu markieren.

Auswirkungsrichtung

Verwenden Sie dieses Feld, um die Auswirkungsrichtung aus folgender Liste auszuwählen:

  • Andere

  • Unbekannt

  • Quelle - Ziel

  • Ziel - Quelle

  • Service - untergeordneter Service

  • Service - Angebot

Verknüpfte Listen oder zusätzliche Informationen

Historie

Verwenden Sie diese Registerkarte zum Registrieren der für das CI erstellten Hinweise zum Aktionsverlauf.

Kunden

Auf dieser Registerkarte werden die mit dem CI verknüpften Kunden aufgelistet. Verwenden Sie diese Registerkarte, um die Kunden auszuwählen, die das CI verwenden oder für die das CI reserviert ist.
Wenn das CI nicht von einem bestimmten Kunden verwendet wird oder für ihn reserviert ist, lassen Sie das Feld leer.

Konten

Auf dieser Registerkarte werden die mit dem CI verknüpften Konten aufgelistet. Verwenden Sie diese Registerkarte, um die Unternehmen auszuwählen, für die das CI reserviert ist.
Wenn das CI nicht von einem bestimmten Unternehmen verwendet wird oder für dieses reserviert ist, lassen Sie das Feld leer.

Incidents

Auf dieser Registerkarte werden die mit dem CI verknüpften Incidents aufgelistet. Verwenden Sie diese Registerkarte zum Verknüpfen von Incidents mit dem CI.

Change Requests

Auf dieser Registerkarte werden die mit dem CI verknüpften Change Requests aufgelistet. Verwenden Sie diese Registerkarte zum Verknüpfen von Change Requests mit dem CI.

Probleme

Auf dieser Registerkarte werden die mit dem CI verknüpften Probleme aufgelistet. Verwenden Sie diese Registerkarte zum Verknüpfen von Problemen mit dem CI.

Releases

Auf dieser Registerkarte werden die mit dem CI verknüpften Releases aufgelistet. Verwenden Sie diese Registerkarte zum Verknüpfen von Releases mit dem CI.

Dokumentation

Auf dieser Registerkarte werden die mit dem CI verknüpften Dokumente aufgelistet. Verwenden Sie diese Registerkarte zum Anhängen von Dateien, die zusätzliche Informationen über das CI enthalten.

Prüfung

Auf dieser Registerkarte werden die an den Informationen zum CI vorgenommenen Änderungen aufgelistet.
