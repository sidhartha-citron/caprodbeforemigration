Prozessbeschreibung

Der Release Management-Prozess besteht aus den folgenden Verfahren:

  • Die Change Request-Verarbeitung wird von den Release-Koordinatoren verwendet, um die Anforderungen und Probleme zu prüfen, die an das Release Management übermittelt wurden.

  • Die Release-Definition wird von Release-Koordinatoren verwendet, um CAB-Besprechungen zu organisieren, und von CAB-Mitarbeitern, wenn diese entscheiden, welche der zur CAB-Überprüfung gesammelten Change Requests im nächsten Release erfüllt werden sollten. Release-Koordinatoren teilen anhand dieses Verfahrens auch die Anforderungen von Releases in logische Gruppen auf, die von den Change-Koordinatoren auf effiziente Weise verarbeitet werden.

  • Die Geschäftsbegründung wird von den Release-Koordinatoren verwendet, wenn für die Implementierung eines Release zusätzliches Budget erforderlich ist.

  • Die Release-Koordination wird von Release-Koordinatoren verwendet, um die Implementierung von Releases zu initiieren und bei Bedarf korrektive Maßnahmen zu beschließen.

Um weitere Informationen zu diesen Verfahren zu erhalten, klicken Sie auf Prozess , um zur grafischen Prozessdarstellung zurückzukehren, und danach auf die Boxen, welche die verschiedenen Verfahren verkörpern. Bei der grafischen Darstellung eines Verfahrens wird dessen Beschreibung im rechten Bereich angezeigt.
