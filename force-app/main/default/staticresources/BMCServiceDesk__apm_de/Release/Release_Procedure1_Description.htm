Verfahren 1, Change Request-Verarbeitung


Nachdem der Change-Koordinator eines Service einen Change Request an den Release-Koordinator eines Service weitergeleitet hat, prüft der Release-Koordinator die Anforderung, um sich einen Überblick zu verschaffen. Der Release-Koordinator weist den Change Request wieder dem Change-Koordinator zu, wenn keine Beteiligung seitens des Release Management erforderlich ist. Nur die Change Requests, die die Implementierung eines nicht standardmäßigen Change erfordern, müssen vom Release Management verarbeitet werden, mit Ausnahme von Change Requests, die zum Verhindern oder Beheben eines Problems eingesendet wurden, vorausgesetzt, dass auf die Implementierung eines solchen Change Folgendes zutrifft:

durch einen einzelnen Change-Koordinator koordiniert werden können

bewirkt keine Änderung der Änderung der Funktionen eines Service

erfordert kein zusätzliches Budget

Wenn der Change Request keine Beteiligung des Release Management erfordert, überprüft der Release-Koordinator, ob eine CAB-Genehmigung erforderlich ist. Der CAB muss den Change Request nicht prüfen, wenn er die Implementierung eines Change zum Verhindern oder Beheben eines Problems erforderte und wenn dieser Change keine Änderungen an den Funktionen eines Service bewirkt. Wenn jedoch eine CAB-Genehmigung erforderlich ist, fügt der Release-Koordinator den Change Request der Liste von Anforderungen hinzu, die während der nächsten CAB-Besprechung des Service geprüft werden sollen.
