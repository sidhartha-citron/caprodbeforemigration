Rollen und Verantwortungsbereiche


In der folgenden Tabelle werden die verschiedenen Rollen im Release Management-Prozess sowie ihre jeweiligen Verantwortungsbereiche aufgeführt. Klicken Sie auf eine Rolle, um ihr Profil anzuzeigen.

Rolle

Verantwortungsbereich

CAB-Mitglied

Entscheidet, welche der Change Requests, die für die CAB-Überprüfung gesammelt wurden, abgelehnt, welche im nächsten Release erfüllt und welche bei der nächsten CAB-Besprechung noch einmal geprüft werden sollten.

Release-Koordinator

  • Überprüft die Change Requests für die Services, für die der Benutzer als Release-Koordinator fungiert, nachdem sie vom Change Management weitergeleitet wurden.

  • Organisiert und unterstützt die CAB-Besprechungen für die Services, für die der Benutzer als Release-Koordinator fungiert.

  • Gliedert die Anforderungen von Releases in logische Gruppen auf, die von Change-Koordinatoren auf effiziente Weise verarbeitet werden können.

  • Erstellt einen Geschäftsfall für einen neuen Release, wenn ein zusätzliches Budget für seine Implementierung erforderlich ist.

  • Initiiert die Implementierung von Releases und beschließt bei Bedarf korrektive Maßnahmen.

  • Organisiert und leitet Implementierungsnachbesprechungen, um Verbesserungsvorschläge für zukünftige Releases zu sammeln.
