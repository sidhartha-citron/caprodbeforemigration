Arbeitsanweisungen für Release-Koordinatoren

3.10.1

Fügen Sie den abgelehnten Geschäftsfall der Geschäftsbegründungsaufgabe hinzu und geben Sie den Grund für die Ablehnung im Feld Lösung an.

3.10.2

Setzen Sie das Feld Status der Geschäftsbegründungsaufgabe auf Fehlgeschlagen.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der Aufgabe ein.

3.10.3

Öffnen Sie die Aufgabe zur Aktualisierung der Anforderer und Genehmiger dieses Release. Diese Aufgabe ist mit demselben Change verknüpft wie die Geschäftsbegründungsaufgabe und wurde ebenfalls von der Change-Vorlage bei Registrierung dieses Change generiert.

3.10.4

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können.

3.10.5

Wählen Sie mithilfe der Option Ändern neben dem Feld Besitzer sich selbst im Feld Besitzer der Aufgabe aus.

3.10.6

Wenn Sie die Aktualisierung des Anforderers und Genehmigers noch nicht durchführen möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie die Aktualisierung des Anforderers und Genehmigers vornehmen möchten, setzen Sie das Feld Status der Aufgabe auf In Bearbeitung.

3.10.7

Öffnen Sie im Bereich Verknüpfte Probleme des Change, mit dem das Problem, für das der Release registriert wurde, verknüpft ist, dieses verknüpfte Problem. Fügen Sie eine Hinweisaktion hinzu, um anzugeben, dass der Geschäftsfall für den Release abgelehnt wurde. Schließen Sie zudem eine Begründung für die Ablehnung ein. Setzen Sie das Feld Status des Problems auf Abgelehnt, damit das Problem wieder dem entsprechenden Koordinator zugewiesen wird.

3.10.8

Entfernen Sie die Verknüpfung zwischen dem Problem und dem Change. Klicken Sie dazu im Bereich Verknüpfte Probleme des Change auf den Link Del (Entf) in der Spalte Aktion.

3.10.9

Senden Sie eine E-Mail an die Genehmiger des abgelehnten Geschäftsfalls, um sie darüber zu informieren, dass der Release abgelehnt wurde und geschlossen wird. Geben Sie den Grund für die Ablehnung in der E-Mail an.

3.10.10

Geben Sie im Feld Lösung der Aufgabe zum Aktualisieren der Anforderer und Genehmiger des Release an, dass der Koordinator des Problems, für das der Release implementiert werden sollte, sowie die Genehmiger des abgelehnten Geschäftsfalls informiert wurden.

Hinweis:

Die CAB-Mitarbeiter müssen nicht informiert werden, da das CAB den Release nicht definiert hat.

3.10.11

Setzen Sie das Feld Status der Aufgabe zum Aktualisieren der Anforderer und Genehmiger des Release auf Fertiggestellt.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die Zeit ein, die Sie zum Informieren der Anforderer und Genehmiger benötigt haben.
