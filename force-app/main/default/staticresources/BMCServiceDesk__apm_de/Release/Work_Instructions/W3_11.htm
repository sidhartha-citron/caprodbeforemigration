Verfahrensschritt

Arbeitsanweisungen für Release-Koordinatoren

3.11.1

Fügen Sie den abgelehnten Geschäftsfall der Geschäftsbegründungsaufgabe bei und geben Sie den Grund für die Ablehnung im Feld Lösung an.

3.11.2

Setzen Sie das Feld Status der Geschäftsbegründungsaufgabe auf Fehlgeschlagen.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der Aufgabe ein.

3.11.3

Öffnen Sie die Aufgabe zur Aktualisierung der Anforderer und Genehmiger dieses Release. Diese Aufgabe ist mit demselben Change verknüpft wie die Geschäftsbegründungsaufgabe und wurde ebenfalls von der Change-Vorlage bei Registrierung dieses Change generiert.

3.11.4

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können.

3.11.5

Wählen Sie mithilfe der Option Ändern neben dem Feld Besitzer sich selbst im Feld Besitzer der Aufgabe aus.

3.11.6

Wenn Sie die Aktualisierung des Anforderers und Genehmigers noch nicht durchführen möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie die Aktualisierung des Anforderers und Genehmigers durchführen möchten, setzen Sie das Feld Status der Aufgabe auf In Bearbeitung.

3.11.7

Zeigen Sie die Change Requests des abgelehnten Release in den Bereichen Verknüpfte Incidents und Verknüpfte Probleme des Change an, der für die Koordination des Release verwendet wird. (Dies ist derselbe Change, mit dem auch der Arbeitsauftrag zum Aktualisieren der Anforderer und Genehmiger des Release verknüpft ist.)

3.11.8

Öffnen Sie über diese Bereiche alle Anforderungen und Probleme. In einer Support-Anforderung geben Sie im Bereich Aktionsverlauf an, dass der Geschäftsfall für den verknüpften Release abgelehnt wurde. Geben Sie außerdem den Grund für die Ablehnung an und legen Sie fest, dass der Change Request bei der nächsten CAB-Besprechung für den Service erneut geprüft wird. Verknüpfen Sie den als Platzhalter verwendeten Change des Service im Feld Verknüpfte Change Requests der Anforderung. Aktualisieren Sie auf dieselbe Weise die verknüpften Probleme. Verknüpfen Sie das Problem im Bereich Verknüpfte Change Requests wieder mit dem als Platzhalter dienenden Change, indem Sie auf die Schaltfläche Neuer Link klicken.

3.11.9

Entfernen Sie die Verknüpfungen zwischen allen abgelehnten Change Requests und dem Change, der zur Koordinierung des Release verwendet wird. Um eine Verknüpfung zu einer Anforderung aufzuheben, klicken Sie im Bereich Verknüpfte Incidents des Change auf den Link Del (Entf) in der Spalte Aktion. Um eine Verknüpfung zu einem Problem aufzuheben, klicken Sie im Bereich Verknüpfte Probleme des Change auf den Link Del (Entf) in der Spalte Aktion.

3.11.10

Senden Sie eine E-Mail an alle Anforderer der Change Requests (Support-Anforderung oder Problem), die Sie zurück zu dem Change verschoben haben, der als Platzhalter für Anforderungen fungiert, die während der nächsten CAB-Besprechung des Service geprüft werden sollen. Nehmen Sie in jede dieser E-Mails die Informationen auf, die Sie im Feld Information Update (Informationen aktualisieren) für den Change Request des E-Mail-Empfängers eingegeben haben.

3.11.11

Senden Sie eine ähnliche E-Mail an die Genehmiger des abgelehnten Geschäftsfalls, um sie davon in Kenntnis zu setzen, dass der Release abgelehnt und geschlossen wurde.
