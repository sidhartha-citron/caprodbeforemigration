Arbeitsanweisungen für Release-Koordinatoren

2.9.1

Öffnen Sie einen neuen Release, indem Sie auf der Registerkarte Releases auf die Schaltfläche Neu klicken.

2.9.2

Geben Sie eine Beschreibung für den Release in das Feld Release-Titel ein und fassen Sie die Anforderungen des Release im Feld Beschreibung zusammen.

2.9.3

Wählen Sie die korrekte Kategorie im Feld Kategorie. Wählen Sie den korrekten Release-Typ, -Subtyp und Bereitstellungstyp aus.

2.9.4

Geben Sie die Auswirkung und Dringlichkeit an.

2.9.5

Setzen Sie das Feld Status des Release auf Registriert.

2.9.6

Geben Sie realistische Termine in die Felder Fälligkeitsdatum, Start Implementation (Implementierung starten), End Implementation (Implementierung beenden), Start Deployment (Bereitstellung starten) und End Deployment (Bereitstellung beenden) ein und speichern Sie den Release.

2.9.7

Wählen Sie den Service, für den der Release implementiert werden soll, im Bereich Verknüpfte Services und CIs aus.

2.9.8

Öffnen Sie einen neuen Change, indem Sie auf die Schaltfläche Neu im Bereich Verknüpfte Change Requests klicken.

2.9.9

Wählen Sie im Feld Vorlage die Vorlage für die Release-Koordination aus und speichern Sie die Änderung. Mit dieser Aktion wird sichergestellt, dass die erforderlichen Felder des Change ausgefüllt und die folgenden Aufgaben erstellt und mit dem Change verbunden werden, wenn dieser gespeichert wird:

  • Aufgliedern des Release in Changes

  • Geschäftsbegründung für den Release

  • Vorbereitung der Anforderungsdokumente für Change-Koordinatoren

  • Aktualisieren der Anforderer und Genehmiger des Release

  • Implementierungsnachbesprechung

2.9.10

Vergewissern Sie sich, dass Sie in den Feldern Initiator-ID und „Besitzer“ des Change ausgewählt sind.

2.9.11

Bearbeiten Sie die Kurzbeschreibung des Change Request im Feld Beschreibung.

2.9.12

Setzen Sie das Feld Status des Change Request auf In Bearbeitung und speichern Sie den Change.

2.9.13

Wählen Sie den Service, für den der Release implementiert werden soll, im Bereich Verknüpfte Services und CIs des Change Request aus.

2.9.14

Klicken Sie im Bereich Linked Release (Verknüpfter Release) auf die Schaltfläche Neuer Link.

2.9.15

Klicken Sie auf die Schaltfläche Neuer Link im Bereich Verknüpfte Requests des Change, um Requests zu verknüpfen, deren Anforderungen durch den neuen Release erfüllt werden müssen. Klicken Sie dementsprechend auf die Schaltfläche Neuer Link im Bereich Verknüpfte Probleme des Change, um Probleme zu verknüpfen, für die durch den neuen Release eine vorgeschlagene strukturelle Lösung implementiert werden soll.

2.9.16

Öffnen Sie den Change, der als Platzhalter für die Change Requests des Service dient, die vom CAB geprüft werden sollen. Entfernen Sie die Verknüpfung zwischen dem als Platzhalter verwendeten Change Request und Problem, die Sie zur Koordination des neuen Release mit dem Change verknüpft haben. Um eine Verknüpfung mit einer Support-Anforderung zu entfernen, klicken Sie im Bereich Verknüpfte Requests des als Platzhalter verwendeten Change auf Del (Löschen) in der Spalte Aktion. Um eine Verknüpfung mit einem Problem zu entfernen, klicken Sie im Abschnitt Verknüpfte Probleme des als Platzhalter verwendeten Change auf Del (Löschen) in der Spalte Aktion.
