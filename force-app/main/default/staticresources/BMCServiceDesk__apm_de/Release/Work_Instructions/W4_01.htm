Arbeitsanweisungen für Release-Koordinatoren

4.1.1

Öffnen Sie die Aufgabe zur Vorbereitung der Anforderungsdokumente für Change-Koordinatoren. Diese Aufgabe wird durch die Change-Vorlage generiert, wenn der Change für die Koordination des Release registriert wird.

4.1.2

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können.

4.1.3

Vergewissern Sie sich, dass Sie im Feld Besitzer der Aufgabe ausgewählt sind.

4.1.4

Wenn Sie die Anforderungsdokumente für die Change-Koordinatoren noch nicht vorbereiten möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie die Anforderungsdokumente vorbereiten möchten, setzen Sie das Feld Status der Aufgabe auf In Bearbeitung.

4.1.5

Bereiten Sie ein Dokument für jeden Change-Koordinator vor. Geben Sie in jedem Dokument die Anforderungen an, die der Change-Koordinator vereinbarungsgemäß erfüllen muss.

4.1.6

Senden Sie jedes Dokument per E-Mail an den Change-Koordinator, für den Sie es erstellt haben. Bitten Sie den Change-Koordinator in jeder E-Mail darum, einen Change zur Erfüllung der Anforderungen zu planen. Geben Sie zudem das Datum an, zu dem Sie die Implementierung des Change erwarten. Stellen Sie sicher, dass dieses Datum realistisch ist. Achten Sie schließlich darauf, die Release-Nummer anzugeben, damit alle Change-Koordinatoren sie dazu verwenden können, ihre Changes mit dem Release zu verknüpfen.

Hinweis:

Wenn Sie die Implementierung mehrerer Changes anfordern, stellen Sie sicher, dass die angeforderten Changes in der richtigen Reihenfolge und zum richtigen Zeitpunkt implementiert werden (falls dies für den Gesamterfolg der Release-Implementierung von Bedeutung ist).

4.1.7

Fügen Sie eine Hinweisaktion zur Liste mit den Namen der Anforderungsdokumente hinzu. Geben Sie außerdem an, welches Dokument an welchen Change-Koordinator gesendet wurde und dass Sie auf die Registrierung und die Verknüpfung der Changes mit dem Release warten.

4.1.8

Setzen Sie das Feld Status der Aufgabe auf Waiting for… (Warten auf)

4.1.9

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für das Vorbereiten der Anforderungsdokumente aufgewendete Zeit ein.

4.1.10

Setzen Sie das Feld Status des Release auf In Bearbeitung.
