Arbeitsanweisungen für Release-Koordinatoren

3.12.1

Senden Sie eine E-Mail an alle Mitarbeiter des Service-CAB, um sie davon in Kenntnis zu setzen, dass der Geschäftsfall für das nächste Release abgelehnt wurde.

3.12.2

Diskutieren Sie mit den CAB-Mitgliedern, ob sie die nächste Besprechung vorverlegen möchten, um Möglichkeiten zur Verringerung der durch die Ablehnung des Release bedingten Auswirkungen zu beraten.

3.12.3

Geben Sie im Feld Lösung der Aufgabe zum Aktualisieren der Anforderer und Genehmiger des Release das Ergebnis der Beratungen mit den CAB-Mitgliedern an.

3.12.4

Setzen Sie das Feld Status dieser Aufgabe auf Fertiggestellt.

3.12.5

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die Zeit ein, die Sie zum Informieren der Anforderer und Genehmiger aufgewendet haben.
