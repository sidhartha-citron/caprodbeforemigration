Arbeitsanweisungen für Release-Koordinatoren

4.8.1

Fügen Sie dem Release eine Hinweisaktion hinzu, um zusammenzufassen, zu welchem Ergebnis die abgeschlossene Change-Implementierung geführt hat. Falls die Change-Implementierung nicht ganz erfolgreich war, geben Sie an, welche Anforderungen durch die Implementierung des Change nicht erfüllt wurden, warum sie nicht erfüllt wurden und warum dem Release keine weiteren Changes hinzugefügt wurden, um sicherzustellen, dass die nicht erfüllten Anforderungen noch erfüllt werden.

Hinweis:

Wenn ein Change-Koordinator Sie darüber informiert, dass ein weiterer verknüpfter Change abgeschlossen wurde, kehren Sie zu Schritt 4.3.1 zurück.
