Arbeitsanweisungen für Release-Koordinatoren

2.3.1

Wenn die CAB-Besprechung beginnt, setzen Sie das Feld Status der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung auf In Bearbeitung. Diese Aufgabe ist mit dem Change verknüpft, der als Platzhalter für die zum Service gehörigen Change Requests fungiert, die während der Besprechung geprüft werden sollen.

2.3.2

Prüfen Sie die Fortschritte (hinsichtlich der Release Management-Aktivitäten zu den Services des CAB), die seit der letzten CAB-Besprechung erzielt wurden. Wenn seit der letzten Besprechung ein Release geschlossen wurde, präsentieren Sie die aus diesem Release zu ziehenden Schlüsse den bei der Besprechung anwesenden CAB-Mitarbeitern.

2.3.3

Prüfen und besprechen Sie jeden Change Request auf der fertigen Liste mit den CAB-Mitgliedern. Stellen Sie sicher, dass allen CAB-Mitgliedern die Konsequenzen der Implementierung eines angeforderten Change bekannt sind, bevor Sie zum nächsten Change Request übergehen.

2.3.4

Sobald alle Change Requests in der fertigen Liste geprüft wurden, bitten Sie die CAB-Mitglieder darum, über die Change Requests abzustimmen, die sie ablehnen bzw. beim nächsten Release integrieren möchten.

2.3.5

Machen Sie während der Besprechung Notizen, um diese später mit der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung zu verknüpfen.
