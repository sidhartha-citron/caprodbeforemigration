Verfahrensschritt

Arbeitsanweisungen für Release-Koordinatoren

4.9.1

Fügen Sie dem Release eine Hinweisaktion hinzu, um zusammenzufassen, ob und welche Anforderungen welcher Change Requests nicht erfüllt wurden.

4.9.2

Öffnen Sie den Change Request, mit dem die Change Requests für den Release verknüpft sind. Öffnen Sie über diesen Change Request die Aufgabe zur Aktualisierung der Anforderer und Genehmiger dieses Release. Diese Aufgabe wird von der Change-Vorlage bei Registrierung des Change generiert.

4.9.3

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können.

4.9.4

Vergewissern Sie sich, dass Sie im Feld Besitzer der Aufgabe ausgewählt sind.

4.9.5

Wenn Sie die Aktualisierung des Anforderers und Genehmigers noch nicht durchführen möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie bereit sind, setzen Sie das Feld Status der Aufgabe auf In Bearbeitung.

4.9.6

Öffnen Sie im Bereich Verknüpfte Incidents des Change Request, mit dem Change Requests für den Release verknüpft sind, nacheinander die Requests, deren Anforderungen durch die Implementierung des Release vollständig erfüllt wurden. (Dieser Change Request ist zugleich der Change Request, mit dem die Aufgabe zum Aktualisieren der Anforderer und Genehmiger des Release verknüpft ist.) Geben Sie im Feld Lösung der jeweiligen Anforderung eine Zusammenfassung zur Implementierung des Release ein und legen Sie im Feld Abschlusskategorie den Wert Solved – Root Cause Analysis not Required (Gelöst – Ursachenanalyse nicht erforderlich) fest.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der einzelnen Anforderungen ein.

4.9.7

Setzen Sie das Feld Status der Anforderung auf Fertiggestellt, um sicherzustellen, dass das Service Desk den Kunden informiert, der im Feld „Kunden-ID“ mit der Anforderung verknüpft ist.

4.9.8

Öffnen Sie im Bereich Verknüpfte Probleme des Change Request, mit dem Change Requests für den Release verknüpft sind, nacheinander die Probleme, deren Anforderungen durch die Implementierung des Release vollständig erfüllt wurden. (Dieser Change Request ist zugleich der Change Request, mit dem die Aufgabe zum Aktualisieren der Anforderer und Genehmiger des Release verknüpft ist.) Geben Sie im Bereich Aktionsverlauf des jeweiligen Problems eine Zusammenfassung zur Implementierung des Release ein und setzen Sie das Feld Status auf Change fertiggestellt.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der einzelnen Probleme ein.

Hinweis:

Wenn das Feld Status eines Problems auf Change fertiggestellt gesetzt wird, wird das Problem wieder dem entsprechenden Problem-Koordinator zugewiesen.

4.9.9

Öffnen Sie nacheinander die Requests, deren Anforderungen nicht (vollständig) erfüllt wurden, und schätzen Sie den zur vollständigen Implementierung der Anforderungen verbleibenden Arbeitsaufwand für interne Mitarbeiter und langfristige Vertragspartner. Schätzen Sie auch die Kosten für CIs, Services und Support, die für jede dieser Support-Anfragen von Anbietern eingeholt werden müssten, um sicherzugehen, dass die Anforderung vollständig erfüllt wird. Fügen Sie jeder dieser Anforderungen eine Hinweisaktion hinzu, um Folgendes anzugeben:

  • welche Anforderungen nicht erfüllt wurden und warum dies der Fall war

  • die neuen Schätzungen für Kosten und Arbeitsaufwand und die Annahmen, auf denen sie basieren

  • dass die nicht erfüllten Anforderungen bei der nächsten CAB-Besprechung für den Service geprüft werden

Verknüpfen Sie jede dieser Anforderungen mit dem Change, der als Platzhalter für Anforderungen dient, die während der nächsten CAB-Besprechung für den Service geprüft werden müssen, und trennen Sie die Verknüpfung zu dem Change, der mit dem Release verknüpft ist. Senden Sie außerdem eine E-Mail an den Kunden, der im Feld Kunden-ID der jeweiligen Anforderung ausgewählt ist. Nehmen Sie in jede dieser E-Mails die Informationen auf, die Sie in die Hinweisaktionen für die vom Empfänger der E-Mail eingesendete Anforderung eingegeben haben.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der einzelnen Anforderungen ein.

4.9.10

Öffnen Sie jedes Problem, das durch die Implementierung des Release nicht (vollständig) gelöst wurde. Fügen Sie diesen Problemen Hinweisaktionen hinzu, in denen Sie angeben, warum die Anforderungen nicht erfüllt wurden. Setzen Sie das Feld Status dieser Probleme auf Change fertiggestellt, um sicherzustellen, dass sie wieder dem entsprechenden Problem-Koordinator zugewiesen werden. Trennen Sie die Verknüpfung dieser Probleme mit dem Change Request, mit dem die Change Requests für den Release verknüpft sind.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der einzelnen Anforderungen ein.

4.9.11

Verfassen Sie eine E-Mail an die CAB-Mitarbeiter (falls der Release durch CAB definiert wurde) und die Genehmiger des Geschäftsfalls (falls ein solcher für den Release erstellt wurde). Geben Sie in dieser E-Mail an, ob alle Anforderungen erfüllt wurden oder nicht. Wenn nicht alle Anforderungen erfüllt wurden, geben Sie die nicht erfüllten Anforderungen sowie den Grund für die Entscheidung an, den Release trotz der nicht erfüllten Anforderungen fertigzustellen.

4.9.12

Setzen Sie das Feld Status der Aufgabe zum Aktualisieren der Anforderer und Genehmiger des Release auf Fertiggestellt.

4.9.13

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für die Aktualisierung der Anforderer und Genehmiger aufgewendete Zeit ein.

4.9.14

Wählen Sie die passende Option im Feld Abschlusskategorie des Release. Setzen Sie das Feld Status der Anforderung auf Fertiggestellt.
