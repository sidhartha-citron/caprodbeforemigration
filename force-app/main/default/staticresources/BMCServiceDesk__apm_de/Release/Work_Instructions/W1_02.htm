Arbeitsanweisungen für Release-Koordinatoren

1.2.1

Falls ein Change Request keine Beteiligung des Release Management erfordert, fahren Sie fort mit Punkt 1.3.1. Andernfalls gehen Sie weiter zu Punkt 1.4.1.

Hinweis:

Change Requests zur Implementierung eines nicht standardmäßigen Change müssen durch das Release Management verarbeitet werden. Ausgenommen sind Change Requests, die zur Vermeidung oder Behebung eines Problems eingereicht wurden, sofern für ihre Implementierung die folgenden Bedingungen zutreffen:

  • Sie kann durch einen einzelnen Change-Koordinator koordiniert werden

 

  • Sie bewirkt keine Veränderung der Funktionen eines Service.

 

  • Sie erfordert kein zusätzliches Budget.
