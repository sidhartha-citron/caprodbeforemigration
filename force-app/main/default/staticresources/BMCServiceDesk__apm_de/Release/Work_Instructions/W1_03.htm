Arbeitsanweisungen für Release-Koordinatoren

1.3.1

Senden Sie den Change Request zurück an den Change Management-Prozess.

1.3.2

Falls der Change Request in Form einer Support-Anforderung eingeht, fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, dass der Change ohne den Einbezug des Release Management-Prozesses implementiert werden kann. Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Prüfung der Anforderung ein.

Verwenden Sie die Option Ändern neben dem Feld Besitzer für die Anforderung, um den Change-Koordinator auszuwählen, der Ihnen den Change Request zugewiesen hat. Setzen Sie das Feld Status der Support-Anforderung auf Zugewiesen.

Hinweis:

Setzen Sie das Feld Status der Anforderung nicht auf Abgelehnt.

1.3.3

Falls der Change Request in Form eines Problems eingeht, fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, dass der Change ohne den Einbezug des Release Management-Prozesses implementiert werden kann. Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Prüfung des Problems ein.

Verwenden Sie die Option Ändern neben dem Feld Besitzer für die Anforderung, um den Change-Koordinator auszuwählen, der Ihnen den Change Request zugewiesen hat. Setzen Sie das Feld Status des Problems auf Change Requested (Change angefordert).

Hinweis:

Sie dürfen das Feld Status des Problems nicht auf Abgelehnt setzen.
