Arbeitsanweisungen für Release-Koordinatoren

3.13.1

Öffnen Sie den Change, mit dem die Change Requests für den Release verknüpft waren. Öffnen Sie von diesem Change aus die Aufgabe zur Vorbereitung von Anforderungsdokumenten für die Change-Koordinatoren sowie die Aufgabe für die Implementierungsnachbesprechung. Diese Aufgaben wurden durch die Change-Vorlage bei der Registrierung des Change generiert.

3.13.2

Setzen Sie das Feld Status für diese Aufgaben auf Storniert. Geben Sie im Feld Lösung in beiden Aufgaben an, dass der Geschäftsfall für den Release abgelehnt wurde.

3.13.3

Wählen Sie im Feld Closure Category (Abschlusskategorie) des Change, mit dem diese Aufgaben verknüpft sind, die Option Rejected by Approvers (Von Genehmigern abgelehnt).

3.13.4

Setzen Sie das Feld Status des Change auf Fertiggestellt.

3.13.5

Fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, dass der Release nicht implementiert werden konnte, da sein Geschäftsfall abgelehnt wurde. Schließen Sie zudem eine Begründung für die Ablehnung ein.

3.13.6

Wählen Sie Geschäftsfall abgelehnt im Feld Closure Category (Abschlusskategorie) des Release und setzen Sie das Feld Status auf Geschlossen.
