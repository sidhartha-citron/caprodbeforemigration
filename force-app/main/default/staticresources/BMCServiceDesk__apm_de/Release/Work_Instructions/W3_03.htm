Arbeitsanweisungen für Release-Koordinatoren

3.3.1

Setzen Sie das Feld Status des Release auf To Be Approved (Zur Genehmigung).

3.3.2

Öffnen Sie den Change, mit dem die Change Requests für den Release verknüpft sind. Öffnen Sie von diesem Change aus die Geschäftsbegründungsaufgabe, die von der Change-Vorlage bei der Registrierung des Change generiert wurde.

3.3.3

Fügen Sie in dieser Aufgabe eine Hinweisaktion hinzu, in der Sie angeben, weshalb ein genehmigter Business Case für die Implementierung des Release erforderlich ist.

3.3.4

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können. Berücksichtigen Sie die zur Genehmigung des Business Case erforderliche Zeit.

3.3.5

Vergewissern Sie sich, dass Sie im Feld Besitzer der Aufgabe ausgewählt sind.

3.3.6

Wenn Sie den Geschäftsfall für den Release noch nicht vorbereiten möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie bereit sind, setzen Sie das Feld Status der Aufgabe auf In Bearbeitung.

3.3.7

Bitten Sie die CAB-Mitglieder um die Namen und Positionen der Personen, die als Sponsoren für den neuen Release auftreten.

3.3.8

Fügen Sie eine Hinweisaktion mit den Namen und Positionen der Sponsoren in der Geschäftsbegründungsaufgabe hinzu.

3.3.9

Arbeiten Sie bei Bedarf mit den CAB-Mitgliedern zusammen, um einen Geschäftsfall zur Begründung der Investition (in puncto Kosten und Aufwand) für die Sponsoren des neuen Release vorzubereiten.

3.3.10

Wenn ein Budget verfügbar ist, aktivieren Sie im Bereich Business and Financial Details (Geschäftliche und finanzielle Informationen) das Kontrollkästchen Budget Available (Budget verfügbar). Wählen Sie die zutreffende Geschäftsbegründung und Risikobewertung. Falls benötigt, aktivieren Sie außerdem die Kontrollkästchen Required Contracts Verified (Erforderliche Verträge überprüft), Required Licenses Verified (Erforderliche Lizenzen überprüft) und Required CIs Available (Erforderliche CIs verfügbar).
