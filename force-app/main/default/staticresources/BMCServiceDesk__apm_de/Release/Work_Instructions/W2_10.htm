Arbeitsanweisungen für Release-Koordinatoren

2.10.1

Öffnen Sie im Abschnitt Verknüpfte Aufgaben des Change Request, mit dem die Change Requests für den Release verknüpft sind, die Aufgabe zur Aufgliederung des Release in Changes. Diese Aufgabe wird von der Change-Vorlage bei Registrierung des Change generiert.

2.10.2

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können.

2.10.3

Vergewissern Sie sich, dass Sie im Feld Besitzer der Aufgabe ausgewählt sind.

2.10.4

Wenn Sie den Release noch nicht in Changes aufgliedern möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie bereit sind, setzen Sie das Feld Status der Aufgabe auf In Bearbeitung.

2.10.5

Prüfen Sie die Anforderungen in den Change Requests, die Sie mit dem neuen Release des Service verknüpft haben, um festzustellen, welche der unterstützenden Services geändert werden müssen.

2.10.6

Planen Sie eine Besprechung mit dem Change-Koordinator des Service, für den der neue Release implementiert werden soll, sowie mit den Change-Koordinatoren der unterstützenden Services, die ebenfalls geändert werden müssen.

2.10.7

Fügen Sie eine Hinweisaktion in der Aufgabe für die Aufgliederung der Releases in Changes hinzu und geben Sie das Datum und die Namen der teilnehmenden Change-Koordinatoren ein. Setzen Sie das Feld Status der Aufgabe auf Waiting for (Warten auf).

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Planung der Besprechung ein.

2.10.8

Setzen Sie zu Beginn der Besprechung mit den Change-Koordinatoren das Feld Status auf In Bearbeitung.

2.10.9

Prüfen Sie die Change Requests, die mit diesen Change-Koordinatoren mit dem neuen Release verknüpft wurden. Vergewissern Sie sich, dass die geschätzten Kosten bzw. der geschätzte Aufwand für die verschiedenen Change Requests realistisch sind.

2.10.10

Arbeiten Sie mit den Change-Koordinatoren zusammen, um die Verantwortung für die Erfüllung der unterschiedlichen Aufgaben zu koordinieren. Verteilen Sie die Verantwortung so, dass allen Change-Koordinatoren die Verantwortung für Anforderungen zufällt, die ihre jeweilige Gruppe implementieren kann.

2.10.11

Entwerfen Sie einen umfassenden Implementierungsplan, in dem die Dauer für jeden Change angegeben ist, der durch die Change-Koordinatoren zur Erfüllung ihrer Anforderungen koordiniert wird, sowie die verschiedenen Abhängigkeiten.

2.10.12

Fügen Sie den Planentwurf der Aufgabe zum Aufgliedern des Release in Changes bei und geben Sie im Feld Lösung an, dass Sie den Planentwurf beigefügt haben.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die zum Vorbereiten des Planentwurfs aufgewendete Zeit ein.

Setzen Sie das Feld Status der Aufgabe auf Fertiggestellt.
