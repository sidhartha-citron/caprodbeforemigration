Arbeitsanweisungen für Release-Koordinatoren

4.2.1

Halten Sie sich in Bezug auf die Change Requests, die Sie an die Change-Koordinatoren gesendet haben, auf dem Laufenden, indem Sie dafür sorgen, dass die angeforderten Changes in der Service Management-Anwendung registriert und mit dem Release verknüpft werden.

4.2.2

Falls ein Change noch nicht registriert ist, erinnern Sie den verantwortlichen Change-Koordinator daran, dies nachzuholen.

4.2.3

Falls ein Change-Koordinator es versäumt hat, den Change mit dem Release zu verknüpfen, öffnen Sie den Change und verknüpfen Sie ihn im Abschnitt Verknüpfte Releases mit dem Release.

4.2.4

Geben Sie im Feld Lösung der Aufgabe für die Vorbereitung der Anforderungsdokumente an, dass die angeforderten Changes in der Service Management-Anwendung registriert und mit dem Release verknüpft sind. Setzen Sie das Feld Status auf Fertiggestellt.

4.2.5

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die Zeit an, die Sie aufgewendet haben, um sich hinsichtlich der Anforderungsdokumente auf dem Laufenden zu halten.

Hinweis:

Wenn ein Change-Koordinator Sie darüber informiert, dass ein verknüpfter Change fertiggestellt wurde, gehen Sie weiter zu Punkt 4.3.1.
