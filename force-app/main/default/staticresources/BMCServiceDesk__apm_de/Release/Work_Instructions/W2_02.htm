Arbeitsanweisungen für Release-Koordinatoren

2.2.1

Nachdem die Deadline zum Einsenden von Change Requests für die nächste CAB-Besprechung verstrichen ist, setzen Sie das Feld Status der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung auf In Bearbeitung.

2.2.2

Bereiten Sie eine Liste mit den Details zu allen Change Requests vor, die vom CAB geprüft werden sollen. Diese Requests sind ebenso wie die Aufgaben mit dem Change verknüpft, der als Platzhalter für die zum Service gehörigen Change Requests fungiert, für die eine CAB-Prüfung ansteht.

Hinweis:

Die Liste mit Change Requests muss die folgenden Spalten mit Informationen enthalten:

  • Anforderungs- oder Problemnummer

  • Name und Unternehmen des Anforderers

  • Beschreibung des Change Request

  • Geschätzter Implementierungsaufwand

  • Geschätzte Implementierungskosten

2.2.3

Senden Sie die endgültige Liste der bei der nächsten CAB-Besprechung zu prüfenden Change Requests per E-Mail an alle CAB-Mitarbeiter. So können sie die Change Requests vor der Besprechung intern diskutieren.

2.2.4

Fügen Sie der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung eine Hinweisaktion hinzu, in der Sie angeben, dass Sie die endgültige Liste der Change Requests an alle CAB-Mitarbeiter gesendet haben und dass die Arbeit mit dem Beginn der CAB-Besprechung fortgesetzt wird.

2.2.5

Setzen Sie das Feld Status in der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung auf Waiting for… (Warten auf).

2.2.6

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für das Fertigstellen der Liste der Change Requests benötigte Zeit ein.
