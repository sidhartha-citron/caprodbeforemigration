Arbeitsanweisungen für Release-Koordinatoren

1.5.1

Schätzen Sie den Aufwand für interne und externe Mitarbeiter, der zur Implementierung des angeforderten Change notwendig ist.

1.5.2

Schätzen Sie die Kosten für Konfigurationselemente, Services und Support, die von Lieferanten bezogen werden müssen, damit der angeforderte Change implementiert werden kann.

Hinweis:

Beraten Sie sich mit den Change-Koordinatoren und Spezialisten, um die Kosten für den Change realistisch einschätzen zu können.

1.5.3

Falls der Change Request in Form einer Anforderung eingeht, fügen Sie eine Hinweisaktion hinzu, in der Sie die geschätzten Werte eingeben und angeben, dass die Verantwortung für die Koordination vom Release Management übernommen wurde. Geben Sie zudem die Annahmen ein, auf welchen die Schätzungen beruhen. Setzen Sie das Feld Status der Anforderung auf Change ausstehend.

1.5.4

Falls der Change Request in Form eines Problems eingeht, fügen Sie eine Hinweisaktion hinzu, in der Sie die geschätzten Werte eingeben und angeben, dass die Verantwortung für die Koordination vom Release Management übernommen wurde. Geben Sie zudem die Annahmen ein, auf welchen die Schätzungen beruhen. Setzen Sie das Feld Status des Problems auf Change ausstehend.
