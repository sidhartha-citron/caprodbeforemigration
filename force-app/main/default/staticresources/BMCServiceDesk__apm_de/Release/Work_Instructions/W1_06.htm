Arbeitsanweisungen für Release-Koordinatoren

1.6.1

Wenn der Change Request in Form einer Anforderung eingeht, wählen Sie im Bereich Verknüpfte Change Requests der Anforderung den Change Request, der als Platzhalter für Anforderungen fungiert, die während der nächsten CAB-Besprechung des Service geprüft werden müssen.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen der Anforderung ein.

1.6.2

Wenn der Change Request in Form eines Problems eingeht, klicken Sie im Bereich Verknüpfte Change Requests auf die Schaltfläche Neuer Link und verknüpfen Sie das Problem mit dem Change Request, der als Platzhalter für Anforderungen fungiert, die während der nächsten CAB-Besprechung des Service geprüft werden müssen.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen des Problems ein.

Hinweis:

Durch die Verknüpfung des Change Request mit diesem Change Request wird sichergestellt, dass er während der nächsten CAB-Besprechung des Service in Verfahren 2, Release-Definition geprüft wird.
