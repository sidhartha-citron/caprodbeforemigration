Arbeitsanweisungen für Release-Koordinatoren

2.1.1

Setzen Sie das Feld Status der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung auf In Bearbeitung. Diese Aufgabe ist mit dem Change verknüpft, der als Platzhalter für die Change Requests des Service fungiert, für die eine CAB-Prüfung aussteht.

2.1.2

Senden Sie die Einladungen für die nächste CAB-Besprechung des Service per E-Mail an alle betroffenen Mitarbeiter. Geben Sie die Deadline an, bis zu der Change Requests zur Prüfung bei der nächsten CAB-Besprechung eingesendet werden können.

2.1.3

Fügen Sie der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung eine Hinweisaktion hinzu, in der Sie angeben, dass Sie die Einladungen verschickt haben. Geben Sie außerdem die Deadline für das Einsenden von Change Requests an.

2.1.4

Setzen Sie das Feld Status der Aufgabe zur Vorbereitung und Unterstützung der CAB-Besprechung auf Waiting for… (Warten auf).

2.1.5

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die zum Einladen der CAB-Mitglieder benötigte Zeit ein.
