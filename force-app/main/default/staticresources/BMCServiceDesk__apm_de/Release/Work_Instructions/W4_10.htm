Arbeitsanweisungen für Release-Koordinatoren

4.10.1

Öffnen Sie den Change Request, mit dem die Change Requests für den Release verknüpft sind oder waren. Öffnen Sie von hier aus die Aufgabe für die Implementierungsnachbesprechung. Diese Aufgabe wird von der Change-Vorlage bei Registrierung des Change generiert.

4.10.2

Aktualisieren Sie das Feld Fälligkeitsdatum der Aufgabe, indem Sie ein Datum und eine Zeit eingeben, die erreicht werden können.

4.10.3

Vergewissern Sie sich, dass Sie im Feld Besitzer der Aufgabe ausgewählt sind.

4.10.4

Wenn Sie noch nicht mit der Organisation der Implementierungsnachbesprechung beginnen möchten, ändern Sie das Feld Status der Aufgabe von Registriert in Angenommen. Wenn Sie mit der Aufgabe beginnen möchten, setzen Sie den Status der Aufgabe auf In Bearbeitung.

4.10.5

Laden Sie den Servicebesitzer des Service, für den der Release durchgeführt wurde, und die Change-Koordinatoren, die einen oder mehrere Changes des Release koordiniert haben, zur Implementierungsnachbesprechung ein.

4.10.6

Prüfen Sie zusammen mit den Teilnehmern der Besprechung den gesamten Lebenszyklus des Release und diskutieren Sie aufgetretene Probleme.

4.10.7

Sammeln Sie alle Verbesserungsvorschläge.

4.10.8

Vereinbaren Sie Punkte, die für alle zukünftigen Releases des Service berücksichtigt werden sollen. Legen Sie außerdem Aktionen fest, die ausgeführt werden können, um Verbesserungen zu implementieren, und bestimmen Sie, wer für welche Aktionen verantwortlich ist.

4.10.9

Fertigen Sie Besprechungsnotizen an, um sicherzustellen, dass alle Verbesserungsvorschläge und Entscheidungen dokumentiert werden und nach der Besprechung veröffentlicht werden können.
