Verfahren 3, Geschäftsbegründung


Wenn ein genehmigter Geschäftsfall für den Release nicht erforderlich ist, storniert der Release-Koordinator den Arbeitsauftrag für die Geschäftsbegründung, der Teil des Change ist, mit dem die Change Requests des Release verknüpft sind. Wenn andererseits der nächste Release nicht mit dem dem Servicebesitzer bereits zur Verfügung stehenden Budget implementiert werden kann, erstellt der Release-Koordinator einen Geschäftsfall, um den Release zu rechtfertigen. Nachdem er den Geschäftsfall erstellt hat, fordert der Release-Koordinator eine Genehmigung für den Geschäftsfall an, um sicherzustellen, dass ein Budget für die Implementierung des neuen Release bereitgestellt wird.

Wenn der Geschäftsfall abgelehnt wird, passt ihn der Release-Koordinator an, wenn er dadurch noch genehmigt werden kann. Wenn es nicht möglich ist, den Geschäftsfall zu genehmigen, informiert der Release-Koordinator die Anforderer des Release darüber, dass der Geschäftsfall für den Release abgelehnt wurde. Wenn der Release durch das CAB definiert wurde, verschiebt der Release-Koordinator die Change Requests, die mit dem Release verknüpft sind, zurück zu dem Change, der als Platzhalter für Change Requests fungiert, die während der nächsten CAB-Besprechung geprüft werden müssen. Bevor er den Release abschließt, kontaktiert der Release-Koordinator außerdem die CAB-Mitarbeiter, um sie davon in Kenntnis zu setzen, dass der Geschäftsfall Ihres Release abgelehnt wurde, und gibt ihnen die Möglichkeit, die nächste CAB-Besprechung vorzuverlegen.
