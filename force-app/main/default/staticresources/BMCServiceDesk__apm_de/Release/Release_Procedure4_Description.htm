Verfahren 4, Release-Koordination


Der Release-Koordinator erstellt ein Dokument für jeden Change-Koordinator, das Einzelheiten zu den Release-Anforderungen enthält, für die der Change-Koordinator verantwortlich ist. Der Release-Koordinator sendet die Anforderungsdokumente an die Change-Koordinatoren und stellt sicher, dass die angeforderten Changes registriert und mit dem Release verknüpft sind.

Wenn ein Change-Koordinator den Release-Koordinator darüber informiert hat, dass einer der Changes des Release abgeschlossen wurde, fügt der Release-Koordinator dem Release ein kurzes Update hinzu.

Wenn der Change nicht erfolgreich implementiert werden konnte, bestimmt der Release-Koordinator in Zusammenarbeit mit den Koordinatoren der mit dem Release verknüpften Change Requests, wie das (teilweise) Misslingen der Change-Implementierung sich auf die Implementierung des Release insgesamt auswirkt. Wenn noch ein oder mehrere zusätzliche Changes implementiert werden können, um sicherzustellen, dass alle oder die meisten Anforderungen des Release erfüllt sind, oder wenn ein oder mehrere zusätzliche Changes erforderlich sind, um sicherzugehen, dass das Zurücksetzen des Release abgeschlossen wird, dokumentiert der Release-Koordinator die Anforderungen für zusätzliche Changes, bevor er diese an die entsprechenden Change-Koordinatoren weiterleitet.

Nach Abschluss des letzten Change, um dessen Implementierung im Release der Change-Koordinator gebeten wurde stellt der Release-Koordinator sicher, dass die Anforderer darüber informiert werden, ob ihre Anforderungen durch die Release-Implementierung erfüllt wurden oder nicht. Wenn der Release durch CAB definiert wurde, verschiebt der Release-Koordinator sämtliche Change Requests, deren Anforderungen nicht vollständig erfüllt wurden, zurück zu dem Change, der als Platzhalter für Change Requests fungiert, die während der nächsten CAB-Besprechung geprüft werden müssen. Der Release-Koordinator informiert auch die CAB-Mitarbeiter (wenn der Release von ihnen definiert wurde) und die Genehmiger über den Geschäftsfall (wenn ein solcher für den Release erstellt werden musste).

Nachdem den Anforderern und Genehmigern die neuesten Informationen zum Release mitgeteilt wurden, organisiert und veranstaltet der Release-Koordinator die Implementierungsnachbesprechung. Während dieser Besprechung prüft der Release-Koordinator den gesamten Lebenszyklus des Release zusammen mit dem Servicebesitzer des Service, für den der Release implementiert wurde, und den Change-Koordinatoren, die ein oder mehrere Change Requests des Release koordiniert haben. Nach der Besprechung veröffentlicht der Release-Koordinator die Verbesserungsvorschläge und Entscheidungen, auf die sich die Besprechungsteilnehmer geeinigt haben. Am Ende schließt der Release-Koordinator den Release.
