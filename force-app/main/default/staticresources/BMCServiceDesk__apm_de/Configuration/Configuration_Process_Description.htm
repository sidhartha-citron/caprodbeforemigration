Prozessbeschreibung


Der Configuration Management-Prozess besteht aus den folgenden Verfahren:

  • CI-Anforderung: Wird vom Konfigurationsadministrator verwendet, wenn neue Konfigurationselemente (CIs) bestellt werden müssen.

  • CI-Eingang: Wird von Konfigurationsadministratoren verwendet, wenn neue CIs geliefert wurden.

  • Lieferanteninformationspflege: Wird von Konfigurationsadministratoren bei der Registrierung neuer Lieferanten und der Aktualisierung der Kontaktdaten bereits registrierter Lieferanten verwendet.

  • CI-Aktualisierung: Wird von Konfigurationsadministratoren bei der Aktualisierung der Attribute und Beziehungen von bereits in der CMDB registrierten CIs verwendet.

Weitere Informationen zu diesen Verfahren erhalten Sie, wenn Sie auf Prozess klicken, um zur grafischen Prozessdarstellung zurückzukehren, und danach auf die Felder, welche die Verfahren verkörpern. Bei der grafischen Darstellung dieses Verfahrens wird deren Beschreibung im rechten Bereich angezeigt.
