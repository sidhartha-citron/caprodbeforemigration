Arbeitsanweisungen für Konfigurationsadministratoren

2.8.1

Registrieren Sie die Details des neuen Konfigurationselements (CI). Wenn eine Bestellanforderung für das neue CI übermittelt wurde, ist das CI bereits in der CMDB registriert und das Feld Status ist auf Bestellt gesetzt. Aktualisieren Sie in diesem Fall den Status der CIs und fügen Sie die fehlenden Informationen (z. B. die Seriennummer) hinzu.

Wenn jedoch keine formelle Bestellanforderung gesendet wurde (z. B. im Falle intern entwickelter Software), registrieren Sie die neuen CIs in der CMDB.

Verknüpfen Sie bei Bedarf alle neuen CIs mit anderen CIs, Kunden und Konten. Die Verknüpfung können Sie in Übereinstimmung mit den Feldverwendungsrichtlinien für das Konfigurationselement-Formular durchführen.

2.8.2

Drucken Sie den eindeutigen CI-Code für jedes neue Hardware-CI auf ein Etikett. Fügen Sie jede Bezeichnung an ihr entsprechendes Hardware-CI an, sodass die Bezeichnung nach der Installation des CI deutlich sichtbar ist.

2.8.3

Drucken Sie den eindeutigen CI-Code für jedes neue Software-CI auf ein Etikett. Fügen Sie jede Bezeichnung so an die Masterkopie des entsprechenden Software-CI an, dass die Bezeichnung deutlich sichtbar ist, nachdem die Masterkopie in das DML platziert wurde. Legen Sie die Masterkopie jedes neuen Software-CI in der DML ab.

2.8.4

Drucken Sie den eindeutigen CI-Code für jedes neue Software-Lizenzzertifikat auf ein Etikett. Bringen Sie das Etikett so an das entsprechende Zertifikat an, dass das Zertifikat nach dem Ablegen in der DML leicht zu finden ist. Legen Sie alle neuen Software-Lizenzzertifikate in der DML ab.
