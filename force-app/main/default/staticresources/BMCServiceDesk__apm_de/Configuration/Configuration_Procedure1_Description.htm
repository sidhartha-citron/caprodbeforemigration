Verfahren 1, CI-Anforderung


Jedes Mal, wenn neue Konfigurationselemente (CIs) bestellt werden müssen oder die Konfigurationsmanagement-Datenbank (CMDB) aktualisiert werden muss, erhält der zuständige Konfigurationsadministrator eine Aufgabe vom Change Management oder Request Management.

Wenn eine Aufgabe keine neue CI anfordert, fährt der Konfigurationsadministrator direkt mit Verfahren 2, CI-Eingang fort. Wenn jedoch eine Aufgabe eine oder mehrere neue CIs anfordert, sendet der Konfigurationsadministrator zunächst eine Bestellanforderung für die erforderlichen CIs. Nachdem eine Bestellanforderung an den Einkauf gesendet wurde, prüft der Konfigurationsadministrator deren Fortschritt regelmäßig.

Wenn die CIs vom Einkauf bestellt werden, registriert der Konfigurationsadministrator die bestellten CIs in der CMDB. Wenn jedoch die Bestellanforderung abgelehnt wurde, fährt der Konfigurationsadministrator mit Verfahren 4, CI-Aktualisierung fort.
