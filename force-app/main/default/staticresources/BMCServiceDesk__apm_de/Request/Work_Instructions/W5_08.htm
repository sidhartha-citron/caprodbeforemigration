Arbeitsanweisungen für Spezialisten

5.8.1

Geben Sie im Feld Lösung die Schritte zur Erfüllung der Anforderung an. Verwenden Sie dieses Feld, um genaue Anweisungen für eine Lösung zu geben (wenn z. B. der Benutzer eine Einstellung ändern muss), um eine Informationsanfrage zu beantworten oder um zu begründen, weshalb die Anforderung nicht gelöst werden kann (falls keine Lösung oder Umgehungslösung angeboten werden kann).

5.8.2

Wählen Sie die zutreffende Abschlusskategorie im Feld Closure Category (Abschlusskategorie) und setzen Sie das Feld Status der Anforderung auf Fertiggestellt.

5.8.3

Wählen Sie das betreffende CI im Feld CI aus, falls das Feld leer ist oder das falsche CI angegeben ist.

Hinweis:

Falls das CI nicht in der CMDB registriert ist, jedoch in den Bereich des Konfigurationsmanagements fällt, wählen Sie das CI mit dem Code NORECORD aus.

Das CI mit dem Code NORECORD steht möglicherweise nicht standardmäßig zur Verfügung.

Hinweis:

Falls die Anforderung kein bestimmtes CI betrifft oder das CI nicht unter den Konfigurationsmanagement-Prozess fällt, wählen Sie das CI mit dem Code N/A aus.

5.8.4

Wenn im Feld Auswirkung Hoch oder Schwerwiegend angegeben ist, geben Sie Datum und Uhrzeit des Ausfalls des Service im Feld Ausfallstartdatum und Datum und Uhrzeit der Wiederaufnahme des Service im Feld Ausfallenddatum ein.

5.8.5

Wenn Sie einen Incident gelöst haben und feststellen, dass ähnliche Incidents mit großer Wahrscheinlichkeit auftreten werden, wenn die Ursache nicht beseitigt wird, prüfen Sie, ob das Problem, das den Incident verursacht hat, bereits registriert ist. Falls das Problem bereits registriert ist, verknüpfen Sie die von Ihnen gelöste Anforderung mit dem Problem.

Falls das Problem, das zum Incident geführt hat, noch nicht registriert ist, benachrichtigen Sie den Problem-Koordinator des betroffenen Service per E-Mail oder telefonisch, wobei Sie die Anforderungsnummer sowie die Begründung für das wahrscheinliche Wiederauftreten des Incidents angeben.

5.8.6

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Lösung der Anforderung ein.

5.8.7

Führen Sie eine letzte Prüfung der Anforderung durch, um sich zu vergewissern, dass die Informationen vollständig und korrekt sind.

5.8.8

Falls es wahrscheinlich ist, dass vergleichbare Anforderungen eingereicht werden, erstellen Sie einen Knowledge-Artikel für die bereitgestellte Lösung.
