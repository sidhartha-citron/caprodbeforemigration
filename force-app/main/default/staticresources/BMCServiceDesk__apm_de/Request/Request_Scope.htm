Umfang


Der Umfang des Request Management-Prozesses ist auf die folgenden Support-Anforderungskategorien begrenzt.

Kategorie

Beschreibung

Incident

Anforderung von einem Benutzer zur Wiederherstellung eines Service, der nicht ordnungsgemäß funktioniert

Hinweis:

Anforderungen zum Zurücksetzen von Kennwörtern von Benutzern, die ihr Kennwort vergessen haben, und Anforderungen zum Wiederherstellen von Backups von Benutzern, die Daten verloren haben

Change-Anforderung

Anforderung von einem Benutzer zum Ändern, Hinzufügen, Verschieben oder Entfernen von Funktionen, Zugriff oder Komponenten

Hinweis:

Anforderungen zum Kopieren von Daten (zum Beispiel zum Erstellen eines Backups oder Wiederherstellen eines Backups an einem anderen Ort) und Anforderungen für einen Stapel-Job fallen unter diese Kategorie

Serviceanforderung

Anforderung von einem Benutzer, die durch Verwenden eines vordefinierten Workflows in folgenden Szenarien gelöst werden kann:

  • Vorlage für diesen Workflow ist vorhanden
  • Keine Risiko- und Auswirkungsanalyse erforderlich
  • Keine Genehmigung außer der des Kunden erforderlich

Informationsanforderung

Anforderung von einem Benutzer für eine Antwort auf eine servicebezogene Frage

Lob

Lob oder Unterstützung durch einen Benutzer

Beschwerde

Anforderung von einem Benutzer zur Verbesserung der Unterstützung des Benutzers durch den Serviceanbieter
