Arbeitsanweisungen für Spezialisten

7.2.1

Implementieren Sie den Change in der Entwicklungsumgebung wie mit dem Servicebesitzer besprochen (bzw. dem diensthabenden Manager, falls der Servicebesitzer nicht verfügbar ist). Testen Sie den Change in einer separaten Testumgebung, falls eine solche vorhanden ist. Andernfalls führen Sie die Tests in der Entwicklungsumgebung durch. Wenn (automatisierte) Skripte für Regressionstests verfügbar sind, führen Sie diese Skripte aus Beheben Sie mögliche Fehler, die während der Tests auftreten, in der Entwicklungsumgebung. Fordern Sie bei Bedarf Hilfe von anderen Spezialisten an.

7.2.2

Kontaktieren Sie den Release-Administrator, nachdem Sie bestätigt haben, dass die Korrektur funktioniert und keine neuen Fehler eingeführt wurden, und fordern Sie die Übertragung des neuen Release in die Testumgebung an.

7.2.3

Fügen Sie eine Hinweisaktion zur Support-Anforderung hinzu, in der Sie Changes in der Anwendung, von Ihnen durchgeführte Tests und die Person, die das neue Release übermitteln soll, angeben.

7.2.4

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Entwicklung und das Testen der Lösung ein.
