Arbeitsanweisungen für Change-Koordinatoren

5.4.1

Wenn der neue Release den Kundentest bestanden hat, fahren Sie mit 5.8.1 fort. Falls der neue Release den Test nicht bestanden hat, fahren Sie fort mit 5.5.1.

Hinweis:

Wenn während des Tests ein oder mehrere Probleme aufgetreten sind, die Kundenvertreter das Übertragen des neuen Release an die Produktion dennoch bestätigen, stellen Sie sicher, dass für jeden Problempunkt ein Problem registriert wird. Indem ein Problem in der Service Management-Anwendung für jeden Problempunkt registriert wird, werden die Probleminformationen für sämtliche Personen zugänglich gemacht, die für den Support des neuen Release zuständig sind. Das Erstellen eines Problems gewährleistet außerdem, dass ein Mitarbeiter mit der Lösung des Problems beauftragt wird.

Hinweis:

Wenn die Lösung der Problempunkte als neuer Knowledge-Artikel verfügbar gemacht werden soll, klicken Sie auf die Registerkarte Knowledge-Artikel und anschließend auf Neu.

Wählen Sie den Record-Typ Problemlösung aus. Wählen Sie die passende Kategorie und geben Sie den Titel ein. Geben Sie die Einzelheiten zum Knowledge-Artikel in den verfügbaren Feldern ein. Wählen Sie Zur Genehmigung übermitteln aus, um die Genehmigung für den neuen Knowledge-Artikel zu erhalten.
