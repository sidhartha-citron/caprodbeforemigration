Arbeitsanweisungen für Change-Koordinatoren

6.8.1

Wenn Sie benachrichtigt werden, dass die Arbeit am Change Request abgeschlossen ist, und Sie nicht daran arbeiten können, ändern Sie das Feld Status der Aufgabe zum Aktualisieren von Anforderern und Genehmigern der Änderung von Registriert in Akzeptiert. Wenn Sie die Anforderer und Genehmiger des Change Request aktualisieren können, legen Sie das Feld Status der Aufgabe auf In Bearbeitung fest.

6.8.2

Falls die Change-Implementierung nicht erfolgreich war (z. B. weil die Change-Anforderung abgelehnt wurde oder die Change-Implementierung zurückgesetzt werden musste), fügen Sie eine Hinweisaktion im Change mit der Begründung hinzu.

6.8.3

Teilen Sie den Genehmigern des Change Request per E-Mail mit, ob der Change implementiert wurde. Falls die Change-Implementierung nicht erfolgreich war, geben Sie den Grund für das Scheitern an.

6.8.4

Öffnen Sie im Abschnitt Verknüpfte Incidents des Change Request, alle vom Request Management eingegangenen Requests (falls vorhanden). Hat die Change-Implementierung die angegebenen Change-Anforderungen erfüllt, geben Sie eine Übersicht über die Implementierung im Feld Lösung des Request an und legen Sie das Feld Abschlusskategorie auf Gelöst fest. Hat die Change-Implementierung die Anforderungen nicht erfüllt, geben Sie den Grund im Feld Lösung des Request an und legen Sie das Feld Abschlusskategorie auf Unable to solve or in conflict with standards or policy (Keine Lösung möglich oder Konflikt mit Standard oder Richtlinie) fest.

6.8.5

Setzen Sie das Feld Status der Anforderung auf Abgeschlossen, um sicherzustellen, dass der Service Desk die mit der Support-Anforderung im Feld Kunden-ID verknüpfte Person informiert.

6.8.6

Öffnen Sie im Abschnitt Verknüpfte Probleme des Change Request alle vom Problem Management eingegangenen Probleme (falls vorhanden). Fügen Sie, falls die Change-Implementierung die in einem Problem angegebenen Change-Anforderungen erfüllt, eine Hinweisaktion hinzu, in der Sie angeben, wie der Change implementiert wurde, und setzen Sie das Feld Status auf Change fertiggestellt. Falls die Change-Implementierung die in einem Problem angegebenen Change-Anforderungen nicht erfüllt, fügen Sie eine Hinweisaktion mit dem Grund dafür hinzu. Wurde der Change von den Genehmigern abgelehnt, setzen Sie das Feld Status solcher Probleme auf Abgelehnt. Hat die Change-Implementierung die angegebenen Anforderungen nicht erfüllt (z. B. die Implementierung war unvollständig oder musste zurückgesetzt werden), setzen Sie das Feld Status auf Change fertiggestellt.

Hinweis:

Wenn Sie das Feld Status eines Problem auf Abgelehnt oder Change fertiggestellt setzen, wird das Problem an den Problem-Koordinator zurückgeleitet.

6.8.7

Falls die Anforderungen für den Change vom Release Management in Form eines Dokuments gesendet wurden, senden Sie eine E-Mail an den Release-Koordinator, um ihr/ihm mitzuteilen, ob die Implementierung des Change erfolgreich war. Falls die Change-Implementierung nicht erfolgreich war, geben Sie den Grund an.

6.8.8

Wenn Sie das Aktualisieren der Anforderer und Genehmiger abgeschlossen haben, setzen Sie das Feld Status der Aufgabe auf Fertiggestellt.

6.8.9

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für den Abschluss der Aufgabe benötigte Zeit ein.

Hinweis:

Falls die Implementierung unvollständig war und bereits eine Lösung für bekannte Probleme existiert, erstellen Sie einen Knowledge-Artikel für die bereitgestellte Lösung.
