Arbeitsanweisungen für Release-Administratoren

5.9.1

Wenn das Feld Status Ihrer Aufgabe zur Übertragung des neuen Release sich in Zugewiesen ändert, öffnen Sie die Aufgabe und lesen Sie die Anweisungen im Feld Beschreibung und in den Hinweisaktionen.

5.9.2

Wenn Sie nicht an der Aufgabe arbeiten können, legen Sie das Feld Status der Aufgabe auf Akzeptiert fest.

Hinweis:

Falls die Aufgabe einem anderen Release-Administrator zugewiesen werden soll, bitten Sie den Change-Koordinator, die Zuweisung zu ändern. Dafür legen Sie das Feld Status der Aufgabe auf Abgelehnt fest und fügen eine Hinweisaktion hinzu, in der Sie angeben, warum sie abgelehnt wurde.

5.9.3

Wenn Sie die Übertragung des neuen Release in die Produktionsumgebung durchführen wollen, legen Sie das Feld Status der Aufgabe auf In Bearbeitung fest und führen Sie die Übertragung aus.

Hinweis:

Wenn Sie an dieser Aufgabe nicht weiterarbeiten können (weil Sie zum Beispiel nicht die erforderlichen Zugriffsrechte erhalten haben), legen Sie das Feld Status auf Warten auf… fest und fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, worauf Sie warten und wann Sie die Aufgabe wahrscheinlich abschließen werden.

5.9.4

Wenn Sie die Übertragung des neuen Release in die Produktionsumgebung abgeschlossen haben, legen Sie das Feld Status der Aufgabe auf Fertiggestellt fest.

Wenn Sie die Übertragung nicht erfolgreich abschließen konnten, legen Sie das Feld Status der Aufgabe auf Fehlgeschlagen fest und geben Sie im Feld Lösung an, warum sie fehlgeschlagen ist.

5.9.5

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für den Abschluss der Aufgabe benötigte Zeit ein.
