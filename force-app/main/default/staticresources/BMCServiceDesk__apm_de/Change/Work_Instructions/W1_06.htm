Arbeitsanweisungen für Change-Koordinatoren

1.6.1

Setzen Sie den Anforderer davon in Kenntnis, dass der Change Request zu einem Konflikt mit einem internen Standard bzw. einer internen Richtlinien führt.

1.6.2

Falls der Change Request vom Request Management in Form einer Anforderung erhalten wurde, geben Sie im Feld Lösung an, mit welchem internen Standard oder mit welcher Richtlinie die Support-Anforderung in Konflikt steht. Setzen Sie das Feld Closure Category (Schließungskategorie) für die Anforderung auf Unable to solve or in conflict with standards or policy (Keine Lösung möglich oder Konflikt mit Standard oder Richtlinie) und das Feld Status auf Fertiggestellt. Indem das Feld Status auf „Abgeschlossen“ gesetzt wird, wird gewährleistet, dass der Service Desk die Person informiert, deren Record mit der Anforderung im Feld Kunden-ID verknüpft ist.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aktualisierung der Anforderung ein.

1.6.3

Wenn der Change Request vom Problem Management in Form eines Problems erhalten wurde, fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, welcher interne Standard oder welche Richtlinie mit der vorgeschlagenen strukturellen Lösung in Konflikt steht. Setzen Sie das Feld Status des Problems auf Abgelehnt, sodass das Problem zurück an den Problem-Koordinator geht.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Aktualisieren des Problems ein.
