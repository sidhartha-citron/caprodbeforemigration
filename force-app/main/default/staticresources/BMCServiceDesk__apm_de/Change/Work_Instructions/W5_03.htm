Arbeitsanweisungen für Kundenvertreter

5.3.1

Wenn das Feld Status der Aufgabe zum Test des neuen Release sich in Zugewiesen ändert, öffnen Sie die Aufgabe und lesen Sie die Anweisungen im Feld Beschreibung und in den Hinweisaktionen.

5.3.2

Wenn Sie noch nicht an der Aufgabe arbeiten können, legen Sie das Feld Status der Aufgabe auf Akzeptiert fest.

Hinweis:

Falls die Aufgabe einer anderen Person zugewiesen werden soll, bitten Sie den Change-Koordinator, die Zuweisung zu ändern. Dafür legen Sie das Feld Status der Aufgabe auf Abgelehnt fest und fügen eine Hinweisaktion hinzu, in der Sie angeben, warum sie abgelehnt wurde.

5.3.3

Wenn Sie mit dem Test des neuen Release beginnen möchten, legen Sie das Feld Status der Aufgabe auf In Bearbeitung fest und führen Sie die Tests in der Testumgebung aus.

5.3.4

Beschreiben Sie die Tests und die entsprechenden Ergebnisse in einem Dokument und hängen Sie dieses an die Aufgabe an.

Hinweis:

Wenn Sie nicht mit der Arbeit an der Aufgabe fortfahren können (z. B. weil die Testumgebung vorübergehend nicht verfügbar ist), legen Sie das Feld Status auf Warten auf… fest und fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, worauf Sie warten und wann Sie die Tests wahrscheinlich abschließen werden.

5.3.5

Wenn Sie den Test des neuen Release abgeschlossen haben und er alle Tests bestanden hat, legen Sie das Feld Status der Aufgabe auf Fertiggestellt fest.

Wenn der neue Release nicht alle Tests bestanden hat, legen Sie das Feld Status der Aufgabe auf Fehlgeschlagen fest und geben Sie im Feld Lösung an, welche Tests fehlgeschlagen sind.

5.3.6

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für den Abschluss der Aufgabe benötigte Zeit ein.
