Arbeitsanweisungen für Change-Koordinatoren

1.12.1

Klicken Sie auf die Registerkarte Change Requests und öffnen Sie einen neuen Change Request, indem Sie auf Neu klicken.

1.12.2

Vergewissern Sie sich, dass Sie im Feld Initiator-ID des Change Request ausgewählt sind.

1.12.3

Wählen Sie über das Feld Vorlage die Change-Vorlage, die Sie bei der Registrierung des angeforderten Change verwenden. Durch die Verwendung einer Vorlage wird sichergestellt, dass die Felder des Change Request mit den Werten der ausgewählten Change-Vorlage ergänzt sowie die erforderlichen Aufgaben erstellt und mit dem Change verknüpft werden, wenn dieser gemäß der ausgewählten Vorlage gespeichert wird.

1.12.4

Wählen Sie im Feld Kategorie die Option Non-Standard - Approved Change Template not Available (Kein Standard – Genehmigte Change-Vorlage nicht verfügbar) . Geben Sie außerdem im Feld Gründe des Change den Hauptgrund für die Implementierung des Change an .

1.12.5

Fügen Sie im Feld Beschreibung eine Kurzbeschreibung des Change hinzu.

1.12.6

Wählen Sie im Feld Change-Typ den geeigneten Typ aus.

1.12.7

Wählen Sie im Feld Change-Kategorie die geeignete Kategorie aus.

1.12.8

Speichern Sie den Change Request, um die dazugehörigen Aufgaben zu generieren und die Verknüpfung mit einer Anforderung, einem Problem, einem Release oder einer Anlage zu ermöglichen.

1.12.9

Fügen Sie eine Hinweisaktion hinzu, um die Bedingungen für den Change zusammenzufassen.

1.12.10

Wählen Sie im Abschnitt Verknüpfte Services und CIs die Services aus, die durch die Implementierung des Change beeinträchtigt werden würden.

1.12.11

Falls der Change Request vom Request Management in Form einer Anforderung erhalten wurde, öffnen Sie die Anforderung und verknüpfen Sie den Change Request im Abschnitt „Verknüpfte Change Requests“ der Anforderung. Fügen Sie der Anforderung die folgende Hinweisaktion hinzu: Responsibility for coordination has been accepted by Change Management (Verantwortung für die Koordination wurde vom Change Management akzeptiert).

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für das Prüfen der Anforderung aufgewendete Zeit ein.

Setzen Sie das Feld Status der Anforderung auf Change ausstehend.

1.12.12

Falls der Change Request vom Problem Management in Form eines Problems erhalten wurde, öffnen Sie das Problem und verknüpfen Sie den Change Request im Abschnitt Verknüpfte Change Requests des Problems. Fügen Sie dem Problem die folgende Hinweisaktion hinzu: Responsibility for coordination has been accepted by Change Management (Verantwortung für die Koordination wurde vom Change Management akzeptiert).

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen des Problems ein.

Setzen Sie das Feld Status des Problems auf Change ausstehend.

1.12.13

Falls der Change Request vom Release Management erhalten wurde, verknüpfen Sie den Release im Abschnitt Verknüpfte Releases des Change Request. Fügen Sie die Anforderungsdokumente im Abschnitt Hinweise und Anlagen des Change Request an.
