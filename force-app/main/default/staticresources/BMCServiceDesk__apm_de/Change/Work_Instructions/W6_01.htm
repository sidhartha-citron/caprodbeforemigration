Arbeitsanweisungen für Spezialisten

6.1.1

Wenn das Feld Status der Aufgabe zum Testen des implementierten Change in Zugewiesen geändert wird, öffnen Sie sie und lesen Sie die Anweisungen im Feld Beschreibung und in den Hinweisaktionen.

6.1.2

Wenn Sie noch nicht an der Aufgabe arbeiten können, legen Sie das Feld Status der Aufgabe auf Akzeptiert fest.

Hinweis:

Falls die Aufgabe einer anderen Person zugewiesen werden soll, bitten Sie den Change-Koordinator, die Zuweisung zu ändern. Dafür legen Sie das Feld Status der Aufgabe auf Abgelehnt fest und fügen eine Hinweisaktion hinzu, in der Sie angeben, warum sie abgelehnt wurde.

6.1.3

Wenn Sie mit der Arbeit beginnen können, legen Sie das Feld Status der Aufgabe auf In Bearbeitung fest und führen Sie einen letzten Test in der Produktionsumgebung aus, um zu überprüfen , ob alle Change-Anforderungen erfüllt sind.

Hinweis:

Falls Sie nicht über die entsprechenden Zugriffsrechte verfügen, um einen Test in der Produktionsumgebung durchzuführen, arbeiten Sie mit einem Benutzer zusammen, der über diese Zugriffsrechte verfügt.

6.1.4

Beschreiben Sie im Feld Lösung der Aufgabe den Test und das Ergebnis.

Hinweis:

Wenn Sie an dieser Aufgabe nicht weiterarbeiten können (z. B. wenn die Produktionsumgebung vorübergehend nicht verfügbar ist), legen Sie das Feld Status auf Warten auf… fest und fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, worauf Sie warten und wann Sie die Aufgabe voraussichtlich abschließen werden.

6.1.5

Wenn Sie nach Abschluss des Tests der Meinung sind, dass alle Change-Anforderungen erfüllt sind, setzen Sie das Feld Status der Aufgabe auf Fertiggestellt.

Falls der Test der Implementierung nicht erfolgreich war, bestimmen Sie, ob die Change-Implementierung zurückgesetzt werden muss. Geben Sie im Feld Lösung der Aufgabe Ihr Fazit und den Grund für den Produktionstest an. Lassen Sie in diesem Fall das Feld Status auf In Bearbeitung und lassen Sie die Aufgabe offen, während Sie die nächsten Schritte ausführen.

Hinweis:

Eine Change-Implementierung muss nicht zurückgesetzt werden, falls sie eine Verbesserung gegenüber der vorherigen Situation darstellt und keine Sicherheits- oder Datenintegritätsrisiken birgt.

6.1.6

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für die Aufgabe ein.
