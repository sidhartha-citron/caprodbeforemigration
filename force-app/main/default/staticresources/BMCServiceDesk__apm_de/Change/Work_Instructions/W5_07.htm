Arbeitsanweisungen für Change-Koordinatoren

5.7.1

Fügen Sie eine Hinweisaktion zum Change Request hinzu, in der Sie darauf hinweisen, dass der neue Release den Kundentest nicht bestanden hat, dass zusätzliche Arbeit erforderlich ist, um sicherzustellen, dass der Release den Test besteht, und dass der Change erneut genehmigt werden muss, weil das Timing zum Übertragen an die Produktion angepasst werden muss. Legen Sie die Felder Geplanter Start und Geplantes Ende auf die neuen Start- und Enddaten der Implementierung fest.

5.7.2

Setzen Sie das Feld Status des Change Request auf To Be Approved (Zur Genehmigung). Der Change Request wird in der Liste von Change Requests angezeigt, die auf die Genehmigung des Change Managers warten.
