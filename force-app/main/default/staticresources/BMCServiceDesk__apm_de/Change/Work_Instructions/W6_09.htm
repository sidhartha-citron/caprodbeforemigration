Arbeitsanweisungen für Change-Koordinatoren

6.9.1

Wenn die Arbeit am Change abgeschlossen ist, setzen Sie das Feld Status einer beliebigen Aufgabe, die nicht abgeschlossen werden muss (z. B. wenn die Implementierung von den Genehmigern abgelehnt wurde) auf Storniert. Geben Sie im Feld Lösung an, warum die Aufgabe storniert wurde.

6.9.2

Prüfen Sie für alle Aufgaben, die mit dem Change Request verknüpft sind, ob die Aufgaben den Endstatus (Fehlgeschlagen, Fertiggestellt oder Storniert) erreicht haben. Wählen Sie anschließend im Feld Abschlusskategorie des Change die entsprechende Option.

Hinweis:

Sie können den Change erst auf „Fertiggestellt“ setzen, wenn alle mit dem Change verknüpften Aufgaben ihren Endstatus (Fehlgeschlagen, Storniert oder Fertiggestellt) erreicht haben.

6.9.3

Setzen Sie das Feld Status des Change Request auf Fertiggestellt.

6.9.4

Geben Sie das tatsächliche Enddatum der Implementierung im Feld Tatsächliches Enddatum ein.
