Arbeitsanweisungen für Change-Koordinatoren

1.2.1

Suchen Sie nach den Changes für den zu ändernden Service.

1.2.2

Stellen Sie fest, ob bereits ein Change für einen Change Request mit denselben Anforderungen registriert wurde. Falls ein solcher Change existiert, fahren Sie fort mit Punkt 1.3.1.

1.2.3

Wenn Sie einen Change Request verwalten, bei dem die Verhinderung oder Behebung eines Problems angefordert wird, stellen Sie fest, ob bereits ein Change registriert wurde, dem Sie die Change-Anforderungen für das Problem hinzufügen können. Dies ist unter den folgenden Bedingungen möglich:

Die Funktionen des Service ändern sich nicht, wenn die Change-Anforderungen des Problems erfüllt sind.

Zum Erfüllen der Change-Anforderungen des Problems ist kein zusätzliches Budget erforderlich.

Das Feld Status des bestehenden Change ist nicht auf „Zur Genehmigung“ gesetzt.

Falls das Hinzufügen des Problems zu einem bestehenden Change möglich und effizient ist, gehen Sie weiter zu Punkt 1.3.1.

Hinweis:

Durch die Kombination von Change Requests vom Problem Management kann Zeit gespart werden, z. B. wenn mehrere Probleme zur Behebung von Mängeln in einer Anwendung an das Change Management weitergeleitet wurden. Das Verknüpfen der Change Requests mit einem einzelnen Change stellt sicher, dass nur ein Change genehmigt werden muss und nur ein neuer Release der Anwendung getestet und übertragen wird.

1.2.4

Für alle anderen Change Requests gehen Sie weiter zu Punkt 1.4.1.
