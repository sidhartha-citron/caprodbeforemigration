Arbeitsanweisungen für Spezialisten

5.1.1

Wenn das Feld Status Ihrer Aufgabe zur Entwicklung des neuen Release sich in Zugewiesen ändert, öffnen Sie die Aufgabe und lesen Sie die Anweisungen im Feld Beschreibung und in den Hinweisaktionen.

5.1.2

Wenn Sie noch nicht an der Aufgabe arbeiten können, legen Sie das Feld Status der Aufgabe auf Akzeptiert fest.

Hinweis:

Falls die Aufgabe einem anderen Entwickler zugewiesen werden soll, bitten Sie den Change-Koordinator, die Zuweisung der Aufgabe zu ändern. Dafür legen Sie das Feld Status der Aufgabe auf Abgelehnt fest und fügen eine Hinweisaktion hinzu, in der Sie angeben, warum sie abgelehnt wurde.

5.1.3

Wenn Sie mit der Entwicklung des neuen Release beginnen möchten, legen Sie das Feld Status der Aufgabe auf In Bearbeitung fest und entwickeln Sie den neuen Release.

Hinweis:

Wenn Sie nicht weiter an der Aufgabe arbeiten können (z. B. weil die Entwicklung schwieriger als erwartet ist und Sie auf die Hilfe eines Kollegen warten). legen Sie das Feld Status der Aufgabe auf Warten auf… fest und fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, worauf Sie warten und wann Sie die Aufgabe wahrscheinlich abschließen werden.

5.1.4

Wenn Sie die Entwicklung des neuen Release abschließen, testen Sie den Release, um sicherzugehen, dass die Change Requests erfüllt wurden. Testen Sie ihn in einer getrennten Testumgebung, falls eine solche vorhanden ist. Andernfalls verwenden Sie dazu die Entwicklungsumgebung. Falls (automatisierte) Skripte für Regressionstests für die Anwendung vorhanden sind, aktualisieren Sie die Skripte und führen Sie sie aus. Beheben Sie mögliche Fehler, die während der Tests aufgetreten sind, in der Entwicklungsumgebung. Nachdem Sie sich vergewissert haben, dass die Change Requests erfüllt wurden und keine neuen Probleme aufgetreten sind, geben Sie an, welche Test Sie im Feld Lösung der Aufgabe ausgeführt haben, und legen Sie das Feld Status auf Fertiggestellt fest.

Falls die Change Requests nicht erfüllt werden konnten, legen Sie das Feld Status der Aufgabe auf Fehlgeschlagen fest und geben Sie im Feld Lösung an, warum die erforderlichen Funktionen nicht entwickelt werden können.

5.1.5

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für den Abschluss der Aufgabe benötigte Zeit ein.
