Arbeitsanweisungen für Change-Koordinatoren

1.8.1

Leiten Sie den Change Request wieder an den Prozess, von dem aus er an das Change Management gesandt wurde.

1.8.2

Falls der Change Request vom Request Management in Form einer Anforderung erhalten wurde, fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, dass der Change im Rahmen des Request Management-Prozesses implementiert werden kann.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen der Anforderung ein.

Wählen Sie Ihren Gruppenkoordinator im Feld Besitzer der Support-Anforderung aus. Setzen Sie das Feld Status der Support-Anforderung auf Zugewiesen.

Hinweis:

Setzen Sie das Feld Status der Support-Anforderung nicht auf Abgelehnt, weil die Service Management-Anwendung die Anforderung im Dashboard des Service Desk anzeigt.

1.8.3

Falls der Change Request vom Problem Management in Form eines Problems erhalten wurde, fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, dass der Change im Rahmen des Problem Management-Prozesses implementiert werden kann.

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen des Problems ein.

Setzen Sie das Feld Status des Problems auf Abgelehnt, damit das Problem wieder dem Problemkoordinator zugewiesen wird.
