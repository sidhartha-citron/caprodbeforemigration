Arbeitsanweisungen für Change-Koordinatoren

1.10.1

Weisen Sie den Change Request dem Release-Koordinator des Service zu, für den ein Change angefordert wurde.

1.10.2

Falls der Change Request in Form einer Anforderung vom Request Management eingegangen ist, geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen der Anforderung ein.

Wählen Sie im Feld Besitzer der Anforderung den Release-Koordinator des Service, für den der Change angefordert wurde. Setzen Sie das Feld Status auf Zugewiesen.

1.10.3

Falls der Change Request in Form eines Problems vom Problem Management eingegangen ist, geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Prüfen des Problems ein.

Wählen Sie im Feld Besitzer des Problems den Release-Koordinator des Service, für den der Change angefordert wurde. Setzen Sie das Feld Status des Problems auf Change Requested (Change angefordert).
