Arbeitsanweisungen für Change-Koordinatoren

6.7.1

Prüfen Sie für die Produktionstestaufgabe, deren Feld Status vom Spezialisten auf Fehlgeschlagen festgelegt wurde, die Angaben im Feld Lösung .

6.7.2

Fügen Sie die erforderlichen zusätzlichen Implementierungsaufgaben ein, um alle Change-Bedingungen zu erfüllen.

Hinweis:

Wenn mehrere Aufgaben einer Person zugeteilt werden und diese Aufgaben aufeinanderfolgend und ohne Unterbrechung erledigt werden müssen, können sie eine einzige Aufgabe für alle diese Aufgaben erstellen.

6.7.3

Fügen Sie, falls die zusätzliche Arbeit dazu führt, dass eine oder mehrere Kundenunternehmen unterschiedliche Funktionen erhalten oder es zeitweise zu Serviceverschlechterungen bzw. -unterbrechungen kommt (egal ob inner- oder außerhalb der Servicezeiten), Informationen zu diesen betroffenen Unternehmen den Implementierungsaufgaben, deren Ausführung eine Beeinträchtigung bedeutet, in der Hinweisaktion der Aufgaben hinzu.

6.7.4

Verknüpfen Sie jede neue Implementierungsaufgabe mit den CIs, die durch das Ausführen der Aufgabe verändert werden. Klicken Sie dazu auf Neuer Link im Bereich Verknüpfte Services und CIs der Aufgaben.

6.7.5

Wählen Sie für jede Implementierungsaufgabe einen geeigneten Mitarbeiter aus Ihrer Gruppe (in puncto Kenntnisse, Verfügbarkeit und Zugriffsrechte) aus. Wählen Sie diese Person im Feld Besitzer der Aufgabe aus.

Falls die Implementierungsaufgabe von einer anderen Gruppe erledigt werden muss, arbeiten Sie mit dem Koordinator der Gruppe zusammen, um die für diese Aufgabe geeignetsten Personen (in puncto Kenntnisse, Verfügbarkeit und Zugriffsrechte) zu bestimmen und ein realistisches Abschlussziel festzulegen. Wählen Sie diese Person im Feld Besitzer der Aufgabe aus. Wenn der Gruppenkoordinator jedoch noch nicht weiß, wer diese Aufgabe abschließen soll, wählen Sie die Gruppe im Feld Besitzer aus. In diesem Fall kann der Gruppenkoordinator die geeignete Person im Feld Besitzer auswählen, sobald der Status der Aufgabe auf Zugewiesen festgelegt wurde.

Falls eine Implementierungsaufgabe von einem externen Unternehmen abgeschlossen werden muss, erarbeiten Sie zusammen mit Ihrer Kontaktperson in diesem Unternehmen ein realistisches Implementierungsdatum. Wählen Sie sich selbst im Feld Besitzer der Aufgabe aus und fügen Sie eine Hinweisaktion hinzu, um das für die Arbeit vorgesehene Unternehmen auszuwählen. Sobald mit der Arbeit begonnen werden kann, informieren Sie den Lieferanten und geben Sie die Referenznummer, unter welcher der Lieferant diese Zuweisung registriert hat, in derselben Hinweisaktion ein.

6.7.6

Vergewissern Sie sich, dass das Datum und die Uhrzeit, die im Feld Fälligkeitsdatum der neuen Aufgaben angegeben wurden, realistisch sind. Aktualisieren Sie außerdem bei Bedarf das Feld Fälligkeitsdatum für Aufgaben, die zu einem früheren Zeitpunkt geplant waren und nach Durchführung der neuen Aufgaben erledigt werden müssen.

6.7.7

Legen Sie das Feld Status der neuen Aufgabe auf Registriert fest.

6.7.8

Fügen Sie eine Hinweisaktion hinzu, die darauf hinweist, dass der Change Request erneut genehmigt werden muss, da zusätzliche Arbeiten erforderlich sind, damit alle Change-Anforderungen erfüllt sind.

6.7.9

Legen Sie das Feld Status des Change Request auf To Be Approved (Zur Genehmigung). Der Change Request wird in der Liste der Changes angezeigt, die vom Change Manager genehmigt werden müssen.
