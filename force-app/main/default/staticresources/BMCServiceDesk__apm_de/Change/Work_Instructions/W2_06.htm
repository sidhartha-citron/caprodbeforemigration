Arbeitsanweisungen für Change-Koordinatoren

2.6.1

Wenn der Change dazu führt, dass es für ein oder mehrere Kundenunternehmen zu Veränderungen an den bereitgestellten Funktionen oder vorübergehend zu einer Serviceverschlechterung oder zu Serviceausfällen kommt (gleichgültig, ob dies innerhalb oder außerhalb der Servicezeiten geschieht), fügen Sie den Implementierungsaufgaben, deren Ausführung Auswirkungen für diese Unternehmen haben wird, eine Hinweisaktion hinzu und geben Sie die Unternehmen an.

2.6.2

Verknüpfen Sie jede Implementierungsaufgabe mit den CIs, die sich ändern (falls vorhanden), wenn die Aufgabe ausgeführt wird. Klicken Sie dazu auf Neuer Link im Abschnitt Verknüpfte Services und CIs der Aufgabe.

2.6.2

Fügen Sie eine Hinweisaktion hinzu, um die erwartete Ausfallzeit für jede Implementierungsaufgabe anzugeben.

2.6.3

Wählen Sie für jede Implementierungsaufgabe die geeignetsten Mitarbeiter aus Ihrer Gruppe (in puncto Kenntnisse, Verfügbarkeit und Zugriffsrechte) aus. Wählen Sie diese Person im Feld Besitzer für die Aufgabe aus.

Falls die Implementierungsaufgabe von einer anderen Gruppe erledigt werden muss, arbeiten Sie mit dem Koordinator der Gruppe zusammen, um die für diese Aufgabe geeignetsten Personen (in puncto Kenntnisse, Verfügbarkeit und Zugriffsrechte) zu bestimmen und ein realistisches Abschlussziel festzulegen. Wählen Sie diese Person im Feld Besitzer für die Aufgabe aus. Wenn der Koordinator jedoch angibt, dass noch nicht bekannt ist, wer für die Aufgabe bestimmt werden soll, wählen Sie die Gruppe im Feld Besitzer der Aufgabe aus. In diesem Fall kann der Gruppenkoordinator die geeignete Person im Feld Besitzer auswählen, sobald der Aufgabenstatus auf „Zugewiesen“ gesetzt wurde.

Falls eine Implementierungsaufgabe durch eine externe Organisation erledigt werden muss, klären Sie mit Ihrer Kontaktperson im Unternehmen, wer sich am besten für diese Aufgabe eignet. Wählen Sie sich selbst im Feld Besitzer der Aufgabe aus und fügen Sie eine Hinweisaktion hinzu, um die für die Arbeit vorgesehene externe Organisation einzugeben. Sobald mit der Arbeit begonnen werden kann, informieren Sie den Lieferanten und geben Sie die Referenznummer, unter welcher der Lieferant diese Zuweisung erfasst hat, in der Hinweisaktion ein.

2.6.4

Vergewissern Sie sich, dass Datum und Uhrzeit, die im Feld Fälligkeitsdatum der Aufgaben angegeben wurden, realistisch sind. Bei nicht standardmäßigen Changes oder bei Standard-Changes, die eine Risiko- und Auswirkungsanalyse erfordern, geben Sie das Startdatum der ersten Implementierungsaufgabe im Feld Geplanter Start und das Zieldatum der letzten Implementierungsaufgabe im Feld Geplantes Ende an.

2.6.5

Setzen Sie das Feld Status der Implementierungsaufgabe auf Registriert.

2.6.6

Geben Sie im Feld Backout-Plan für den Change Request an, wie der Change, falls erforderlich, zurückgesetzt werden soll. Beispiel: „Falls ein Zurücksetzen erforderlich ist, stellen Sie das Backup wieder her, das vor dem Übertragen des neuen Release an die Produktion erstellt wurde.“
