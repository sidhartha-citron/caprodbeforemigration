Prozessbeschreibung


Der Change Management-Prozess besteht aus acht Verfahren. Sechs davon dienen zum Implementieren geplanter Changes, zwei zum Implementieren von Notfall-Changes.

  • Die Change-Registrierung wird von Change-Koordinatoren zum Verwalten von Change Requests verwendet.

  • Die Change-Planung wird von Change-Koordinatoren und Spezialisten zum Erstellen von Implementierungsplänen für geplante Changes verwendet.

  • Die Change-Genehmigung wird vom Change Manager und von Change-Genehmigern (Kundenvertretern und Servicebesitzern) zum Genehmigen geplanter Changes verwendet.

  • Die Implementierung von Infrastruktur-Changes wird von Spezialisten zur Implementierung von Infrastruktur-Changes verwendet.

  • Die Implementierung des Anwendungs-Change wird von Spezialisten, Release-Administratoren, Kundenvertretern und Change-Koordinatoren zur Implementierung von Anwendungs-Changes verwendet.

  • Die geplante Change-Schließung wird von Spezialisten verwendet, wenn diese nach Implementierung eines Change einen abschließenden Produktionstest durchführen, und von Change-Koordinatoren, wenn sie einen Change abschließen.

  • Die Implementierung von Notfall-Changes wird von Spezialisten und Release-Administratoren zur Implementierung von Changes zum Lösen von Incidents verwendet.

  • Die Schließung von Notfall-Changes wird von Spezialisten und Change-Koordinatoren zum Fertigstellen und Schließen von Notfall-Changes verwendet.
