Prozessbeschreibung

Der Problem Management-Prozess besteht aus vier Verfahren:

  • Die Anforderungsprüfung wird von Problem-Koordinatoren verwendet, wenn sie Anforderungen prüfen, um Probleme innerhalb des Services, für den sie verantwortlich sind, zu identifizieren.

  • Die Ursachenanalyse wird von Spezialisten zur Problemanalyse verwendet.

  • Die Analyseprüfung wird von Problem-Koordinatoren ausgeführt, wenn Sie die Ergebnisse einer von einem Spezialisten durchgeführten Ursachenanalyse prüfen.

  • Die Problemschließung wird von Problem-Koordinatoren bei der Schließung von Problemen verwendet.

Um weitere Informationen zu diesen Verfahren zu erhalten, klicken Sie auf Prozess, um zur grafischen Prozessdarstellung zurückzukehren, und danach auf die Boxen, welche die verschiedenen Verfahren verkörpern. Bei der grafischen Darstellung eines Verfahrens wird deren Beschreibung im rechten Bereich angezeigt.
