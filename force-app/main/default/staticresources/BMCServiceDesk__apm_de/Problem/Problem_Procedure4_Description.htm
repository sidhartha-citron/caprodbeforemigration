Verfahren 4, Problemschließung


Nachdem ein Change zur strukturellen Lösung des Problems implementiert wurde, überprüft der Problem-Koordinator, ob das Problem dadurch behoben wurde.

Wenn das Problem durch die Implementierung gelöst wurde, schließt der Problem-Koordinator das Problem. Wenn das Problem reaktiv identifiziert wurde, informiert die Servicemanagement-Anwendung den Service Desk per E-Mail, dass das Problem aus dem Service entfernt wurde.

Wenn der Change hingegen nicht implementiert wurde oder das Problem durch die Implementierung nicht behoben werden konnte, entscheidet der Problem-Koordinator, ob eine andere strukturelle Lösung für das Problem gefunden werden kann. Ist dies der Fall, weist der Problem-Koordinator das Problem einem Spezialisten innerhalb der Gruppe neu zu.

Wenn derzeitig jedoch keine praktischen Maßnahmen zur langfristigen Behebung oder Umgehung der Ursache vorhanden sind, aktualisiert der Problem-Koordinator das Problem, um die Dringlichkeit anzuzeigen. Der Problem-Koordinator sucht regelmäßig nach neuen Lösungen und bittet um eine weitere Analyse wenn es wahrscheinlich ist, dass durch eine neue Herangehensweise oder den Einsatz einer anderen Technologie eine praktische strukturelle Lösung gefunden werden kann.

Wenn die Lösung oder die Umgehungslösung des Problems in der Knowledge Base aufgeführt werden soll, wird ein neuer Knowledge-Artikel erstellt.
