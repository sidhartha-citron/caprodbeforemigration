Arbeitsanweisungen für Problem-Koordinatoren

1.7.1

Um ein neues Problem zu öffnen, navigieren Sie zur Registerkarte Probleme und klicken Sie auf Neu.

1.7.2

Vergewissern Sie sich, dass Sie im Feld Besitzer der Aufgabe ausgewählt sind.

1.7.3

Wählen Sie im Feld Quelle des Problems die richtige Quelle.

1.7.4

Wählen Sie im Feld Kategorie die richtige Kategorie.

1.7.5

Geben Sie im Feld Beschreibung eine kurze Beschreibung ein.

1.7.6

Wählen Sie in den Feldern Auswirkung und Dringlichkeit die entsprechende Auswirkung und Dringlichkeit. Klicken Sie auf Speichern, um das Problem zu speichern.

1.7.7

Fügen Sie eine Hinweisaktion mit einer detaillierten Beschreibung der Problemsymptome ein. Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die für das Identifizieren und Registrieren des Problems aufgewendete Zeit ein.

1.7.8

Um den vom Problem betroffenen Service zu verknüpften, klicken Sie im Abschnitt Verknüpfte Services und CIs auf Neuer Link.

1.7.9

Um das vom Problem betroffene CI zu verknüpften, klicken Sie im Abschnitt Verknüpfte Services und CIs auf Neuer Link.

1.7.10

Klicken Sie im Abschnitt Verknüpfte Change Requests auf Change Request mit Problem verknüpfen. Sie können auch eine zuvor geprüfte Anforderung verknüpfen, die bisher nicht bedeutend war, aber eine Serviceunterbrechung verursacht hat.
