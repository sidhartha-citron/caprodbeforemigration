Arbeitsanweisungen für Problem-Koordinatoren

4.5.1

Fügen Sie eine Hinweisaktion hinzu, um anzugeben, warum eine erneute Analyse des Problems angefordert wird.

4.5.2

Bestimmen Sie, welcher Spezialist sich (in puncto Kenntnisse, Verfügbarkeit und Zugriffsrechte) am ehesten zur Durchführung einer besseren Analyse eignet. Dies kann auch der Spezialist sein, der die frühere Analyse des Problems durchgeführt hat. Verwenden Sie die Option Ändern neben dem Feld Besitzer, um den Spezialisten auszuwählen.

Hinweis:

Falls das Problem ein CI zu sein scheint, das Teil einer Serviceinstanz ist, die den Service unterstützt, für den Sie als Problem-Koordinator agieren, kontaktieren Sie den Problem-Koordinator des unterstützenden Service. Besprechen Sie das Problem. Wenn Sie mit dem Problem-Koordinator einer Meinung sind, weisen Sie das Problem dem anderen Problem-Koordinator zu, indem Sie den Namen im Feld Besitzer ändern. Fassen Sie im Feld Informationen zusammen, warum Sie die Verantwortung für das Problem an den anderen Problem-Koordinator übergeben haben.

4.5.3

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion den Zeitaufwand für das Identifizieren und Registrieren des Problems ein.

4.5.4

Setzen Sie das Feld Status auf Zugewiesen.
