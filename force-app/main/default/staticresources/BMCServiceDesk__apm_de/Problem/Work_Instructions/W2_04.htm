Arbeitsanweisungen für Spezialisten

2.4.1

Um die Ursache des Problems zu finden, sehen Sie sich die Protokolldateien an oder recherchieren Sie im Internet.

2.4.2

Wenn Sie die Ursache ermittelt haben, fügen Sie diese Informationen in eine Hinweisaktion ein und setzen Sie das Feld Status auf Bekannter Fehler. Alternativ können Sie eine Hinweisaktion hinzufügen, wenn die Ursache nicht ermittelt werden konnte.

Hinweis:

Wenn Sie nicht mit der Analyse fortfahren können (weil Sie zum Beispiel auf Informationen eines Lieferanten warten), fügen Sie eine Hinweisaktion hinzu, in der Sie angeben, worauf Sie warten, und setzen Sie das Feld Status auf Waiting for… (Warten auf).

2.4.3

Verknüpfen Sie das CI, das die Ursache darstellt, im Abschnitt Verknüpfte Services und CIs. Wenn die Ursache nicht ermittelt werden konnte und nicht bekannt ist, welches CI das Problem verursacht hat, wählen Sie das Sonder-CI mit dem Code N/A.

2.4.4

Wenn die Ursache ermittelt werden konnte, lesen Sie den im Feld Umgehungslösung angegebenen Knowledge-Artikel. Wenn Sie eine bessere Umgehungslösung vorschlagen können, fügen Sie die Informationen im Knowledge-Artikel hinzu.

2.4.5

Geben Sie im Feld Dauer (HH:MM) in der Hinweisaktion die Zeit ein, die Sie für die Bearbeitung des Problems seit der letzten Aktualisierung benötigt haben.
