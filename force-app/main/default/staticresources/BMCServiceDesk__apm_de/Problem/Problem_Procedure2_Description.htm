Verfahren 2, Ursachenanalyse


Der Spezialist überprüft die Details des Problems. Wenn das Problem identifiziert wurde, nachdem es einen oder mehrere Incidents ausgelöst hat, versucht der Spezialist, eine Umgehungslösung bereitzustellen. Es wird ein Knowledge-Artikel mit den Informationen zur Umgehungslösung und ihrer Implementierung erstellt. Dem Problem wird eine Referenz zum Artikel hinzugefügt.

Diese Umgehungslösung kann zur Lösung zukünftiger, durch dieses Problem verursachter Incidents verwendet werden, bis eine strukturelle Lösung gefunden und implementiert wurde.

Sobald die Ursachenanalyse vom Spezialisten durchgeführt wurde, wird das Problem mit seiner Beschreibung aktualisiert.

Anschließend erwägt der Spezialist mögliche strukturelle Lösungen und bewertet diese. Dem Problem wird neben einer Empfehlung der bevorzugten strukturellen Lösung eine Beschreibung jeder Option hinzugefügt.

Wenn Change Management zum Entfernen einer Ursache nicht erforderlich ist, implementiert der Spezialist die bevorzugte strukturelle Lösung.

Wenn der Spezialist die Ursache nicht finden oder keine strukturelle Lösung vorschlagen kann, erwähnt der Spezialist den Grund dafür in dem Problem.

Der Spezialist aktualisiert den Status des Problems unabhängig davon, ob eine strukturelle Lösung vorgeschlagen oder bereits implementiert wurde, um den Problem-Koordinator über den Abschluss der Arbeit zu informieren.
