Schrittweise Anleitung zum Erstellen einer Anforderungsdefinition
 Nicht als Registerkarte für Startseite anzeigen
(Zum Neustart klicken Sie auf in der Anforderungsdefinitionsliste.)

Sie können Serviceanforderungsdefinitionen (SRDs) zum Definieren von angebotenen Services verwenden und einen Servicekatalog für Ihre Selbstbedienungskunden erstellen. Mithilfe von SRDs können Sie die Erfüllung dieser Services automatisieren. SRDs werden in der Selbstbedienung als Serviceanforderungen angezeigt. Serviceanforderungen von Kunden werden als Incidents gespeichert. Dabei wird bei Is Service Request (Ist Serviceanforderung) die Option „True“ angegeben.
Serviceanforderungen werden in der Remedyforce-Konsole im Modul „Serviceanforderungen“ angezeigt.

Schritt 1. Identifizieren Sie die Kategorie, den Service und das Serviceangebot, für das Sie eine SRD erstellen. Mithilfe von Kategorien können Sie die angebotenen Services in Ihrem Servicekatalog entsprechend gruppieren.
 
Schritt 2. Klicken Sie hier und erstellen Sie den Service und das Serviceangebot, die Sie Ihren Kunden zur Verfügung stellen möchten.
 
Schritt 3. Klicken Sie hier und erstellen Sie die Kategorie, die Sie in der SRD verwenden möchten.
 
Schritt 4. Klicken Sie hier und erstellen Sie Vorlagen, die Sie in der SRD verwenden möchten: Details anzeigen... Details ausblenden...
  • Erstellen Sie eine Incident-Vorlage, die Standardwerte für Felder, wie zum Beispiel „Auswirkung“ und „Dringlichkeit“ im Incident-Record enthält.
    Mithilfe dieser Vorlage werden Incident-Records erstellt. Dabei wird bei Is Service Request (Ist Serviceanforderung) die Option „True“ ausgewählt.
  • Erstellen Sie die Anforderungsdetailvorlage, die Sie in der SRD verwenden möchten. In dieser Vorlage müssen Sie die Incident-Vorlage auswählen, die Sie zuvor erstellt haben.
    Mithilfe der Servicedetailvorlage können Sie den Standardwert des Felds Menge festlegen. Beispiel: Zum Beantragen eines Laptops können Sie den Standardwert bei Menge auf 1 festlegen.
 
Schritt 5. Auf der Seite „Anforderungsdefinition“ müssen Sie Informationen auf mehreren Registerkarten eingeben: Allgemeine Informationen, Berechtigung, Display Options (Anzeigeoptionen), Eingabe und Erfüllung. Details anzeigen...Details ausblenden...
Geben Sie auf der Registerkarte Allgemeine Informationen folgende Informationen ein: 
  • Geben Sie einen Namen und eine Beschreibung ein, mit denen Kunden ermitteln können, ob die SRD die Anforderungen erfüllt.
    Geben Sie beispielsweise „Serviceanforderung für iPhone“ ein.
  • Wählen Sie den Service, das Serviceangebot, die Kategorie, die Incident-Vorlage und die Anforderungsdetailvorlage für die zuvor erstellte SRD oder erstellen Sie diese neu. 
 
Schritt 6. Wählen Sie auf der Registerkarte Berechtigung die Konten, Profile oder Berechtigungssätze, deren Kunden auf die SRD in der Selbstbedienung zugreifen können.
Sie können die SRD für alle Kunden, für Kunden mit bestimmten Konten oder für Kunden mit bestimmten Profilen oder Berechtigungssätzen zur Verfügung stellen.
 
Schritt 7. Konfigurieren Sie auf der Registerkarte Display Options (Anzeigeoptionen) die Serviceanforderungsinformationen, die den Kunden in der Selbstbedienung angezeigt werden sollen.
Um beispielsweise den Preis für das iPhone und die Bearbeitungszeit bis zur Lieferung des Telefons an die Kunden anzuzeigen, aktivieren Sie die Kontrollkästchen Preis und Bearbeitungszeit .
 
Schritt 8. Definieren Sie auf der Registerkarte Eingabe die zusätzlichen Eingabefelder, über die Mitarbeiter die Serviceanforderung erfolgreich abschließen können. Damit beispielsweise ein Laptop mithilfe einer Serviceanforderung bereitgestellt werden kann, können Sie Aufgabenvorlagen für die Beschaffung des Laptops und für die Installation von Betriebssystem und Anwendungen auswählen. 
Schritt 9. Wählen Sie auf der Registerkarte Erfüllung die zur Erfüllung der Serviceanforderung erforderlichen Vorlagen Beispiel: Zum Beantragen eines Laptops können Sie Aufgabenvorlagen zur Anschaffung eines Laptops und Installation eines Betriebssystems und von Anwendungen auswählen.  Details anzeigen...Details ausblenden...
Definieren Sie den Erfüllungsprozess für eine SRD, indem Sie Informationen in folgenden Bereichen bereitstellen:  
  • Vorlage — Klicken Sie hier, um Aufgaben- oder Change Request-Vorlagen für jeden Schritt des Erfüllungsprozesses zu erstellen. Wählen Sie dann die Aufgaben- oder Change Request-Vorlagen, die zur Erfüllung der Serviceanforderung erforderlich sind.
  • Zuordnung — Zuordnen von Eingabefeldern zu den Erfüllungsvorlagen.
    Zuordnung ist für die Bereitstellung von Informationen für Aufgaben und Change-Anforderungen erforderlich, die Sie für die Erfüllung der Serviceanforderung erstellen.
 
Schritt 10. Führen Sie auf der Registerkarte Allgemeine Informationen folgende Schritte aus, um die Anforderungsdefinition in der Selbstbedienung anzuzeigen: Details anzeigen...Details ausblenden...
  •  Wählen Sie den Status Bereitgestellt der Anforderungsdefinition.
  •  Um die Anforderungsdefinition in der Selbstbedienung anzuzeigen, aktivieren Sie das Kontrollkästchen Online.
  •  Um die Anforderungsdefinition in der Mobilanwendung von Remedyforce-Selbstbedienung oder BMC MyIT anzuzeigen, aktivieren Sie das Kontrollkästchen Mobil.
 
Schritt 11. Klicken Sie hier, um zu überprüfen, ob die SRD in der Selbstbedienung verfügbar ist.
