Step-by-step guide for creating a request definition
 Don't show as landing tab
(To launch again, click  from the Request Definition list)

You can use service request definitions (SRDs) to define offered services and build a service catalog for your Self Service clients. Using SRDs, you can automate the fulfillment of those services. SRDs are displayed in Self Service as service requests. Service requests that are raised by clients are saved as incidents with the Is Service Request set to "True".
Service requests are shown in the Service Requests module in the Remedyforce Console.

Step 1. Identify the category, service, and service offering for which you are creating an SRD. Categories enable you to appropriately group the offered services in your service catalog.
 
Step 2. Click here and create the service and service offering that you are providing to your clients.
 
Step 3. Click here and create the category that you will use in the SRD.
 
Step 4. Click here  and create templates that you will use in the SRD: Show details... Hide details...
  • Create the Incident template that will contain default values for fields, such as Impact and Urgency, in the incident record.
    This template is used to create incident records with the Is Service Request set to “True.”
  • Create the Request Detail template that you will use in the SRD. In this template, you must select the incident template that you created previously.
    You can use the Request Detail template to set the default value of the Quantity field. For example, for a service request to issue a laptop, you can set the default value of Quantity to 1.
 
Step 5. On the Request Definition page, you are required to provide information in multiple tabs – General Information, Input, Fulfillment, Display Options and Entitlement. Show details...Hide details...
On the General Information tab, provide information as follows: 
  • Enter a name and description that will enable clients to determine if the SRD meets their requirements.
    For example, enter “Service Request for iPhone”
  • Select or create new the service, service offering, category, incident template, and request detail template for the SRD that you created in the previous steps. 
 
Step 6. On the General Information tab, perform the following steps to show the request definition in Self Service: Show details...Hide details...
  •  Select the Deployed status of the request definition.
  •  To show the request definition in Self Service, select the Online check box.
  •  To show the request definition in the Remedyforce Self Service mobile application or in BMC MyIT, select the Mobile check box.
 
Step 7. On the Input tab, define the additional input fields that will enable staff members to successfully complete the service request. For example, for a service request to issue a laptop, you can select task templates to procure a laptop and install an operating system and applications. 
Step 8. On the Fulfillment tab select the templates that are required to fulfill the service request For example, for a service request to issue a laptop, you can select task templates to procure a laptop and install an operating system and applications.  Show details...Hide details...
Define the fulfillment process for an SRD by proving information in the following areas:  
  • Template—Click here  to create task or change request templates for each step of the fulfillment process. Then select the task or change request templates that are required to fulfill the service request.
  • Mapping—Map the input fields to the fulfillment templates.
    Mapping is required to provide information for the tasks and change requests that you create to fulfill the service request.
 
Step 9. On the Display Options tab, configure the service request information to display to clients in Self Service.
For example, to display the price of the iPhone and the turnaround time to deliver the phone to clients, select the Price and Turnaround Time check boxes.
 
Step 10. On the Entitlement tab select accounts, profiles, or permission sets whose clients can access the SRD in Self Service.
You can make the SRD available to all clients, to clients associated with specific accounts, or to clients associated with specific profiles or permission sets.
 
Step 11. Click here to verify that the SRD is available in Self Service.
