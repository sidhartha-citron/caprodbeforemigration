Guía paso a paso para crear una definición de petición
 No mostrar como ficha de aterrizaje
(Para volver a iniciar, haga clic en  desde la lista de definición de petición)

Puede usar definiciones de petición de servicio (SRD) para definir servicios ofrecidos y crear un catálogo de servicios para sus clientes de Autoservicio. Al usar SRD, puede automatizar el cumplimiento estos servicios. Las SRD se muestran en Autoservicio como peticiones de servicio. Las peticiones de servicio creadas por clientes se guardan como incidencias con Es petición de servicio establecido como "Verdadero".
Las peticiones de servicio se muestran en el módulo Peticiones de servicio de la Consola de RemedyForce.

Paso 1. Identifique la categoría, el servicio y la oferta de servicio para la que crea la SRD. Las categorías le permiten agrupar los servicios ofrecidos adecuadamente en el catálogo de servicio.
 
Paso 2. Haga clic aquí y cree el servicio y la oferta de servicio que proporciona a sus clientes.
 
Paso 3. Haga clic aquí y cree la categoría que usará en la SRD.
 
Paso 4. Haga clic aquí  y cree las plantillas que usará en la SRD: Mostrar detalles... Ocultar detalles...
  • Cree la plantilla de incidencia que contendrá los valores predeterminados de los campos, como el impacto y la urgencia, en el registro de incidencia.
    Esta plantilla se usa para crear registros de incidencias con Es petición de servicio establecido como “Verdadero”.
  • Cree la plantilla Detalle de petición que usará en la SRD. En esta plantilla, debe seleccionar la plantilla de incidencia que creó con anterioridad.
    Puede usar la plantilla Detalle de petición para establecer el valor predeterminado en el campo Cantidad . Por ejemplo, para una petición de servicio para proveer un portátil, pude establecer el valor predeterminado de Cantidad como 1.
 
Paso 5. En la página Definición de petición, debe proporcionar información en varias fichas: Información general, Derecho, Opciones de visualización, Entrada y Cumplimiento. Mostrar detalles...Ocultar detalles...
En la ficha Información general, proporcione la siguiente información: 
  • Introduzca un nombre y una descripción para que los clientes puedan determinar si la SRD cumple con sus requisitos.
    Por ejemplo, introduzca “Petición de servicio para iPhone”.
  • Seleccione o cree un nuevo servicio, la oferta de servicio, la categoría, la plantilla de incidencia y la de detalle de petición para la SRD que creó en los pasos anteriores. 
 
Paso 6. En la ficha Derecho, seleccione cuentas, perfiles o conjuntos de permisos cuyos clientes puedan acceder a la SRD en Autoservicio.
Puede poner la SRD a disposición de todos los clientes, clientes asociados con cuentas específicas o clientes asociados con conjuntos de permisos o perfiles específicos.
 
Paso 7. En la ficha Opciones de visualización, configure la información de petición de servicio para mostrar a los clientes en Autoservicio.
Por ejemplo, para mostrar el precio de un iPhone y el tiempo de respuesta para enviar el teléfono a clientes, seleccione las casillas de verificación Precio y Tiempo de respuesta .
 
Paso 8. En la ficha Entrada, defina los campos de entrada adicionales que permitirán a los miembros del personal completar la petición de servicio correctamente. Por ejemplo, para una petición de servicio para proveer un portátil, puede seleccionar plantillas de tarea para obtener un portátil e instalar un sistema operativo y aplicaciones. 
Paso 9. En la ficha Cumplimiento, seleccione las plantillas requeridas para cumplir la petición de servicio Por ejemplo, para una petición de servicio cuya finalidad sea obtener un ordenador portátil, puede seleccionar plantillas de tarea para suministrar un portátil e instalar un sistema operativo y aplicaciones.   Mostrar detalles...Ocultar detalles...
Defina el proceso de cumplimiento para una SRD proporcionando información en las siguientes áreas:  
  • Plantilla: haga clic  aquí para crear plantillas de tarea o petición de cambio para cada paso del proceso de cumplimiento. A continuación, seleccione las plantillas de tarea o petición de cambio necesarias para completar la petición de servicio.
  • Asignación: asigne los campos de entrada a las plantillas de cumplimiento.
    La asignación se necesita para proporcionar información sobre las tareas y peticiones de cambio que crea para completar la petición de servicio.
 
Paso 10. En la ficha Información general, siga los siguientes pasos para mostrar la definición de petición en Autoservicio: Mostrar detalles...Ocultar detalles...
  •  Seleccione el estatus Desplegado de la definición de petición.
  •  Para mostrar la definición de petición en Autoservicio, seleccione la casilla de verificación Con conexión.
  •  Para mostrar la definición de petición en la aplicación móvil Autoservicio de Remedyforce o BMC MyIT, seleccione la casilla de verificación Móvil.
 
Paso 11. Haga clic aquí para verificar que la SRD está disponible en Autoservicio.
