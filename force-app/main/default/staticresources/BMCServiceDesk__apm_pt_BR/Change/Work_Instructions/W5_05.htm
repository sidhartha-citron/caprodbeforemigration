Work Instructions for Change Coordinators

5.5.1

Review the test result documentation from the customer representatives who discovered that the new release is not functioning as required.

5.5.2

Insert the necessary additional implementation tasks for correcting the new release and for getting it tested again by the customer representatives.

Note:

When multiple tasks must be assigned to the same staff, and if these tasks are to be completed in sequence without interruption, you can create one task to cover all these tasks.

5.5.3

Link each new task to the CIs that will be changed (if any) when the task is executed. To do so, click Select and Link Services or Configuration Items to Task in the Linked Services and Configuration Items section of such tasks.

5.5.4

For each new implementation task, determine the most appropriate staff of your group (in terms of skills, availability, and access rights) to execute it. Select this person in the Owner field of the task.

If an implementation task must be completed by another group, work with the coordinator of the group to determine who is the most appropriate staff (in terms of skills, availability, and access rights) to do the work and agree on a realistic completion date. Select the person in the Owner field of the task. However, if the group coordinator does not yet know who should complete the task, select the queue in the Owner field. In that case, the group coordinator can select the most appropriate staff in the Owner field after the status of the task is set to Assigned.

If an implementation task must be completed by an external organization, work with your contact person at this organization to determine a realistic implementation date. Select yourself in the Owner field of the task and add a Notes action to specify the external organization that will be asked to do the work. Later, when the work can start, inform the supplier and enter the reference number under which the supplier has registered this assignment, in the same Notes action record.

5.5.5

Ensure that the date and time specified in the Due Date field of the new tasks is achievable. If necessary, also update the Due Date field of tasks that were planned earlier and that must be completed after the new tasks have been executed.

5.5.6

Ensure that the Status field of the new tasks is set to Registered. If it is not necessary to adjust the due date of the task for transfer to production, set the Status field of the tasks to Assigned. Change implementation continues with these tasks.
