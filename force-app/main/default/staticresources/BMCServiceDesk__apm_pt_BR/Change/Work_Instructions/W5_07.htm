Work Instructions for Change Coordinators

5.7.1

Add a Notes action in the change request to specify that the new release did not pass customer testing, additional work must be done to ensure that the release passes testing, and the change must be approved again because the timing of the transfer to production must be adjusted. Set the Scheduled Start and Scheduled End fields to the new implementation start and end dates.

5.7.2

Set the Status field of the change request to To Be Approved. The change request appears in the list of change requests awaiting approval of change manager.
