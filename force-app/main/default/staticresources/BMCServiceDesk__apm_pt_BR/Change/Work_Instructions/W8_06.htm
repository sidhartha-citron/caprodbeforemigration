Work Instructions for Change Coordinators

8.6.1

Review the details of the requests for which the emergency change was implemented.

8.6.2

Open a new change request by clicking Create and Link Change Request to Incident in the Linked Change Requests section of the request.

8.6.3

Select yourself in the Initiator ID field of the change.

8.6.4

Select the emergency change template in the Template field of the change request.

8.6.5

Enter a short description of the emergency change in the Description field and enter Correction in the Reason field.

8.6.6

Select Emergency in the Change Type field and an appropriate category in the Change Category field.

8.6.7

Select In Progress in the Status field of the new change and save the change request.

8.6.8

Add a Notes action to summarize what has changed.

8.6.9

In the Linked Services and Configuration Items section, select the service that was affected by the incident for which the emergency change was implemented.

8.6.10

If the CMDB needs to be updated, assign the task to update the CMDB to the configuration administrator of your group and set its Status field to Assigned. If CMDB does not require an update, set the Status field of this task to Cancelled, and in its Resolution field specify why the CMDB update is not required.

8.6.11

Ensure that you are selected in the Owner field of the task for informing the service owner of the implemented emergency change.
