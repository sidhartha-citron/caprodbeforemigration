Work Instructions for Change Coordinators

5.4.1

If the new release passed customer testing, go to 5.8.1. If the new release failed the testing, continue with 5.5.1.

Note:

If one or more issues were found during the test, but the customer representatives still decides to approve the transfer of the new release to production, ensure that a problem is registered for each issue. Registering a problem in the service management application for each issue makes the issue information available to the people who are supporting the new release. Creating a problem also ensures that a staff is assigned to resolve the problem.

Note:

If the solution to the issues must be available as a new knowledge article, click the Knowledge Articles tab, and click New.

Select the record type as Problem Solution. Select the appropriate Category, and enter the title. Enter the details of the knowledge article in the available fields. Select Submit for Approval to gain an approval for the new knowledge article.
