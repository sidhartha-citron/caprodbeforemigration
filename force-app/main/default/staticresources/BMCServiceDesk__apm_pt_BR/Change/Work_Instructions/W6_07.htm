Work Instructions for Change Coordinators

6.7.1

For the production test task whose Status field was set to Failed by the specialist, review the information in the Resolution field.

6.7.2

Insert the necessary additional implementation tasks to ensure that all change requirements are met.

Note:

When multiple tasks must be assigned to the same staff, if these tasks are to be completed in a sequence without interruption, you can create one task to cover all these tasks.

6.7.3

If the additional work will cause one or more customer organizations to receive different functionality or to experience a period of service degradation or service outage (regardless of whether it occurs during service hours), add information about these organizations to the implementation tasks whose execution affects them in the Notes action of the tasks.

6.7.4

Link each new implementation task to the CIs that will change when the task is executed. To do so, click Select and Link Services or Configuration Items to Task in the Linked Services and Configuration Items section of the tasks.

6.7.5

For each implementation task, determine the appropriate member of your group (in terms of skills, availability, and access rights) to execute it. Select this person in the Owner field of the task.

If an implementation task must be completed by another group, work with the coordinator of the group to determine the appropriate staff (in terms of skills, availability, and access rights) to do the work and agree on a realistic completion date. Select the person in the Owner field of the task. However, if the group coordinator does not yet known who should complete the task, select the group in the Owner field. In that case, the group coordinator can select the most appropriate staff in the Owner field after the status of the task is set to Assigned.

If an implementation task must be completed by an external organization, work with your contact person at this organization to determine a realistic implementation date. Select yourself in the Owner field of the task and add a Notes action to specify the external organization that will do the work. Later, when the work can start, inform the supplier and enter the reference number, under which the supplier has registered this assignment, in the same Notes action.

6.7.6

Ensure that the date and time specified in the Due Date field of the new tasks is achievable. Also, update the Due Date field of tasks that were planned earlier and must be completed after the new tasks have been executed, as needed.

6.7.7

Set the Status field of the new tasks to Registered.

6.7.8

Add a Notes action indicating that the change request needs to be approved again because additional work is required to ensure that all change requirements are met.

6.7.9

Set the Status field of the change request to To Be Approved. The change request appears in the list of changes that are awaiting approval of change manager.
