Work Instructions for Release Administrators

5.9.1

When the Status field of your task to transfer the new release changes to Assigned, open the task and read the instructions in the Description field and Notes actions.

5.9.2

If you cannot work on the task, set the Status of the task to Accepted.

Note:

If the task should be assigned to another release administrator, ask the change coordinator to change its assignment. To do so, set the Status field of the task to Rejected, and add a Notes action saying why it has been rejected.

5.9.3

When you are ready to transfer the new release to the production environment, set the Status field of the task to In Progress, and transfer.

Note:

If you cannot continue working on the task (for example, you have not been provided with the required access rights), set its Status field to Waiting for…, and add a Notes action to specify what you are waiting for and when you expect to complete the task.

5.9.4

When you have completed the transfer of the new release to the production environment, set the Status field of the task to Completed.

If you could not complete the transfer successfully, set the Status field of the task to Failed, and in the Resolution field specify why it failed.

5.9.5

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent completing the task.
