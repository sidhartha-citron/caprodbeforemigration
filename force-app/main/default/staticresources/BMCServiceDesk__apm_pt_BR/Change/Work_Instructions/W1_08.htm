Work Instructions for Change Coordinators

1.8.1

Return the change request to the process from which it was passed to Change Management.

1.8.2

If the change request was received from Request Management as a request, add a Notes action to specify that the change can be implemented within the Request Management process.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the request.

Select your group coordinator in the Owner field of the support request. Set the Status field of the support request to Assigned.

Note:

Do not set the Status field of the support request to Rejected because the service management application shows the request in the dashboard of the servicedesk.

1.8.3

If the change request was received from Problem Management as a problem, add a Notes action to specify that the change can be implemented within the Problem Management process.

In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the problem.

Set the Status field of the problem to Rejected that causes the problem to be returned to its problem coordinator.
