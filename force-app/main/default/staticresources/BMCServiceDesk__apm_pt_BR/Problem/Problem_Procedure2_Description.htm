Procedure 2, Root Cause Analysis


The specialist reviews the details of the problem. If the problem was identified after one or more incidents were created by it, the specialist attempts to provide a workaround. A knowledge article is created that contains the information about the workaround and how to implement the workaround. A reference to the article is added to the problem.

This workaround can be used to resolve future incidents caused by the problem until a structural solution has been found and implemented.

Once the root cause analysis is done by the specialist, the problem is updated with its description.

Next, the specialist considers possible structural solutions and evaluates them. A description of each option is added to the problem along with the recommendation for the preferred structural solution.

If Change Management is not required to remove the root cause, the specialist implements the preferred structural solution.

If the specialist is not able to find the root cause or is not able to propose a structural solution, the specialist mentions the reason in the problem.

Regardless of whether a structural solution was proposed or even already implemented, the specialist updates the status of the problem to inform the problem coordinator that the work has been completed.
