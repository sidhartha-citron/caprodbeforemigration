Work Instructions for Problem Coordinators

4.1.1

Read the recent information in the Notes actions of the problem.

4.1.2

If implemented, verify the structural solution (for example, review the production test results).

Note:

If you do not have the access rights to verify the structural solution, work with a user that does.

Note:

If it was not possible to reproduce the symptoms of incidents caused by the problem before the solution was implemented, the only way to verify the solution is to monitor the service requests to ensure no new incidents are caused.
