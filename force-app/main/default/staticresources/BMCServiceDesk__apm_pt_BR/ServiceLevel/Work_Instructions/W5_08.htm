Work Instructions for Service Level Managers

5.8.1

From the Remedyforce Console tab, click View > Service Requests. To open a new service request, click New.

5.8.2

In the Client ID field, select the customer representative. To do fast, select the Client ID lookup and enter the last or first name in the Search field. If the characters entered in the search field are not unique, a list of persons whose last name or first name start with the entered characters is displayed. Select the customer representative from the list.

5.8.3

In the Request Type field, select the request type. The options for requests submitted by a customer representative to a service level manager are:

  • Request for Change

  • Service Request

  • Complaint

  • Compliment

This field is not provided out-of-the-box. You must customize the Incident form and add it to the required section.

5.8.4

In the Description field, enter a short meaningful description of the request.

5.8.5

In the Incident Source field, select Personal Visit.

This field is not provided out-of-the-box. You must customize the Incident form and add it to the required section.

5.8.6

In the Category field, select the appropriate request category.

5.8.7

In the Impact and Urgency fields, select the appropriate impact and urgency.

5.8.8

Set the Status field of the incident to Assigned.

5.8.9

In the Services and CIs Details, select the service that the request relates to.

Note:

If it concerns a complaint, a compliment or a request for the development of a new service, select the special service N/A - Service does not (yet) exist or is out of scope.

This field is not provided out-of-the-box. You must customize the Incident form and add it to the required section.

5.8.10

In the Services and CIs Details section , select the specific configuration item (CI) that the request relates to.

Note:

If the CI is not registered in the CMDB, but falls within the scope of the Configuration Management process, select the special CI with the code NORECORD.

This field is not provided out-of-the-box. You must customize the Incident form and add it to the required section.

Note:

If the incident does not concern a specific CI (for example, in case of a complaint or compliment), or if the CI falls outside the scope of the Configuration Management process, select the special CI with the code N/A.

This field is not provided out-of-the-box. You must customize the Incident form and add it to the required section.

5.8.11

Save the incident.

5.8.12

In the Owner field of the request, select specific queue. Do not select a specific staff member as it is up to the group coordinator of the group to decide who is the most appropriate Staff to work on the request.

5.8.13

Add a Notes action to describe the customer's business requirements in terms that are within the service provider organization to avoid misunderstandings.

Note:

If the customer representative requested a new service, or a change in service level, add the SLRs in this field to ensure that the customer's business requirements is not misinterpreted by the staff of the service provider organization.

5.8.14

In the Notes action, Duration (HH:MM) field, enter the time that you spent registration of the request.

5.8.15

Go back to 5.8.1 until a separate request has been registered for each request submitted by the customer representative.
