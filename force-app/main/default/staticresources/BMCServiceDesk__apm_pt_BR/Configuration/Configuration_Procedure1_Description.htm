Procedure 1, CI Requisition


Whenever new configuration items (CIs) need to be ordered, or when the configuration management database (CMDB) needs to be updated, the responsible configuration administrator receives a task from Change Management or Request Management.

If a task does not request a new CI, the configuration administrator goes straight to Procedure 2, CI Receipt. On the other hand, if a task orders one or more new CIs, the configuration administrator first submits a purchase requisition for the required CIs. Having submitted a purchase requisition to purchasing, the configuration administrator periodically checks up on its progress.

If the CIs are ordered by purchasing, the configuration administrator registers the ordered CIs in the CMDB. On the other hand, if the purchase requisition has been rejected, the configuration administrator continues to Procedure 4, CI Update.
