Work Instructions for Configuration Administrators

2.8.1

Register the details of the new configuration item (CI). If a purchase requisition is submitted for the new CI, the CI will already be registered in the CMDB with the Status field set to Ordered. In this case, update the status of the CIs and add any missing information (for example, the serial number).

If a formal purchase requisition is not submitted (for example, internally developed software), register the new CI in the CMDB.

Link each new CI to other CIs, clients and accounts as needed. Do this by following the field utilization guidelines for the Configuration Item form.

2.8.2

Print the unique CI code of each new hardware CI on a label. Attach each label to its corresponding hardware CI in a way that the label is clearly visible after the CI is installed.

2.8.3

Print the unique CI code of each new software CI on a label. Attach each label to the master copy of its corresponding software CI in such a way that the label is clearly visible after the master copy is placed in the DML. Place the master copy of each new software CI in the DML.

2.8.4

Print the unique CI code of each new software license certificate on a label. Attach each label to its corresponding certificate in such a way that the certificate is easy to find after it has been placed in the DML. Place each new software license certificate in the DML.
