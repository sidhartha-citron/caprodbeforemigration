Work Instructions for Release Coordinators

1.3.1

Return the change request to the Change Management process.

1.3.2

If the change request is received in the form of a support request, add a Notes action saying that the change can be implemented without the involvement of the Release Management process. In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the request.

Use the Change option next to the Owner field of the request to select the change coordinator who assigned the change request to you. Set the Status field of the support request to Assigned.

Note:

Do not set the Status field of the request to Rejected.

1.3.3

If the change request is received in the form of a problem, add a Notes action saying that the change can be implemented without the involvement of the Release Management process. In the Duration (HH:MM) field, in the Notes action, enter the time that you spent reviewing the problem.

Use the Change option next to the Owner field of the request to select the change coordinator who assigned the change request to you. Set the Status field of the problem to Change Requested.

Note:

Do not set the Status field of the problem to Rejected.
