Process description

The Release Management process consists of the following procedures:

  • Change Request Handling is used by the release coordinators to review the requests and problems that have been passed to Release Management.

  • Release Definition is used by release coordinators to organize CAB meetings and by CAB staff members when they decide which of the change requests that have been collected for CAB review should be fulfilled by the next release. Release coordinators also use this procedure to split the requirements of releases into logical groups that can be handled efficiently by change coordinators.

  • Business Justification is used by the release coordinators when additional funding needs to be obtained for the implementation of a release.

  • Release Coordination is used by release coordinators to initiate the implementation of releases and to decide on corrective actions as needed.
