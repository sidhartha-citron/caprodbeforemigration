Procedure 6: Request Fulfilment Handling


When the request is created by a template that contains the workflow for implementing the request, the service desk analyst or specialist reviews the tasks that are created by the template and determines if additional tasks are required. If required, the additional tasks are created and linked to the request.

Next, the service desk analyst or specialist determines which tasks can be completed without involving others and assigns these tasks to themselves.
For each remaining task, the service desk analyst or specialist determines the appropriate person to complete this task. Tasks that should be completed by other service desk analysts or specialists are assigned to:

  • Appropriate staff when the staff is in the same group as the service desk analyst or specialist handling the request
  • Group coordinator of the target group if the appropriate specialist is in another group
  • Group coordinator of the target group if the appropriate specialist is not known


When assigning the tasks, consider the order of completing these tasks. The order ensures that the assigned tasks do not require other tasks to be completed first.

The service desk analyst or specialist responsible for handling the request completes the tasks assigned to them and monitors the completion of the remaining tasks. When required, they can create additional tasks and assign the tasks. When all tasks are completed successfully, the request is completed.
