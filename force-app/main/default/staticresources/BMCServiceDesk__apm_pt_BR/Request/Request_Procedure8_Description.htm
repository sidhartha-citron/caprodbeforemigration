Procedure 8: Request Closure


If the service desk analyst can resolve the request (in terms of skills, access rights, and time restrictions), the service desk analyst resolves and completes the request.

After the request is completed, the service desk analyst informs the user who submitted the request and asks the user to verify its solution.

If the user informs the service desk that the request has not been resolved to the user's satisfaction, the service desk analyst again works on resolving the request.

Otherwise, the service desk analyst checks if the request is for a service outage that affects multiple users. In this case, the service desk analyst informs the affected users about how the incident is resolved.

The affected users can be:

  • Owner of the affected service
  • Customer representatives of the affected service
  • Service level managers of the affected customer organizations
  • On-duty manager

If the user confirms that the request is satisfactorily resolved or the user does not respond within 28 days, the request is closed.
