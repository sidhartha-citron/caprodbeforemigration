Scope


The scope of the Request Management process is limited to the following support request categories.

Category

Description

Incident

Request from a user to restore a service that is not functioning correctly

Note:

Requests for password resets from users who have forgotten their passwords and requests for restoring backup from users who have lost data

Request for Change

Request from a user for modifying, adding, moving, or removing some or all service functionality, access, or components

Note:

Requests for copying data (for example, for making a backup or restoring a backup at a different location) and requests for batch job runs belong to this category

Service Request

Request from a user that can be resolved by using a predefined workflow in the following scenarios:

  • Template exists for this workflow
  • No risk and impact analysis is required
  • No approval is required other than approval from the customer

Information Request

Request from a user for an answer to a service-related question

Compliment

Praise or encouragement from a user

Complaint

Request from a user for the improvement of how the service provider organization supports the user
