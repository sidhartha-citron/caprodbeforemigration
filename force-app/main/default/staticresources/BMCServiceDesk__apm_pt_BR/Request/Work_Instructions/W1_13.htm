Work Instructions for Service Desk Analysts

1.13.1

Select the applicable request category in the Category field.

1.13.2

If you are creating a service request, select a request
definition that you want to use in the Request Definition field. The request definitions for
the selected category are displayed.

1.13.3

Enter a short but meaningful description of the request in the Description field.

1.13.4

Select how the request is submitted to the service desk in the Incident Source field.

1.13.5

Select the related service in the Service field.

Note:

If the request is a complaint, compliment, or a request for the development of a new service, select the N/A - Service does not (yet) exist or is out of scope service.

1.13.6

If the request is about (or is suspected to be caused by) a specific configuration item (CI), select the CI in the Configuration Item field.

Note:

If the CI is not registered in the CMDB, but in the scope of the Configuration Management process, select the CI with the NORECORD code.

Note:

If the request is not for a specific CI, or if the CI is outside the scope of the Configuration Management process, select the CI with the N/A code.

1.13.7

Save the request.

1.13.8

If the request type is set to service request, the accompanying work orders are generated and a popup appears where you must answer the questions related to the service request.

1.13.9

Add a Notes action to enter any additional information that the user can provide and can prove useful when resolving the request.

1.13.10

If the request type is set to incident, check service management to see which changes, implemented during the past 24 hours, could have caused the incident. If you find a change that appears to have caused the incident, link the change in the Linked Change Requests related list. Add a Notes action to specify why the implementation of this change has caused the incident.
