Problem


The following table lists the fields of the Problem form and provides utilization guidelines for each field.

Field

Utilization

Problem #

This field contains a unique problem number. This number is generated by the application.

Problem Details

Problem Source

Use this field to select the source from the following options:

  • Incident

  • Event Trigger

  • Client

  • Staff

Category

Use this field to select the appropriate category.

Known Error

Use this field to indicate that the problem is a known error.

Description

Use this field to enter a short description of the symptoms that are caused by the problem.

Resolution

Use this field to summarize how the problem has been resolved, that is how the root cause has been removed or fixed by using a workaround.

Workaround

Use this field to describe the workaround that can be applied to resolve incidents caused by this problem until a structural solution has been implemented.

Status and Priority

Impact

Use this field to select the extent to which the service is impacted. The following options are available:

  • Low - Service Degraded for 1 User

  • Medium - Service Down for 1 User or Degraded for Several

  • High - Service Down for Several Users

Note that a service is degraded when some of its functionality is not functioning, or when the response time of the service is slow. A service is down when none of its functionality can be accessed.

Urgency

Use this field to select the urgency from the following options:

  • Low - Responding using standard operating procedures and as time allows

  • Medium - Responding using standard procedures and operating within normal supervisory management structures

  • High - Immediate response required

Priority

The application selects priority after the impact and urgency levels have been selected.

Status

Use this field to select the appropriate status for the problem. The following options are available:

Opened

The initial status of the problem.

Rejected

The problem must be assigned to another queue or Staff.

Assigned

The analysis of the problem can start.

Accepted

The analysis of the problem starts when the assigned staff is ready to start working on it.

In Progress

The problem is being analyzed.

Known Error

The underlying cause of the problem has been found and, in case of a reactively identified problem, a temporary workaround has been proposed.

Waiting for…

It is temporarily not possible to make any further progress with the analysis of the problem.

Analyzed

The underlying cause of the problem has been found and a structural solution has been proposed, or it is not possible to propose a practical structural solution.

Change Requested

The problem has been passed to the Change Management process for the implementation of the proposed structural solution.

Change Pending

A change has been registered for implementing the proposed structural solution.

Change Completed

The change that was registered for implementing the proposed structural solution has been completed.

Dead-End

It is not possible to solve the problem because either its root cause cannot be found or it is currently not possible to propose a practical structural solution.

No Resolution

No resolution is found.

Solved

A structural solution for the problem has been implemented.

Closed

The problem is closed.

Closure Category

Use this field to select the closure category from the following options:

  • Duplicate

  • Resolved/Fixed

  • Unable to resolve

  • Workaround created

Broadcast

Use this field to select the broadcast to which this problem must be related.

Inactive

Select this field if the problem is no longer valid but the record has to be retained for compliance with the records retention policy.

Date and Time Details

Opened Date

This field is set to the date and time at which the problem is created.

Due Date

This field is set to the date and time at which the root cause analysis must be completed after the severity has been set.

Closed Date

This field is set to the date and time at which the problem status is set to Dead-End or Solved.

Staff Assignment Details

Owner

This field displays the queue or staff to whom the problem is assigned.

Related Lists

Action History

This section lists the history actions that have been registered for the problem.

Linked Incidents

This section lists the incidents that are related to the problem. Use this section to link requests that have been related to the problem because either those requests have been caused by the problem (when the problem category is Reactive - Existing Problem) or have been registered to warn about the problem (when the problem category is Proactive - Anticipated Problem).

Linked Change Requests

This section lists the change requests that are related to the problem. Use this section to link change requests to the problem.

Linked Services and CIs

This section lists the services and CI that are related to the problem. Use this section to link services or CIs to the problem.

Linked Tasks

This section lists the tasks that are related to the problem. Use this section to link tasks to the problem.

Notes & Attachments

This section lists the documents that are related to the problem. Use this section to attach any files that provide additional information about the analysis and solution of the problem.

Problem History

This section lists the modifications that have been made to the problem information.

Linked Releases

This section lists the releases that are related to the problem. Use this section to link releases to the problem.
