Configuration Item


The following table lists the fields of the Configuration Item (CI) form and provides utilization guidelines for each field.

Field

Utilization

Attributes: General

Instance name

Use this field to enter the unique name for the CI.

Description

Use this field to enter the description for the CI.

Short Description

Use this field to enter a short description of the CI that is displayed in the CI Explorer.

Notes

Use this field to enter additional information about the CI.

Manufacturer Name

Use this field to select the manufacturer of the CI.

Model

Use this field to enter the model name if it concerns a hardware CI.

Version Number

  • For a Software or Documentation CI, use this field to enter the version number.

  • For a Software License Certificate CI, use this field to enter the versions that the licenses cover.

Serial #

Use this field to enter the serial number of the CI. For a Software License Certificate CI, enter the license certificate number.
For a Database, Software, Software Distribution Package, Software/Interface Configuration Settings, Web Site, or WAN Link CI, leave the field blank.

Asset #

Use this field to enter the number under which the CI has been registered in the financial application (for example, in SAP).

Token Id

Use this field to enter the unique identifier of the CI.

Priority

Use this field to select the priority of the CI indicating its importance in the CMDB.

Primary Client

Use this field to select a client who primarily uses the CI.

Created Date

This field is set to the date and time at which the CI is registered.

Last Modified By

The login name of the staff who last updated the CI is saved in this field.

Last Modified Date

This field is set to the date and time at which the CI is last updated.

Class ID

This field is set to the class of the element that is used to register the CI.

Class Name

This field is set to the class name of the element that is used to register the CI.

Supported

Use this field to specify whether technical support is provided for this CI.

Warranty Exp Date

Use this field to specify the date on which the warranty coverage for the CI expires.

Site

  • For a Hardware CI, use this field to select the facility where the CI is located.

  • For a Software or Software Distribution Package CI, use this field to select the facility where the master copy of the DML is located.

  • For a Software License Certificate CI, use this field to select the facility where the DML in which the certificate is stored is located.

  • For a Database, Software/Interface Configuration Settings, or Web Site CI, set this field to N/A.

CI Status

Use this field to select the status of the CI. You can use this field to record information about the decommissioning, transfer, and disposal of CIs.

Stage

Use this field to select the appropriate stage for the CI from the following list of options:

  • Requirements

  • Defined

  • Analyzed

  • Approved

  • Chartered

  • Designed

  • Developed

  • Build

  • Tested

  • Released

  • Operational

  • Published

  • Unpublished

  • Retired

Mark As Deleted

Use this field to mark the CI as soft deleted.

Instance ID

This field is set to the instance of the element that is used to register the CI.

Submitter

The login name of the staff who registered the CI is saved in this field.

This field is not provided out-of-the-box. You must customize the form to show the field in the Custom section.

Attributes: Specifications

Specifications (several element specific fields)

Each element has different fields in the Specifications tab. Therefore, these fields are not described in this form.

Relationships

Relationship Name

Use this field to enter the unique name for the relationship.

Description

Use this field to enter the description for the relationship.

Mark As Deleted

Use this field to mark the relationship as soft deleted.

Select Class Instance

Use this button to open the Select From Class Instances window and select the CI that you want to relate to the current CI.

Type

Use this field to select the appropriate type of relationship from the following list of options:

  • Other

  • All Related

  • Component

  • Dependency

  • Has Impact

  • Member of Collection

  • Service - Offering

  • Source - Destination

  • Destination - Source

  • Service - Subservice

  • Unknown

  • Account On System

  • Application System Services

  • Base Relationship

  • Contract Component

  • Element Location

  • Genealogy

  • Hosted Access Point

  • Hosted Service

  • Hosted System Components

  • In IP Subnet

  • In Segment

  • IP Subnets In Collection

  • LNs in Collection

  • Offering Measured By

  • Segments In Collection

  • Setting Of

Direction

Use this field to select the direction arrow that is updated between the two CI icons.

When you select the Source-Destination or Destination-Source relationship type, the direction arrow is automatically updated.

HasImpact

Use this field to indicate if one CI in the relationship impacts the other.

Details

History

Use this tab to register the action history notes created for the CI.

Clients

This tab lists the clients that are related to the CI. Use this tab to select the clients who use the CI or for whom the CI is reserved.
If the CI is not used by, or reserved for, a specific client, leave this field blank.

Accounts

This tab lists the accounts that are related to the CI. Use this tab to select the organizations for which the CI is reserved.
If the CI is not used by, or reserved for, a specific organization, leave this field blank.

Incidents

This tab lists the incidents that are related to the CI. Use this tab to link incidents to the CI.

Service Requests

This tab lists the service requests that are related to the CI. Use this tab to link service requests to the CI.

Change Requests

This tab lists the change requests that are related to the CI. Use this tab to link change requests to the CI.

Problems

This tab lists the problems that are related to the CI. Use this tab to link problems to the CI.

Releases

This tab lists the releases that are related to the CI. Use this tab to link releases to the CI.

Tasks

This tab lists the tasks that are related to the CI. Use this tab to link tasks to the CI.

Knowledge articles

This tab lists the knowledge articles that are related to the CI. Use this tab to link knowledge articles to the CI.

Documentation

This tab lists the documents that are related to the CI. Use this tab to attach any files that provide additional information about the CI.

Auditing

This tab lists the modifications that have been made to the CI information.
