Task


The following table lists the fields of the Task form and provides utilization guidelines for each field.

Field

Utilization

Task #

This field contains the unique task number. This number is generated by the application.

Client Details

Client ID

Use this field to enter the client ID of the client who submitted the task.

Client Type

Use this field to select the correct client type.

This field is not provided out-of-the-box. You must customize the Task form and add it to the required section.

Task Details

Template

Use this field to select the template used for the task.

Category

Use this field to select the task category.

Description

Use this field to enter a short description of the task.

Resolution

  • After the task has been set to the Failed status, enter information that might be useful to the change coordinator. For example, specify why the risk and impact task could not be answered.

  • After the task has been set to the Cancelled status, enter the reason for the cancellation in this field. For example, the task requires cancellation because the task was generated by the change template used to register the change request but is not required for this specific implementation.

  • After the task has been set to the Completed status and when the result is slightly different, or if the result was achieved in a different fashion than requested, enter the information in this field. If the task was executed as requested, and the objectives have been met, leave the field blank (unless otherwise specified in the applicable work instruction).

Incident

Use this field to select the incident to which the task is linked.

Change Request

Use this field to select the change request to which the task is linked.

Problem

Use this field to select the problem to which the task is linked.

Release

Use this field to select the release to which the task is linked.

Task Type

Use this field to select the task type from the following options:

  • General

  • Testing

  • Operational rule

This field is not provided out-of-the-box. You must customize the Task form and add it to the required section.

Status and Priority Details

Impact

Use this field to select the extent to which the service is impacted. The following options are available:

  • Low - Service Degraded for One User

  • Medium - Service Down for One User or Degraded for Several

  • High - Service Down for Several Users

Urgency

Use this field to select the urgency from the following options:

  • Low - Responding using standard operating procedures and as time allows

  • Medium - Responding using standard procedures and operating within normal supervisory management structures

  • High - Immediate response required

Priority

The application selects priority after the impact and urgency levels have been selected.

Status

Use this field to select the appropriate status for the task from the following options:

Opened

The initial status of the task.

Registered

The task should not be executed yet.

Rejected

The task must be assigned to another queue or staff.

Assigned

The execution of the task can start, and it should be executed at the date and time specified in the Scheduled Start field.

Accepted

The task will be executed when the assigned staff is ready to start working on it.

In Progress

The task is currently being worked on.

Waiting for…

The task is currently waiting for more information or for another task to be completed.

Failed

An answer could not be found for the risk and impact task, the staff to whom the approval task was assigned cannot approve the change, or the objective of the implementation task cannot be achieved.

Completed

The answer has been provided for the risk and impact task, the change has been approved by the staff to whom the approval task was assigned, or the execution of the implementation task has achieved its objective.

Cancelled

The task cannot be completed. For example, the task cannot be completed because the change has been rejected, or the task was generated by a change template but does not apply to this specific change implementation.

Closed

The task is closed.

Closed By

The login name of the person who closed the task is saved in this field.

Inactive

Select this field if the task is no longer valid, but the record has to be retained for compliance with the records retention policy.

Date and Time Details

Opened Date

This field is set to the date and time at which the task is created.

Due Date

Use this field to select the date and time at which the execution of the task must be completed.

Scheduled Start

Use this field to select the date and time at which the task must start.

Scheduled End

Use this field to select the date and time at which the task must be finished.

Closed Date

This field is set to the date and time at which the task is closed.

Assignment Details

Owner

This field displays the queue or staff to whom the task is assigned.

Related Lists

Action History

This section lists the actions that have been registered for the task.

Linked Services and CIs

This section lists the services and CIs that are related to the task. Use this section to link all the CIs and services that are affected by the execution of the task.

Notes & Attachments

This section lists the documents that are related to the task. Use this section to attach any files that provide additional information about the completion of the task.

Task History

This section lists the modifications that have been made to the task information.
