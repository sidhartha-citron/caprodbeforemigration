Work Instructions for Service Desk Analysts

2.4.1

Enter the number of minutes you spent on the registration or update of the support request in the Time spent (min.) field.

2.4.2

If this concerns a new support request assign the request to the approrpriate queue.

2.4.3

If the support request was rejected (i.e. its Status field is set to "Rejected"), or if the user is dissatisfied with its solution (i.e. the Status field is set to "Completed"), change its status to "Assigned" and manually select the most appropriate assignment queue using the "Assign To" button on the toolbar.

Note:

Do not select a specific staff to assign the support request to. It is up to the group coordinator of the group to decide who the most appropriate staff is (in terms of skills, availability and access rights) to work on the support request.

2.4.4

If you still have the user on the telephone, provide him/her the support request number. This number is displayed on the title bar. By communicating the support request number, the user is assured that his/her request has been registered.

Note:

The user is also automatically informed by email of the support request number upon saving the support request.

2.4.5

If it concerns a support request that must be dealt with as soon as possible (e.g. in case of a virus attack), telephone or page the coordinator of the group to which you have assigned the support request to ensure that he/she is aware of the urgency. Do the same when it is currently after the group's normal office hours and it is important that a specialist already starts to work on the support request.
