Procedimiento 3, Justificación comercial


Si no se requiere un caso de negocio aprobado para el despliegue, el coordinador de despliegue cancelará la tarea de justificación comercial que forma parte del cambio al que están vinculadas las peticiones de cambio del despliegue. Por otro lado, si el próximo despliegue no puede implementarse usando los presupuestos que ya están disponibles para el propietario del servicio, el coordinador de despliegue prepara un caso de negocio para justificar el despliegue. Tras preparar el caso de negocio, el coordinador de despliegue pide aprobación para el caso de negocio para asegurarse de que haya un presupuesto disponible para la implementación del nuevo despliegue.

Si se rechaza el caso de negocio, el coordinador de despliegue lo ajusta si aún se podría obtener aprobación de este modo. Si no es posible obtener aprobación para el caso de negocio, el coordinador de despliegue comunicará a los solicitantes del despliegue que el caso de negocio del despliegue fue rechazado. Si CAB definió el despliegue, el coordinador de despliegue devolverá las peticiones de cambio vinculadas al despliegue al cambio que actúa como marcador de posición para peticiones de cambio que se deben revisar durante la próxima reunión CAB. Antes de cerrar el despliegue, el coordinador de despliegue también se comunica con los miembros de personal CAB para informarles que el caso de negocio de su despliegue fue rechazado y les da la opción de adelantar la próxima reunión CAB.
