Procedimiento 3, Revisión de análisis


Una vez un especialista haya finalizado el análisis de la causa origen de un problema, el coordinador de problemas revisa los resultados para ver si se ha propuesto o implementado una solución estructural. Si el especialista ha solucionado el problema porque el proceso de gestión de cambios no era necesario para coordinar la implementación de la solución estructural, el coordinador de problemas pasa directamente al Procedimiento 4, Cierre de problema.

El coordinador de problemas también pasa al Procedimiento 4, Cierre de problema si el especialista llevó a cabo un buen análisis pero no pudo proponer una solución estructural práctica.

Si el especialista propuso una solución estructural práctica pero no la implementó porque creyó necesario recurrir al proceso de gestión de cambios, el coordinador de problemas lo comprueba para ver si dicha gestión es realmente necesaria. De no ser el caso, el coordinador de problemas asigna el problema al especialista más indicado para su resolución. Si es necesaria la gestión de cambios, el coordinador de problemas pasa el problema al coordinador de cambios de servicio afectado.

Sin embargo, si el análisis del especialista no es aceptable, el coordinador de problemas vuelve a asignar el problema al mismo especialista o a otro distinto para que realice un análisis mejor.
