Procedimiento 4, Cierre de problema


Tras implementar un cambio para proporcionar una solución estructural al problema, el coordinador de problemas verifica si el cambio ha resuelto realmente el problema o no.

Si parece que la implementación ha resuelto el problema, el coordinador de problemas lo cierra. Si el problema se identificó de manera recurrente, la aplicación de gestión de servicios, a continuación, informa al servicio de asistencia mediante un correo electrónico de que se ha eliminado del servicio.

Por otro lado, si no se implementa el cambio o si la implementación no ha arreglado el problema, el coordinador de problemas decide si es posible la propuesta de una solución estructural diferente para el problema. De ser el caso, el coordinador de problemas reasigna el problema a un especialista dentro del grupo.

Si está claro que en el momento no hay medios prácticos disponibles para evitar o eliminar la causa origen de manera permanente, el coordinador de problemas actualiza el problema para indicar un punto muerto. El coordinador de problemas comprobará periódicamente nuevas posibilidades y pedirá otro análisis si es probable que un enfoque o una tecnología diferente pueda proporcionar una solución estructural práctica.

Si la solución final o provisional del problema debe estar disponible en la base de conocimientos, se crea un artículo de conocimiento nuevo.
