Procedimiento 2, Análisis de causa origen


El especialista revisa los detalles del problema. Si el problema se ha identificado tras provocar una o más incidencias, este intenta proporcionar una solución provisional. Se crea un artículo de conocimiento nuevo que contiene la información sobre la solución provisional y el modo de implementarla. Se añade una referencia a dicho artículo al problema.

Esta solución provisional se puede utilizar para resolver futuras incidencias provocadas por un problema hasta que se encuentre e implemente una solución estructural.

Una vez que los especialistas realizan el análisis de la causa origen, el problema se actualiza con su descripción.

Después, el especialista considera las posibles soluciones estructurales y las evalúa. Se agrega una descripción de cada opción al problema junto con la recomendación de la solución estructural preferida.

Si no se necesita el proceso de gestión de cambios para eliminar la causa origen, el especialista implementa la solución estructural preferida.

Si el especialista no encuentra la causa origen o no puede proponer una solución estructural, especifica el motivo en el problema.

Sin tener en cuenta si se ha propuesto una solución estructural o si se ha implementado, el especialista actualiza el estatus del problema para informar al coordinador de problemas de que su trabajo ya ha concluido.
