Instrucciones de trabajo para coordinadores de problemas

3.6.1

Determine si la solución estructural propuesta se puede implementar sin tener que involucrar a gestión de cambios. Incluso aunque el especialista que haya propuesto la solución haya establecido que se necesita el proceso de gestión de cambios, es bueno verificarlo a fin de evitar retrasos innecesarios si el problema se puede solucionar en realidad sin tener que recurrir a dicho proceso.

Nota:

Gestión de cambios debe coordinar un cambio cuya implementación pueda provocar:

  • que el servicio no esté disponible o se degrade durante las horas de servicio;

  • que la funcionalidad de un servicio cambie;

  • que sea necesario actualizar la CMDB.

3.6.2

Vaya al paso 3.9.1 si se necesita la gestión de cambios. De lo contrario, continúe con el 3.8.1.
