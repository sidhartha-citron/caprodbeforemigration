Procedimiento 1, Revisión de petición


El coordinador de problemas utiliza la información de la petición para identificar problemas en los servicios asignados. El coordinador de problemas puede identificar problemas utilizando las funcionalidades de búsqueda y generación de informes de la aplicación de gestión de servicios. Un especialista también puede informar a un coordinador de problemas sobre determinadas peticiones que, en opinión del primero, representen un problema.

Tras la identificación de un problema, el coordinador de problemas lo registra en la aplicación de gestión de servicios. A continuación, el coordinador de problemas selecciona el nivel de impacto y urgencia adecuado y vincula las peticiones provocadas por el problema.

El coordinador de problemas asigna el problema al especialista más indicado (según destrezas, disponibilidad y derechos de acceso) para que realice el análisis.

Una vez guardado el problema, se puede informar automáticamente al servicio de asistencia por correo electrónico de que se ha identificado un problema nuevo.

Si el coordinador de problemas encuentra peticiones que no se han vinculado aún a un problema pero que están provocadas por un problema ya identificado, vincula dichas peticiones al problema.
