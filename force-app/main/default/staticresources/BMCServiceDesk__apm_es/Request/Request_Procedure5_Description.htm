Procedimiento 5: Resolución de petición por un especialista


Una vez que se haya asignado una petición al especialista de un grupo, este último revisa la información de la petición y determina cómo resolver la petición.

Si resolución de peticiones puede procesar una petición, el especialista coordina la implementación de la petición.

El especialista escala la petición al propietario del servicio cuando la petición corresponde a una incidencia que no se puede resolver sin gestión de cambios porque la resolución provoca que:

  • el servicio no esté disponible o se degrade durante las horas de servicio
  • la funcionalidad de un servicio cambie
  • sea necesario actualizar la CMDB

Un coordinador de grupo no puede determinar si es necesario aplicar el proceso de gestión de campos para resolver incidencias porque un especialista debe investigar previamente tales incidencias.

Si no se necesita gestión de cambios y la petición es una incidencia que ocurrió porque falló un servicio complementario, el especialista registra una petición relacionada y espera a que la petición se complete antes de continuar resolviendo la petición.

El especialista informa al coordinador de grupo correspondiente en caso de que determine que la petición debe resolverla otro especialista. De lo contrario, resuelve la petición y la actualiza.
