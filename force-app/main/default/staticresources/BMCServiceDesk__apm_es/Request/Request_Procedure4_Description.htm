Procedimiento 4: Seguimiento de peticiones


El servicio de asistencia sigue las peticiones abiertas comprobando su estatus periódicamente para asegurarse de que las peticiones abiertas continúen avanzando hacia su finalización. Cuando una petición no avanza, el analista de servicio de asistencia se comunica con el coordinador del grupo asignado para pedirle que se asegure de que la petición se procese.

Si una petición describe una indisponibilidad de servicio que afecta a varios usuarios, el analista del servicio de asistencia informa a las siguientes personas implicadas:

  • Usuarios afectados
  • Propietario del servicio afectado
  • Representantes del cliente del servicio afectado
  • Gestores de nivel de servicio de las organizaciones de clientes afectados
  • Gestor de guardia

Si ha transcurrido la mitad del tiempo de resolución disponible y no se ha resuelto la indisponibilidad de servicio, el analista de servicio de asistencia proporciona una actualización del progreso a las personas implicadas.

Si la incidencia no se resuelve cuando se alcanzan la fecha y hora de vencimiento, el analista del servicio de asistencia proporciona otra actualización de progreso a las personas implicadas. Después de la segunda actualización, el analista de servicio de asistencia escala la incidencia al propietario de servicio.
