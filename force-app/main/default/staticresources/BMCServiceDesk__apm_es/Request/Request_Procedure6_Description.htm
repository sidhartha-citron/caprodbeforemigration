Procedimiento 6: Tratamiento de resolución de peticiones


Cuando una petición se crea mediante la plantilla que contiene el flujo de trabajo para la implementación de la petición, el analista de servicio de asistencia o el especialista revisa las tareas creadas por la plantilla y determina si se necesitan tareas adicionales. De ser necesarias, las tareas adicionales se crean y se vinculan a la petición.

A continuación, el analista de servicio de asistencia o el especialista determina qué tareas pueden completarse sin involucrar a otros y se las asignan a ellos mismos.
El analista de servicio de asistencia o el especialista determina quién es la persona adecuada para completar cada una de las tareas restantes. Las tareas que deberían completar otros analistas de servicio de asistencia u otros especialistas se asignan a:

  • personal adecuado que se encuentra en el mismo grupo que el analista de servicio de asistencia o el especialista que procesa la petición
  • el coordinador de grupo del grupo en cuestión si el especialista adecuado está en otro grupo
  • el coordinador de grupo del grupo en cuestión si se desconoce quién es el especialista adecuado


Al asignar las tareas, considere el orden en el que deben completarse. El orden garantiza que las tareas asignadas no requieran que primero se completen otras tareas.

El analista de servicio de asistencia o el especialista responsable de procesar la petición completa las tareas que se le asignaron y monitoriza la finalización de las tareas restantes. Cuando sea necesario, pueden crear tareas adicionales y asignarlas. Se finaliza la petición cuando todas las tareas se completan correctamente.
