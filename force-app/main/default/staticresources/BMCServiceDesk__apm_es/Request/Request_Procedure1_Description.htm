Procedimiento 1: Registro de petición


Cuando un usuario se comunica con el servicio de asistencia, el analista de servicio de asistencia vincula el registro del usuario a una petición nueva. A continuación, el analista de servicio de asistencia determinará la naturaleza de la petición del usuario preguntándole cómo puede ayudarle si la comunicación es por teléfono o en persona. En cambio, si el usuario se ha comunicado con el servicio de asistencia por escrito (por ejemplo, por correo electrónico fax, formulario en papel, etc.), el analista determinará la naturaleza de la petición leyendo la información proporcionada.

Si el usuario se comunicó con el servicio de asistencia sobre una petición enviada anteriormente, el analista de servicio de asistencia busca la petición. Si la petición ya se ha completado o si se ha establecido como "ClosedMail" (Correo cerrado), pero el usuario no está satisfecho con la solución provista, el analista de servicio de asistencia establece el estatus como "Asignado". La petición aparecerá en la lista de tareas del personal y el grupo que proporcionaron la solución original. Si el usuario desea una actualización sobre el estatus de la petición, el analista de servicio de asistencia informa al usuario acerca del progreso.

Si el usuario se comunicó con el servicio de asistencia para realizar una petición nueva, el analista de servicio de asistencia determina si hay una plantilla disponible para la petición.

Si la hay, el analista de servicio de asistencia usa la plantilla para crear la petición. El analista de servicio de asistencia lee las instrucciones de la plantilla y las sigue. Las instrucciones especifican la información que se debe obtener del usuario y cómo proceder después de recibirla (por ejemplo, cómo resolver la petición o a qué grupo asignarla).

Si no hay una plantilla disponible para la petición, el analista de servicio de asistencia crea la petición. El analista del servicio de asistencia vincula el servicio aplicable y el elemento de configuración a la petición.

Si el usuario presenta una incidencia, el analista del servicio de asistencia determina el nivel de impacto y urgencia según lo siguiente:

  • Información proporcionada por el usuario

  • Información sobre la configuración de la instancia de servicio

  • Otras incidencias actualmente abiertas para el servicio afectado

El nivel de impacto y urgencia determina el nivel de prioridad que se asigna a las peticiones de resolución de incidencia.

Si el usuario indica que un servicio crítico o principal ha fallado para varios usuarios, el analista de servicio de asistencia escala la incidencia al propietario de servicio. El propietario del servicio activa el equipo de incidencia grave para resolver la incidencia grave.
