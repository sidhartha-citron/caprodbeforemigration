Procedimiento 8: Cierre de petición


Si el analista del servicio de asistencia puede resolver la petición (en cuanto a habilidades, derechos de acceso y restricciones de tiempo), la resolverá y la completará.

Una vez que se haya completado la petición, el analista de servicio de asistencia informará al usuario que envió la petición y le pedirá que verifique la solución.

Si el usuario informa al servicio de asistencia que no está satisfecho con la solución, el analista de servicio de asistencia vuelve a trabajar para resolver la petición.

De lo contrario, el analista del servicio de asistencia comprueba si la petición es por una indisponibilidad de servicio que afecta a varios usuarios. En este caso, el analista de servicio de asistencia informa a los usuarios afectados sobre cómo se resuelve la incidencia.

Los usuarios afectados pueden ser los siguientes:

  • Propietario del servicio afectado
  • Representantes del cliente del servicio afectado
  • Gestores de nivel de servicio de las organizaciones de clientes afectados
  • Gestor de guardia

Si el usuario confirma que la petición se ha resuelto satisfactoriamente o si no responde dentro de los siguientes 28 días, se cierra la petición.
