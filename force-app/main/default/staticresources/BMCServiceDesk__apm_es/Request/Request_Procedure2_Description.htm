Procedimiento 2: Asignación de petición


Si la petición es una queja o un cumplido, el analista de servicio de asistencia asigna la petición al gestor del servicio de asistencia.

Si resolución de peticiones puede procesar una petición, el analista de servicio de asistencia coordina la implementación de la petición.

El analista de servicio de asistencia considera todas las categorías de petición de resoluciones. Si el analista de servicio de asistencia no puede resolver la petición (en cuanto a habilidades, derechos de acceso y restricciones de tiempo), la asignará al grupo adecuado.

El coordinador del grupo revisa todas las peticiones asignadas a un grupo por analistas del servicio de asistencias, operadores, gestores de continuidad, y usuarios y representantes del cliente que envían un formulario de petición web.

Si el coordinador de grupo determina que una petición se ha asignado al grupo incorrecto o que la petición no contiene toda la información que el analista de servicio de asistencia, el operador o el gestor de continuidad deberían proporcionar para completarla, el coordinador de grupo rechaza la petición y especifica por qué.

Las peticiones rechazadas se envían automáticamente al servicio de asistencia. Un analista de servicio de asistencia se asegura de que se recopile la información que falta y de que estas peticiones rechazadas se reasignen al grupo correspondiente si el servicio de asistencia no puede resolverlas.

Una vez que las peticiones se asignan al grupo correcto, las peticiones con la información necesaria se asignan al coordinador de cambios del servicio relacionado si las peticiones son de un cambio cuya implementación se coordina con gestión de cambios.. Todas las demás peticiones se asignan al especialista adecuado del grupo (en cuanto a habilidades, disponibilidad y derechos de acceso).
