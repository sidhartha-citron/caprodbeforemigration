Instrucciones de trabajo para analistas del servicio de asistencia

1.14.1

Si el tipo de petición está establecido como Incidencia, busque las peticiones abiertas para el mismo servicio y use la información del usuario para determinar el alcance del impacto en el servicio. Seleccione el nivel de impacto adecuado en el campo Urgencia. Encontrará las siguientes opciones:

  • Bajo

-

Servicio degradado para un usuario

  • Medio

-

Fuera de servicio para un usuario o degradado para varios usuarios

  • Alto

-

Fuera de servicio para varios usuarios

  • Principal

-

Fuera de servicio o fallo crítico para varios usuarios Este nivel de impacto podría no estar disponible de manera predeterminada.

Seleccione el nivel de urgencia adecuado en el campo Urgencia. Encontrará las siguientes opciones:

  • Bajo

-

Responder utilizando procedimientos de funcionamiento estándar y según el tiempo lo permita.

  • Medio

-

Responder utilizando procedimientos de funcionamiento estándar según estructuras habituales de gestión y supervisión.

  • Alto

-

Respuesta inmediata obligatoria


Nota:

Si el tipo de petición es Incidencia, no seleccione el nivel de impacto Ninguno: sin degradación del servicio.

Este nivel de impacto podría no estar disponible de manera predeterminada.

Nota:

Si el tipo de petición no es Incidencia, establezca el nivel de impacto como Ninguno: sin degradación del servicio.

Este nivel de impacto podría no estar disponible de manera predeterminada.

Nota:

Gestión de servicios determina automáticamente la prioridad adecuada para las peticiones usando la matriz de prioridad estándar o el SLA del cliente. Tras determinar la prioridad adecuada para la petición, gestión de servicios usa la prioridad para calcular la fecha y hora de vencimiento para la petición.
