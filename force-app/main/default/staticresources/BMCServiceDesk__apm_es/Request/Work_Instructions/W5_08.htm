Instrucciones de trabajo para especialistas

5.8.1

En el campo Resolución, especifique qué hacer para completar la petición. Use este campo para proporcionar instrucciones de solución detalladas (por ejemplo, si el usuario debe ajustar un parámetro), para proporcionar la respuesta a una petición de información, o para describir por qué la petición no se puede resolver si no es posible proporcionar una solución o solución provisional.

5.8.2

Seleccione la categoría de cierre correspondiente en el campo Categoría de cierre y establezca el campo Estatus de la petición como Terminado.

5.8.3

Seleccione el CI adecuado en el campo CI si este campo no está rellenado o si lo está con el CI incorrecto.

Nota:

Si el CI no está registrado en la CMDB, pero está dentro del alcance de la gestión de configuración, seleccione el CI con el código NORECORD (No registrado).

El CI con el código NORECORD podría no estar disponible de forma predeterminada.

Nota:

Si la petición no se trata de un CI específico o si el CI está fuera del alcance del proceso de gestión de configuración, seleccione el CI con el código N/A (No disponible).

5.8.4

Si el campo Impacto se establece como Alto o Grave, rellene la fecha y hora en las que el servicio dejó de estar disponible en el campo Inicio de indisponibilidad, y la fecha y hora en que el servicio volvió a estar disponible en el campo Fin de indisponibilidad.

5.8.5

Si resolvió una incidencia y cree que es probable que ocurran otras similares si no se elimina la causa origen, compruebe si el problema que provocó la incidencia ya se ha registrado. Si el problema está registrado, vincule la petición que resolvió con el problema.

Si el problema que causó la incidencia no está registrado, notifique por correo electrónico o por teléfono al coordinador de problemas del servicio afectado, y asegúrese de proporcionarle el número de petición de la incidencia y la razón por la que es probable que se repita.

5.8.6

En el campo Duración (HH:MM), en la acción de Notas, introduzca el tiempo que empleó para resolver la petición.

5.8.7

Realice una revisión final de la petición para garantizar que la información sea correcta y esté completa.

5.8.8

Si es probable que se envíen peticiones similares, cree un artículo de conocimientos para la solución provista.
