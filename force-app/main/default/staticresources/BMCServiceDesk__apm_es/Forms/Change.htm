Petición de cambio


En la siguiente tabla encontrará los campos del formulario Petición de cambio y las pautas de uso de cada campo.

Campo

Uso

N.º de cambio

Este campo contiene un número único de cambio.
La aplicación genera este número automáticamente cuando usted guarda una petición de cambio.

Detalles de iniciador

ID de iniciador

Use este campo para seleccionarse a usted mismo como coordinador del cambio.

Detalles de cambio

Plantilla

Use este campo para seleccionar una plantilla que lo ayude a registrar la petición de cambio y sus tareas.

Tenga en cuenta que las tareas que vincule a la petición de cambio después de aplicar la plantilla no se visualizarán hasta que guarde la petición de cambio.

Categoría

Use este campo para seleccionar la categoría correcta de la petición de cambio.

Descripción

Use este campo para introducir una descripción breve de la petición de cambio.

Reasons (Motivos)

Use este campo para introducir el motivo del cambio.

Plan de despliegue

Use este campo para introducir el motivo del cambio.

Plan de restauración

Use este campo para introducir el plan para restaurar el elemento de configuración a su estado inicial antes de la implementación del cambio.

Aprobado

Use este campo para especificar si desea crear una petición de cambio con aprobación previa para garantizar una implementación rápida.

Comentarios de revisión

Use este campo para introducir todos los comentarios de los que desee realizar un seguimiento para el cambio actual de acuerdo con la revisión. En estas notas puede incluir los motivos por los que se ha producido un error en el cambio y la forma de evitar que la situación se repita en el futuro.

Tipo de cambio

Use este campo para seleccionar el tipo de cambio entre las siguientes opciones:

  • Emergencia

  • Normal

  • Estándar

Categoría de cambio

Use este campo para asignar la categoría del cambio entre las siguientes opciones:

  • Principal

  • Secundario

  • Significativo

Impacto

Use este campo para seleccionar el alcance del impacto sobre el servicio. Encontrará las siguientes opciones:

  • Bajo: servicio degradado para un usuario

  • Medio: fuera de servicio para un usuario o degradado para varios usuarios

  • Alto: fuera de servicio para varios usuarios

Urgencia

Use este campo para seleccionar el grado de urgencia entre las siguientes opciones:

  • Baja: responder utilizando procedimientos estándar y según el tiempo lo permita.

  • Media: responder utilizando procedimientos de estándar según estructuras habituales de gestión y supervisión.

  • Alta: respuesta inmediata obligatoria

Prioridad

La aplicación asigna la prioridad una vez seleccionados los niveles de impacto y de urgencia.

Estatus

Use este campo para seleccionar el estatus adecuado para la petición de cambio. Encontrará las siguientes opciones:

Abierto

El estatus inicial de la petición de cambio.

Registrado

La petición de cambio aún no está lista para someterse a un análisis de riesgos e impacto.

Riesgos e impacto

Se está llevando a cabo un análisis de riesgos e impacto de la petición de cambio y se está preparando un plan de implementación.

To Be Approved (Para aprobar)

El plan de implementación está preparado y debe revisarlo el gestor de cambios, después de lo cual habrá que solicitar aprobación.

Aprobación solicitada

El gestor de cambios ha solicitado la aprobación para el cambio.

Aprobación pendiente

La aprobación está pendiente.

Aprobado

La petición de cambio ha sido aprobada.

En curso

La petición de cambio está en proceso de implementación.

Rechazado

Se ha rechazado la petición de cambio.

Terminado

No se puede tomar ninguna medida más con respecto a este cambio porque ha sido implementado, revertido o rechazado por sus aprobadores, tal y como se especifica en el campo Categoría de cierre.

Cerrado

Se ha cerrado la petición de cambio.

Categoría de cierre

Una vez completada la petición de cambio, use este campo para seleccionar la categoría de cierre adecuada para la petición. Encontrará las siguientes opciones:

  • Retirado por solicitante

  • Rechazado

  • Revertido

  • Implantado parcialmente

  • Implantado

Inactivo

Seleccione este campo si el cambio ya no es válido, pero el registro debe conservarse para respetar la directiva de retención de registros.

Cambiar detalles de fecha y hora de cambio

Fecha de apertura

Este campo se establece en la fecha y hora en que se crea la petición de cambio.

Fecha de respuesta

Este campo se establece en la fecha y hora en que se recibe la respuesta. Este campo registra la fecha y hora en que el personal ha informado al iniciador acerca del cambio.

Fecha de revisión

Use este campo para seleccionar la fecha en que se debe revisar la petición de cambio después de su cierre.

Fecha de cierre

Use este campo para seleccionar la fecha en que se cierra la petición de cambio.

Inicio programado

Use este campo para seleccionar la fecha para la que se ha programado la implementación de la petición de cambio.

Finalización programada

Use este campo para seleccionar la fecha en que se debe completar la petición de cambio.

Fecha de inicio real

Use este campo para seleccionar la fecha de comienzo de la implementación del cambio.

Fecha de finalización real

Use este campo para seleccionar la fecha real de finalización de la implementación.

Detalles de asignación

Propietario

Use este campo para asignar la petición de cambio a una cola o un miembro del personal.

Listas relacionadas

Historial de acciones

Use esta sección para añadir al historial las acciones registradas para la petición de cambio.

Problemas vinculados

Use esta sección para añadir y vincular problemas relacionados. En esta sección encontrará los problemas para los que se ha preparado la petición de cambio.

Servicios y CI vinculados

En esta sección encontrará los elementos de configuración (CI) y los servicios relacionados con la petición de cambio. Use esta sección para vincular los servicios y CI relacionados al cambio.

Evaluaciones de cambios

En esta sección encontrará las evaluaciones de cambio relacionadas con la petición de cambio. Use esta sección para crear y vincular las evaluaciones de cambio que desee registrar para el cambio.

Historial de aprobaciones

Use esta lista para aprobar una petición de cambio o para ver el historial de aprobaciones registrado para la petición de cambio.

Tareas vinculadas

En esta sección encontrará las tareas relacionadas con la petición de cambio. Use esta sección para crear y vincular las tareas que se han registrado para la petición de cambio.

Incidencias vinculadas

En esta sección encontrará las incidencias relacionadas con la petición de cambio. Use esta sección para vincular las incidencias para las que se ha preparado la petición de cambio.

Notas y adjuntos

En esta sección encontrará los documentos relacionados con la petición de cambio. Use esta sección para adjuntar cualquier archivo que contenga información adicional para el análisis de riesgos e impacto, las aprobaciones, la implementación y la reversión de la petición de cambio.

Auditoría de petición de cambio

En esta sección encontrará las modificaciones que haya sufrido la información del cambio.

Despliegues vinculados

En esta sección encontrará los despliegues relacionados con la petición de cambio. Use esta sección para vincular los despliegues que se han registrado para la petición de cambio.
