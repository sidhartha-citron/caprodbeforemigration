Problema


En la siguiente tabla encontrará los campos del formulario Problema y las pautas de uso de cada campo.

Campo

Uso

N.º de problema

Este campo contiene un número de problema exclusivo. La aplicación genera este número automáticamente.

Detalles del problema

Origen de problema

Use este campo para seleccionar el origen en la siguiente lista de opciones:

  • Incidencia

  • Activador de evento

  • Cliente

  • Personal

Categoría

Use este campo para seleccionar la categoría adecuada.

Error conocido

Use este campo para indicar que el problema es un error conocido.

Descripción

Use este campo para introducir una breve descripción de los síntomas causados por el problema.

Resolución

Use este campo para resumir el modo en que se ha resuelto el problema, es decir, el modo en que se ha eliminado o solucionado la causa origen.

Solución provisional

Use este campo para describir la solución provisional que se debe aplicar para resolver incidencias causadas por este problema hasta haber diseñado una solución estructural.

Estatus y prioridad

Impacto

Use este campo para seleccionar el alcance del impacto sobre el servicio. Encontrará las siguientes opciones:

  • Bajo: servicio degradado para 1 usuario

  • Medio: fuera de servicio para 1 usuario o degradado para varios usuarios

  • Alto: fuera de servicio para varios usuarios

Tenga en cuenta que se considera que un servicio está degradado cuando parte de su funcionalidad no está disponible o cuando el tiempo de respuesta del servicio es lento. Un servicio no está disponible cuando no se puede acceder a su funcionalidad.

Urgencia

Use este campo para seleccionar el grado de urgencia entre las siguientes opciones:

  • Baja: responder utilizando procedimientos de funcionamiento estándar y según el tiempo lo permita.

  • Media: responder utilizando procedimientos de funcionamiento estándar según estructuras habituales de gestión y supervisión.

  • Alta: respuesta inmediata obligatoria

Prioridad

La aplicación selecciona la prioridad una vez seleccionados los niveles de impacto y de urgencia.

Estatus

Use este campo para seleccionar el estatus adecuado para el problema. Encontrará las siguientes opciones:

Abierto

El estatus inicial del problema.

Rechazado

El problema se debe asignar a otra cola u otro miembro del personal.

Asignado

El análisis del problema puede comenzar.

Aceptado

El análisis del problema comienza cuando el personal asignado está listo para trabajar en él.

En curso

Se está analizando el problema.

Error conocido

Se ha identificado la causa subyacente del problema y, en caso de ser un problema identificado con recurrencia, se ha propuesto una solución temporal.

Esperando…

Resulta temporalmente imposible avanzar de ningún modo en el análisis del problema.

Analizado

Se ha identificado la causa subyacente del problema y se ha propuesto una solución estructural, o bien no es posible proponer una solución estructural práctica.

Cambio solicitado

El problema se ha remitido al proceso de Gestión de cambios para la implementación de la solución estructural propuesta.

Cambio pendiente

Se ha registrado un cambio para la implementación de la solución estructural propuesta.

Cambio terminado

Se ha completado el cambio registrado para la implementación de la solución estructural propuesta.

Dead-End (Sin más opciones)

No es posible resolver el problema debido a que o bien es imposible identificar la causa origen o bien resulta actualmente imposible proponer una solución estructural práctica.

Sin resolución

No se ha encontrado una resolución.

Resuelto

Se ha implementado una solución estructural para el problema.

Cerrado

El problema está cerrado.

Categoría de cierre

Use este campo para seleccionar la categoría de cierre de la siguiente lista de opciones:

  • Duplicado

  • Resuelto/Corregido

  • No se puede resolver

  • Solución provisional creada

Difusión

Use este campo para seleccionar la difusión con la que debe estar relacionado este problema.

Inactivo

Seleccione este campo si el problema ya no es válido, pero el registro debe retenerse para respetar la directiva de retención de registros.

Detalles de fecha y hora

Fecha de apertura

Este campo se establece en la fecha y hora en que se creó el problema.

Fecha de vencimiento

Este campo se establece en la fecha y la hora en que se debe completar el análisis de la causa origen después de haber establecido el nivel de gravedad.

Fecha de cierre

Este campo se establece en la fecha y la hora en que se estableció el estatus del problema como Dead-End (Sin más opciones) o Resuelto.

Detalles de asignación de personal

Propietario

Este campo muestra la cola o persona a la que se asigna el problema.

Listas relacionadas

Historial de acciones

En esta sección encontrará el historial de acciones registradas para el problema.

Incidencias vinculadas

En esta sección encontrará las incidencias relacionadas con el problema. Use esta sección para vincular las peticiones relacionadas con el problema porque o bien han sido causadas por el problema (cuando la categoría del problema es Reactivo - Problema existente) o bien se han registrado para advertir del problema (cuando la categoría del problema es Proactivo - Problema previsto).

Peticiones de cambio vinculadas

En esta sección encontrará las peticiones de cambio relacionadas con el problema. Úsela para vincular peticiones de cambio al problema.

Servicios y CI vinculados

En esta sección encontrará los servicios y CI relacionados con el problema. Úsela para vincular servicios o CI al problema.

Tareas vinculadas

En esta sección encontrará las tareas relacionadas con el problema. Úsela para vincular tareas al problema.

Notas y adjuntos

En esta sección encontrará los documentos relacionados con el problema. Úsela para adjuntar cualquier archivo que contenga información adicional sobre el análisis y la solución del problema.

Historial de problema

En esta sección encontrará las modificaciones que haya sufrido la información del problema.

Despliegues vinculados

En esta sección encontrará los despliegues relacionados con el problema. Úsela para vincular despliegues al problema.
