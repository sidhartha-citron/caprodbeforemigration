Petición


En la tabla a continuación se enumeran los campos del formulario Petición y se proporcionan directrices de utilización para cada campo.

Campo

Uso

N.º de incidencia

Este campo contiene un número de petición exclusivo. La aplicación genera este número automáticamente.

Información del cliente

ID de cliente

Utilice este campo para seleccionar el cliente que ha enviado la petición.

Tipo de cliente

Utilice este campo para seleccionar el tipo de cliente.

Método de contacto preferido

Use este campo para seleccionar el método de contacto que prefiere el cliente. Encontrará las siguientes opciones:

  • Correo electrónico

  • Teléfono móvil

  • Teléfono del trabajo

Cuenta

Use este campo para introducir la cuenta del cliente.

Nº de llamada

Use este campo para introducir la cantidad de veces que el cliente de una petición ha llamado al servicio de asistencia.

Detalles de incidencia

Tipo de incidencia

Use este campo para seleccionar el tipo de petición en la siguiente lista de opciones:

  • Petición de cambio

  • Incidencia

  • Petición de servicio

  • Queja

  • Cumplido

  • Petición de información

  • Fuera de alcance

Incident Source (Origen de incidencia)

Use este campo para registrar cómo se envía la petición de soporte. Encontrará las siguientes opciones:

  • Mensaje de correo electrónico

  • Otro correo electrónico

  • Evento

  • Fax, carta, formulario de papel o memorándum

  • Escucha de correo

  • Observación propia

  • Visita personal

  • Teléfono

  • Autoservicio

  • Acercamiento

  • Formulario de petición web

Plantilla

Use este campo para seleccionar una plantilla con información que ayude a registrar la petición.

Categoría

Utilice este campo para seleccionar la categoría de la petición.

Descripción

Utilice este campo para introducir una breve descripción de la petición.

Ejemplos para incidencias:

  • Tiempo de respuesta lento en <Servicio>

  • Mensaje de error utilizando <Servicio>

  • No se puede iniciar sesión en (o acceder a) <Servidor o servicio>

  • El trabajo <Nombre de trabajo> ha fallado con <Código de anulación>

  • El router <Código de cliente> en <Ciudad> está fuera de servicio

Ejemplo para peticiones:

Cómo <Información solicitada>

Ejemplos para peticiones de cambio:

  • Instalar <Software y versión> en <Código de estación de trabajo o servidor>

  • Aplicar una actualización de <Software Actual y versión> a <Software solicitado y versión>

Ejemplos para quejas y cumplidos:

  • La petición <Número de petición> no se ha resuelto a tiempo

  • El cambio <Número de cambio> se ha implementado sin aprobación

Definición de petición

Utilice este campo para seleccionar la definición de petición que desea utilizar para crear una petición.

Difusión

Use este campo para seleccionar una difusión existente que ya se ha registrado y que tiene que ver con la misma indisponibilidad. La petición actual heredará toda la información relevante de la difusión.

Resolución

Utilice este campo para especificar lo que se ha hecho para completar la petición. También debe especificar instrucciones detalladas para la solución (por ejemplo, si el cliente necesita configurar un ajuste) para proporcionar la respuesta de resolución de la petición. Si no es posible brindar una solución o una solución provisional, especifique por qué no es posible resolver la petición.

Solución en la primera llamada

Use este campo para indicar si la petición de soporte se resolvió durante la primera llamada al servicio de asistencia.

Inactivo

Use este campo si la petición ya no es válida, pero el registro debe retenerse para respetar la directiva de retención de registros.

Estatus y detalles de prioridad

Impacto

Use este campo para seleccionar el alcance del impacto sobre el servicio. Encontrará las siguientes opciones:

  • Ninguno: sin degradación del servicio

  • Bajo: servicio degradado para 1 usuario

  • Medio: fuera de servicio para 1 usuario o degradado para varios usuarios

  • Alto: fuera de servicio para varios usuarios

  • Incidencia grave

Tenga en cuenta que se considera que un servicio está degradado cuando parte de sus funcionalidades no están disponibles o cuando el tiempo de respuesta del servicio es lento. Un servicio no está disponible cuando no se puede acceder a su funcionalidad.

Urgencia

Use este campo para seleccionar el grado de urgencia entre las siguientes opciones:

  • Baja: responder utilizando procedimientos de funcionamiento estándar y según el tiempo lo permita.

  • Media: responder utilizando procedimientos de funcionamiento estándar según estructuras habituales de gestión y supervisión.

  • Alta: respuesta inmediata obligatoria

Prioridad

La aplicación selecciona la prioridad una vez seleccionados los niveles de impacto y de urgencia.

Estatus

Utilice este campo para seleccionar el estatus de la petición entre las siguientes opciones:

Abierto

Estatus inicial de la petición.

Rechazado

La petición se debe asignar a otra cola u otro miembro del personal.

Asignado

La responsabilidad de la resolución de la petición se ha asignado a una cola o persona específica.

Aceptado

Se resolverá la petición en cuanto el personal asignado esté listo para trabajar en ella.

En curso

La petición se está resolviendo.

Esperando…

La petición está a la espera de que se complete o apruebe una tarea, o de que se proporcione más información.

Waiting for customer (Esperando al cliente)

De momento no es posible progresar en la resolución de la petición.

Pendiente

La petición está pendiente.

Cambio pendiente

Se ha registrado una petición de cambio para la implementación del cambio solicitado.

Terminado

Las labores de resolución de la petición han finalizado por el motivo especificado en el campo Categoría de cierre.

ClosedMail (Correo cerrado)

El cliente ha sido informado por correo electrónico de la forma en que se ha completado la petición. En el mensaje de correo electrónico se solicita que se verifique la solución de la petición.

Cerrado/Sin contacto

El cliente no ha respondido y la petición se ha cerrado.

Cerrado

El cliente no ha respondido a la notificación por correo electrónico "ClosedMail" (Correo cerrado) en el plazo de 28 días. Este valor no se puede seleccionar manualmente, sino que es la aplicación de gestión de servicios la que lo selecciona automáticamente.

Categoría de cierre

Use este campo para seleccionar la categoría de cierre adecuada para la petición una vez que se haya completado. Encontrará las siguientes opciones:

  • Resuelto

  • No se puede reproducir

  • No se puede resolver o está en conflicto con estándares o directivas

  • Retirado por solicitante

  • Solución provisional

Detalles de fecha y hora

Fecha de apertura

Este campo se establece en la fecha y hora en que se crea la petición.

Fecha de respuesta

Este campo calcula la fecha y hora de respuesta de la petición según lo estipulado en el Acuerdo de nivel de servicio (SLA).

Fecha de vencimiento

Después de haber establecido la prioridad, este campo se establece en la fecha y hora en que la petición se debe completar. Si se adjunta un SLA a la petición, la fecha de vencimiento se basará en el SLA.

Fecha de cierre

Este campo se establece en la fecha y hora a la que se estableció el estatus de Cerrado a la petición de soporte.

Detalles de servicio y CI

Servicio

Utilice este campo para seleccionar el servicio para el que el cliente ha enviado la petición.
Si la petición no corresponde a un servicio específico o si corresponde a un servicio que aún no se ofrece, seleccione el servicio especial N/A - Service does not (yet) exist or is out of scope (No disponible: el servicio no existe (aún) o está fuera de alcance).

Oferta de servicio

Utilice este campo para vincular una oferta del servicio asociado.

Elemento de configuración

Utilice este campo para seleccionar el elemento de configuración (CI) que causa la interrupción del servicio (en el caso de una petición de resolución de incidencias), el elemento sobre el que el cliente desea obtener información (en el caso de resolución de peticiones) o el elemento que el cliente desea cambiar (en caso de peticiones de cambio).

Seleccione el CI especial con el código NORECORD (No registrado) si el CI todavía no se ha registrado en la base de datos de gestión de la configuración (CMDB).

Seleccione el CI especial con el código N/A si la petición de soporte no tiene que ver con un CI específico (por ejemplo, en el caso de una queja o un cumplido).

Inicio de indisponibilidad

Utilice este campo para introducir la fecha y la hora exacta a la que el servicio dejó de estar disponible debido a una incidencia descrita en esta petición.

Fin de indisponibilidad

Utilice este campo para introducir la fecha y la hora exacta a la que el servicio volvió a estar disponible tras la solución de la incidencia descrita en esta petición.

Detalles de asignación

Propietario

Este campo muestra la cola o persona a la que se asigna la petición.

Listas relacionadas

Historial de acciones

En esta sección encontrará el historial de acciones registradas para la petición.

Tareas vinculadas

Esta sección enumera las tareas que se han registrado para la petición y se utiliza para resolución de peticiones. Úsela para vincular tareas al despliegue.

Historial de aprobaciones

En esta sección encontrará el historial de aprobaciones registradas para la petición.

CI

En esta sección encontrará los CI relacionados con la petición. Úsela para vincular CI al despliegue.

Peticiones de cambio vinculadas

En esta sección encontrará las peticiones de cambio relacionadas con la petición. Úsela para vincular peticiones de cambio a la petición.

Problemas vinculados

En esta sección encontrará los problemas relacionados con la petición. Úsela para vincular problemas a la petición.

Notas y adjuntos

Use esta sección para adjuntar archivos que proporcionen información adicional para la resolución de la petición.

Historial de incidencias

En esta sección encontrará las modificaciones que haya sufrido la información de la petición.

Despliegues vinculados

Esta sección enumera los despliegues que se han registrado para el problema. Úsela para vincular despliegues a la petición.
