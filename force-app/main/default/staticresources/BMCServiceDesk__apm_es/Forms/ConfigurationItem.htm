Elemento de configuración


En la siguiente tabla encontrará los campos del formulario Elemento de configuración (CI) y las pautas de uso de cada campo.

Campo

Uso

General

Nombre de instancia

Use este campo para introducir el nombre único de CI.

Descripción

Use este campo para introducir la descripción de CI.

Descripción breve

Use este campo para introducir una breve descripción del CI que se muestra en el Explorador de CI.

Notas

Use este campo para introducir información adicional sobre el CI.

Nombre de fabricante

Use este campo para seleccionar el fabricante de CI.

Modelo

Use este campo para introducir el nombre del modelo en el caso de que se trate de un CI de hardware.

Número de versión

  • Use este campo para introducir el número de versión en el caso de que se trate de un CI de software o de documentación.

  • Cuando el CI sea un certificado de licencia de software, use este campo para introducir las versiones cubiertas por la licencia.

Nº de serie

Use este campo para introducir el número de serie del CI. Si el CI es un certificado de licencia de software, introduzca el número de certificado de licencia.
Deje el campo vacío para CI de Base de datos, Software, Paquete de distribución de software, Ajustes de configuración de software/interfaz, Sitio web o Enlace WAN.

Nº de activo

Use este campo para introducir el número con el que se ha registrado el CI en la aplicación financiera (por ejemplo, en SAP).

ID de testigo

Use este campo para introducir el identificador único del CI.

Prioridad

Use este campo para seleccionar la prioridad del CI indicando su importancia en la CMDB.

Fecha de creación

Este campo se ajusta a la fecha y la hora en que se registró el CI.

Última modificación por

En este campo aparece el nombre de usuario del último miembro del personal que actualizó el CI.

Fecha de última modificación

Este campo se ajusta a la fecha y hora en que se actualizó el CI por última vez.

ID de clase

Este campo se ajusta a la clase del elemento empleado para registrar el CI.

Nombre de clase

Este campo se ajusta al nombre de clase del elemento empleado para registrar el servicio.

En mantenimiento

Use este campo para especificar si se proporciona soporte técnico para este CI.

Fecha de caducidad de la garantía

Use este campo para especificar la fecha en que caduca la garantía que cubre el CI.

Sitio

  • Si se trata de un CI de hardware, use este campo para seleccionar la instalación en la que se encuentra el CI.

  • En el caso de que se trate de un CI de software o de paquete de distribución de software, use este campo para seleccionar la instalación en la que se guarda la copia maestra del DML.

  • Cuando el CI sea un certificado de licencia de software, use este campo para seleccionar la instalación de la DML en la que se almacena el certificado.

  • Establezca el campo como N/A en el caso de CI de Base de datos, Ajustes de configuración de software/interfaz o Sitio web.

Fase

Use este campo para seleccionar en la siguiente lista de opciones la fase adecuada del CI:

  • Requisitos

  • Definido

  • Analizado

  • Aprobado

  • Decretado

  • Diseñado

  • Desarrollado

  • Compilación

  • Probado

  • Desplegado

  • Operacional

  • Publicado

  • Sin publicar

  • Retirado

Marcar como eliminado

Use este campo para marcar el CI como eliminado temporalmente.

ID de instancia

Este campo se ajusta a la instancia del elemento empleado para registrar el CI.

Remitente

En este campo aparece el nombre de usuario del miembro del personal que registró el CI.

No está disponible de manera predeterminada. Debe personalizar el formulario para que se muestre el campo en la sección personalizada.

Especificaciones

Especificaciones (varios campos de elementos específicos)

Cada elemento tiene campos diferentes en la ficha Especificaciones. Por lo tanto, estos campos no se describen en el presente formulario.

Relaciones

Marcar como eliminado

Use este campo para marcar la relación como eliminada temporalmente.

Dirección de impacto

Use este campo para seleccionar en la siguiente lista de opciones la dirección de impacto adecuada:

  • Otro

  • Desconocido

  • Origen - Destino

  • Destino - Origen

  • Servicio - Subservicio

  • Servicio - Oferta

Listas relacionadas o información adicional

Historial

Use esta ficha para registrar las notas del historial de acción creado para el CI.

Clientes

En esta ficha se enumeran los clientes relacionados con el CI. Use esta ficha para seleccionar los clientes que usan el CI o lo tienen reservado.
Deje este campo vacío si ningún cliente específico usa o tiene reservado el CI.

Cuentas

En esta ficha se enumeran las cuentas relacionadas con el CI. Use esta ficha para seleccionar las organizaciones que tienen el CI reservado.
Deje este campo vacío si ninguna organización específica usa o tiene reservado el CI.

Incidencias

En esta ficha se enumeran las incidencias relacionadas con el CI. Úsela para vincular incidencias al CI.

Peticiones de cambio

En esta ficha se enumeran las peticiones de cambio relacionadas con el CI. Úsela para vincular peticiones de cambio al CI.

Problemas

En esta ficha se enumeran los problemas relacionados con el CI. Úsela para vincular problemas al CI.

Despliegues

En esta ficha se enumeran los despliegues relacionados con el CI. Úsela para vincular despliegues al CI.

Documentación

En esta ficha se enumeran los documentos relacionados con el CI. Úsela para adjuntar archivos que proporcionen información adicional sobre el CI.

Auditoría

En esta ficha se enumeran las modificaciones que haya sufrido la información del CI.
