Tarea


La siguiente tabla enumera los campos del formulario Tarea y proporciona pautas de uso para cada campo.

Campo

Uso

Nº de tarea

Este campo contiene el número único de tarea. La aplicación genera este número automáticamente.

Información del cliente

ID de cliente

Use este campo para introducir el ID de cliente del cliente que envió la tarea.

Tipo de cliente

Utilice este campo para seleccionar el tipo de cliente correcto.

No está disponible de manera predeterminada. Debe personalizar el formulario Tarea y agregarlo a la sección requerida.

Detalles de tarea

Plantilla

Use este campo para seleccionar la plantilla usada para la tarea.

Categoría

Use este campo para seleccionar la categoría de la tarea.

Descripción

Use este campo para introducir una descripción breve de la tarea.

Resolución

  • Una vez que el estatus de la tarea se haya establecido como Fallido, introduzca información útil para el coordinador de cambios. Por ejemplo, especifique por qué no se pudo responder a la tarea de riesgos e impacto.

  • Una vez que el estatus de la tarea se haya establecido como Cancelado, introduzca el motivo de la cancelación en este campo. Por ejemplo, podría ser necesario cancelar la tarea porque la plantilla de cambios utilizada para registrar la petición de cambio la genera, pero no es necesaria para esta implementación específica.

  • Una vez que el estatus de la tarea se haya establecido como Terminado y, si el resultado es ligeramente distinto o si el resultado no se logró de la manera solicitada, introduzca la información en este campo. Si la tarea se ejecutó según lo solicitado y se cumplieron los objetivos, deje el campo vacío (a menos que se indique lo contrario en las instrucciones de trabajo pertinentes).

Incidencia

Use este campo para seleccionar la incidencia a la que está vinculada la tarea.

Petición de cambio

Use este campo para seleccionar el cambio al que está vinculada la tarea.

Problema

Use este campo para seleccionar el problema al que está vinculada la tarea.

Despliegue

Use este campo para seleccionar el despliegue al que está vinculada la tarea.

Tipo de tarea

Use este campo para seleccionar el tipo de tarea entre las siguientes opciones:

  • General

  • Prueba

  • Regla operacional

No está disponible de manera predeterminada. Debe personalizar el formulario Tarea y agregarlo a la sección requerida.

Estatus y detalles de prioridad

Impacto

Use este campo para seleccionar el alcance del impacto sobre el servicio. Encontrará las siguientes opciones:

  • Bajo: servicio degradado para un usuario

  • Medio: fuera de servicio para un usuario o degradado para varios usuarios

  • Alto: fuera de servicio para varios usuarios

Urgencia

Use este campo para seleccionar el grado de urgencia entre las siguientes opciones:

  • Baja: responder utilizando procedimientos de funcionamiento estándar y según el tiempo lo permita.

  • Media: responder utilizando procedimientos de funcionamiento estándar según estructuras habituales de gestión y supervisión.

  • Alta: respuesta inmediata obligatoria

Prioridad

La aplicación selecciona la prioridad una vez seleccionados los niveles de impacto y de urgencia.

Estatus

Use este campo para seleccionar en la siguiente lista de opciones el estatus adecuado para la tarea:

Abierto

El estatus inicial de la tarea.

Registrado

La tarea no debe ejecutarse todavía.

Rechazado

La tarea se debe asignar a otra cola u otro miembro del personal.

Asignado

Se puede dar comienzo a la ejecución de la tarea en la fecha y hora especificadas en el campo Inicio programado.

Aceptado

La tarea se ejecutará cuando el personal asignado esté listo para comenzar a trabajar en ella.

En curso

Se está trabajando en la tarea actualmente.

Esperando…

La tarea está a la espera de que se proporcione más información o se complete otra tarea.

Fallido

No puede encontrarse una respuesta para la tarea de riesgos e impacto, la persona a la que se le asignó la aprobación no puede aprobar el cambio o el objetivo de la tarea de implementación no puede alcanzarse.

Terminado

Se ha proporcionado la respuesta para la tarea de riesgos e impacto, el cambio ha sido aprobado por la persona a la que se le asignó la aprobación o la ejecución de la tarea de implementación ha alcanzado su objetivo.

Cancelado

No se pudo completar la tarea. Por ejemplo, la tarea no pudo completarse porque el cambio se ha rechazado o porque la plantilla de cambio generó la tarea, pero no se aplica a esta implementación de cambio especifica.

Cerrado

La tarea se ha cerrado.

Cerrada por

En este campo se guarda el nombre de usuario de la persona que cerró la tarea.

Inactivo

Seleccione este campo si la tarea ya no es válida, pero el registro debe conservarse para respetar la directiva de retención de registros.

Detalles de fecha y hora

Fecha de apertura

Este campo se establece en la fecha y hora en que se crea la tarea.

Fecha de vencimiento

Use este campo para establecer la fecha y hora en que debe completarse la ejecución de la tarea.

Inicio programado

Use este campo para seleccionar la fecha y hora en que la tarea debe comenzar.

Finalización programada

Use este campo para seleccionar la fecha y hora en que se debe finalizar la tarea.

Fecha de cierre

Este campo se establece en la fecha y hora en que se cierra la tarea.

Detalles de asignación

Propietario

Este campo muestra la cola o persona a la que se ha asignado la tarea.

Listas relacionadas

Historial de acciones

En esta sección encontrará las acciones que se han registrado para la tarea.

Servicios y CI vinculados

En esta sección encontrará los servicios y CI relacionados con la tarea. Úsela para vincular todos los CI y servicios que se verán afectados por la ejecución de la tarea.

Notas y adjuntos

En esta sección encontrará los documentos relacionados con la tarea. Úsela para adjuntar archivos que contengan información adicional sobre la finalización de la tarea.

Historial de tarea

En esta sección encontrará las modificaciones que haya sufrido la información de la tarea.
