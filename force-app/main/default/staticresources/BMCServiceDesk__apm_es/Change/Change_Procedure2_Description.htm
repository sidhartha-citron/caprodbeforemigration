Procedimiento 2, Planificación de cambios


Tras haber registrado el cambio, el coordinador de cambios inicia la fase de análisis de riesgos e impacto. Durante esta fase, el coordinador de cambios recopila la información necesaria para crear un plan de implementación de cambio que minimice el riesgo de error y el impacto en los clientes.

El coordinador de cambios crea una tarea de riesgos e impacto separada (Evaluación de cambio) para cada pregunta que se necesite responder.

Si se necesita crear una instancia de servicio nueva o modificar una instancia de servicio existente para cumplir los requisitos del cambio, el coordinador de cambios también asigna una tarea de riesgos e impacto al gestor de disponibilidad para que diseñe la creación de una instancia de servicio nueva o la modificación de una existente.

Las tareas de riesgos e impacto que el coordinador de cambios no puede responder se asignan al especialista más adecuado dentro del grupo del coordinador de cambios o a otro grupo.

Usando las respuestas de las tareas de riesgos e impacto (Evaluación de cambio), el coordinador de cambios ajusta el plan de implementación que se copió de la plantilla cuando se registró el cambio.
