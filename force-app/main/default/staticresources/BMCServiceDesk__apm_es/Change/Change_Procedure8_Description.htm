Procedimiento 8, Cierre de cambios de emergencia


Cuando se finaliza la implementación, el especialista (con ayuda de un usuario si el especialista no posee derechos de acceso suficientes) realiza una prueba en el entorno de producción para verificar si la implementación se realizó correctamente. Si la implementación no se realizó correctamente, el especialista se comunica con el propietario del servicio (o el gestor de guardia si el propietario del servicio no está disponible) para analizar la situación.

Y a la inversa, si la implementación es correcta, el especialista completa la petición de soporte. Además, el especialista pide al coordinador de cambios del servicio afectado que registre el cambio de emergencia.

El coordinador de cambios registra el cambio y comprueba que CMDB esté actualizada de ser necesario. A continuación, el coordinador de cambios
informa al propietario del servicio que el cambio de emergencia se implementó correctamente. Finalmente, el coordinador de cambios cierra la petición de emergencia.
