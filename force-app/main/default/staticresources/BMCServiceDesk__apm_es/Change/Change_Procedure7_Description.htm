Procedimiento 7, Implementación de cambios de emergencia

El especialista iniciará el trabajo de implementación una vez que el propietario del servicio (o el gestor de guardia, si el propietario del servicio no está disponible) le haya pedido que solucione una incidencia mediante la implementación de un cambio de emergencia. Si el cambio de emergencia supone un cambio de infraestructura, el especialista realizará la implementación como si se tratara de una petición de soporte normal.

Si el cambio de emergencia supone un cambio de aplicación, el especialista creará un nuevo despliegue en el entorno de desarrollo y garantizará que la incidencia se soluciona sin introducir nuevos defectos (errores). A continuación, el especialista pide a un administrador de despliegues que transfiera el nuevo despliegue al entorno de prueba. El especialista probará el nuevo despliegue en el entorno de prueba.

Si el nuevo despliegue no supera la prueba, el especialista deberá volver al entorno de desarrollo para corregirlo. Si el nuevo despliegue supera la prueba, el especialista pedirá al administrador de despliegues que lo transfiera al entorno de producción.
