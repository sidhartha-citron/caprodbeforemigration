Instrucciones de trabajo para gestores de cambios

3.7.1

Determine si se requiere aprobación de todos los miembros CAB (el representante del cliente y el propietario del servicio que se debe cambiar) o solo del propietario del servicio.

3.7.2

La aprobación del representante del cliente solo se necesita si la implementación del cambio hará:

  • que el servicio no esté disponible o se degrade durante las horas de servicio,

  • que cambie la funcionalidad de un servicio

 

Nota:

Cada cambio no estándar coordinado por gestión de cambios necesita la aprobación del propietario del servicio independientemente de si se requiere la aprobación del representante del cliente.

3.7.3

Cuando el cambio esté listo para enviarse para aprobación, haga clic en Enviar para aprobación en la sección Historial de aprobaciones. El flujo de aprobación se establece y se informa a los usuarios relacionados automáticamente.

Nota:

Se puede cancelar una petición de aprobación seleccionando Retirar petición de aprobación en el Historial de aprobaciones de la petición de cambio.

Nota:

La información sobre el usuario cuya aprobación se requiere para la petición de cambio se puede indicar en una definición de flujo de trabajo de aprobación.

3.7.4

Especifique las fechas de Inicio programado y Finalización programada para la planificación del cambio.

Nota:

Para comprobar la planificación completa del cambio, haga clic en Programa de cambio.

3.7.5

Establezca el campo Estatus del cambio como Aprobación solicitada.
