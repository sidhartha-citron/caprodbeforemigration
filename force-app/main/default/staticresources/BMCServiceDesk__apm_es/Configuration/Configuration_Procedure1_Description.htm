Procedimiento 1, Petición de CI


Cuando haya que pedir nuevos elementos de configuración (CI), o cuando haya que actualizar la base de datos de gestión de la configuración (CMDB), el administrador de configuración responsable recibirá una tarea de gestión de cambios o de gestión de peticiones.

Si la tarea no requiere un nuevo CI, el administrador de configuración pasará directamente al procedimiento 2, Recepción de CI. Por otro lado, si una tarea pide uno o varios nuevos CI, el administrador de configuración enviará en primer lugar un requerimiento de compra por los CI necesarios. Una vez enviado el requerimiento de compra a compras, el administrador de configuración comprobará su progreso periódicamente.

Si compras pide los CI, el administrador de configuración registrará los CI solicitados en la CMDB. Por otro lado, si el requerimiento de compra se rechaza, el administrador de configuración pasará al procedimiento 4, Actualización de CI.
