Instrucciones de trabajo para administradores de configuración

2.8.1

Registre los detalles del elemento de configuración (CI) nuevo. Si se envía un requerimiento de compra para el CI nuevo, dicho CI ya estará registrado en la CMDB con el campo Estatus establecido como Pedido. En este caso, actualice el estatus de los CI y añada la información que falte (como el número de serie).

Si no se envía un requerimiento de compra formal (como en el caso de software desarrollado internamente), registre el CI nuevo en la CMDB.

Vincule todos los nuevos CI a los otros CI, clientes y cuentas según proceda. Para ello, siga las directrices de utilización de campos para el formulario Elemento de configuración.

2.8.2

Imprima el código de CI único de cada nuevo CI de hardware en una etiqueta. Pegue cada etiqueta en el CI de hardware correspondiente de forma que esté claramente visible tras su instalación.

2.8.3

Imprima el código de CI único de cada nuevo CI de software en una etiqueta. Pegue las etiquetas en la copia maestra de su CI de software correspondiente, de forma que esté claramente visible cuando coloque dicha copia maestra en la. DML. Coloque la copia maestra de cada nuevo CI de software en la DML.

2.8.4

Imprima el código de CI único de cada nuevo certificado de licencia de software en una pegatina. Pegue las pegatinas en su certificado correspondiente de forma que resulte fácil de encontrar el certificado después de haberlo colocado en la DML. Coloque todos los nuevos certificados de licencia de software en la DML.
