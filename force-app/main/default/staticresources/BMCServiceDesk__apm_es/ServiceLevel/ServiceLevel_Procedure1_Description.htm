Procedimiento 1, Mantenimiento del catálogo de servicios


Los cambios que requieren que se actualice información de gestión de nivel de servicio incluyen una tarea para la actualización. La tarea se asigna al administrador de nivel de servicio.

Cuando se asigna dicha tarea al administrador de nivel de servicio, este determina si es necesario un nuevo elemento de catálogo. Si no se necesita un elemento de catálogo nuevo, el administrador de nivel de servicio omite el resto del procedimiento y va al Procedimiento 2, Activación del servicio. Sin embargo, si se requiere un elemento de catálogo nuevo, el administrador de nivel de servicio lo prepara con la plantilla de elemento de catálogo o copia un elemento de catálogo similar.

Cuando el administrador de nivel de servicio ha terminado de preparar el nuevo elemento de catálogo, debe obtener la aprobación de:

El propietario del servicio responsable de cumplir los SLT especificados en el elemento de catálogo

Los gestores de nivel de servicio de las organizaciones clientes para las que se ha preparado el elemento de catálogo para garantizar que los SLR de los clientes estén cubiertos por el elemento de catálogo

El gestor financiero de la organización proveedora de servicios cuyo compromiso se necesita para los aspectos financieros del elemento de catálogo.

Si los aprobadores necesitan que se hagan ajustes antes de firmar el elemento de catálogo nuevo, el administrador de nivel de servicio ajusta el elemento de catálogo según sea necesario hasta que se obtengan todas las firmas de aprobación. Sin embargo, si los aprobadores no llegan a un acuerdo, el administrador de nivel de servicio va al Procedimiento 4, Finalización del servicio y menciona la razón del fracaso de la creación del elemento de catálogo nuevo en la tarea.

Cuando todos los aprobadores hayan firmado el elemento de catálogo nuevo, el administrador de nivel de servicio lo agrega al catálogo de servicios basado en la Web de la organización proveedora de servicios. Si el elemento de catálogo nuevo está preparado para cumplir con SLR específicos de un cliente, el gestor de nivel de servicio se asegura de que los clientes no tengan acceso a la versión publicada.
