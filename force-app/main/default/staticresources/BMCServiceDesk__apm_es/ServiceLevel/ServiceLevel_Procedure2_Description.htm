Procedimiento 2, Activación del servicio


Si la tarea para gestión de nivel de servicio solicita que se registre una instancia de servicio nueva, el administrador de nivel de servicio completa el formulario Servicio en la aplicación de gestión de servicios.

Si la tarea solicita que se prepare un SLA nuevo, el administrador de nivel de servicio prepara un SLA nuevo. Para ello, completa una Plantilla de acuerdo de nivel de servicio, imprime dos copias y adjunta a cada copia el elemento de catálogo relevante impreso. A continuación, el administrador de nivel de servicio recopila las firmas del representante del cliente y el propietario de servicio responsables de cumplir los SLT especificados en el SLA.

Si el representante del cliente o el propietario del servicio decidieran no firmar el SLA, el administrador de nivel de servicio va al Procedimiento 4, Finalización del servicio para mencionar la razón por la que no se estableció el SLA nuevo en la tarea. Si se firmó el SLA nuevo, el administrador de nivel de servicio archiva una copia del SLA firmado y devuelve la otra al representante del cliente para sus registros.

Antes de registrar un SLA nuevo, el administrador de nivel de servicio comprueba la aplicación de gestión de servicio para saber si los detalles de contacto del cliente están actualizados. Si el cliente del SLA aún no está registrado o si la información de contacto del cliente ya no es actual, el administrador de nivel de servicio se asegura de que la información del cliente se registre o se actualice en el Procedimiento 3, Mantenimiento de la información de cliente.

Cuando la información de contacto del cliente sea actual, el administrador de nivel de servicio registra el SLA nuevo en la aplicación de gestión de servicio y lo vincula a la instancia de servicio usada para proporcionar el servicio al cliente.
