Instrucciones de trabajo para gestores de nivel de servicio

5.10.1

Analice cada petición con el representante del cliente para asegurarse de comprender los requisitos de negocios del cliente. Para entender los requisitos del cliente, cree un conjunto conciso de SLR cuando el representante del cliente pida un servicio nuevo o solicite que un servicio se proporcione a otro nivel.

Nota:

Para reflejar los requisitos de negocio del cliente como SLR, consulte el elemento de catálogo seleccionado por el cliente. Pero si el cliente solicita un servicio nuevo, puede que tenga que ayudar al representante a decidir los requisitos de nivel de asistencia, disponibilidad, fiabilidad, etc. Se puede usar una Continuity Risk Assessment Scorecard (Tabla de puntuación para evaluar riesgos de continuidad) para ayudar al representante del cliente a decidir cuál es el nivel adecuado de cobertura de continuidad para el servicio.

Nota:

Si el representante del cliente envía una queja, es importante comprender el impacto en el negocio del cliente. Use la información para evitar malentendidos dentro de la organización proveedora de servicios.

Nota:

El representante del cliente debe informar de incidencias directamente al servicio de asistencia. Cuando el representante del cliente solicite un cambio, pídale que use el formulario de petición web para enviar la petición de cambio. Estas peticiones de soporte no deberían ser procesadas por gestores de nivel de servicio.

Nota:

No registre la petición hasta que no se hayan aclarado todos los requisitos de negocio.
